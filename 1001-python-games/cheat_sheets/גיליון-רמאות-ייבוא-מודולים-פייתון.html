# ייבוא מודולים בפייתון

ייבוא ספריות בפייתון מאפשר להשתמש במודולים ופונקציות חיצוניים או משלך, המוגדרים בקבצים או ספריות אחרות. זה עוזר לארגן את הקוד, לשפר את קריאותו ולמנוע כפילויות.

מהם מודולים?

מודול בפייתון הוא פשוט קובץ עם סיומת <span dir="ltr"><code>.pycode>span>, המכיל קוד (פונקציות, מחלקות, משתנים וכו'). כאשר אתה רוצה להשתמש בקוד ממודול אחר, עליך לייבא אותו לקובץ הנוכחי.

מודולים חיצוניים ופנימיים

  • **מודולים חיצוניים** — אלו מודולים שאינם חלק מהספרייה הסטנדרטית של פייתון ויש להתקין אותם בנפרד. לדוגמה, מודולים לעבודה עם שרת אינטרנט, עיבוד נתונים או למידת מכונה, כגון <span dir="ltr"><code>numpycode>span>, <span dir="ltr"><code>requestscode>span> ואחרים.
  • **מודולים פנימיים** — אלו מודולים שכבר כלולים בספרייה הסטנדרטית של פייתון. לדוגמה, מודולים לעבודה עם קבצים, זמן, פעולות מתמטיות: <span dir="ltr"><code>oscode>span>, <span dir="ltr"><code>mathcode>span>, <span dir="ltr"><code>datetimecode>span>.

    • התקנת מודולים חיצוניים באמצעות pip

    כדי להתקין מודול חיצוני, נשתמש בפקודה <span dir="ltr"><code>pipcode>span> — זהו כלי לניהול חבילות פייתון. לדוגמה, כדי להתקין את ספריית <span dir="ltr"><code>requestscode>span>, יש להריץ את הפקודה הבאה בטרמינל:

    
pip install requests

    פקודה זו תוריד ותתקין את הספרייה בפרויקט שלך.

    ייבוא מודולים

    כאשר אתה רוצה להשתמש במודול בפייתון, עליך לייבא אותו:

  • **ייבוא מודול שלם**:

    • ```python

    import os

    ```

    לאחר מכן תוכל להשתמש בכל הפונקציות והמשתנים מהמודול, לדוגמה:

    ```python

    os.list<span dir="ltr">dir()span> # רשימת קבצים בספרייה

    ```

  • **ייבוא אלמנטים ספציפיים ממודול**:

    • ```python

    from math import sqrt

    ```

    כעת ניתן להשתמש בפונקציה <span dir="ltr"><code>sqrtcode>span> ללא צורך לפנות למודול <span dir="ltr"><code>mathcode>span>.

  • **ייבוא עם כינוי**:

    • ```python

    import pandas as pd

    ```

    במקרה זה, כדי להשתמש בפונקציות של ספריית <span dir="ltr"><code>pandascode>span>, תכתוב <span dir="ltr"><code>pdcode>span>, ולא את השם המלא <span dir="ltr"><code>pandascode>span>.

    ייבוא מודולים משלך

    אם אתה כותב מספר קבצים בפרויקט אחד, ניתן ליצור מודולים משלך. לדוגמה, אם יש לך קובץ <span dir="ltr"><code>utils.pycode>span> עם פונקציות שימושיות, תוכל לייבא אותו לקבצים אחרים כך:

    
import utils

    או לייבא פונקציה ספציפית:

    
from utils import my_function

    למה ליצור מודול אחד ולייבא אותו לחלקים אחרים של הפרויקט?

  • **ארגון קוד**: הפרדת לוגיקה למודולים שונים עוזרת למנוע בלאגן בפרויקט גדול. לדוגמה, ניתן ליצור מודול לעבודה עם נתונים, אחר לעיבוד בקשות, ושלישי לממשק.
  • **שימוש חוזר בקוד**: כאשר לוגיקה מופרדת למודולים, ניתן להשתמש בה בחלקים שונים של הפרויקט. זה מפחית כפילויות קוד ומקל על תחזוקה.
  • **קריאות ותחזוקתיות**: כאשר קוד מופרד למודולים, מפתחים אחרים (או אתה בעצמך בעתיד) יוכלו להבין ולתחזק את הפרויקט בקלות רבה יותר.

    • דוגמה:

    
# utils.py
def greet(name):
    return f"Hello, {name}!"
    
# main.py
from utils import greet

print(greet("Alice"))

    בדוגמה זו, הפונקציה <span dir="ltr"><code>greetcode>span> מוגדרת במודול אחד (<span dir="ltr"><code>utils.pycode>span>), אך משמשת במודול אחר (<span dir="ltr"><code>main.pycode>span>), מה שמשפר את המבנה והופך את הקוד למודולרי ונוח יותר לשינוי.
