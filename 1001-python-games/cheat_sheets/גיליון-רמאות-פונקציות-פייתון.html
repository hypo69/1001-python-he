# פונקציות בפייתון

פונקציות בשפת פייתון הן בלוקים של קוד בעלי שם, המבצעים משימה ספציפית. הן מאפשרות לארגן את הקוד, להפוך אותו למובנה יותר ונוח לשימוש חוזר.

    • הצהרת פונקציה

    פונקציה מוצהרת באמצעות מילת המפתח <span dir="ltr"><code>defcode>span>, ואחריה שם הפונקציה, רשימת פרמטרים בסוגריים עגולים ונקודתיים. גוף הפונקציה נכתב עם הזחה.

    
def שם_פונקציה(פרמטרים):
    # פעולות
    return תוצאה

    דוגמה:

    
def חיבור(a: int, b: int) -> int:
    """מחזירה את סכום שני מספרים."""
    return a + b

    כאן:

    - <span dir="ltr"><code>a: intcode>span> ו-<span dir="ltr"><code>b: intcode>span> — פרמטרים של פונקציה עם הערות סוג.

    - <span dir="ltr"><code>-> intcode>span> — הערת סוג של ערך מוחזר.

    פרמטרים של פונקציה

    פונקציות יכולות לקבל פרמטרים, המהווים נתוני קלט. הם מצוינים בסוגריים לאחר שם הפונקציה.

    דוגמה עם פרמטר אחד:

    
def ברכה(שם: str) -> str:
    """מברכת את המשתמש בשמו."""
    return f"שלום, {שם}!"

    סוגי פרמטרים:

  • **פרמטרים חובה** — חייבים להיות מועברים בעת קריאה לפונקציה.
  • **פרמטרים אופציונליים** — יכולים להיות בעלי ערכי ברירת מחדל.

    • ```python

    def ברכה(שם: str, גיל: int = 18) -> str:

    return f"שלום, {שם}! אתה בן {גיל} שנים."

    ```

    החזרת ערך

    פונקציה יכולה להחזיר ערך באמצעות מילת המפתח <span dir="ltr"><code>returncode>span>. אם <span dir="ltr"><code>returncode>span> אינו בשימוש, הפונקציה מחזירה <span dir="ltr"><code>Nonecode>span> כברירת מחדל.

    דוגמה:

    
def כפל(a: int, b: int) -> int:
    """מחזירה את מכפלת שני מספרים."""
    return a * b

    משתנים מקומיים וגלובליים

    - **משתנה מקומי** — זהו משתנה הקיים רק בתוך פונקציה. הוא נוצר ונהרס בכל קריאה לפונקציה.

    - **משתנה גלובלי** — זהו משתנה הנגיש בכל הקוד, כולל פונקציות.

    דוגמה לשימוש במשתנה גלובלי:

    
x = 10  # משתנה גלובלי

def הצג_x() -> int:
    return x  # גישה למשתנה גלובלי

    אם בתוך פונקציה צריך לשנות משתנה גלובלי, יש להשתמש במילת המפתח <span dir="ltr"><code>globalcode>span>:

    
x = 10  # משתנה גלובלי

def שנה_x() -> None:
    global x
    x = 20

    פונקציות מקוננות

    בפייתון, פונקציות יכולות להיות מקוננות, כלומר פונקציה אחת יכולה להיות מוגדרת בתוך אחרת. פונקציה מקוננת יכולה לגשת למשתנים של הפונקציה החיצונית.

    דוגמה:

    
def חיצונית(a: int, b: int) -> int:
    """פונקציה המשתמשת בפונקציה מקוננת לחישוב ההפרש."""
    
    def מקוננת(x: int, y: int) -> int:
        """פונקציה מקוננת המחזירה את ההפרש."""
        return x - y
    
    return מקוננת(a, b)

    רקרוסיה

    רקרוסיה — זה כאשר פונקציה קוראת לעצמה. זה שימושי למשימות שניתן לפרק למשימות קטנות יותר דומות (לדוגמה, עצרת).

    דוגמה לרקורסיה:

    
def עצרת(n: int) -> int:
    """מחשבת את עצרת המספר באמצעות רקורסיה."""
    if n == 0:
        return 1  # מקרה בסיס
    return n * עצרת(n - 1)  # קריאה רקורסיבית

    טיפול בחריגים עם <span dir="ltr"><code>trycode>span> ו-<span dir="ltr"><code>exceptcode>span>

    פייתון מספקת מנגנון טיפול בשגיאות באמצעות בלוקים <span dir="ltr"><code>trycode>span> ו-<span dir="ltr"><code>exceptcode>span>. קוד שעלול לגרום לשגיאה ממוקם בבלוק <span dir="ltr"><code>trycode>span>, והשגיאות מטופלות בבלוק <span dir="ltr"><code>exceptcode>span>.

    דוגמה לטיפול בשגיאות:

    
def חלוקה(a: int, b: int) -> float:
    """מחלקת מספר אחד באחר, תוך טיפול בשגיאות אפשריות."""
    try:
        result = a / b
    except ZeroDivisionError:
        return "שגיאה: חלוקה באפס"
    except Exception as e:
        return f"אירעה שגיאה: {e}"
    return result

    כאן:

    - בלוק <span dir="ltr"><code>trycode>span> מנסה לבצע את פעולת החלוקה.

    - בלוק <span dir="ltr"><code>except ZeroDivisionErrorcode>span> לוכד את שגיאת החלוקה באפס.

    - בלוק <span dir="ltr"><code>except Exception as ecode>span> לוכד חריגים אחרים ומציג הודעת שגיאה.

    דוגמה לשימוש בפונקציות

    
# חיבור שני מספרים
print(חיבור(5, 3))  # 8

# פונקציה מקוננת
print(חיצונית(10, 4))  # 6

# רקורסיה לחישוב עצרת
print(עצרת(5))  # 120

# טיפול בחריגים בחלוקה
print(חלוקה(10, 2))  # 5.0
print(חלוקה(10, 0))  # שגיאה: חלוקה באפס
