פריסת Jekyll אוטומטית ל-GitHub Pages

כדי להפוך את תהליך הפריסה לאוטומטי, נשתמש ב-GitHub Actions, המאפשרים לבצע משימות שונות, כולל בנייה ופרסום אתרים, ישירות במאגר שלך.

1: סקירת קובץ Workflow

ראשית, בואו נסתכל על קובץ ה-workflow הראשי השולט בתהליך הבנייה והפריסה. קובץ זה כתוב ב-YAML ונמצא בדרך כלל בספריית <span dir="ltr"><code>.github/workflowscode>span>. הנה תוכנו:


# Sample workflow for building and deploying a Jekyll site to GitHub Pages
name: Deploy Jekyll with GitHub Pages dependencies preinstalled

on:
  # Runs on pushes targeting the default branch
  push:
    branches: ["master"]

  # Allows you to run this workflow manually from the Actions tab
  workflow_dispatch:

# Sets permissions of the GITHUB_TOKEN to allow deployment to GitHub Pages
permissions:
  contents: read
  pages: write
  id-token: write

# Allow only one concurrent deployment, skipping runs queued between the run in-progress and latest queued.
# However, do NOT cancel in-progress runs as we want to allow these production deployments to complete.
concurrency:
  group: "pages"
  cancel-in-progress: false

jobs:
  # Build job
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
      - name: Setup Pages
        uses: actions/configure-pages@v5
      - name: Build with Jekyll
        uses: actions/jekyll-build-pages@v1
        with:
          source: ./docs/gemini/consultant/ru/src
          destination: ./_site
      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3

  # Deployment job
  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    needs: build
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4

2: פירוק מבנה Workflow

כעת נפרק כל חלק בקובץ זה:

2.1. מידע כללי

- <span dir="ltr"><code>name: Deploy Jekyll with GitHub Pages dependencies preinstalledcode>span>: שם ה-workflow שתראה ברשימת Actions במאגר.

- <span dir="ltr"><code>oncode>span>: מתאר מתי ה-workflow אמור לרוץ:

- <span dir="ltr"><code>pushcode>span>: ה-workflow רץ בכל push לענף <span dir="ltr"><code>mastercode>span>.

- <span dir="ltr"><code>workflow_dispatchcode>span>: מאפשר לך להריץ את ה-workflow ידנית דרך ממשק GitHub.

- <span dir="ltr"><code>permissionscode>span>: מגדיר הרשאות להפעלת ה-workflow:

- <span dir="ltr"><code>contents: readcode>span>: הרשאה לקרוא קוד מהמאגר.

- <span dir="ltr"><code>pages: writecode>span>: הרשאה לפרסם ב-GitHub Pages.

- <span dir="ltr"><code>id-token: writecode>span>: הרשאה לקבל אסימון אימות (נדרש עבור GitHub Actions).

- <span dir="ltr"><code>concurrencycode>span>: מגדיר ביצוע מקביל של workflow:

- <span dir="ltr"><code>group: "pages"code>span>: מבטיח שרק workflow אחד עבור GitHub Pages ירוץ בכל פעם.

- <span dir="ltr"><code>cancel-in-progress: falsecode>span>: מונע ביטול של הרצת workflow נוכחית כאשר מופעלת חדשה.

2.2. סעיף <span dir="ltr"><code>jobscode>span>

סעיף זה מתאר אילו משימות יש לבצע. יש לנו שתי משימות: <span dir="ltr"><code>buildcode>span> ו-<span dir="ltr"><code>deploycode>span>.

2.2.1. <span dir="ltr"><code>buildcode>span>: בניית אתר

- <span dir="ltr"><code>runs-on: ubuntu-latestcode>span>: מציין שהמשימה רצה על שרת עם אובונטו.

- <span dir="ltr"><code>stepscode>span>: מפרט את השלבים המבוצעים במהלך הבנייה:

- <span dir="ltr"><code>name: Checkoutcode>span>: מוציא את קוד המקור של המאגר.

- <span dir="ltr"><code>uses: actions/checkout@v4code>span>: משתמש בפעולה מוכנה להוצאת קוד.

- <span dir="ltr"><code>name: Setup Pagescode>span>: מגדיר את הסביבה לעבודה עם GitHub Pages.

- <span dir="ltr"><code>uses: actions/configure-pages@v5code>span>: משתמש בפעולה מוכנה להגדרה.

- <span dir="ltr"><code>name: Build with Jekyllcode>span>: מתחיל לבנות אתר Jekyll.

- <span dir="ltr"><code>uses: actions/jekyll-build-pages@v1code>span>: משתמש בפעולה מוכנה לבנייה.

- <span dir="ltr"><code>with:code>span>: מגדיר פרמטרים לפעולה:

- <span dir="ltr"><code>source: ./docs/gemini/consultant/ru/srccode>span>: מציין היכן נמצאים קבצי המקור של האתר שלך. **שים לב**: הנתיב לקבצים שלך <span dir="ltr"><code>docs/gemini/consultant/ru/srccode>span>.

- <span dir="ltr"><code>destination: ./_sitecode>span>: מציין היכן למקם את הקבצים הבנויים.

- <span dir="ltr"><code>name: Upload artifactcode>span>: מעלה את הקבצים הבנויים כדי להעביר אותם למשימה הבאה.

- <span dir="ltr"><code>uses: actions/upload-pages-artifact@v3code>span>: משתמש בפעולה מוכנה להעלאת ארטיפקטים.

2.2.2. <span dir="ltr"><code>deploycode>span>: פרסום אתר

- <span dir="ltr"><code>environmentcode>span>: מגדיר את סביבת הפרסום.

- <span dir="ltr"><code>name: github-pagescode>span>: שם הסביבה.

- <span dir="ltr"><code>url: ${{ steps.deployment.outputs.page_url }}code>span>: מקבל את כתובת ה-URL של האתר שפורסם.

- <span dir="ltr"><code>runs-on: ubuntu-latestcode>span>: מציין שהמשימה רצה על שרת עם אובונטו.

- <span dir="ltr"><code>needs: buildcode>span>: מציין שהמשימה <span dir="ltr"><code>deploycode>span> צריכה לרוץ לאחר השלמה מוצלחת של המשימה <span dir="ltr"><code>buildcode>span>.

- <span dir="ltr"><code>stepscode>span>: מפרט את השלבים המבוצעים במהלך הפרסום:

- <span dir="ltr"><code>name: Deploy to GitHub Pagescode>span>: מפרסם את האתר ל-GitHub Pages.

- <span dir="ltr"><code>id: deploymentcode>span>: מגדיר מזהה לפעולה.

- <span dir="ltr"><code>uses: actions/deploy-pages@v4code>span>: משתמש בפעולה מוכנה לפריסה.

3: מה עושים קבצי Markdown?

קבצי <span dir="ltr"><code>.mdcode>span> (Markdown) הם הבסיס לאתר Jekyll. Markdown היא שפת סימון פשוטה המאפשרת לך לעצב טקסט.

Jekyll מעבד אוטומטית קבצי <span dir="ltr"><code>.mdcode>span>, והופך אותם לדפי HTML. הקבצים שלך צריכים להיות ממוקמים בתיקייה <span dir="ltr"><code>docs/gemini/consultant/ru/srccode>span> המצוינת ב-workflow.

5: איך זה עובד

  • **שינוי קוד:** אתה מבצע שינויים בקבצי <span dir="ltr"><code>.mdcode>span> או <span dir="ltr"><code>.htmlcode>span> שלך הממוקמים בתיקייה <span dir="ltr"><code>docs/gemini/consultant/ru/srccode>span>.
  • **Push:** אתה שולח (push) את השינויים לענף <span dir="ltr"><code>mastercode>span> של מאגר GitHub שלך.
  • **הפעלת Workflow:** GitHub Actions מפעיל אוטומטית את ה-workflow המתואר בקובץ YAML.
  • **בנייה:** ה-workflow מוריד תחילה את הקוד מהמאגר, ולאחר מכן בונה אתר Jekyll מקבצי המקור שלך לתיקייה <span dir="ltr"><code>_sitecode>span>.
  • **פרסום:** האתר הבנוי מפורסם ל-GitHub Pages.
  • **האתר מוכן:** האתר שלך זמין בכתובת ה-URL המצוינת בהגדרות GitHub Pages שלך.
