### **זמן פולינומי**

**זמן פולינומי** הוא מונח המשמש בתורת הסיבוכיות החישובית לתיאור זמן הריצה של אלגוריתם, אשר גדל כפולינום של גודל הקלט. אם זמן הריצה של אלגוריתם ניתן לביטוי כ-\(O(n^k)\), כאשר \(n\) הוא גודל הקלט ו-\(k\) הוא קבוע, אז אלגוריתם כזה פועל בזמן פולינומי.

#### **דוגמאות:**
1.  **מיון רשימה**: אלגוריתמים כגון מיון מיזוג (merge sort) או מיון מהיר (quick sort) פועלים בזמן \(O(n \log n)\), הנחשב לזמן פולינומי.
2.  **מציאת מסלול קצר ביותר בגרף**: אלגוריתם דייקסטרה (Dijkstra) פועל בזמן \(O(n^2)\) או \(O(n \log n)\) בהתאם למימוש, וגם זהו זמן פולינומי.

#### **מאפיינים:**
*   אלגוריתמים הפועלים בזמן פולינומי נחשבים ל**יעילים** ו**ישימים בפועל**.
*   בעיות שניתן לפתור בזמן פולינומי שייכות ל**מחלקה P**.

---

### **זמן אקספוננציאלי**

**זמן אקספוננציאלי** הוא זמן הריצה של אלגוריתם, אשר גדל באופן אקספוננציאלי כתלות בגודל הקלט. אם זמן הריצה ניתן לביטוי כ-\(O(k^n)\), כאשר \(n\) הוא גודל הקלט ו-\(k\) הוא קבוע, אז אלגוריתם כזה פועל בזמן אקספוננציאלי.

#### **דוגמאות:**
1.  **בעיית הסוכן הנוסע (Travelling Salesperson Problem - TSP)**: פתרון בשיטת הסריקה המלאה של כל המסלולים האפשריים דורש זמן \(O(n!)\), שהוא גרוע יותר מזמן אקספוננציאלי.
2.  **סריקת כל תתי-הקבוצות**: אלגוריתם הבודק את כל תתי-הקבוצות האפשריות של קבוצה בגודל \(n\) פועל בזמן \(O(2^n)\).

#### **מאפיינים:**
*   אלגוריתמים הפועלים בזמן אקספוננציאלי נחשבים ל**לא יעילים** עבור קלטים גדולים, מכיוון שזמן הריצה הופך לגדול באופן בלתי מעשי אפילו עבור \(n\) קטן יחסית.
*   בעיות שניתן לפתור רק בזמן אקספוננציאלי שייכות לרוב למחלקות **NP-קשות** או **NP-שלמות**.

---

### **השוואת זמן פולינומי וזמן אקספוננציאלי**

| **מאפיין**           | **זמן פולינומי**                    | **זמן אקספוננציאלי**                |
| -------------------- | ----------------------------------- | ------------------------------------ |
| **גידול זמן הריצה**  | איטי (למשל, \(n^2\), \(n^3\))        | מהיר (למשל, \(2^n\), \(3^n\))        |
| **דוגמאות לבעיות**   | מיון, מציאת מסלול קצר ביותר        | בעיית הסוכן הנוסע, סריקת תתי-קבוצות |
| **יישימות בפועל**    | יעיל עבור נתונים גדולים            | לא ישים עבור נתונים גדולים         |
| **מחלקה סיבוכיות** | P                                   | NP-קשות, NP-שלמות                   |

---

### **מדוע זה חשוב?**

1.  **זמן פולינומי**:
    *   אלגוריתמים הפועלים בזמן פולינומי נחשבים ל**ישימים בפועל**, מכיוון שהם יכולים לעבד כמויות גדולות של נתונים בזמן סביר.
    *   בעיות השייכות למחלקה **P** (פתירות בזמן פולינומי) מהוות בסיס ליישומים רבים במדעי המחשב, כגון עיבוד נתונים, רשתות, קריפטוגרפיה ובינה מלאכותית.

2.  **זמן אקספוננציאלי**:
    *   אלגוריתמים הפועלים בזמן אקספוננציאלי הופכים ל**בלתי ישימים בפועל** אפילו עבור קלטים קטנים יחסית. לדוגמה, עבור \(n = 100\), \(2^{100}\) כבר עולה על מספר האטומים ביקום הנראה.
    *   בעיות שניתן לפתור רק בזמן אקספוננציאלי דורשות לרוב שימוש ב**שיטות קירוב**, **היוריסטיקות** או **חישובים מקביליים**.

### **דוגמה להבנה**

דמיינו שיש לכם בעיה ואתם רוצים לפתור אותה עבור \(n = 10\) ו-\(n = 100\):

*   **זמן פולינומי (\(n^2\))**:
    *   עבור \(n = 10\): \(10^2 = 100\) פעולות.
    *   עבור \(n = 100\): \(100^2 = 10,000\) פעולות.

*   **זמן אקספוננציאלי (\(2^n\))**:
    *   עבור \(n = 10\): \(2^{10} = 1,024\) פעולות.
    *   עבור \(n = 100\): \(2^{100} \approx 1.26 \times 10^{30}\) פעולות.

כפי שניתן לראות, עבור \(n = 100\), אלגוריתם פולינומי יבצע 10,000 פעולות, שזה בהחלט אפשרי, בעוד אלגוריתם אקספוננציאלי ידרוש כ-\(1.26 \times 10^{30}\) פעולות, שזה למעשה בלתי אפשרי.

לצורך בניית גרפים הממחישים את ההבדל בין זמן פולינומי לזמן אקספוננציאלי, ניתן להשתמש בפונקציות מתמטיות שונות. להלן דוגמאות לפונקציות שניתן להשתמש בהן לצורך הדמיה:

### **פונקציות פולינומיות**
1.  **פונקציה לינארית**:
    \( f(n) = n \)
    דוגמה: זמן ריצה של אלגוריתם המעבד כל איבר פעם אחת.

2.  **פונקציה ריבועית**:
    \( f(n) = n^2 \)
    דוגמה: זמן ריצה של אלגוריתם עם לולאות מקוננות, לדוגמה, מיון בועות (bubble sort).

3.  **פונקציה קובית**:
    \( f(n) = n^3 \)
    דוגמה: זמן ריצה של אלגוריתם המעבד נתונים תלת-ממדיים.

4.  **פונקציה לוגריתמית**:
    \( f(n) = \log n \)
    דוגמה: זמן ריצה של חיפוש בינארי.

5.  **פונקציה לינארית-לוגריתמית (או לוג-לינארית)**:
    \( f(n) = n \log n \)
    דוגמה: זמן ריצה של מיון מהיר או מיון מיזוג.

---

### **פונקציות אקספוננציאליות**
1.  **פונקציה אקספוננציאלית**:
    \( f(n) = 2^n \)
    דוגמה: זמן ריצה של אלגוריתם הסורק את כל תתי-הקבוצות של קבוצה.

2.  **פונקציית העצרת**:
    \( f(n) = n! \)
    דוגמה: זמן ריצה של אלגוריתם הסורק את כל התמורות (לדוגמה, בעיית הסוכן הנוסע).

3.  **פונקציה אקספוננציאלית עם בסיס אחר**:
    \( f(n) = 3^n \)
    דוגמה: זמן ריצה של אלגוריתם החוקר את כל הקומבינציות האפשריות.

### **דוגמת קוד לבניית גרפים (Python, Matplotlib)**

```python
import matplotlib.pyplot as plt
import numpy as np

# Range of n values
n = np.linspace(1, 20, 100)

# Polynomial functions
linear = n
quadratic = n**2
cubic = n**3
logarithmic = np.log(n)
nlogn = n * np.log(n)

# Exponential functions
exponential = 2**n
factorial = [np.math.factorial(int(i)) for i in n]  # Factorial is defined only for integers

# Plotting the graphs
plt.figure(figsize=(10, 6))

# Polynomial functions
plt.plot(n, linear, label='Linear: $f(n) = n$')
plt.plot(n, quadratic, label='Quadratic: $f(n) = n^2$')
plt.plot(n, cubic, label='Cubic: $f(n) = n^3$')
plt.plot(n, logarithmic, label='Logarithmic: $f(n) = \log n$')
plt.plot(n, nlogn, label='Linearithmic: $f(n) = n \log n$')

# Exponential functions
plt.plot(n, exponential, label='Exponential: $f(n) = 2^n$')
plt.plot(n, factorial, label='Factorial: $f(n) = n!$')

# Graph settings
plt.yscale('log')  # Logarithmic scale for convenience
plt.xlabel('Input size (n)')
plt.ylabel('Time complexity')
plt.title('Comparison of Polynomial and Exponential Time Complexity')
plt.legend()
plt.grid(True)
plt.show()
```
