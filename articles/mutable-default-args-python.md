
# 🐍 טעות נפוצה בפייתון: ערכים משתנים כברירת מחדל

🎯 **מטרה:**
להבין באג בפייתון שלא זורק חריגה — אבל כן שובר את הלוגיקה של התוכנית שלנו.

---

## 📍 תרחיש:

נניח שיש לנו פונקציה לרישום אירועים (לוג), שאפשר להעביר לה מטא-דאטה (מידע נלווה). אם לא נשלח מידע כזה — היא תשתמש בערך ברירת מחדל:

```python
def log_event(event, metadata={}):
    metadata["event"] = event
    print(metadata)
```

נראה תקין, נכון? אבל בוא נריץ כמה קריאות לפונקציה הזו:

```python
log_event("start")
log_event("stop")
log_event("error", {"code": 500})
log_event("retry")
```

## 👀 הפלט המפתיע:

```python
{'event': 'start'}
{'event': 'stop'}
{'code': 500, 'event': 'error'}
{'code': 500, 'event': 'retry'}
```

### ❗ רגע, למה `code: 500` מופיע בקריאה האחרונה?!

הרי לא שלחנו אותו בפעם הרביעית בכלל...

---

## 🔍 מה בעצם קרה כאן?

### 🧩 הסבר:

הבעיה נעוצה בשימוש **בערך משתנה (`dict`) כברירת מחדל**.
בפייתון, ערכי ברירת מחדל לפונקציות נוצרים **פעם אחת בלבד** — בזמן הגדרת הפונקציה, **ולא** בכל קריאה שלה.

כלומר, `metadata={}` נוצר פעם אחת ומשמש שוב ושוב בכל קריאה לפונקציה — כל עוד לא מעבירים פרמטר חדש במפורש.

### 🔬 בדיקה פשוטה:

```python
def f(d={}):
    print(id(d))
    d["x"] = 1
    print(d)

f()
f()
```

🔁 הפלט יראה ששני הקריאות השתמשו באותו מזהה (`id`) — כלומר באותו האובייקט בדיוק.

---

## ✅ הפתרון הפייתוני:

הדרך הנכונה להתמודד עם זה היא להשתמש ב-`None` כברירת מחדל, ואז לאתחל את המילון בתוך גוף הפונקציה:

```python
def log_event(event, metadata=None):
    if metadata is None:
        metadata = {}
    metadata["event"] = event
    print(metadata)
```

👆 ככה כל קריאה מקבלת מילון חדש, נקי, אם לא נשלח אחד ידנית.

---

## 🧠 איפה עוד זה קורה?

אותה טעות בדיוק יכולה לקרות גם עם מבנים משתנים אחרים:

* רשימות: `items=[]`
* קבוצות: `visited=set()`
* מופעים של מחלקות: `obj=MyClass()`

---

## 📌 סיכום חשוב:

שימוש בערכים משתנים (`dict`, `list`, `set`) כברירת מחדל הוא אחת מהטעויות הנפוצות בפייתון.

* ❌ לא זורקת חריגה
* ❌ קשה לזיהוי
* ❌ יכולה לגרום לבאגים מתעתעים

🛡 תמיד השתמשו ב-`None` כברירת מחדל, ואתחלו את המבנה **בתוך הפונקציה**.
