## יצירת 'טביעת אצבע' של סגנון הקלדה: זיהוי משתמש באמצעות למידת מכונה ו-Scikit-learn (דוגמה מעשית)

במאמר זה אראה כיצד ליצור מערכת פשוטה לזיהוי משתמש המבוססת על ניתוח סגנון ההקלדה הייחודי שלו (סגנון הקלדה). אנו נשתמש בספריית למידת המכונה Scikit-learn ב-Python. נעבור על כל השלבים הדרושים ליצירת אב טיפוס פועל, אף עם פישוטים מסוימים.

**1. הדמיית איסוף נתונים: יצירת לחיצות מקשים "וירטואליות"**

בתרחיש אידיאלי, השלב הראשון יהיה התחברות למקלדת המשתמש ותיעוד כל לחיצה ושחרור מקש. אך, למרבה הצער, במסגרת מאמר זה אין לי אפשרות ליצור מערכת שתיירט את קלט המקלדת שלכם. לכן אני נאלץ **לדמות** תהליך זה, וליצור נתונים אקראיים המייצגים את משך לחיצות המקשים. דמיינו שכבר ביצעתי איסוף נתונים, ועתה יש ברשותי קבוצת ערכים מסוימת בה אשתמש.

לשם כך, פיתחתי את הפונקציה `generate_simulated_data()`:

```python
import numpy as np
import random

def generate_simulated_data(num_users=2, num_sessions=5, session_length=50):
    """מייצר הדמיית נתונים על משך לחיצות מקשים."""
    data = []
    labels = []
    for user_id in range(num_users):
        for session_id in range(num_sessions):
            mean_duration = random.randint(80, 150)
            std_dev = random.randint(10, 20)
            session_data = np.random.normal(mean_duration, std_dev, session_length)
            data.append(session_data)
            labels.append(f"user{user_id + 1}")
    return data, labels

data, labels = generate_simulated_data()
```

פונקציה זו מייצרת נתונים עבור מספר נתון של משתמשים (`num_users`), כאשר לכל משתמש ישנן מספר "סשנים" של הקלדה (`num_sessions`). כל סשן מורכב מקבוצת ערכים אקראיים המייצגים את משך לחיצת המקש במילישניות. אני משתמש בהתפלגות נורמלית (`np.random.normal()`) כדי לדמות את השונות הטבעית במהירות ההקלדה. לכל משתמש מוקצה סגנון הקלדה "ממוצע" ייחודי (ערך אקראי של `mean_duration`). זהו, כמובן, מודל מפשט ביותר, אך הוא מאפשר להדגים את השלבים העיקריים ביצירת המערכת.

**הנחת יסוד מרכזית:** ביישום אמיתי, עליכם *להחליף* פונקציה זו בקוד שאכן טוען נתונים שנאספו ממשתמשים אמיתיים, תוך שימוש ב-JavaScript, ואף בקצה אחורי (backend) לאחסון המידע.

**2. חילוץ מאפיינים: פישוט התמונה**

כפי שכבר ציינו, קיימים מאפיינים רבים שניתן לחלץ מנתונים על לחיצות מקשים. לשם הפשטות, בדוגמה זו אשתמש במאפיין אחד בלבד: **משך לחיצת המקש הממוצע בסשן**.

```python
X = np.array([np.mean(session) for session in data]).reshape(-1, 1)
y = np.array(labels)
```

קוד זה מחשב את הערך הממוצע עבור כל סשן וממיר את הנתונים לפורמט שמובן ל-Scikit-learn. השימוש ב-`reshape(-1, 1)` נועד להמיר מערך חד-ממדי למערך דו-ממדי, כפי שנדרש על ידי רוב אלגוריתמי למידת המכונה ב-Scikit-learn. נשאיר מאפיינים מורכבים יותר לניסויים עתידיים.

**3. הכנת נתונים: מתכוננים לאימון**

לפני אימון המודל, יש להכין את הנתונים. הדבר כולל:

*   **פיצול נתונים:** פיצול הנתונים לערכת אימון ולערכת בדיקה. ערכת האימון משמשת לאימון המודל, וערכת הבדיקה - להערכת דיוקו.
*   **נרמול נתונים:** קנה מידה (סקלינג) לנתונים, כך שלכל המאפיינים יהיה טווח ערכים זהה. הדבר מסייע בשיפור ביצועי המודל.

```python
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler

X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42)

scaler = StandardScaler()
X_train = scaler.fit_transform(X_train)
X_test = scaler.transform(X_test)
```

אני משתמש ב-`train_test_split` לפיצול הנתונים ל-70% ערכת אימון ו-30% ערכת בדיקה. `StandardScaler` משמש לנרמול הנתונים. `fit_transform` מוחל על ערכת האימון כדי לחשב את פרמטרי הסקלינג, ו-`transform` מוחל על ערכת הבדיקה, תוך שימוש בפרמטרים אלה. שלבים אלה חשובים ביותר כדי שהמודל יפעל כראוי.

**4. אימון מודל: יצירת "טביעת האצבע" של סגנון ההקלדה**

עתה אוכל לאמן את מודל למידת המכונה. אשתמש ב-`RandomForestClassifier`, כפי שציינתי קודם.

```python
from sklearn.ensemble import RandomForestClassifier

model = RandomForestClassifier(n_estimators=100, random_state=42)
model.fit(X_train, y_train)
```

אני יוצר מופע של `RandomForestClassifier` עם 100 עצים (`n_estimators=100`) ומאמן אותו על ערכת האימון. פעולה זו בונה את "טביעת האצבע" שלנו של סגנון ההקלדה, המבוססת על הנתונים שסופקו.

**5. הערכת מודל: מדידת דיוק "טביעת האצבע"**

לאחר אימון המודל, יש להעריך את דיוקו על ערכת הבדיקה.

```python
from sklearn.metrics import accuracy_score

predictions = model.predict(X_test)
accuracy = accuracy_score(y_test, predictions)
print(f"דיוק המודל: {accuracy}")
```

אני משתמש ב-`accuracy_score` לחישוב דיוק המודל, כלומר, שיעור הדוגמאות שסווגו נכון. תוצאה טיפוסית:

```
Точность модели: 0.75
```

(הערך עשוי להיות שונה במקצת עקב אקראיות בנתונים)

**6. שימוש במודל: זיהוי משתמש חדש**

לבסוף, אוכל להשתמש במודל המאומן כדי לזהות משתמש חדש.

```python
def predict_user(session_data):
    """חוזה את המשתמש על בסיס נתוני הסשן."""
    mean_duration = np.mean(session_data)
    scaled_data = scaler.transform([[mean_duration]])
    prediction = model.predict(scaled_data)[0]
    return prediction

new_session_data = np.random.normal(120, 15, 50)
predicted_user = predict_user(new_session_data)
print(f"משתמש חזוי: {predicted_user}")
```

הפונקציה `predict_user` מקבלת נתונים מסשן הקלדה חדש, מחשבת את משך הלחיצות הממוצע, מבצעת סקלינג (קנה מידה) לנתונים ומשתמשת במודל המאומן לחיזוי המשתמש. `scaler.transform([[mean_duration]])` מבצע סקלינג לנתונים, תוך שימוש בפרמטרי הסקלינג שחושבו על ערכת האימון. דוגמה לפלט:

```
Предсказанный пользователь: user2
```

(התוצאה עשויה להשתנות בהתאם לאקראיות)
