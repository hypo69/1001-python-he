**שאלה 29.** כיצד משפיע האופרטור `break` על זרימת הביצוע בלולאות מקוננות?

- א. הוא יוצא רק מהלולאה הפנימית ביותר וממשיך בביצוע מהרמה הבאה של הלולאה.
- ב. הוא מסיים מיידית את כל הלולאות, ללא קשר לרמת הקינון שלהן.
- ג. הוא משעה זמנית את ביצוע כל הלולאות ומחדש את הביצוע מהלולאה החיצונית ביותר.
- ד. הוא גורם לביצוע מיידי של בלוק ה-`else` המשויך ללולאה הפנימית ביותר.

**תשובה נכונה: A**

**הסבר:**

האופרטור `break` ב-Python משמש ליציאה מיידית מהלולאה שבה הוא ממוקם. כאשר `break` נתקל בתוך לולאות מקוננות, הוא משפיע רק על הלולאה שבה הוא נמצא באופן ישיר.

*   **אפשרות A** נכונה: האופרטור `break` מסיים את ביצוע הלולאה הפנימית בלבד, ומאפשר ללולאות החיצוניות להמשיך בפעולתן.

*   **אפשרות B** אינה נכונה: `break` אינו מסיים את כל הלולאות, הוא יוצא רק מהנוכחית.
*   **אפשרות C** אינה נכונה: `break` אינו משעה את הביצוע, הוא קוטע את הלולאה.
*   **אפשרות D** אינה נכונה: `break` אינו גורם לביצוע בלוק ה-`else`. בלוק ה-`else` מבוצע רק אם הלולאה לא נקטעה.

**כיצד פועל `break` בלולאות מקוננות:**

1.  כאשר מפרש Python נתקל ב-`break` בתוך לולאה מקוננת, הביצוע של הלולאה הפנימית *הזאת בלבד* מופסק מיידית.
2.  השליטה מועברת לפקודה הבאה *אחרי* הלולאה הפנימית.
3.  הלולאות החיצוניות ממשיכות בביצוען.

**דוגמה:**

```python
for i in range(3):
  print(f"Внешний цикл: i = {i}")
  for j in range(3):
    if j == 1:
       print("Выход из внутреннего цикла.")
       break
    print(f"  Внутренний цикл: j = {j}")
  print("После внутреннего цикла")
# פלט:
# לולאה חיצונית: i = 0
#   לולאה פנימית: j = 0
# יציאה מהלולאה הפנימית.
# אחרי הלולאה הפנימית
# לולאה חיצונית: i = 1
#   לולאה פנימית: j = 0
# יציאה מהלולאה הפנימית.
# אחרי הלולאה הפנימית
# לולאה חיצונית: i = 2
#   לולאה פנימית: j = 0
# יציאה מהלולאה הפנימית.
# אחרי הלולאה הפנימית
```

**כתוצאה מכך:**

*   הלולאה החיצונית (`i`) מתבצעת 3 פעמים.
*   הלולאה הפנימית (`j`) מודפסת עד שהיא מגיעה לערך 1, לאחר מכן מתרחשת יציאה מהלולאה הפנימית, והקוד ממשיך לביצוע הלולאה החיצונית.

לפיכך, **אפשרות A** היא התשובה הנכונה.
