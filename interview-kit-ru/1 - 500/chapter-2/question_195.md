**שאלה 195.** מהי מטרת הדקורטור `@classmethod` בפייתון?

A. הוא הופך שיטה לשיטה סטטית.
B. הוא משמש ליצירת מופעים של המחלקה.
C. הוא הופך שיטה לשיטת מחלקה, אשר מקבלת את המחלקה כארגומנט הראשון (`cls`).
D. הוא מגביל את הגישה לשיטה רק מתוך מופע של המחלקה.

**תשובה נכונה: C**

**הסבר:**

הדקורטור `@classmethod` בפייתון משמש להגדרת שיטות המקושרות למחלקה עצמה, ולא למופע ספציפי של המחלקה. שיטות כאלו מקבלות את המחלקה כארגומנט הראשון שלהן, המכונה בדרך כלל `cls`.

*   **אפשרות A** שגויה: הדקורטור `@staticmethod` הופך שיטה לשיטה סטטית.
*   **אפשרות B** שגויה: הקונסטרוקטור של המחלקה `__init__()` משמש ליצירת מופעים של המחלקה.
*   **אפשרות C** נכונה: הדקורטור `@classmethod` מגדיר שיטת מחלקה ומעביר אליה הפניה למחלקה (`cls`).
*   **אפשרות D** שגויה: שיטת מופע משמשת רק בתוך אובייקט.

**כיצד פועל `@classmethod`:**

1.  הדקורטור `@classmethod` ממוקם מעל השיטה.
2.  שיטת המחלקה מקבלת באופן אוטומטי הפניה למחלקה (המכונה בדרך כלל `cls`) כארגומנט הראשון.
3.  שיטת המחלקה יכולה לשנות את מצב המחלקה (תכונות המחלקה).
4.  שיטת המחלקה יכולה להיקרא הן דרך המחלקה והן דרך מופע של המחלקה.

**דוגמה:**

```python
class MyClass:
    count: int = 0

    def __init__(self):
        MyClass.count += 1

    @classmethod
    def get_count(cls):
       """Class method returning the number of instances."""
       return cls.count


obj1: MyClass = MyClass()
obj2: MyClass = MyClass()

print(MyClass.get_count()) # Output: 2
print(obj1.get_count()) # Output: 2
```

**כתוצאה מכך:**

*   השיטה `get_count` מסומנת ב-`@classmethod` ומקבלת הפניה למחלקה (`cls`).
*   השיטה `get_count` יכולה להיקרא הן דרך המחלקה והן דרך המופע שלה.

לפיכך, **אפשרות C** היא הנכונה.
