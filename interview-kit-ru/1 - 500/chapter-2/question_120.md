**שאלה 120.** ב-Python, כיצד מבנה `try-except` מטפל בזרימת הביצוע כאשר מתרחשת שגיאה?

- א. היא עוצרת את התוכנית באופן מיידי בעת התרחשות שגיאה, אלא אם כן השגיאה נלכדת בבלוק `except`.
- ב. היא מנתבת מחדש את זרימת הביצוע לבלוק `except`, מטפלת בשגיאה ללא עצירת התוכנית, אם השגיאה תואמת לחריגה המצוינת בבלוק `except`.
- ג. היא מתעלמת מכל השגיאות, ומאפשרת לתוכנית להמשיך את הביצוע ללא הפרעה.
- ד. היא מציגה אוטומטית הודעות שגיאה במסוף, ולאחר מכן ממשיכה לעבוד מהשורה הבאה של הקוד.

**התשובה הנכונה: ב**

**הסבר:**

מבנה `try-except` ב-Python הוא המנגנון העיקרי לטיפול בחריגות. הוא מאפשר ללכוד ולטפל בשגיאות שעשויות להתרחש במהלך ביצוע התוכנית, ובכך למנוע את סיומה הבלתי צפוי (קריסתה).

*   **אפשרות א** אינה נכונה: התוכנית אינה עוצרת באופן מיידי בעת חריגה אם היא נלכדת על ידי `except`.

*   **אפשרות ב** נכונה: חריגות בבלוק `try` מנתבות את זרימת הביצוע לבלוק `except`, שם מתבצע טיפול בשגיאות.

*   **אפשרות ג** אינה נכונה: `try-except` אינו מתעלם משגיאות.

*   **אפשרות ד** אינה נכונה: `try-except` אינו מציג אוטומטית שגיאות במסוף; דבר זה נעשה במפורש בבלוק except.

**כיצד פועל `try-except`:**

1.  **בלוק `try`:** קוד שבו עשויה להתרחש חריגה ממוקם בבלוק `try`.
2.  **בלוק `except`:** אם מתרחשת חריגה בתוך בלוק `try`, Python מחפש בלוק `except` מתאים שיכול לטפל בחריגה שהתרחשה.
3.  **טיפול בחריגה:** אם נמצא בלוק `except` מתאים, הקוד שבתוכו מתבצע, המאפשר לטפל בשגיאה ולהמשיך את עבודת התוכנית (או לסיים אותה באופן תקין יותר).
4.  **חריגות שלא נלכדו:** אם חריגה אינה נלכדת, ביצוע התוכנית יופסק, ותודפס הודעת שגיאה.

**דוגמה:**

```python
def divide(x: int, y: int) -> None:
    """מבצע חלוקה של x ב-y, לוכד את החריגה ZeroDivisionError."""
    try:
        result: float = x / y # עלולה להתרחש חריגה ZeroDivisionError
        print(f"תוצאת החלוקה: {result}")
    except ZeroDivisionError: # לכידת חריגה ZeroDivisionError
        print("שגיאה: חלוקה באפס!")

divide(10, 2)   # פלט: תוצאת החלוקה: 5.0
divide(10, 0)  # פלט: שגיאה: חלוקה באפס!
```
**בתוצאה:**

*   בעת חלוקה `10 / 2` לא תתרחש שגיאה, והתוכנית תציג את התוצאה.
*   בעת חלוקה `10 / 0` תתרחש חריגה `ZeroDivisionError`, שתילכד ותודפס הודעת שגיאה.
*   התוכנית לא תסתיים כתוצאה מכך.

לפיכך, **אפשרות ב** היא התשובה הנכונה.
