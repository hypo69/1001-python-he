**שאלה 106.** מה תפקידו של אופרטור `pass` בשפת פייתון?

- א. משמש כ"ממלא מקום", מבטיח את התקינות התחבירית של התוכנית כאשר אין צורך בפעולה כלשהי, אך מבחינה תחבירית נדרש אופרטור.
- ב. מעביר את השליטה לשורת הקוד הבאה מחוץ לפונקציה או ללולאה הנוכחית.
- ג. גורם לעיכוב בלולאה, מאלץ את התוכנית להמתין זמן מסוים לפני המשך הביצוע.
- ד. בודק שגיאות בבלוק הקוד שקדם לו ומעביר את השליטה למטפל השגיאות אם נתגלה חריג.

**התשובה הנכונה: א**

**הסבר:**

האופרטור `pass` בפייתון הוא אופרטור ריק שאינו מבצע שום פעולה. הוא משמש כ"ממלא מקום" (placeholder) בחלקי קוד בהם נדרש ביטוי כלשהו מבחינה תחבירית, אך אין צורך בפעולה כלשהי בנקודה זו של התוכנית. במקום `pass` ניתן להשתמש גם ב*שלוש נקודות* (`...`), המבצעות את אותה הפעולה, אך עשויות להיחשב תמציתיות ו"נקיות" יותר.

*   **אפשרות א** נכונה: `pass` משמש כ"ממלא מקום" כדי שהקוד יישאר תקני מבחינה תחבירית, גם כאשר אין פעולות לבצע.
*   **אפשרות ב** אינה נכונה: ליציאה מפונקציה משתמשים ב-`return`, ומיציאה מלולאה ב-`break`, בעוד ש-`pass` אינו עושה דבר.
*   **אפשרות ג** אינה נכונה: `pass` אינו יוצר עיכוב.
*   **אפשרות ד** אינה נכונה: לבדיקת שגיאות משתמשים ב-`try...except`.

**מתי משתמשים ב-`pass` (או ב-`...`):**

1.  **בלוקי קוד ריקים:** כאשר יש להגדיר בלוק ריק עבור `if`, `for`, `while`, `try`, `except` או גוף פונקציה.
2.  **שלד (Stub):** כאשר יש להשאיר מקום לקוד עתידי שטרם נכתב, אך התחביר דורש נוכחות של גוף.
3. **ממשקים (Interfaces):** במחלקות אבסטרקטיות, להגדרת מתודות חובה שאין להן מימוש.

**דוגמה:**

```python
# פונקציה ריקה (עם pass)
def my_function() -> None:
  pass

# פונקציה ריקה (עם ... - שלוש נקודות)
def another_function() -> None:
  ...


# מחלקה ריקה
class MyClass:
    def method(self):
        pass # שלד (Stub)

# בלוק if ריק
if 5 > 10:
    pass
else:
    print("5 לא גדול מ-10")

# בלוק except ריק
try:
    raise ValueError("שגיאה")
except ValueError:
    ...  # לא עושים דבר במקרה של שגיאה
```
**תוצאה:**
*   `pass` (או `...`) אינו מבצע שום פעולה, אלא רק מבטיח את התקינות התחבירית של הקוד.
*   בלוקי קוד יבוצעו ללא שום פעולה אם הם מכילים `pass` (או `...`).

לפיכך, **אפשרות א** היא הנכונה.
