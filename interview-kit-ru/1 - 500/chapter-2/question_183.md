**שאלה 183.** מהי "טיפוסיות ברווז" (duck typing) ב-Python?

א. זוהי שיטה להגדרת טיפוסים של משתנים.
ב. זוהי שיטה לתיקון אוטומטי של שגיאות בקוד.
ג. זהו רעיון לפיו טיפוס האובייקט אינו חשוב, העיקר שלאובייקט יהיו המתודות והתכונות הדרושות.
ד. זוהי שיטה להגדרת היררכיית מחלקות.

**תשובה נכונה: ג**

**הסבר:**

"טיפוסיות ברווז" (duck typing) הוא רעיון בשפות בעלות טיפוסיות דינמית, כגון Python, שבה טיפוס האובייקט אינו חשוב. העיקר שהאובייקט תומך במתודות ובתכונות הדרושות. כלומר, "אם משהו הולך כמו ברווז ומגעגע כמו ברווז, אז זה ברווז".

*   **אפשרות א** אינה נכונה: `duck typing` אינו קשור להגדרת טיפוסים של משתנים, אלא לשימוש בהם.
*   **אפשרות ב** אינה נכונה: `duck typing` אינו מתקן שגיאות בקוד.
*   **אפשרות ג** נכונה: `duck typing` בודק רק נוכחות של מתודות ותכונות, ולא את הטיפוס.
*   **אפשרות ד** אינה נכונה: `duck typing` אינו קשור להיררכיית מחלקות.

**כיצד פועלת טיפוסיות ברווז:**

1.  Python אינה בודקת את טיפוס האובייקט לפני ביצוע מתודה או גישה לתכונה.
2.  במקום זאת, Python בודקת את נוכחותן של המתודות והתכונות ההכרחיות, שבהן נעשה שימוש בקוד.
3.  אם לאובייקט יש את המתודות והתכונות ההכרחיות, הוא נחשב תואם לקוד.

**דוגמה:**

```python
class Duck:
    def quack(self):
        print("Quack!")

class Cat:
  def quack(self):
     print("Meow!")

def make_quack(animal):
    animal.quack() # נבדקת רק נוכחות המתודה quack

duck = Duck()
cat = Cat()
make_quack(duck) # פלט: Quack!
make_quack(cat) # פלט: Meow!
```

**כתוצאה מכך:**

* הפונקציה `make_quack()` עובדת עם אובייקטים שונים, כל עוד יש להם מתודה `quack()`.

לפיכך, **אפשרות ג** היא התשובה הנכונה.
