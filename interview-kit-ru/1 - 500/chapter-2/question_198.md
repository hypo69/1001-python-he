**שאלה 198.** מהי "אינקפסולציה" בתכנות מונחה עצמים?

א. זוהי היכולת של מחלקה לרשת מתודות ותכונות ממחלקה אחרת.
ב. זוהי היכולת לעבד אובייקטים ממחלקות שונות באמצעות ממשק משותף.
ג. זהו עקרון הסתרת המימוש הפנימי של אובייקט והספקת גישה אליו באמצעות ממשק ציבורי.
ד. זוהי היכולת להשתמש באותה מתודה עבור סוגי נתונים שונים.

**תשובה נכונה: C**

**הסבר:**

אינקפסולציה היא אחד מעקרונות היסוד של תכנות מונחה עצמים, והיא מתבטאת באיחוד נתונים (תכונות) ומתודות הפועלות על נתונים אלה, לתוך אובייקט אחד (מחלקה). במקביל, הגישה לחלקים מסוימים של האובייקט מוגבלת, תוך הספקת גישה אליהם רק דרך ממשק המוגדר במיוחד לצורך זה.

*   **אפשרות א'** אינה נכונה: זוהי הגדרה של ירושה.
*   **אפשרות ב'** אינה נכונה: זוהי הגדרה של פולימורפיזם.
*   **אפשרות ג'** נכונה: אינקפסולציה משמעותה הסתרת מימוש והספקת גישה באמצעות ממשק ציבורי.
*   **אפשרות ד'** אינה נכונה: זהו תיאור של פולימורפיזם.

**כיצד פועלת אינקפסולציה:**

1.  תכונות (נתונים) של האובייקט יכולות להיות מוכרזות כ"מוגנות" או "פרטיות", מה שמגביל את הגישה אליהן מחוץ למחלקה.
2.  לצורך עבודה עם תכונות כאלה, מסופקות מתודות מיוחדות, המכונות "גטרים" ו"סטררים", הקובעות כיצד ומתי התכונות יכולות להשתנות.
3.  זה מאפשר לשלוט בגישה לנתונים ומבטיח מבנה קוד אמין וגמיש יותר.

**דוגמה:**

```python
class BankAccount:
    def __init__(self, balance: int):
        self._balance = balance # תכונה מוגנת

    def get_balance(self):
        """מתודה לקבלת יתרה"""
        return self._balance
    
    def deposit(self, amount: int):
        """מתודה להפקדת יתרה"""
       if amount > 0:
          self._balance += amount
       else:
         print("סכום הפקדה שגוי")

    def withdraw(self, amount: int):
        """מתודה למשיכת כסף"""
        if amount > 0 and amount <= self._balance:
           self._balance -= amount
        else:
          print("אין מספיק אמצעים")
    
account = BankAccount(1000)
print(f"יתרה: {account.get_balance()}") # פלט: יתרה: 1000
account.deposit(500)
print(f"יתרה לאחר הפקדה: {account.get_balance()}") # פלט: יתרה לאחר הפקדה: 1500
account.withdraw(100)
print(f"יתרה לאחר משיכה: {account.get_balance()}") # פלט: יתרה לאחר משיכה: 1400
# account._balance = -10000 # זוהי אינה דרך נכונה לעשות זאת
```

**בתוצאה:**
*   במחלקה `BankAccount` התכונה `_balance` היא מוגנת ולא ניתן לגשת אליה ישירות, אלא רק באמצעות מתודות.
*   זה מגן על נתוני האובייקט מפני שינוי ישיר ומאפשר לשלוט בהם באמצעות מתודות שתוכננו במיוחד.

לפיכך, **אפשרות C** היא התשובה הנכונה.
