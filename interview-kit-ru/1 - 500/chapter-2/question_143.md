### `question_143.md`

**שאלה 43.** מהי מטרת השימוש ב-`yield` בפייתון?

א. לסיום ביצוע פונקציה.
ב. לשמירת מצב הפונקציה והחזרת ערך.
ג. ליצירת לולאה אינסופית.
ד. לפלט מיידי של תוצאות לקונסול.

**תשובה נכונה: ב**

**הסבר:**

מילת המפתח `yield` בפייתון משמשת ליצירת גנרטורים (generators). גנרטורים הם פונקציות שיכולות להחזיר ערכים בזה אחר זה, תוך שמירת מצבן בין קריאות.

*   **אפשרות א'** אינה נכונה: לסיום פונקציה משתמשים ב-`return`.
*   **אפשרות ב'** נכונה: מילת המפתח `yield` מאפשרת לפונקציה להשהות את ביצועה ולהחזיר ערך. בקריאה הבאה, הפונקציה תמשיך את ביצועה מהמקום שבו היא עצרה ב-`yield`, תוך שמירת מצבה.
*   **אפשרות ג'** אינה נכונה: ליצירת לולאה אינסופית משתמשים בדרך כלל ב-`while True`.
*   **אפשרות ד'** אינה נכונה: לפלט תוצאות לקונסול משתמשים ב-`print()`.

**כיצד פועל `yield`:**

1.  כאשר פונקציה מגיעה לאופרטור `yield`, היא משהה את ביצועה.
2.  בשלב זה מוחזר הערך שצוין לאחר `yield`.
3.  כאשר פונקציית הגנרטור נקראת בפעם הבאה, הביצוע ממשיך מהמקום שבו הוא עצר, והערך הבא מוחזר.
4.  כל המשתנים המקומיים ומצב הפונקציה נשמרים בין הקריאות.
5.  לאחר שכל הערכים נמסרו, האיטרציה על הגנרטור מסתיימת.

**דוגמה:**

```python
def my_generator(n: int):
    for i in range(n):
        yield i * 2
        print(f"אחרי yield: {i}") # לאחר קבלת הערך
gen = my_generator(3)

for value in gen:
    print(f"התקבל ערך: {value}")

# פלט:
# התקבל ערך: 0
# אחרי yield: 0
# התקבל ערך: 2
# אחרי yield: 1
# התקבל ערך: 4
# אחרי yield: 2
```

**בתוצאה:**

*   הפונקציה `my_generator` משהיתה בכל פעם שהגיעה ל-`yield`, והמשיכה לעבוד מאותו מקום.
*   המשתנה `i` המשיך לשמור את ערכו במהלך האיטרציות.

לפיכך, **אפשרות ב'** היא הנכונה.
