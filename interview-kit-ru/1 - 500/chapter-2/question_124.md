**שאלה 24.** איזו טענה נכונה לגבי לולאת `while` עם סעיף `else`?

- א. סעיף `else` מופעל אם הלולאה לעולם אינה מתחילה לפעול עקב תנאי שקר בתחילתה.
- ב. סעיף `else` מתבצע בסוף כל איטרציה מוצלחת של הלולאה.
- ג. סעיף `else` מתבצע לאחר סיום הלולאה, אך רק אם הלולאה הסתיימה באמצעות `break`.
- ד. סעיף `else` מופעל לאחר סיום הלולאה ורק אם לא הייתה יציאה באמצעות `break`.

**תשובה נכונה: ד**

**הסבר:**

בפייתון, ללולאות `while` יכול להיות בלוק `else` אופציונלי, אשר מתבצע רק לאחר שהלולאה מסתיימת באופן טבעי, כלומר כאשר תנאי הלולאה הופך לשקר, ורק אם הלולאה לא נקטעה על ידי הפקודה `break`.

*   **סעיף א** אינו נכון: בלוק `else` מתבצע לאחר הלולאה, גם אם גוף הלולאה לא הופעל כלל בגלל תנאי התחלה שקרי.
*   **סעיף ב** אינו נכון: `else` מתבצע רק פעם אחת, לאחר סיום הלולאה.
*   **סעיף ג** אינו נכון: בלוק `else` מתבצע רק בעת סיום *רגיל* של הלולאה, ללא `break`.
*   **סעיף ד** נכון: `else` מתבצע לאחר סיום הלולאה ורק אם הלולאה לא נקטעה על ידי `break`.

**כיצד פועל `while...else`:**

1.  לולאת `while` מתבצעת כל עוד התנאי שלה אמיתי.
2.  אם תנאי ה-`while` הופך לשקר והלולאה מסתיימת מבלי להשתמש ב-`break`, הבלוק `else` מתבצע.
3.  אם הלולאה נקטעת על ידי הפקודה `break`, אזי בלוק `else` אינו מתבצע.

**שימושים טיפוסיים ב-`else` בלולאות:**

1.  **סיום מוצלח:** `else` מאפשר לבצע קוד אם הלולאה הסתיימה באופן טבעי (לדוגמה, כל האיברים נסרקו), ולא נקטעה בטרם עת.
2.  **בדיקת ביצוע הלולאה**: שימוש ב-`else` לטיפול במקרה שבו יש לבדוק האם הלולאה התבצעה או לא.

**דוגמה:**
```python
# דוגמת while-else כאשר הלולאה מסתיימת באופן תקין
x: int = 0
while x < 5:
  x += 1
else:
   print("לולאת while הסתיימה באופן תקין") # פלט: לולאת while הסתיימה באופן תקין

# דוגמת while-break
x: int = 0
while x < 10:
    x += 1
    if x == 3:
        break
else:
   print("לולאת while הסתיימה באופן תקין") # בלוק else לא יודפס
```
**התוצאה:**
* בדוגמה הראשונה, לולאת `while` מסתיימת מכיוון שהתנאי `x<5` הפך לשקר, ובלוק `else` מתבצע.
* בדוגמה השנייה, לולאת `while` נקטעת על ידי הפקודה `break` כאשר `x == 3`, ובלוק `else` אינו מתבצע.

לפיכך, **סעיף ד** הוא הנכון.
