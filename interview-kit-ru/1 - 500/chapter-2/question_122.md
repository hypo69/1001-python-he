### `question_122.md`

**שאלה 22.** כיצד פועלת הצהרת `finally` בתוך בלוק `try-except`?

- א.  היא מבצעת את הקוד בתוכה רק במקרה שלא נזרקו חריגות כלשהן בבלוק `try`.
- ב.  היא מתבצעת ללא קשר לשאלה אם נזרק חריג או לא, ואפילו אם נתקלים באופרטור `return`.
- ג.  היא משמשת לזריקת חריגה, אם היא לא נזרקה בבלוק `try`.
- ד.  היא מונעת התפשטות של כל חריגה שנזרקה בבלוק `try` הלאה.

**תשובה נכונה: ב**

**הסבר:**

הצהרת `finally` בבלוק `try-except` מיועדת לביצוע קוד ש*חייב* להתבצע *תמיד*, ללא קשר לשאלה אם התרחשה חריגה בבלוק `try` או לא, או אפילו אם בוצע אופרטור `return`.

*   **אפשרות א** אינה נכונה: לשם כך משתמשים ב-`else`, לא ב-`finally`.
*   **אפשרות ב** נכונה: הקוד בבלוק `finally` מתבצע ללא קשר לאופן ביצוע בלוק `try`.
*   **אפשרות ג** אינה נכונה: לזריקת חריגה משתמשים ב-`raise`, לא ב-`finally`.
*   **אפשרות ד** אינה נכונה: `finally` אינו מונע התפשטות חריגות, אלא רק מבטיח ביצוע קוד.

**כיצד פועל `try...except...finally`:**

1.  הקוד בבלוק `try` מתבצע.
2.  אם בבלוק `try` מתרחשת חריגה, Python מחפש בלוק `except` מתאים לטיפול בה.
3.  **ללא קשר** לשאלה אם הייתה חריגה והאם נמצא בלוק `except`, הקוד בבלוק `finally` מתבצע *תמיד*.
4.  אם לא הייתה חריגה, הבלוק `finally` יתבצע לאחר בלוק `try`.
5.  בלוק `finally` מתבצע גם אם בלוק `try` מכיל אופרטור `return`, או אפילו אם `except` מכיל אופרטור `return`.

**שימושים טיפוסיים של `finally`:**

*   **שחרור משאבים:** שחרור משאבים חיצוניים, כגון קבצים, שקעים, או חיבורים למסדי נתונים.
*   **הבטחת סיום תקין:** ביצוע פעולות סופיות שחייבות להתבצע באופן מובטח, ללא קשר לביצוע מוצלח או לא מוצלח של הקוד.

**דוגמה:**

```python
def my_function(num: int) -> int | None:
  """בדיקת זוגיות של מספר וטיפול בחריגות באמצעות try-except-finally"""
  try:
    result: int = 10 // num
    print(f"Результат деления: {result}") # יתורגם להסבר בהמשך
    if num > 5:
       return result
  except ZeroDivisionError:
    print("Нельзя делить на ноль") # יתורגם להסבר בהמשך
    return None # החזרת None אם יש שגיאה
  finally:
      print("Блок finally всегда выполняется") # יתורגם להסבר בהמשך


res1 = my_function(2)
print(f"Результат: {res1}") # יתורגם להסבר בהמשך

res2 = my_function(0) # זריקת ZeroDivisionError
print(f"Результат: {res2}") # יתורגם להסבר בהמשך

res3 = my_function(6)
print(f"Результат: {res3}") # יתורגם להסבר בהמשך

# Вывод: # פלט:
# Результат деления: 5 # תוצאת החלוקה: 5
# Блок finally всегда выполняется # בלוק finally תמיד מתבצע
# Результат: 5 # תוצאה: 5
# Нельзя делить на ноль # אסור לחלק באפס
# Блок finally всегда выполняется # בלוק finally תמיד מתבצע
# Результат: None # תוצאה: None
# Результат деления: 1 # תוצאת החלוקה: 1
# Блок finally всегда выполняется # בלוק finally תמיד מתבצע
# Результат: 1 # תוצאה: 1
```

**בתוצאה:**
* בעת קריאה ל-`my_function(2)`, הקוד מתבצע ללא שגיאות ובלוק `finally` מתבצע ולאחר מכן התוצאה מוחזרת.
* בעת קריאה ל-`my_function(0)` מתרחשת `ZeroDivisionError`, הקוד עובר לבלוק `except`, שם החריגה מטופלת ומוחזר `None`, ובנוסף מתבצע הקוד מבלוק `finally`.
* בעת קריאה ל-`my_function(6)` מוחזר `result`, ובנוסף מתבצע בלוק `finally`.

לפיכך, **אפשרות ב** היא התשובה הנכונה.
