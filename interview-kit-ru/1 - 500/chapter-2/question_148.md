**שאלה 48.** ב-Python, כיצד מילת המפתח `global` משפיעה על השימוש במשתנים בתוך פונקציות?

A. היא יוצרת משתנה חדש בתחום הגלובלי.
B. היא הופכת את המשתנה בתוך הפונקציה לזמין לקריאה בלבד.
C. היא מאפשרת לפונקציה לשנות את ערכו של משתנה גלובלי.
D. היא מגבילה את המשתנה לתחום מקומי בתוך הפונקציה.

**תשובה נכונה: C**

**הסבר:**

מילת המפתח `global` משמשת בתוך פונקציה כדי לציין שהמשתנה אליו הפונקציה ניגשת הוא משתנה גלובלי, המוגדר מחוץ לפונקציה.

*   **אפשרות A** אינה נכונה: `global` אינה יוצרת משתנה חדש.
*   **אפשרות B** אינה נכונה: `global` אינה מגבילה את הגישה לקריאה בלבד.
*   **אפשרות C** נכונה: `global` מאפשרת לשנות את ערכו של משתנה גלובלי.
*   **אפשרות D** אינה נכונה: `global` מעניקה גישה למשתנה חיצוני, ואינה מגבילה את תחום הראות (scope).

**אופן הפעולה של `global`:**

1.  כברירת מחדל, בעת השמת ערך למשתנה בתוך פונקציה, Python יוצרת משתנה מקומי, גם אם קיים משתנה גלובלי בעל אותו שם.
2.  כדי לשנות את ערכו של משתנה גלובלי בתוך פונקציה, יש להכריז עליו עם מילת המפתח `global` בתוך הפונקציה.
3.  לאחר הכרזה על משתנה כ-`global`, הפונקציה יכולה לשנות משתנה זה, ובכך להשפיע על ערך המשתנה בתחום הגלובלי.

**דוגמה:**

```python
global_var: int = 10 # משתנה גלובלי
def modify_global():
    global global_var  # מכריזים ש-global_var הוא משתנה גלובלי
    global_var = 20 # משנים את ערך המשתנה הגלובלי
    print(f"ערך המשתנה בתוך הפונקציה: {global_var}")

modify_global()
print(f"ערך המשתנה מחוץ לפונקציה: {global_var}")

# פלט:
# ערך המשתנה בתוך הפונקציה: 20
# ערך המשתנה מחוץ לפונקציה: 20
```

**לסיכום:**

*   הפונקציה `modify_global` משנה את המשתנה הגלובלי `global_var`, מכיוון שהוא הוכרז כ-`global`.

לפיכך, **אפשרות C** היא הנכונה.
