**שאלה 182.** ב-Python, מהי "דריסת שיטה" (method overriding) ולשם מה היא משמשת?

א. זהו תהליך הוספת שיטה חדשה למחלקה.
ב. זהו תהליך שינוי שם השיטה.
ג. זהו תהליך הגדרת שיטה במחלקת בת באותו שם וחתימה כמו שיטה במחלקת אב.
ד. זהו תהליך מחיקת שיטה ממחלקה.

**התשובה הנכונה: C**

**הסבר:**

"דריסת שיטה" (method overriding) בתכנות מונחה עצמים היא היכולת של מחלקת בת (יורשת) לספק מימוש משלה לשיטה שכבר הוגדרה במחלקת האב (superclass) שלה.

*   **אפשרות A** אינה נכונה: דריסת שיטה אינה מוסיפה שיטה חדשה, אלא משנה שיטה קיימת.
*   **אפשרות B** אינה נכונה: דריסה אינה משנה את שם השיטה, היא משתמשת באותו שם.
*   **אפשרות C** נכונה: דריסה מגדירה שיטה במחלקת הבת עם אותו שם ופרמטרים כמו במחלקת האב.
*   **אפשרות D** אינה נכונה: דריסה אינה מוחקת שיטה, אלא משנה את המימוש שלה.

**כיצד פועלת דריסת שיטה:**

1.  בעת ירושה, מחלקת הבת מקבלת את כל השיטות והתכונות (attributes) של מחלקת האב.
2.  אם מחלקת הבת מגדירה שיטה באותו שם כמו שיטה במחלקת האב (עם אותה חתימה), שיטת מחלקת הבת מחליפה (דורסת) את שיטת האב.
3.  בעת קריאה לשיטה זו דרך מופע של מחלקת הבת, תבוצע שיטת מחלקת הבת.

**דוגמה:**

```python
class Animal:
    def make_sound(self):
        print("Generic animal sound")


class Dog(Animal):
    def make_sound(self):  # דריסת השיטה
        print("Woof!")


animal: Animal = Animal()
dog: Dog = Dog()
animal.make_sound() # פלט: Generic animal sound
dog.make_sound() # פלט: Woof!
```

**התוצאה:**

*   המחלקה `Dog` דורסת את השיטה `make_sound()`, ומספקת מימוש ספציפי משלה.

לפיכך, **אפשרות C** היא הנכונה.
