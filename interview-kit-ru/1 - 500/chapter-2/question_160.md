**שאלה 60.** איזו מהטענות הבאות מתארת בצורה הטובה ביותר את המושג "טיפוסיות ברווזים" (duck typing) ב-Python?

א. Python בודקת את טיפוס האובייקט לפני ביצוע שיטה וגורמת לשגיאה אם הטיפוס אינו תואם.
ב. Python דורשת שכל האובייקטים המשמשים באותו הקשר יהיו מאותו טיפוס.
ג. Python בודקת את קיומן של שיטות ותכונות נחוצות באובייקט, מבלי לדרוש טיפוס ספציפי, וקוראת לשיטות או לתכונות אלה אם הן קיימות.
ד. Python בודקת האם לאובייקט יש היררכיית מחלקות משותפת עם הטיפוס הצפוי.

**תשובה נכונה: ג**

**הסבר:**

"טיפוסיות ברווזים" (duck typing) היא קונספט בשפות בעלות טיפוסיות דינמית, כגון Python, שבה טיפוס האובייקט אינו משמעותי. מה שחשוב הוא שהאובייקט יכיל את השיטות והתכונות הנחוצות שבהן נעשה שימוש בקוד.

*   **אפשרות א** אינה נכונה: בטיפוסיות ברווזים, Python אינה בודקת את טיפוס האובייקט לפני קריאה לשיטה.
*   **אפשרות ב** אינה נכונה: טיפוסיות ברווזים מאפשרת שימוש באובייקטים מטיפוסים שונים באותו הקשר.
*   **אפשרות ג** נכונה: טיפוסיות ברווזים בודקת את קיומן של שיטות ותכונות נחוצות, ולא טיפוס ספציפי.
*   **אפשרות ד** אינה נכונה: טיפוסיות ברווזים אינה תלויה בהיררכיית מחלקות.

**כיצד פועלת טיפוסיות ברווזים:**

1.  Python אינה דורשת שאובייקטים יצהירו במפורש על טיפוסם.
2.  במקום בדיקת טיפוסים, Python בודקת האם לאובייקט יש את השיטות או התכונות הנחוצות עבור פעולה ספציפית.
3.  אם אובייקט "מתנהג כמו ברווז" (יש לו את השיטות והתכונות הנחוצות), אז Python מניחה שזהו "ברווז" ומאפשרת להשתמש באובייקט, גם אם טיפוסו אינו מוגדר במפורש.

**דוגמה:**

```python
class Duck:
    def quack(self):
        print("Quack!")

class Cat:
    def quack(self):
        print("Meow!")

def make_quack(animal):
    # נבדקת רק נוכחות של השיטה quack, טיפוס האובייקט אינו חשוב
    animal.quack()

duck: Duck = Duck()
cat: Cat = Cat()

make_quack(duck) # פלט: Quack!
make_quack(cat) # פלט: Meow!
```

**כתוצאה מכך:**

*   הפונקציה `make_quack()` אינה בודקת האם הארגומנט הוא מופע של המחלקה `Duck` או לא.
*   היא פשוט קוראת לשיטה `quack()`. מכיוון שלגם לאובייקט `duck` וגם לאובייקט `cat` יש שיטה `quack()`, שתי הקריאות פועלות ללא שגיאות.

לפיכך, **אפשרות ג** היא התשובה הנכונה.
