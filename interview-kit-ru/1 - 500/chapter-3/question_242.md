**שאלה 242.** מהי מטרת המשתנה `__name__` בפייתון, כאשר הוא משמש במודול?

A. הוא קובע אם המודול מורץ ישירות או מיובא.
B. הוא מספק את שם הקובץ המבוצע.
C. הוא בודק את סוג המודול.
D. הוא שומר רשימה של פונקציות המוגדרות במודול.

**תשובה נכונה: A**

**הסבר:**

המשתנה `__name__` הוא משתנה מובנה בפייתון, המוגדר אוטומטית על ידי המפרש ומספק מידע על האופן שבו מודול מורץ: ישירות (כתסריט ראשי) או מיובא למודול אחר.

*   **אפשרות A נכונה:** זוהי בדיוק מטרת `__name__`.
*   **אפשרות B אינה נכונה:** `__name__` אינו מכיל את שם הקובץ.
*   **אפשרות C אינה נכונה:** `__name__` אינו בודק את סוג המודול.
*   **אפשרות D אינה נכונה:** רשימת הפונקציות נשמרת במבני נתונים אחרים, לא ב-`__name__`.

**כיצד `__name__` עובד:**

1.  כאשר קובץ פייתון מורץ *ישירות* (לדוגמה, באמצעות `python my_script.py`), המשתנה `__name__` באותו קובץ מוגדר לערך `"__main__"`.
2.  כאשר קובץ פייתון מיובא כ*מודול* בקובץ אחר (לדוגמה, `import my_module`), המשתנה `__name__` באותו קובץ מוגדר לערך השווה ל*שם המודול* (`"my_module"`).

**דוגמה:**

```python
# my_module.py
print(f"__name__ in my_module: {__name__}")

def my_function():
    pass

if __name__ == '__main__':
    print("my_module is running directly")

# main.py
import my_module
print(f"__name__ in main: {__name__}")
```

בעת הפעלת `python my_module.py` הפלט יהיה:
```
__name__ in my_module: __main__
my_module is running directly
```
בעת הפעלת `python main.py` הפלט יהיה:
```
__name__ in my_module: my_module
__name__ in main: __main__
```

**לשם מה משתמשים ב-`__name__`:**

*   ליצירת מודולים שיכולים לפעול הן כתסריטים עצמאיים והן כמודולים הניתנים לייבוא.
*   להגנה על קוד מפני ביצוע בעת ייבוא.
*   להוספת קוד שיבוצע רק כאשר הקובץ מורץ ישירות (לדוגמה, קוד לבדיקות או לקריאה לפונקציות).

**לסיכום:**

המשתנה `__name__` משמש לקביעה האם קובץ פייתון הופעל ישירות (כתסריט ראשי) או יובא כמודול לקובץ אחר.

לפיכך, אפשרות A היא הנכונה.
