**שאלה 211.** מה מהבאים מהווה הגדרה נכונה למתודת מחלקה בפייתון?

A. מתודה הפועלת רק עם תכונות המופע ואין לה גישה לתכונות המחלקה.
B. מתודה המוחלת אוטומטית על כל מופעי המחלקה.
C. מתודה הנקראת עבור המחלקה ויש לה גישה למצב המחלקה באמצעות הפרמטר `cls`.
D. מתודה שהיא סטטית ואין לה גישה לא למחלקה ולא למופעי המחלקה.

**תשובה נכונה: C**

**הסבר:**

מתודת מחלקה (class method) בפייתון היא מתודה הקשורה למחלקה עצמה, ולא למופע ספציפי של מחלקה זו. המאפיין העיקרי של מתודת מחלקה הוא שהיא מקבלת את המחלקה עצמה בתור הארגומנט הראשון שלה (בדרך כלל נקרא `cls`).

*   **אפשרות A** אינה נכונה: למתודת מחלקה יש גישה לתכונות המחלקה, ולא לתכונות המופע.
*   **אפשרות B** אינה נכונה: מתודת מחלקה נקראת בפנייה למחלקה או למופע, ולא מוחלת אוטומטית על כולם.
*   **אפשרות C** נכונה: מתודת מחלקה מקבלת את המחלקה בתור הארגומנט הראשון `cls` ויש לה גישה למצב המחלקה.
*   **אפשרות D** אינה נכונה: מתודה סטטית אינה מקבלת ארגומנטים של מחלקה או אובייקט.

**כיצד עובד `@classmethod`:**

1.  מתודת מחלקה מוגדרת באמצעות הדקורטור `@classmethod`.
2.  הפרמטר הראשון של מתודת המחלקה (על פי הסכם `cls`) מפנה אוטומטית למחלקה עצמה, שדרכה המתודה נקראת.
3.  מתודת מחלקה יכולה לשנות תכונות מחלקה.
4.  ניתן לקרוא למתודת מחלקה הן דרך המחלקה והן דרך מופעיה.

**דוגמה:**

```python
class MyClass:
    count: int = 0 # תכונת מחלקה

    def __init__(self, name: str):
        self.name = name # תכונת מופע
        MyClass.count += 1

    @classmethod
    def get_total_instances(cls): # מתודת מחלקה
        """מחזירה את מספר האובייקטים שנוצרו"""
        return cls.count

obj1: MyClass = MyClass("Первый")
obj2: MyClass = MyClass("Второй")
print(f"מספר מופעים: {MyClass.get_total_instances()}") # פלט: מספר מופעים: 2
```

**כתוצאה מכך:**
*   המתודה `get_total_instances` היא מתודת מחלקה ויש לה גישה לתכונת המחלקה `count` דרך `cls`.

לפיכך, **אפשרות C** היא התשובה הנכונה.
