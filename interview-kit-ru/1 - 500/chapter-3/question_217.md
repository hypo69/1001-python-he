**שאלה 217.** ב-Python, כיצד נעשה שימוש במילת המפתח `as` בעת ייבוא מודולים?

A. היא משמשת לשינוי שם של מודול או פונקציה בעת הייבוא.
B. היא משמשת לייבוא רק של תכונות (attributes) ספציפיות מתוך מודול.
C. היא משמשת ליצירת מודול חדש המבוסס על מודול קיים.
D. היא משמשת לקריאה לפונקציה מתוך מודול, מבלי לייבא אותו.

**תשובה נכונה: A**

**הסבר:**

מילת המפתח `as` ב-Python משמשת בעת ייבוא מודולים (או פונקציות/מחלקות ספציפיות ממודול) ליצירת כינוי (alias). הכינוי מאפשר להתייחס לאובייקט המיובא בשם אחר, קצר יותר או נוח יותר.

*   **אפשרות A** נכונה: `as` משמשת ליצירת כינוי עבור האובייקט המיובא.
*   **אפשרות B** אינה נכונה: לייבוא תכונות ספציפיות משתמשים במבנה `from module import name`.
*   **אפשרות C** אינה נכונה: `as` אינה יוצרת מודולים חדשים.
*   **אפשרות D** אינה נכונה: `as` אינה משמשת לקריאה לפונקציות, אלא לייבוא עם כינוי.

**כיצד `as` פועלת:**

1.  בעת שימוש ב-`import module_name as alias_name`, המודול המיובא הופך לזמין תחת השם `alias_name`.
2.  `from module import name as alias_name`: מאפשר לייבא פונקציה או מחלקה ממודול תחת הכינוי `alias_name`.
3.  שימוש זה מפשט את העבודה עם מודולים, במיוחד אם שמם ארוך.

**דוגמה:**

```python
# יצירת כינוי למודול
import numpy as np
print(np.array([1,2,3])) # פלט: [1 2 3]

# יצירת כינוי לפונקציה
from math import sqrt as square_root
print(square_root(16)) # פלט: 4.0
```

**בתוצאה:**

*   בדוגמה הראשונה, המודול `numpy` מיובא תחת הכינוי `np`.
*   בדוגמה השנייה, הפונקציה `sqrt` מהמודול `math` מיובאת תחת הכינוי `square_root`.

לפיכך, **אפשרות A** היא התשובה הנכונה.
