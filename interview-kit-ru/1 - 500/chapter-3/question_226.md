**שאלה 226.** כיצד Python מטפלת באחסון במטמון (caching) של מודולים מיובאים ואיזו השפעה יש לכך על תהליך טעינת המודולים בעת ייבואם מרובה פעמים?

א. Python טוענת מחדש את המודול בכל פעם שהוא מיובא.
ב. Python שומרת הפניה למודול המיובא ב-`sys.modules`, ולכן המודול נטען רק פעם אחת.
ג. Python אינה מאפשרת לייבא מודול יותר מפעם אחת בתוכנית.
ד. Python מסירה את המודול מהזיכרון לאחר הייבוא הראשון כדי לחסוך במשאבים.

**תשובה נכונה: ב**

**הסבר:**

Python משתמשת במנגנון מטמון עבור מודולים מיובאים, ושומרת אותם במילון `sys.modules`. כאשר מודול מיובא בפעם הראשונה, הוא נטען לזיכרון, והאובייקט שלו ממוקם ב-`sys.modules`. בייבואים עוקבים של אותו מודול, Python פשוט מחזירה הפניה למודול שכבר נטען מ-`sys.modules`, במקום לטעון אותו שוב.

*   **אפשרות א אינה נכונה:** Python אינה טוענת מחדש מודולים בייבואים חוזרים. זה היה לא יעיל ומאט את ביצועי התוכנית.
*   **אפשרות ב נכונה:** `sys.modules` הוא מילון שבו Python שומרת הפניות למודולים שנטענו. זה מונע טעינה מרובה של אותו מודול.
*   **אפשרות ג אינה נכונה:** ניתן לייבא מודול פעמים רבות, אך הוא ייטען רק פעם אחת בזכות מנגנון המטמון.
*   **אפשרות ד אינה נכונה:** Python אינה מסירה מודולים מהזיכרון לאחר הייבוא הראשון. המודול נשאר בזיכרון עד סוף ריצת התוכנית.

**כיצד פועלת מערכת ייבוא המודולים עם אחסון במטמון:**

1.  בפעם הראשונה שמבצעים `import module_name`:
    *   Python מאתרת את הקובץ `module_name.py`.
    *   מקמפלת אותו לבייטקוד (אם יש צורך).
    *   טוענת את הבייטקוד לזיכרון.
    *   יוצרת אובייקט מודול.
    *   מוסיפה את אובייקט המודול ל-`sys.modules` עם המפתח `module_name`.
2.  בייבואים עוקבים של `import module_name`:
    *   Python בודקת את `sys.modules`.
    *   אם המודול כבר קיים, היא מחזירה הפניה לאובייקט המודול הקיים, וטעינה מחדש אינה מתרחשת.

**בתוצאה:**

מנגנון אחסון המטמון ב-Python מונע טעינה מיותרת של מודולים, הופך את הייבוא למהיר ויעיל יותר, וחוסך בזיכרון.

לפיכך, אפשרות ב' היא הנכונה.
