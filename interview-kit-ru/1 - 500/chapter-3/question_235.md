### `question_235.md`

**שאלה 235.** מה ההבדל בין מילת המפתח `yield` ב-Python לבין `return` בפונקציה, במיוחד מבחינת ניהול זיכרון וחישובים עצלים?

א. `yield` גורם לפונקציה להחזיר את כל התוצאות בבת אחת, בעוד ש-`return` מחזיר תוצאות אחת אחת.
ב. `yield` משמש בפונקציות-גנרטורים ומאפשר חישובים עצלים, כלומר הפונקציה יכולה להפיק ערך ולהשהות את פעולתה, לחדש את העבודה מהמקום בו עצרה, מבלי לצרוך זיכרון עבור כל התוצאות.
ג. `yield` פועל באופן דומה ל-`return`, אך שומר את התוצאות בזיכרון במקום לייצר אותן לפי דרישה.
ד. `yield` משמש לניפוי באגים ואינו משפיע על ניהול הזיכרון.

**תשובה נכונה: B**

**הסבר:**

מילות המפתח `return` ו-`yield` משמשות להחזרת ערכים מפונקציות ב-Python, אך הן פועלות בצורות שונות לחלוטין, במיוחד בהקשר של ניהול זיכרון וחישובים עצלים.

*   **`return`:**
    *   מחזירה *ערך יחיד* מהפונקציה.
    *   *מסיימת את ביצוע* הפונקציה.
    *   כל הערכים נוצרים *בבת אחת*, וכל החישובים מבוצעים *בבת אחת*.
    *   מתאים למצבים בהם נדרש לקבל תוצאה אחת של חישובים ולסיים את פעולת הפונקציה.

*   **`yield`:**
    *   משמש ב*פונקציות-גנרטורים*.
    *   מחזירה *ערך יחיד* מהפונקציה, אך *אינה מסיימת* את ביצועה.
    *   הפונקציה *מושהית* לאחר `yield` ו*שומרת* את מצבה.
    *   בבקשת הערך הבא, ביצוע הפונקציה מתחדש מהנקודה בה היא הושהתה.
    *   יוצרת *גנרטור* – אובייקט ניתן לאיטרציה, שערכיו נוצרים *לפי דרישה* (חישובים עצלים).
    *   מאפשרת עיבוד *רצפים גדולים* של נתונים, מבלי לטעון את כולם לזיכרון בבת אחת.

*   **אפשרות א' אינה נכונה:** `yield` מחזיר ערכים אחד אחד, לא את כולם בבת אחת. `return` יכול להחזיר רק *ערך אחד*.
*   **אפשרות ב' נכונה:** זהו ההבדל המרכזי של `yield` – חישובים עצלים, הפקת ערך והשהיה, וזה חוסך זיכרון.
*   **אפשרות ג' אינה נכונה:** `yield` אינו שומר את כל התוצאות בזיכרון, אלא מייצר אותן "תוך כדי תנועה". `return` מחזיר רק ערך אחד.
*   **אפשרות ד' אינה נכונה:** `yield` משפיע ישירות על ניהול הזיכרון ואינו מיועד לניפוי באגים.

**דוגמאות:**

```python
def my_function():
    return [1, 2, 3] # מחזירה רשימה באופן מיידי

def my_generator():
    yield 1
    yield 2
    yield 3

# שימוש ב-return
result_list = my_function()
print(result_list) # פלט: [1, 2, 3]
print(result_list[0]) # פלט: 1
# שימוש ב-yield
generator = my_generator()
print(next(generator)) # פלט: 1
print(next(generator)) # פלט: 2
print(next(generator)) # פלט: 3

for value in my_generator(): # Iterating over a generator
   print(value) # Output 1, then 2 then 3

```

**חישובים עצלים:**

גנרטורים, המשתמשים ב-`yield`, הם איטרטורים. הם אינם שומרים את כל הערכים בזיכרון, אלא מייצרים אותם לפי הצורך. זה שימושי במיוחד בעבודה עם מערכי נתונים גדולים.

**לסיכום:**

`yield` משמש בגנרטורים לחישוב עצל של ערכים, תוך החזרתם אחד אחד והשהיית הפונקציה, מה שחוסך זיכרון. `return` מחזיר ערך יחיד ומסיים את ביצוע הפונקציה.

לפיכך, אפשרות B היא הנכונה.
