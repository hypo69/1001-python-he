### `question_224.md`

**שאלה 224.** בפייתון, איזה דקורטור משמש ליצירת שיטות סטטיות במחלקה?

A. `@classmethod`
B. `@staticmethod`
C. `@property`
D. `@abstractmethod`

**תשובה נכונה: B**

**הסבר:**

הדקורטור `@staticmethod` בפייתון משמש להגדרת שיטות סטטיות במחלקה.

*   **אפשרות א' שגויה:** `@classmethod` משמש להגדרת שיטות מחלקה (class methods), לא שיטות סטטיות.
*   **אפשרות ב' נכונה:** `@staticmethod` יוצר שיטה סטטית.
*   **אפשרות ג' שגויה:** `@property` משמש ליצירת תכונות "מנוהלות" (managed attributes).
*   **אפשרות ד' שגויה:** `@abstractmethod` משמש להגדרת שיטות מופשטות (abstract methods) במחלקות מופשטות.

**כיצד פועל `@staticmethod`:**

1.  הדקורטור `@staticmethod` ממוקם לפני הגדרת השיטה.
2.  שיטה סטטית אינה מקבלת כאף ארגומנט לא הפניה למופע המחלקה (`self`) ולא הפניה למחלקה עצמה (`cls`).
3.  שיטות סטטיות דומות לפונקציות רגילות, אך הן מוגדרות בתוך המחלקה וקשורות אליה מבחינה לוגית.
4.  שיטה סטטית יכולה להיקרא הן באמצעות מופע של האובייקט (`instance.method()`) והן ישירות מהמחלקה עצמה (`ClassName.method()`).

**דוגמה:**

```python
class MathUtils:
    @staticmethod
    def add(x: int, y: int) -> int:
        """שיטה סטטית לסיכום מספרים, שאינה דורשת גישה לתכונות המחלקה או המופע."""
        return x + y

# קריאה לשיטה הסטטית ישירות דרך המחלקה
result_from_class: int = MathUtils.add(5, 10)
print(f"תוצאה של add_numbers, בקריאה דרך המחלקה: {result_from_class}")

# יצירת מופע של המחלקה
obj1: MathUtils = MathUtils()
# קריאה לשיטה הסטטית דרך המופע
result_from_instance: int = obj1.add(10, 20)
print(f"תוצאה של add_numbers, בקריאה דרך המופע: {result_from_instance}")
```

**התוצאה:**
* `add` היא שיטה סטטית, ואינה דורשת גישה לתכונות המחלקה או המופע. היא ניתנת לקריאה הן מהמחלקה והן מהמופע.

לפיכך, **אפשרות ב'** היא הנכונה.
