**הנחיה מערכתית למודל שפה: תרגום מסמכים מרוסית**

עליך לתרגם מסמך מרוסית לשפות שונות בהתאם להקשר התוכן:

1.  **טקסט שאינו קוד** – כל טקסט מחוץ לחלקי קוד יש לתרגם לעברית אקדמית, ברורה ורהוטה. יש לשמור על רמת ניסוח גבוהה, מתאימה למסמכים טכניים או אקדמיים.
2.  **קוד תוכנה** – אין לשנות את הקוד עצמו בשום אופן. אין להוסיף, למחוק או לערוך שורות קוד.
3.  **הערות בתוך הקוד** (כולל תגובות עם `#`, מסמכי docstring, והערות בתוך שורות קוד) – יש לתרגם רק את ההערות וה-docstrings לעברית אקדמית. עם זאת, במקרה שהוראות ניתנות באנגלית בקוד – יש להשאיר אותן באנגלית תקנית, ולהחליף את ההערות הרוסיות באנגלית טכנית ברורה.
4.  **אין לתרגם מילים או שמות קוד, משתנים, שמות קבצים או פונקציות**.
5.  **יש לשמר את המבנה המקורי של המסמך**, כולל רווחים, מבנה קבצים, פסקאות, ותיחום של קטעי קוד.
6.  **יש להקפיד על עקביות בתרגום מונחים טכניים** – מונח ברוסית שמופיע כמה פעמים צריך להיות מתורגם באותה הצורה בכל המקומות, בהתאם להקשר.

שאלה 215. איזה דקורטור ב-Python משמש ליצירת מתודות סטטיות במחלקה?

א. @classmethod
ב. @staticmethod
ג. @property
ד. @abstractmethod

תשובה נכונה: ב

הסבר:

הדקורטור @staticmethod ב-Python משמש להגדרת מתודות סטטיות במחלקה.

אפשרות א' אינה נכונה: @classmethod משמש להגדרת מתודות מחלקה, לא מתודות סטטיות.

אפשרות ב' נכונה: @staticmethod יוצרת מתודה סטטית.

אפשרות ג' אינה נכונה: @property משמש ליצירת מאפיינים "מנוהלים".

אפשרות ד' אינה נכונה: @abstractmethod משמש להגדרת מתודות אבסטרקטיות במחלקות אבסטרקטיות.

כיצד פועל @staticmethod:

הדקורטור @staticmethod ממוקם לפני הגדרת המתודה.

מתודה סטטית אינה מקבלת כאף ארגומנט לא הפניה למופע של המחלקה (self) ולא הפניה למחלקה עצמה (cls).

מתודות סטטיות דומות לפונקציות רגילות, אך הן מוגדרות בתוך מחלקה וקשורות אליה מבחינה לוגית.

ניתן לקרוא למתודה סטטית הן דרך מופע של אובייקט (instance.method()) והן ישירות דרך המחלקה עצמה (ClassName.method()).

דוגמה:

class MathUtils:
    @staticmethod
    def add(x: int, y: int) -> int:
        """מתודה סטטית לסיכום מספרים, שאינה דורשת גישה למאפיינים של המחלקה או של המופע."""
        return x + y

# קריאה למתודה הסטטית ישירות דרך המחלקה
result_from_class: int = MathUtils.add(5, 10)
print(f"תוצאת add_numbers, שנקראה דרך המחלקה: {result_from_class}")

# יצירת מופע של המחלקה
obj1: MathUtils = MathUtils()
# קריאה למתודה הסטטית דרך המופע
result_from_instance: int = obj1.add(10, 20)
print(f"תוצאת add_numbers, שנקראה דרך המופע: {result_from_instance}")
Use code with caution.
Python
כתוצאה מכך:

`add` היא מתודה סטטית, ואינה דורשת גישה למאפיינים של המחלקה או של המופע. ניתן לקרוא לה הן דרך המחלקה והן דרך המופע.

לפיכך, אפשרות ב' היא התשובה הנכונה.
