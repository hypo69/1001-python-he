### `question_306.md`

**שאלה 306.** מהו הייעוד של המשתנה `__name__` במודולי Python, במיוחד בהקשר של הגדרות פונקציות וייבוא מודולים?

א. הוא בודק אם המודול מורץ כתוכנית ראשית או מיובא כמודול.
ב. הוא מגדיר את שם המחלקה בתוך המודול.
ג. הוא שולט בגישה לפונקציות פרטיות.
ד. הוא מייבא אוטומטית מודולים אחרים באותה ספרייה.

**התשובה הנכונה: א**

**הסבר:**

המשתנה `__name__` הוא משתנה מובנה ב-Python, האוצר בתוכו את שם המודול הנוכחי. ייעודו העיקרי הוא לקבוע כיצד קובץ ה-Python הופעל: כתוכנית ראשית או כמודול מיובא.

*   **אפשרות א' נכונה:** זוהי בדיוק מטרת המשתנה `__name__`.
*   **אפשרות ב' אינה נכונה:** `__name__` אינו אוגר את שם המחלקה, אלא את שם המודול הנוכחי.
*   **אפשרות ג' אינה נכונה:** `__name__` אינו משפיע על גישה לפונקציות פרטיות.
*   **אפשרות ד' אינה נכונה:** `__name__` אינו מבצע ייבוא אוטומטי של מודולים אחרים.

**כיצד פועל המשתנה `__name__`:**

1.  כאשר קובץ Python מופעל כ*תוכנית ראשית* (לדוגמה, `python my_script.py`), המשתנה `__name__` בתוך קובץ זה מוגדר לערך `"__main__"`.
2.  כאשר קובץ Python מיובא כ*מודול* בקובץ אחר (לדוגמה, `import my_module`), המשתנה `__name__` בתוך קובץ זה מוגדר לערך שווה לשם המודול (לדוגמה, `"my_module"`).
3.  זה מאפשר למודול להבחין בין הפעלה ישירה לבין ייבוא, ולהריץ את הקוד המתאים.

**דוגמה:**

```python
# my_module.py
print(f"__name__ in my_module: {__name__}")

def my_function():
  pass


if __name__ == "__main__":
  print("This code runs only when my_module is run directly.")
  my_function()


# main.py
import my_module
print(f"__name__ in main: {__name__}")
my_module.my_function()

```
הפעלת `python my_module.py`:
```
__name__ in my_module: __main__
This code runs only when my_module is run directly.
```
הפעלת `python main.py`:
```
__name__ in my_module: my_module
__name__ in main: __main__
```
**לסיכום:**

המשתנה `__name__` משמש ב-Python על מנת לקבוע האם קובץ מופעל כתוכנית ראשית (`__name__ == '__main__'`) או מיובא כמודול בקובץ אחר. זה מאפשר שליטה בביצוע קוד בתוך המודול.

לפיכך, אפשרות א' היא התשובה הנכונה.
