### `question_303.md`

**שאלה 303.** בעת ייבוא מודול Python, מה יקרה אם למודול יש פונקציה, ואתם מנסים לקרוא לה לפני ייבוא המודול?

A. הדבר יגרום לשגיאת `ImportError`.
B. הפונקציה תיקרא באופן אוטומטי.
C. הפונקציה תתבצע רק אם היא נקראת ישירות.
D. מתורגמן Python יתעלם מהפונקציה.

**תשובה נכונה: A**

**הסבר:**

ב-Python, אם אתם מנסים להשתמש בפונקציה או בכל רכיב אחר ממודול, *לפני* שייבאתם את המודול הזה, מתורגמן Python יחזיר שגיאת `ImportError`. זה קורה מכיוון ש-Python אינו יודע על קיומו של המודול ותוכנו עד שהוא מיובא במפורש.

*   **אפשרות A נכונה:** זהו תיאור מדויק של התנהגות Python כאשר מנסים להשתמש ברכיבי מודול לפני ייבואו.
*   **אפשרות B אינה נכונה:** פונקציות אינן נקראות אוטומטית בעת ייבוא מודול (אלא אם כן קריאה אליהן כתובה במפורש מחוץ לבלוק `if __name__ == "__main__":`).
*   **אפשרות C אינה נכונה:** הפונקציה לא תתבצע עד שהמודול ייוצא ותיקרא לפונקציה דרכו.
*   **אפשרות D אינה נכונה:** Python אינו מתעלם מהפונקציה, אלא מחזיר שגיאה.

**כיצד פועל ייבוא מודולים ושגיאות `ImportError`:**

1.  כאשר Python נתקל ב-`import module_name` או `from module_name import ...`, הוא מנסה למצוא את הקובץ `module_name.py` (או החבילה המתאימה).
2.  אם הקובץ לא נמצא או שמתרחשות בעיות אחרות בעת טעינת המודול, Python מחזיר חריגה מסוג `ImportError`.
3.  אם אתם מנסים להשתמש ברכיב (פונקציה, משתנה, מחלקה) של מודול, *לפני* שהמודול יובא, Python לא יוכל למצוא את הרכיב הזה וגם יחזיר חריגה מסוג `ImportError`.

**דוגמה:**

```python
# my_module.py
def my_function():
    return "Hello from my_module"

# main.py

# הדפס(my_function()) # Error: my_function not defined, NameError
# ייבוא my_module
# הדפס(my_module.my_function()) # Output: Hello from my_module

try:
  print(my_function())
except NameError as e:
  print(e)
  # Output: name 'my_function' is not defined
try:
  import my_module
  print(my_module.my_function())
except ImportError as e:
  print(e)

```

**כתוצאה מכך:**

אם אתם מנסים לקרוא לפונקציה מתוך מודול *לפני* ייבוא המודול הזה, Python יחזיר שגיאת `ImportError`, המציינת שהמודול ותוכנו לא נטענו למרחב השמות הנוכחי.

לפיכך, אפשרות A היא הנכונה.
