**שאלה 13.** אילו מהאפשרויות הבאות מדגימות *באופן המדויק והמלא ביותר* את הגמישות ה*מותרת* של טיפוסיות דינמית ב-Python, המאפשרת למשתנים לשנות את סוג הנתונים שלהם במהלך ביצוע התוכנית *מבלי לגרום לחריגות* ועם שימוש מוצלח לאחר מכן במשתנה בפעולות התואמות את הסוג החדש? (בחרו *את כל* האפשרויות המתאימות)

- A.  `x = 10; x = "ten"; print(x)`
- B.  `x = 10; x = x + "10"; print(x)`
- C.  `x = "10"; x = int(x); print(x + 10)`
- D.  `x = "10"; x = int(x); x = x + "10"; print(x)`

**תשובה נכונה: A, C**

**הסבר:**

טיפוסיות דינמית ב-Python פירושה שסוג המשתנה נקבע בזמן הריצה ויכול להשתנות. שאלה זו בוחנת את ההבנה אילו הקצאות מחדש *מותרות* (אינן גורמות לשגיאות), וכיצד הדבר משפיע על השימוש במשתנה בהמשך.

*   **אפשרות A:** `x = 10; x = "ten"; print(x)` - *תקין*. למשתנה `x` מוקצה תחילה מספר שלם, ולאחר מכן מחרוזת. Python מאפשר הקצאה מחדש זו *ללא שגיאות*. אולם, הוא *אינו מדגים* שימוש במשתנה לאחר הקצאת הסוג מחדש בפעולה כלשהי.

*   **אפשרות B:** `x = 10; x = x + "10"; print(x)` - *לא תקין*. זה יגרום ל-`TypeError`, שכן Python אינו מבצע המרת סוגים מרומזת בעת ניסיון לחבר מספר שלם ומחרוזת.

*   **אפשרות C:** `x = "10"; x = int(x); print(x + 10)` - *תקין ודוגמה מלאה יותר*. מדגים טיפוסיות דינמית באופן הבא:
    *   `x = "10"`: למשתנה x מוקצית מחרוזת.
    *   `x = int(x)`: המשתנה x מומר למספר שלם.
    *   `print(x + 10)`: הערך של x (שכעת הוא מספר שלם) *משמש בהצלחה* בפעולת חיבור עם מספר שלם.

*   **אפשרות D:** `x = "10"; x = int(x); x = x + "10"; print(x)` - *לא תקין*. לאחר ההמרה למספר שלם מתבצע ניסיון לחבר מספר שלם עם מחרוזת, מה שיגרום ל-`TypeError`.
