**שאלה 13.** אילו מהאפשרויות הבאות מדגימות *בצורה המדויקת והמלאה ביותר* את הגמישות ה*מותרת* של טיפוּת דינמית ב-Python, בכך שהן מאפשרות למשתנים לשנות את סוג הנתונים שלהם במהלך ביצוע התוכנית *מבלי לגרום לחריגות* ועם שימוש מוצלח לאחר מכן במשתנה בפעולות התואמות את הסוג החדש? (בחר/י *את כל* האפשרויות המתאימות)

- A.  `x = 10; x = "ten"; print(x)`
- B.  `x = 10; x = x + "10"; print(x)`
- C.  `x = "10"; x = int(x); print(x + 10)`
- D.  `x = "10"; x = int(x); x = x + "10"; print(x)`

**תשובה נכונה: A, C**

**הסבר:**

טיפוּת דינמית ב-Python פירושה שסוג הנתונים של משתנה נקבע במהלך זמן ריצה ויכול להשתנות. שאלה זו מעריכה את ההבנה אילו פעולות השמה חוזרת *מותרות* (אינן גורמות לחריגות), וכיצד הדבר משפיע על השימוש הבא במשתנה.

*   **אפשרות A:** `x = 10; x = "ten"; print(x)` - *תקין*. למשתנה `x` מושם תחילה מספר שלם, ולאחר מכן מחרוזת. Python מתירה השמה חוזרת זו *ללא שגיאות*. עם זאת, הוא *אינו מדגים* שימוש במשתנה לאחר שינוי הסוג בפעולה כלשהי.

*   **אפשרות B:** `x = 10; x = x + "10"; print(x)` - *לא תקין*. זה יגרום ל-`TypeError`, מכיוון ש-Python אינה מבצעת המרה משתמעת של סוגים בעת ניסיון לחבר מספר שלם ומחרוזת.

*   **אפשרות C:** `x = "10"; x = int(x); print(x + 10)` - *תקין ודוגמה מלאה יותר*. מדגים טיפוּת דינמית באופן הבא:
    *   `x = "10"`: למשתנה x מושמת מחרוזת.
    *   `x = int(x)`: המשתנה x מומר למספר שלם.
    *   `print(x + 10)`: הערך של x (כעת מספר שלם) *משמש בהצלחה* בפעולת חיבור עם מספר שלם.

*   **אפשרות D:** `x = "10"; x = int(x); x = x + "10"; print(x)` - *לא תקין*. לאחר ההמרה למספר שלם, מתרחש ניסיון לחבר מספר שלם עם מחרוזת, דבר שיגרום ל-`TypeError`.
