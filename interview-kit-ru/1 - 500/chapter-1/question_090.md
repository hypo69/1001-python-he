### `question_90.md`

**שאלה 90.** מה ההבדל בין `==` לבין `is` בפייתון?

- א.  `==` בודק שוויון בערך, בעוד `is` בודק שוויון באזור הזיכרון.
- ב.  `==` בודק האם המשתנים מצביעים על אותו אובייקט, בעוד `is` בודק שוויון בערך.
- ג.  `==` משמש עבור מחרוזות, ו-`is` עבור מספרים.
- ד.  אין הבדל; שניהם ניתנים לשימוש חליפי.

**תשובה נכונה: א**

**הסבר:**

בפייתון, האופרטורים `==` ו-`is` משמשים להשוואה, אך הם מבצעים זאת באופן שונה:

*   **`==` (שווה):** אופרטור זה משווה את *הערכים* של אובייקטים. הוא מחזיר `True` אם ערכי האובייקטים שווים, ו-`False` אחרת.
*   **`is` (הוא):** אופרטור זה משווה את *זהות* האובייקטים. הוא מחזיר `True` אם שני המשתנים מצביעים על אותו אובייקט בזיכרון, ו-`False` אחרת.

*   **אפשרות א** נכונה: `==` בודק שוויון בערכים, ו-`is` בודק זהות (האם המשתנים מצביעים על אותו אובייקט בזיכרון).
*   **אפשרות ב** אינה נכונה: להפך, `==` בודק שוויון בערכים, ו-`is` בודק זהות.
*   **אפשרות ג** אינה נכונה: `==` ו-`is` משמשים לכל סוג נתונים.
*   **אפשרות ד** אינה נכונה: `==` ו-`is` אינם ניתנים לשימוש חליפי, יש להם ייעוד שונה.

**הסבר מפורט:**

*   **`==`**: משווה את ערכי האובייקטים, תוך שימוש במתודה `__eq__()`, שניתן לדרוס עבור מחלקות מותאמות אישית.
*   **`is`**: משווה את מזהי האובייקטים, כלומר כתובות הזיכרון. אם שני משתנים מפנים לאותו אובייקט בזיכרון, אזי `is` יחזיר `True`.

**דוגמה:**

```python
a: list[int] = [1, 2, 3]
b: list[int] = [1, 2, 3]
c: list[int] = a

print(f"a == b: {a == b}") # פלט: a == b: True
print(f"a is b: {a is b}")  # פלט: a is b: False

print(f"a == c: {a == c}")  # פלט: a == c: True
print(f"a is c: {a is c}") # פלט: a is c: True

num1: int = 1000
num2: int = 1000
print(f"num1 == num2: {num1 == num2}") # פלט: num1 == num2: True
print(f"num1 is num2: {num1 is num2}") # פלט: num1 is num2: False

num3: int = 100
num4: int = 100
print(f"num3 == num4: {num3 == num4}") # פלט: num3 == num4: True
print(f"num3 is num4: {num3 is num4}") # פלט: num3 is num4: True
```

**בסיכום:**

*   `a` ו-`b` מכילים רשימות עם ערכים זהים, ולכן `a == b` מחזיר `True`, אך אלו שני אובייקטים שונים בזיכרון, ולכן `a is b` מחזיר `False`.
*  `a` ו-`c` מפנים לאותו אובייקט, ולכן `a is c` מחזיר `True`
* עבור מספרים קטנים, פייתון עשוי ליצור אובייקטים זהים בזיכרון, למטרות אופטימיזציה, ולכן עבור num3 ו-num4 האופרטור `is` החזיר `True`.

לפיכך, **אפשרות א** היא הנכונה.
