**שאלה 90.** מה ההבדל בין `==` לבין `is` ב-Python?

- A. `==` בודק שוויון בערך, בעוד ש-`is` בודק שוויון באזור הזיכרון.
- B. `==` בודק אם המשתנים מצביעים על אותו אובייקט, בעוד ש-`is` בודק שוויון בערך.
- C. `==` משמש למחרוזות, ו-`is` משמש למספרים.
- D. אין הבדל; שניהם ניתנים לשימוש זה במקום זה.

**תשובה נכונה: A**

**הסבר:**

ב-Python, האופרטורים `==` ו-`is` משמשים להשוואה, אך הם עושים זאת בדרכים שונות:

*   **`==` (שווה):** אופרטור זה משווה את ה*ערכים* של אובייקטים. הוא מחזיר `True` אם ערכי האובייקטים שווים, ו-`False` אחרת.
*   **`is` (הוא):** אופרטור זה משווה את ה*זהות* של אובייקטים. הוא מחזיר `True` אם שני המשתנים מצביעים על אותו אובייקט בזיכרון, ו-`False` אחרת.

*   **אפשרות A** נכונה: `==` בודק שוויון ערכים, ו-`is` בודק זהות (האם המשתנים מצביעים על אותו אובייקט בזיכרון).
*   **אפשרות B** אינה נכונה: להפך, `==` בודק שוויון ערכים, ו-`is` בודק זהות.
*   **אפשרות C** אינה נכונה: `==` ו-`is` משמשים לכל סוג נתונים.
*   **אפשרות D** אינה נכונה: `==` ו-`is` אינם ניתנים לשימוש זה במקום זה, ויש להם ייעוד שונה.

**הסבר מפורט:**

*   **`==`**: משווה את ערכי האובייקטים, תוך שימוש במתודה `__eq__()`, הניתנת לדריסה עבור מחלקות מותאמות אישית.
*   **`is`**: משווה את מזהי האובייקטים, כלומר כתובות הזיכרון. אם שני משתנים מתייחסים לאותו אובייקט בזיכרון, אז `is` יחזיר `True`.

**דוגמה:**

```python
a: list[int] = [1, 2, 3]
b: list[int] = [1, 2, 3]
c: list[int] = a

print(f"a == b: {a == b}") # פלט: a == b: True
print(f"a is b: {a is b}")  # פלט: a is b: False

print(f"a == c: {a == c}")  # פלט: a == c: True
print(f"a is c: {a is c}") # פלט: a is c: True

num1: int = 1000
num2: int = 1000
print(f"num1 == num2: {num1 == num2}") # פלט: num1 == num2: True
print(f"num1 is num2: {num1 is num2}") # פלט: num1 is num2: False

num3: int = 100
num4: int = 100
print(f"num3 == num4: {num3 == num4}") # פלט: num3 == num4: True
print(f"num3 is num4: {num3 is num4}") # פלט: num3 is num4: True
```

**בתוצאה:**

*   `a` ו-`b` מכילים רשימות עם אותם ערכים, לכן `a == b` נותן `True`, אך הם שני אובייקטים שונים בזיכרון, לכן `a is b` נותן `False`.
*   `a` ו-`c` מתייחסים לאותו אובייקט, לכן `a is c` מחזיר `True`.
*   עבור מספרים קטנים, Python יכול ליצור אובייקטים זהים בזיכרון, לצורך אופטימיזציה, לכן עבור num3 ו-num4, האופרטור `is` החזיר `True`.

לפיכך, **אפשרות A** היא הנכונה.
