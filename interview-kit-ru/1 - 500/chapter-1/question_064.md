### `question_064.md`

**שאלה 064.** מהי התוצאה שתודפס על המסך על ידי קוד הפייתון הבא?

```python
def outer_func(y):
    x = 2
    def inner_func():
        return x + y
    x = x + 2
    y = 2
    return inner_func

results = outer_func(3)
print(results())
```

-   א. 7
-   ב. 6
-   ג. 5
-   ד. 4

**תשובה נכונה: B**

**הסבר:**

קוד זה מדגים את אופן הפעולה של סגורים (closures) ותחומי ההיקף (scope) של משתנים בפייתון.

1.  **הפונקציה `outer_func(y)`:**
    *   מקבלת את הארגומנט `y` (בדוגמה שווה ל-3).
    *   מאתחלת את המשתנה המקומי `x` בערך `2`.
    *   מגדירה פונקציה מקוננת `inner_func()`.
        *   `inner_func()` "נסגרת" (closures) על המשתנים מתחום ההיקף של `outer_func` ומחזירה את הסכום `x + y`.
    *   משנה את `x` ל-`x + 2`, כלומר `x` שווה כעת ל-4.
    *   משנה את `y` ל-`2`.
    *   מחזירה את הפונקציה `inner_func` כתוצאה.

2.  **קריאה ל-`outer_func(3)`:**
    *   קוראת ל-`outer_func` עם הארגומנט `3`.
    *   הפונקציה המוחזרת `inner_func` מושמת למשתנה `results`.
    *   חשוב לשים לב שהפונקציה המקוננת `inner_func` זוכרת לא את ערכי המשתנים `x` ו-`y` ברגע ההגדרה, אלא את המשתנים עצמם, וערכיהם יעודכנו ברגע הקריאה ל-`inner_func`.

3.  **קריאה ל-`results()`:**
    *   קוראת לפונקציה `inner_func`, שהוחזרה מ-`outer_func` והושמה ל-`results`.
    *   בתוך `inner_func`:
        *   ל-`x` יש את הערך `4`, שחושב ב-`outer_func` לפני שהיא הוחזרה.
        *   ל-`y` יש את הערך `2`, שהוּשַׂם ב-`outer_func` לפני שהיא הוחזרה.
    *   מוחזר הערך `x + y = 4 + 2 = 6`.
4.  **פלט:** `print(results())` מדפיסה את תוצאת הקריאה ל-`inner_func`, כלומר `6`.

**ניתוח האפשרויות:**
*   **א. 7:** שגוי.
*   **ב. 6:** נכון.
*   **ג. 5:** שגוי.
*   **ד. 4:** שגוי.

**לסיכום:**
*   פונקציות מקוננות יכולות ללכוד (capture) משתנים מתחום ההיקף של הפונקציה ההורה - זה נקרא סְגוֹר (closure).
*   לכידת המשתנה מתרחשת לפי הפניה (by reference) ולא לפי ערך (by value). ברגע הקריאה ל-`inner_func`, המשתנים `x` ו-`y` יכילו את הערך האחרון שהוּשַׂם להם.

לפיכך, התשובה הנכונה היא **ב. 6**.
