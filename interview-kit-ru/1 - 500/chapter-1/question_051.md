**שאלה 051.** בתמונה מוצגים שני עצים בינאריים. פתחו אלגוריתם לחישוב סכום נטיות כל צומת בעץ הבינארי, וסכמו אותן.
*   **נטיית צומת** - זהו ההפרש המוחלט בין סכום כל הערכים בתת-העץ השמאלי של הצומת לבין סכום כל הערכים בתת-העץ הימני של הצומת. אם לצומת אין צאצא שמאלי או ימני, יש להתייחס לסכום המקביל כ-0.

**תמונות העצים ב-ASCII:**

**Figure A:**

```
    4
   / \
  2   9
 / \   \
3   5   7
```

**Figure B:**

```
    6
   / \
  2  7
 / \  \
0   0  0
```

**דוגמאות:**

```
קלט: root = [1,2,3]
פלט: 1

קלט: root = [4,2,9,3,5,null,7]
פלט: 15
```

-   א. כדי לפתור את הבעיה, יש לחשב את סכום כל הצמתים, ואת ההפרש בין סכומי התת-עץ השמאלי והימני, וכן לסכום את כל ההפרשים.
-   ב. כדי לפתור את הבעיה, נדרש אלגוריתם המבוסס על מעבר לרוחב (BFS) וחישוב הפרש סכומי העץ השמאלי והימני בכל רמה.
-   ג. כדי לפתור את הבעיה, נדרש אלגוריתם רקורסיבי, שמחזיר את סכום תת-העץ (תוך כדי סיכום נטיית הצומת הנוכחי), ומחזיר את סכום נטיות כל הצמתים.
-  ד. כדי לפתור את הבעיה, יש להשתמש באלגוריתם רקורסיבי שיחשב את סכום הצמתים עבור כל תת-עץ, תוך שימוש באלגוריתם מציאת המסלול הקצר ביותר.

**תשובה נכונה: ג**

**הסבר:**

כדי לפתור את בעיית חישוב סכום נטיות כל הצמתים בעץ בינארי, האופטימלי ביותר הוא שימוש באלגוריתם רקורסיבי, המאפשר חישוב יעיל של סכומי תת-עצים ונטיית כל צומת.

*   **אלגוריתם (DFS רקורסיבי):**
    1.  **פונקציה רקורסיבית:** ניצור פונקציה רקורסיבית, שמחזירה את סכום צמתי תת-עץ, ומשתמשת בו לחישוב נטיית כל צומת.
         *  אם הצומת ריק, מחזירה 0.
         * מחשבים רקורסיבית את סכום תת-העץ השמאלי והימני.
       *  נטיית הצומת הנוכחי - זהו הערך המוחלט של הפרש סכומי תת-העץ השמאלי והימני.
         *   מוסיפים את הנטייה לערך הכולל.
       *  מחזירים את סכום צמתי תת-העץ, ששווה לסכום ערכי תת-העץ השמאלי, תת-העץ הימני וערך הצומת הנוכחי.
    2. **אתחול:** מאתחלים משתנה `total_tilt`, שישמור את סכום הנטיות.
    3. **קריאה רקורסיבית:** קוראים לפונקציה הרקורסיבית עבור השורש, ולאחר הקריאה מחזירים את `total_tilt`.

*   **יתרונות האלגוריתם:**
    *   **מעבר רקורסיבי:** עובר על כל צמתי העץ.
    *  **שמירת סכומים חלקיים:** סכומים חלקיים של תת-עצים משמשים בחישובים הרקורסיביים.
    *    **חישוב נטייה:** מאפשר לחשב את נטיית כל צומת ולהוסיף אותה לסכום הכולל.
    *   **יעילות:**  מבטיח מימוש יעיל (O(n)).

**דוגמאות (פסאודו-קוד):**

```
function calculate_tree_tilt(node):
  if node is null:
      return 0
  left_sum = calculate_tree_tilt(node.left)
```

**דוגמאות מימוש ב-Python:**
```python
#  מימוש BST לבדיקה
class TreeNode:
   def __init__(self, val=0, left=None, right=None):
     self.val = val
     self.left = left
     self.right = right

def calculate_tree_tilt(node, total_tilt):
        if not node:
            return 0
        left_sum = calculate_tree_tilt(node.left, total_tilt)
        right_sum = calculate_tree_tilt(node.right, total_tilt)
        total_tilt[0] += abs(left_sum - right_sum)
        return node.val + left_sum + right_sum

root = TreeNode(1,TreeNode(2), TreeNode(3))
total_tilt = [0]
calculate_tree_tilt(root, total_tilt)
print(f"קלט: root = [1,2,3]")
print(f"פלט: {total_tilt[0]}") #  יוצג: פלט: 1


root2 = TreeNode(4,TreeNode(2,TreeNode(3),TreeNode(5)), TreeNode(9,None, TreeNode(7)))
total_tilt = [0]
calculate_tree_tilt(root2,total_tilt)
```
