**שאלה 12.** פייתון תומך במספר טיפוסי נתונים, ומחרוזות הן מהנפוצים ביותר בשימוש. בעת התחשבות בטיפוסי הנתונים מספר שלם (`int`), מספר נקודה צפה (`float`) ומחרוזת (`str`), איזה מבין קטעי הקוד הבאים ממיר נכונה ייצוג מחרוזת של מספר למספר נקודה צפה, ולאחר מכן מוסיף אותו למספר שלם לפני הדפסת התוצאה?

```python
- A. result = int("10.5") + 5; print(result)
- B. result = float("10.5") + 5; print(result)
- C. result = str(10.5 + 5); print(result)
- D. result = "10.5" + str(5); print(result)
```

**תשובה נכונה: B**

**הסבר:**

*   **המרה של טיפוסים:** פייתון מאפשרת להמיר ערכים מטיפוס נתונים אחד לאחר באמצעות פונקציות ייעודיות.
*   **`int()`**: ממירה מחרוזת או מספר למספר שלם.
*   **`float()`**: ממירה מחרוזת או מספר למספר נקודה צפה.
*   **`str()`**: ממירה ערך מכל טיפוס למחרוזת.

כעת ננתח כל אפשרות:

*   **אפשרות A**: `result = int("10.5") + 5; print(result)` - אפשרות שגויה, מכיוון שהפונקציה `int()` אינה יכולה להמיר ישירות את המחרוזת `"10.5"` למספר שלם, כיוון שהיא מכילה חלק עשרוני. פעולה זו תגרום לשגיאת `ValueError`.

*   **אפשרות B**: `result = float("10.5") + 5; print(result)` - זו האפשרות הנכונה. הפונקציה `float()` ממירה באופן תקין את המחרוזת `"10.5"` למספר נקודה צפה `10.5`. לאחר מכן, למספר זה מתווסף המספר השלם `5`, והתוצאה (מספר נקודה צפה `15.5`) מודפסת למסך.

*   **אפשרות C**: `result = str(10.5 + 5); print(result)` - אפשרות שגויה, מכיוון שכאן תחילה מחוברים `10.5` ו-`5`, ולאחר מכן התוצאה `15.5` מומרת למחרוזת. במקום מספר נקודה צפה, נקבל את המחרוזת `"15.5"`.

*   **אפשרות D**: `result = "10.5" + str(5); print(result)` - אפשרות שגויה, מכיוון שכאן המחרוזת `"10.5"` משרשרת (concatenated) עם המחרוזת `"5"`, וכתוצאה מכך מתקבלת המחרוזת `"10.55"`. אין כאן המרה למספר נקודה צפה או חיבור.

**דוגמה:**

```python
# דוגמה עם האפשרות הנכונה (B)
result: float = float("10.5") + 5
print(f"Результат: {result}") # פלט: Результат: 15.5
print(f"Тип результата: {type(result)}") # פלט: Тип результата: <class 'float'>

# דוגמה עם אפשרות שגויה (A) - תגרום לשגיאה
try:
  result: int = int("10.5") + 5
  print(result)
except ValueError as e:
   print(e) # פלט: invalid literal for int() with base 10: '10.5'
```

**לסיכום:**

*   **אפשרות B** ממירה בהצלחה את המחרוזת `"10.5"` ל-`float`, מוסיפה לה `5` ומדפיסה את התוצאה כ-`15.5` מטיפוס `float`.
*   **אפשרות A** גורמת לשגיאה, מכיוון שלא ניתן להמיר את המחרוזת `"10.5"` ל-`int`.

לפיכך, רק **אפשרות B** מבצעת נכונה את המרת המחרוזת למספר נקודה צפה ואת החיבור לאחר מכן עם מספר שלם.
