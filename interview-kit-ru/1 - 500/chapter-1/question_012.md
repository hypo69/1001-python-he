**שאלה 12.** Python תומכת במספר סוגי נתונים, ומחרוזות הן בין הנפוצים ביותר בשימוש. בהתייחס לסוגי הנתונים מספר שלם (`int`), מספר נקודה צפה (`float`) ומחרוזתי (`str`), איזה מקטע קוד מבין הבאים ממיר נכונה ייצוג מחרוזתי של מספר למספר נקודה צפה ולאחר מכן מוסיף אותו למספר שלם לפני הדפסת התוצאה?

```python
- A. result = int("10.5") + 5; print(result)
- B. result = float("10.5") + 5; print(result)
- C. result = str(10.5 + 5); print(result)
- D. result = "10.5" + str(5); print(result)
```

**תשובה נכונה: B**

**הסבר:**

*   **המרת סוגים:** Python מאפשרת המרה של ערכים מסוג נתונים אחד לאחר באמצעות פונקציות מיוחדות.
*   **`int()`**: ממירה מחרוזת או מספר למספר שלם.
*   **`float()`**: ממירה מחרוזת או מספר למספר נקודה צפה.
*   **`str()`**: ממירה ערך מכל סוג למחרוזת.

כעת ננתח כל אפשרות:

*   **אפשרות A**: `result = int("10.5") + 5; print(result)` - אפשרות שגויה, מכיוון שהפונקציה `int()` אינה יכולה להמיר ישירות את המחרוזת `"10.5"` למספר שלם, כיוון שהיא מכילה חלק עשרוני. פעולה זו תגרום לשגיאת `ValueError`.

*   **אפשרות B**: `result = float("10.5") + 5; print(result)` - זו האפשרות הנכונה. הפונקציה `float()` ממירה באופן תקין את המחרוזת `"10.5"` למספר נקודה צפה `10.5`. לאחר מכן, נוסף למספר זה המספר השלם `5`, והתוצאה (מספר נקודה צפה `15.5`) מודפסת על המסך.

*   **אפשרות C**: `result = str(10.5 + 5); print(result)` - אפשרות שגויה, מכיוון שכאן סוכמים תחילה את `10.5` ואת `5`, ולאחר מכן התוצאה `15.5` מומרת למחרוזת. במקום מספר נקודה צפה נקבל את המחרוזת `"15.5"`.

*   **אפשרות D**: `result = "10.5" + str(5); print(result)` - אפשרות שגויה, מכיוון שכאן המחרוזת `"10.5"` משרשרת (מתחברת) למחרוזת `"5"`, ומתקבלת המחרוזת `"10.55"`. לא מתרחשת לא המרה למספר נקודה צפה ולא חיבור.

**דוגמה:**

```python
# דוגמה עם האפשרות הנכונה (B)
result: float = float("10.5") + 5
print(f"תוצאה: {result}") # פלט: תוצאה: 15.5
print(f"סוג התוצאה: {type(result)}") # פלט: סוג התוצאה: <class 'float'>

# דוגמה עם האפשרות השגויה (A) - תגרום לשגיאה
try:
  result: int = int("10.5") + 5
  print(result)
except ValueError as e:
   print(e) # פלט: invalid literal for int() with base 10: '10.5'
```

**לסיכום:**

*   **אפשרות B** ממירה בהצלחה את המחרוזת `"10.5"` ל-`float`, מחברת עם `5` ומדפיסה את התוצאה כ-`15.5` מסוג `float`.
*   **אפשרות A** גורמת לשגיאה, כיוון שלא ניתן להמיר את המחרוזת `"10.5"` ל-`int`.

לפיכך, רק **אפשרות B** מבצעת באופן תקין את המרת המחרוזת למספר נקודה צפה ואת החיבור העוקב עם מספר שלם.
