### `question_037.md`

**שאלה 37.** כיצד טיפול בחריגות ב-Python באמצעות בלוקים `try-except` מסייע בפיתוח תוכנה אמין, במיוחד מבחינת רציפות פעולה וניהול שגיאות?

- א. בלוקים `try-except` ב-Python נועדו רק ללכידת שגיאות תחביר בקוד, מה שמאפשר לתוכנות להיות נכונות תחבירית וללא שגיאות.
- ב. בלוקים אלו משמשים לבדיקת קטע קוד לגילוי שגיאות, לכידת החריגות המתעוררות וטיפול בהן בבלוק `except`, מה שמונע סיום פתאומי של התוכנה.
- ג. מבנה `try-except` מתקן אוטומטית שגיאות לוגיות בקוד, ומבטיח ביצוע פעולות מתוכננות ללא התערבות ידנית.
- ד. מבנה זה משמש להגברת מהירות ביצוע הקוד על ידי התעלמות מחריגות שאינן קריטיות לפונקציונליות העיקרית של היישום.

**התשובה הנכונה: ב**

**הסבר:**

בלוקים `try-except` הם מנגנון בסיסי ב-Python לטיפול בחריגות (exceptions) שיכולות להתרחש במהלך ביצוע התוכנה. בלוקים אלו מאפשרים ליצור תוכנות אמינות ועמידות לשגיאות.

*   **בלוק `try`:** מכיל את הקוד שבו עלולה להתרחש חריגה (שגיאת זמן ריצה).
*   **בלוק `except`:** מכיל את הקוד שמבוצע אם מתרחשת חריגה בבלוק `try`. בבלוק except ניתן להגדיר כיצד לטפל בחריגה, כדי למנוע סיום בלתי תקין של התוכנה.

כעת נסקור את אפשרויות התשובה:

*   **אפשרות א** אינה נכונה: `try-except` נועדו ללכידת שגיאות *זמן ריצה*, ולא שגיאות תחביר.
*   **אפשרות ב** נכונה: בלוקים `try-except` אכן מאפשרים ללכוד חריגות ולטפל בהן באופן תקין, מה שמונע סיום פתאומי של התוכנה.
*   **אפשרות ג** אינה נכונה: `try-except` אינם מתקנים שגיאות לוגיות אוטומטית, הם לוכדים חריגות.
*   **אפשרות ד** אינה נכונה: `try-except` אינם מתעלמים מחריגות, אלא מטפלים בהן.

**כיצד `try-except` מסייעים בפיתוח אמין:**

1.  **מניעת קריסות:** `try-except` מאפשר ללכוד חריגות ולהמשיך בביצוע התוכנה, במקום שהיא תסתיים באופן בלתי תקין.
2.  **טיפול בשגיאות:** בלוק `except` מאפשר למפתח לטפל בחריגה, לדוגמה, להציג הודעת שגיאה, לבצע פעולות התאוששות כלשהן, או לסיים את פעולת התוכנה באופן תקין.
3.  **שליטה על זרימת ביצוע:** מאפשר להמשיך את ביצוע התוכנה כאשר מתרחשת שגיאה, על ידי הפניית זרימת הביצוע לבלוק `except`.
4.  **שיפור חווית המשתמש:** בלוקים `try-except` מאפשרים להציג למשתמשים הודעות שגיאה אינפורמטיביות יותר, במקום שהם פשוט יראו סיום בלתי תקין של התוכנה.

**דוגמה:**

```python
def divide(x: int, y: int) -> float:
    try:
      result: float = x / y # ניסיון לבצע חילוק
      print(f"תוצאה: {result}") # הדפסת התוצאה אם אין שגיאה
      return result
    except ZeroDivisionError: # לכידת שגיאת חלוקה באפס
       print("שגיאה: חלוקה באפס!") # טיפול בשגיאה: הדפסת הודעה
       return None # החזרת ערך המציין כי הפעולה נכשלה

divide(10, 2)   # פלט:  תוצאה: 5.0
divide(10, 0) # פלט: שגיאה: חלוקה באפס!
```

**בתוצאה:**

*   בדוגמה לעיל, הפונקציה `divide()` מחלקת נכון את המספר `x` ב-`y`.
*   אם `y` יהיה שווה ל-0, תילכד החריגה `ZeroDivisionError`, והתוכנה תדפיס את ההודעה `"שגיאה: חלוקה באפס!"` ותחזיר `None`, במקום לסיים את עבודתה באופן בלתי תקין.

לפיכך, **אפשרות ב** היא התשובה הנכונה.
