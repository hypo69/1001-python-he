**שאלה 37.** כיצד טיפול בחריגות ב-Python באמצעות בלוקים `try-except` מסייע בפיתוח תוכנה אמינה, במיוחד מנקודת המבט של רציפות פעולה וניהול שגיאות?

- A. בלוקים `try-except` ב-Python מיועדים רק ללכידת שגיאות תחביר בקוד, ומאפשרים לתוכניות להיות נכונות תחבירית וללא שגיאות.
- B. בלוקים אלה משמשים לבדיקת בלוק קוד לנוכחות שגיאות, לכידת החריגות המתעוררות וטיפול בהן בבלוק `except`, מה שמונע סיום פתאומי של התוכנית.
- C. מבנה `try-except` מתקן אוטומטית שגיאות לוגיות בקוד, ומבטיח ביצוע פעולות מתוכננות ללא התערבות ידנית.
- D. מבנה זה משמש להגברת מהירות ביצוע הקוד על ידי התעלמות מחריגות שאינן קריטיות לפונקציונליות העיקרית של היישום.

**תשובה נכונה: B**

**הסבר:**

בלוקים `try-except` הם מנגנון יסודי ב-Python לטיפול בחריגות שעלולות להתרחש במהלך ביצוע התוכנית. בלוקים אלה מאפשרים ליצור תוכניות אמינות ועמידות בפני שגיאות.

*   **בלוק `try`:** מכיל קוד שבו עלולה להתרחש חריגה (שגיאת זמן ריצה).
*   **בלוק `except`:** מכיל קוד המבוצע אם מתרחשת חריגה בבלוק `try`. בבלוק except ניתן להגדיר כיצד לטפל בחריגה באופן ספציפי, כדי למנוע סיום תוכנית באופן פתאומי.

כעת נבחן את אפשרויות התשובה:

*   **אפשרות A** אינה נכונה: `try-except` מיועדים ללכידת שגיאות *זמן ריצה*, לא שגיאות תחביר.
*   **אפשרות B** נכונה: בלוקים `try-except` אכן מאפשרים ללכוד חריגות ולטפל בהן כראוי, מה שמונע סיום תוכנית באופן פתאומי.
*   **אפשרות C** אינה נכונה: `try-except` אינם מתקנים שגיאות לוגיות אוטומטית, הם לוכדים חריגות.
*   **אפשרות D** אינה נכונה: `try-except` אינם מתעלמים מחריגות, אלא מטפלים בהן.

**כיצד `try-except` מסייעים בפיתוח אמין:**

1.  **מניעת קריסות:** `try-except` מאפשר ללכוד חריגות ולהמשיך את ביצוע התוכנית, במקום לסיים אותה באופן פתאומי (באופן לא תקין).
2.  **טיפול בשגיאות:** בלוק `except` מאפשר למפתח לטפל בחריגה, לדוגמה, להציג הודעת שגיאה, לבצע פעולות שחזור כלשהן, או לסיים את עבודת התוכנית בצורה מסודרת.
3.  **שליטה על זרימת הביצוע:** מאפשר להמשיך את ביצוע התוכנית כאשר מתרחשת שגיאה, על ידי הפניית זרימת הביצוע לבלוק `except`.
4.  **שיפור חווית המשתמש:** בלוקים `try-except` מאפשרים להציג למשתמשים הודעות שגיאה אינפורמטיביות יותר, במקום פשוט לראות סיום תוכנית באופן לא תקין.

**דוגמה:**

```python
def divide(x: int, y: int) -> float:
    try:
      result: float = x / y
      print(f"תוצאה: {result}")
      return result
    except ZeroDivisionError:
       print("שגיאה: חלוקה באפס!")
       return None

divide(10, 2)   # פלט: תוצאה: 5.0
divide(10, 0) # פלט: שגיאה: חלוקה באפס!
```

**לסיכום:**

*   בדוגמה שלעיל, הפונקציה `divide()` מחלקת כראוי את המספר `x` ב-`y`.
*   אם `y` יהיה שווה ל-0, תיתפס החריגה `ZeroDivisionError`, והתוכנית תדפיס את ההודעה `"שגיאה: חלוקה באפס!"` ותחזיר `None`, במקום לסיים את עבודתה באופן לא תקין.

לפיכך, **אפשרות B** היא התשובה הנכונה.
