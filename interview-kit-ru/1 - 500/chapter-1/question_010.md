**שאלה 10.** התייחס/י לקטע קוד פייתון המטפל בחריגות בעת ניסיון להמיר מספר שלם מקלט משתמש, תוך שימוש בפונקציה `input()`. איזו מימוש מטפל בצורה נכונה בקלט שאינו מספר שלם חוקי, למשל 'пять', ומדפיס הודעת שגיאה?

```python
- A. try: num = int(input("Введите число: ")) except ValueError: print("Это не является действительным числом!")
- B. try: num = int(input("Введите число: ")) if not num: print("Это не является действительным числом!")
- C. num = int(input("Введите число: ")) except ValueError: print("Это не является действительным числом!")
- D. try: num = int(input("Введите число: ")) catch (ValueError): print("Это не является действительным числом!")
```

**תשובה נכונה: A**

**הסבר:**

לשם טיפול בחריגות המתרחשות בעת ניסיון להמיר קלט משתמש שאינו תקין למספר שלם (לדוגמה, בעת הזנת מילה במקום מספר), נעשה שימוש במבנה `try...except`.

*   בלוק `try`: מכיל קוד שעלול לגרום לחריגה. במקרה זה מדובר בניסיון המרה של המחרוזת שהתקבלה מהמשתמש למספר שלם באמצעות הפונקציה `int()`.
*   בלוק `except`: מכיל קוד שמבוצע אם בבלוק `try` מתרחשת החריגה המצוינת. במקרה שלנו זוהי החריגה `ValueError`, המתרחשת בעת ניסיון כושל להמיר מחרוזת למספר שלם.

כעת ננתח כל אפשרות:

*   **אפשרות A:** `try: num = int(input("Введите число: ")) except ValueError: print("Это не является действительным числом!")` - זהו מימוש נכון. הוא מנסה להמיר את הקלט למספר בצורה תקינה, ואם מתרחשת שגיאת `ValueError` (לדוגמה, אם המשתמש יזין 'пять'), התוכנית לוכדת את החריגה ומדפיסה את הודעת השגיאה המתאימה.
    
*   **אפשרות B:** `try: num = int(input("Введите число: ")) if not num: print("Это не является действительным числом!")` - מכיל שגיאת תחביר ולא יוכל לטפל בחריגה. התנאי `if not num:` לא יבוצע אם מתרחשת חריגת `ValueError`, מכיוון שלמשתנה `num` לא יוקצה ערך.
    
*   **אפשרות C:** `num = int(input("Введите число: ")) except ValueError: print("Это не является действительным числом!")` - חסרה מילת המפתח `try` לפתיחת הבלוק שעלול לגרום לחריגה, ולכן מתרחשת שגיאת תחביר.
    
*   **אפשרות D:** `try: num = int(input("Введите число: ")) catch (ValueError): print("Это не является действительным числом!")` - מכיל שגיאת תחביר: מילת המפתח `catch` אינה משמשת בפייתון לטיפול בחריגות; משתמשים ב-`except`.

**דוגמה:**

```python
# דוגמה למימוש הנכון (אפשרות A)
try:
    num: int = int(input("הכנס מספר: "))
    print(f"הכנסת את המספר: {num}")
except ValueError:
    print("זה אינו מספר חוקי!")

# דוגמה להכנסת ערך שגוי
try:
    num: int = int(input("הכנס מספר: "))
    print(f"הכנסת את המספר: {num}")
except ValueError:
    print("זה אינו מספר חוקי!")
```

**תוצאה:**

בדוגמה הראשונה אם יוזן "123" - יודפס "הכנסת את המספר: 123". בדוגמה השנייה אם יוזן "пять" - תיקלט חריגת `ValueError`, ויוצג המסר "זה אינו מספר חוקי!".

לפיכך, רק **אפשרות A** מטפלת בצורה נכונה בחריגת `ValueError` ומדפיסה את הודעת השגיאה המתאימה.
