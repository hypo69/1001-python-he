### `question_010.md`

**שאלה 10.** התבונן בבלוק קוד Python לטיפול בחריגות בעת ניסיון להמיר מספר שלם מתוך קלט משתמש, באמצעות הפונקציה `input()`. איזו מימוש מטפל נכונה בקלט שיכול להיות מספר שלם לא תקין, לדוגמה 'חמש', ומציג הודעת שגיאה?

```python
- A. try: num = int(input("Введите число: ")) except ValueError: print("Это не является действительным числом!")
- B. try: num = int(input("Введите число: ")) if not num: print("Это не является действительным числом!")
- C. num = int(input("Введите число: ")) except ValueError: print("Это не является действительным числом!")
- D. try: num = int(input("Введите число: ")) catch (ValueError): print("Это не является действительным числом!")
```

**תשובה נכונה: A**

**הסבר:**

לטיפול בחריגות המתחוללות בעת ניסיון להמיר קלט משתמש שגוי למספר שלם (לדוגמה, בעת הזנת מילה במקום מספר), משתמשים במבנה `try...except`.

*   **`try`** בלוק: מכיל קוד שיכול לגרום לחריגה. במקרה זה, מדובר בניסיון להמיר את המחרוזת המתקבלת מהמשתמש למספר שלם באמצעות הפונקציה `int()`.
*   **`except`** בלוק: מכיל קוד שמבוצע אם בבלוק ה-`try` מתרחשת החריגה המצוינת. במקרה שלנו זו החריגה `ValueError`, המתחוללת בעת ניסיון כושל להמיר מחרוזת למספר שלם.

כעת ננתח כל אפשרות:

*   **אפשרות A:** `try: num = int(input("Введите число: ")) except ValueError: print("Это не является действительным числом!")` - זהו המימוש הנכון. הוא מנסה נכונה להמיר את הקלט למספר, ואם מתרחשת שגיאה מסוג `ValueError` (לדוגמה, אם המשתמש יזין 'חמש'), התוכנית לוכדת את החריגה ומציגה את הודעת השגיאה המתאימה.

*   **אפשרות B:** `try: num = int(input("Введите число: ")) if not num: print("Это не является действительным числом!")` - מכיל שגיאת תחביר ולא יוכל לטפל בחריגה. `if not num:` לא יבוצע אם מתרחשת חריגה מסוג `ValueError`, מכיוון שלמשתנה `num` לא הוקצה ערך.

*   **אפשרות C:** `num = int(input("Введите число: ")) except ValueError: print("Это не является действительным числом!")` - חסרה מילת המפתח `try` לתחילת הבלוק שיכול לגרום לחריגה, ולכן מתרחשת שגיאת תחביר.

*   **אפשרות D:** `try: num = int(input("Введите число: ")) catch (ValueError): print("Это не является действительным числом!")` - מכיל שגיאת תחביר: מילת המפתח `catch` אינה משמשת ב-Python לטיפול בחריגות, אלא מילת המפתח `except`.

**דוגמה:**

```python
# דוגמה עם מימוש נכון (אפשרות A)
try:
    num: int = int(input("Введите число: "))
    print(f"Вы ввели число: {num}")
except ValueError:
    print("Это не является действительным числом!")

# דוגמה להזנת ערך לא תקין
try:
    num: int = int(input("Введите число: "))
    print(f"Вы ввели число: {num}")
except ValueError:
    print("Это не является действительным числом!")
```

**התוצאה:**

בדוגמה הראשונה, אם יוכנס "123" - יוצג "Вы ввели число: 123". בדוגמה השנייה, אם יוכנס 'חמש' - תילכד חריגה מסוג `ValueError`, ויוצג המסר "Это не является действительным числом!".

לסיכום, רק **אפשרות A** מטפלת נכונה בחריגה מסוג `ValueError` ומציגה את הודעת השגיאה המתאימה.
