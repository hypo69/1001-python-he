### `question_67.md`

**שאלה 67.** בפייתון, מהי התוצאה של המרת טיפוסים כאשר משתמשים בפונקציה `int()` על מספר נקודה צפה (float), כגון 7.7?

- א. היא מעגלת את המספר למספר השלם הקרוב ביותר.
- ב. היא משמיטה את החלק העשרוני ומחזירה מספר שלם.
- ג. היא מחזירה את המספר השלם הקרוב הקטן יותר אם החלק העשרוני נמוך מ-0.5, ואת הגדול יותר אם הוא גבוה יותר.
- ד. זה גורם לשגיאת `ValueError`, אלא אם כן החריגה מטופלת במפורש.

**תשובה נכונה: ב'**

**הסבר:**

הפונקציה `int()` בפייתון משמשת להמרת ערך למספר שלם. בעת המרת מספר נקודה צפה (`float`) למספר שלם (`int`), הפונקציה `int()` משמיטה את החלק העשרוני, ולא מעגלת.

*   **אפשרות א'** אינה נכונה: `int()` אינה מבצעת עיגול, אלא משמיטה את החלק העשרוני.
*   **אפשרות ב'** נכונה: `int()` אכן משמיטה את החלק העשרוני ומחזירה מספר שלם.
*   **אפשרות ג'** אינה נכונה: `int()` אינה מבצעת עיגול לפי הכללים "פחות מ-0.5 כלפי מטה, יותר - כלפי מעלה".
*   **אפשרות ד'** אינה נכונה: הפונקציה `int()` יכולה להמיר מחרוזת שמייצגת מספר שלם לטיפוס integer, אך לא מחרוזת עם נקודה צפה (לדוגמה "7.7"), והיא לא תגרום ל-`ValueError` בעת המרת מספר `float` ל-`int`.

**דוגמה:**

```python
float_number: float = 7.7
integer_number: int = int(float_number)
print(f"Исходное число: {float_number}")
print(f"Целое число после int(): {integer_number}")  # פלט: מספר שלם לאחר int(): 7

float_number: float = -7.7
integer_number: int = int(float_number)
print(f"Целое число после int(): {integer_number}") # פלט: מספר שלם לאחר int(): -7

float_number: float = 7.2
integer_number: int = int(float_number)
print(f"Целое число после int(): {integer_number}") # פלט: מספר שלם לאחר int(): 7
```

**תוצאה:**
* בעת המרה של `7.7` באמצעות `int()`, מוחזר `7`, החלק העשרוני מושמט.
* בעת המרה של `-7.7`, מוחזר `-7`.
* בעת המרה של `7.2`, מוחזר `7`.

לפיכך, **אפשרות ב'** היא התשובה הנכונה.
