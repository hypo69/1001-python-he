**שאלה 95.** כיצד מנוהל הזיכרון בפייתון?

- א. רק באמצעות ניהול זיכרון ידני.
- ב. באמצעות שימוש בערימה פרטית, המכילה את כל אובייקטי פייתון ומבני הנתונים.
- ג. באופן בלעדי דרך מערכת ההפעלה.
- ד. באמצעות מודל הקצאת זיכרון מבוסס ערימה (stack).

**תשובה נכונה: ב**

**הסבר:**

פייתון משתמש בניהול זיכרון אוטומטי ברוב המקרים. משמעות הדבר היא שמפתחים אינם צריכים להקצות ולשחרר זיכרון במפורש.

*   **אפשרות א** אינה נכונה: פייתון אינו משתמש בניהול זיכרון ידני, כמו שפת C למשל.
*   **אפשרות ב** נכונה: פייתון משתמש בערימה פרטית, המכילה את כל אובייקטי פייתון ומבני הנתונים.
*   **אפשרות ג** אינה נכונה: פייתון אינו מעביר את ניהול הזיכרון כולו למערכת ההפעלה.
*   **אפשרות ד** אינה נכונה: פייתון אינו משתמש במודל הקצאת זיכרון מבוסס ערימה (stack).

**מנגנוני ניהול זיכרון בפייתון:**

1.  **ערימה פרטית (private heap):** פייתון משתמש באזור זיכרון פרטי, שבו מאוחסנים כל אובייקטי פייתון ומבני הנתונים. למפתחים אין גישה ישירה לזיכרון זה.
2.  **איסוף אשפה אוטומטי (automatic garbage collection):** פייתון משתמש באוסף אשפה כדי לזהות ולשחרר זיכרון שאינו בשימוש עוד. אוסף האשפה פועל במצב אוטומטי.
3.  **ספירת הפניות (reference counting):** פייתון עוקב אחר מספר ההפניות לכל אובייקט בזיכרון. כאשר מספר ההפניות מגיע ל-0, האובייקט נמחק מהזיכרון.
4.  **ניהול דרך מנהל הזיכרון של פייתון:** הקצאת הזיכרון מנוהלת על ידי מנהל זיכרון פנימי, המנהל ביעילות את הקצאת ושחרור הזיכרון.

**דוגמה:**

```python
# יצירת אובייקט
my_list: list[int] = [1, 2, 3]
print(f"Объект my_list: {my_list}") # פלט: אובייקט my_list: [1, 2, 3]

# יצירת הפניה חדשה לאובייקט
my_list_2: list[int] = my_list

del my_list # מחיקת הפניה

# האובייקט יימחק על ידי אוסף האשפה כאשר לא יהיו עליו עוד הפניות
print(f"Объект my_list_2: {my_list_2}") # פלט: אובייקט my_list_2: [1, 2, 3]
```

**כתוצאה מכך:**
*   בעת יצירת `my_list`, זיכרון מוקצה בערימה הפרטית, ופייתון עוקב אחר זיכרון זה.
*   המשתנה `my_list_2` מפנה אף הוא לאזור זיכרון זה.
*   לאחר מחיקת `my_list`, האובייקט עדיין נגיש דרך ההפניה `my_list_2`. אוסף האשפה (GC) ישחרר את הזיכרון כאשר מונה ההפניות של האובייקט יגיע ל-0.

לפיכך, **אפשרות ב** היא הנכונה.
