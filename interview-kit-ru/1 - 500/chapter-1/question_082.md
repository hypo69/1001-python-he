### `question_082.md`

**שאלה 82** מה תהיה התוצאה של ביצוע הקוד הבא?

```python
def my_function(n):
    if n == 0:
       return 1
    else:
        return n * my_function(n-1)

print(my_function(5))
```

- א. 1
- ב. 5
- ג. 120
- ד. יגרום לשגיאת גלישת מחסנית (Stack Overflow).

**תשובה נכונה: ג**

**הסבר:**

קוד זה מממש פונקציה רקורסיבית לחישוב עצרת (פקטוריאל) של מספר. הפונקציה `my_function(n)` מחזירה n! (עצרת של n).

*   **נקודת יציאה מהרקורסיה:** אם `n` שווה ל-0, הפונקציה מחזירה 1 (עצרת של 0 שווה ל-1). תנאי זה הכרחי לסיום הקריאות הרקורסיביות.
*   **מקרה רקורסיבי:** אם `n` גדול מ-0, הפונקציה מחזירה את `n` כפול התוצאה של קריאה לעצמה עם `n-1`.

לפיכך, הקריאה ל-`my_function(5)` תתבצע באופן הבא:

```
my_function(5) = 5 * my_function(4)
my_function(4) = 4 * my_function(3)
my_function(3) = 3 * my_function(2)
my_function(2) = 2 * my_function(1)
my_function(1) = 1 * my_function(0)
my_function(0) = 1
```

בהצבת הערכים מלמטה למעלה, אנו מקבלים:

`1 * 1 * 2 * 3 * 4 * 5 = 120`

לפיכך, התוצאה של הקריאה ל-`print(my_function(5))` תהיה `120`.

**הערות נוספות:**
- פונקציות רקורסיביות חייבות לכלול תנאי יציאה מהרקורסיה על מנת למנוע קריאה אינסופית לפונקציות ושגיאת גלישת מחסנית (Stack Overflow).
- מימוש רקורסיבי זה של עצרת מתאים בהחלט למספרים קטנים, אך עבור מספרים גדולים מאוד הוא עשוי להיות פחות יעיל ממימוש איטרטיבי. להלן דוגמה למימוש איטרטיבי של עצרת:

```python
def factorial_iterative(n):
  result = 1
  for i in range(1, n + 1):
    result *= i
  return result
```

- מימוש איטרטיבי יעיל יותר לרוב מבחינת זיכרון, מאחר ואינו משתמש במחסנית הקריאות (Call Stack).
