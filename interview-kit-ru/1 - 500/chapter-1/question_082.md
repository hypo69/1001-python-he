### `question_082.md`

**שאלה 82** מהי תוצאת ההרצה של הקוד הבא?

```python
def my_function(n):
    if n == 0:
       return 1
    else:
        return n * my_function(n-1)

print(my_function(5))
```

- א. 1
- ב. 5
- ג. 120
- ד. תגרום לשגיאת גלישת מחסנית.

**תשובה נכונה: ג**

**הסבר:**

קוד זה מממש פונקציה רקורסיבית לחישוב עצרת של מספר. הפונקציה `my_function(n)` מחזירה את n! (עצרת של n).

*   **נקודת יציאה מהרקורסיה:** אם `n` שווה ל-0, הפונקציה מחזירה 1 (עצרת של 0 שווה ל-1). תנאי זה הכרחי לסיום הקריאות הרקורסיביות.
*   **המקרה הרקורסיבי:** אם `n` גדול מ-0, הפונקציה מחזירה את `n` מוכפל בתוצאת הקריאה לעצמה עם `n-1`.

לפיכך, קריאה ל-`my_function(5)` תתבצע באופן הבא:

```
my_function(5) = 5 * my_function(4)
my_function(4) = 4 * my_function(3)
my_function(3) = 3 * my_function(2)
my_function(2) = 2 * my_function(1)
my_function(1) = 1 * my_function(0)
my_function(0) = 1
```

בהצבת הערכים מלמטה למעלה, מקבלים:

`1 * 1 * 2 * 3 * 4 * 5 = 120`

לפיכך, תוצאת הקריאה ל-`print(my_function(5))` תהיה `120`.

**הערות נוספות:**
- פונקציות רקורסיביות חייבות לכלול תנאי יציאה מהרקורסיה, על מנת למנוע קריאות אינסופיות לפונקציה ושגיאת גלישת מחסנית.
- מימוש רקורסיבי זה של עצרת מתאים למספרים קטנים, אך למספרים גדולים מאוד הוא עשוי להיות פחות יעיל ממימוש איטרטיבי. הנה דוגמה למימוש איטרטיבי של עצרת:

```python
def factorial_iterative(n):
  result = 1
  for i in range(1, n + 1):
    result *= i
  return result
```

- מימוש איטרטיבי הוא לרוב יעיל יותר מבחינת זיכרון, מאחר שאינו משתמש במחסנית הקריאות.
