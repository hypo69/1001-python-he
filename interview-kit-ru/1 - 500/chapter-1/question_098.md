### `question_98.md`

**שאלה 98.** איזו פונקציה מובנית בפייתון מתאימה ביותר להערכה בטוחה של ביטוי מקלט משתמש?

- A.  `eval()`
- B.  `exec()`
- C.  `input()`
- D.  `ast.literal_eval()`

**תשובה נכונה: D**

**הסבר:**

בפייתון, להערכת ביטויים מקלט משתמש, חשוב להשתמש בפונקציות המספקות אבטחה כדי להימנע מחולשות.

*   **אפשרות A** אינה נכונה: `eval()` מבצעת מחרוזת קוד פייתון, מה שעלול להיות לא בטוח אם המחרוזת התקבלה ממשתמש (מכיוון שהמשתמש יכול להכניס קוד שרירותי)
*   **אפשרות B** אינה נכונה: `exec()` דומה ל-`eval()`, אך מבצעת קוד שרירותי, גם היא לא בטוחה בשימוש בקלט משתמש.
*   **אפשרות C** אינה נכונה: `input()` פשוט קוראת קלט משתמש כמחרוזת, אך אינה מעריכה את הביטוי.
*   **אפשרות D** נכונה: `ast.literal_eval()` מעריכה בבטחה רק ביטויים המהווים ליטרלים בפייתון (לדוגמה, מחרוזות, מספרים, רשימות, מילונים, טאפלים). היא אינה מאפשרת ביצוע קוד שרירותי.

**`eval()` לעומת `ast.literal_eval()`:**

*   **`eval()`:**
    *   מבצעת את המחרוזת כקוד פייתון.
    *   עלולה להיות מסוכנת, מכיוון שהיא מאפשרת לבצע כל קוד המועבר על ידי המשתמש.
*   **`ast.literal_eval()`:**
    *   מעריכה בבטחה רק ביטויים המהווים ליטרלים בפייתון.
    *   מוגבלת ביכולותיה, אך בטוחה יותר לקלט משתמש.

**דוגמה:**

```python
import ast

# דרך בטוחה באמצעות ast.literal_eval()
user_input: str = "{'a': 1, 'b': 2}"
try:
    result_safe = ast.literal_eval(user_input)
    print(f"Результат literal_eval: {result_safe}")
    print(f"Тип результата literal_eval: {type(result_safe)}") # פלט: Тип результата literal_eval: <class 'dict'>
except ValueError as e:
    print(f"Ошибка: {e}")

# דרך מסוכנת באמצעות eval()
user_input: str = "10 + 20"
result_unsafe: int = eval(user_input)
print(f"Результат eval: {result_unsafe}") # פלט: Результат eval: 30

# דרך מסוכנת באמצעות eval() - ביצוע קוד שרירותי
user_input_dangerous: str = "import os; os.system('rm -rf /')" # לעולם אל תעשו זאת
# eval(user_input_dangerous) # הרצת קוד זה עלולה לגרום לבעיות חמורות

```

**לסיכום:**
*   `ast.literal_eval()` ממירה בבטחה מחרוזת למילון.
*   `eval()` מבצעת את המחרוזת `"10 + 20"` ומחזירה את התוצאה `30`.
*   שימוש ב-`eval()` לביצוע מחרוזת המכילה פקודת מחיקה, יכול להיות מסוכן מאוד.

לפיכך, **אפשרות D** היא הנכונה.
