### `question_98.md`

**שאלה 98.** איזו פונקציה מובנית ב-Python מתאימה ביותר להערכה בטוחה של ביטוי מקלט משתמש?

- A.  `eval()`
- B.  `exec()`
- C.  `input()`
- D.  `ast.literal_eval()`

**התשובה הנכונה: D**

**הסבר:**

ב-Python, להערכת ביטויים מקלט משתמש, חשוב להשתמש בפונקציות שמבטיחות אבטחה כדי להימנע מחולשות.

*   **אפשרות A** אינה נכונה: `eval()` מבצעת מחרוזת קוד Python, מה שיכול להיות לא בטוח אם המחרוזת מתקבלת ממשתמש (מכיוון שהמשתמש יכול להכניס קוד שרירותי).
*   **אפשרות B** אינה נכונה: `exec()` דומה ל-`eval()`, אך מבצעת קוד שרירותי, וגם היא לא בטוחה בעת שימוש בקלט משתמש.
*   **אפשרות C** אינה נכונה: `input()` פשוט קוראת את קלט המשתמש כמחרוזת, אך אינה מעריכה ביטוי.
*   **אפשרות D** נכונה: `ast.literal_eval()` מעריכה בבטחה רק ביטויים המייצגים ליטרלים של Python (לדוגמה, מחרוזות, מספרים, רשימות, מילונים, טאפלים). היא אינה מאפשרת ביצוע קוד שרירותי.

**`eval()` לעומת `ast.literal_eval()`:**

*   **`eval()`:**
    *   מבצעת מחרוזת כקוד Python.
    *   יכולה להיות מסוכנת, מכיוון שהיא מאפשרת לבצע כל קוד שמסופק על ידי המשתמש.
*   **`ast.literal_eval()`:**
    *   מעריכה בבטחה רק ביטויים המייצגים ליטרלים של Python.
    *   מוגבלת באפשרויות, אך בטוחה יותר עבור קלט משתמש.

**דוגמה:**

```python
import ast

# שיטה בטוחה באמצעות ast.literal_eval()
user_input: str = "{'a': 1, 'b': 2}"
try:
    result_safe = ast.literal_eval(user_input)
    print(f"Результат literal_eval: {result_safe}")
    print(f"Тип результата literal_eval: {type(result_safe)}") # פלט: Тип результата literal_eval: <class 'dict'>
except ValueError as e:
    print(f"Ошибка: {e}")

# שיטה מסוכנת באמצעות eval()
user_input: str = "10 + 20"
result_unsafe: int = eval(user_input)
print(f"Результат eval: {result_unsafe}") # פלט: Результат eval: 30

# שיטה מסוכנת באמצעות eval() - ביצוע קוד שרירותי
user_input_dangerous: str = "import os; os.system('rm -rf /')" # לעולם אל תעשו זאת
# eval(user_input_dangerous) # הפעלת קוד זה עלולה לגרום לבעיות חמורות

```

**כתוצאה מכך:**
*   `ast.literal_eval()` ממירה בבטחה את המחרוזת למילון.
*   `eval()` מבצעת את המחרוזת `"10 + 20"` ומחזירה את התוצאה `30`.
*   שימוש ב-`eval()` לביצוע מחרוזת המכילה פקודת מחיקה עלול להיות מסוכן ביותר.

לפיכך, **אפשרות D** היא הנכונה.
