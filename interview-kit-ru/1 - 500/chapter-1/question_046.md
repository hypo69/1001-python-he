### `question_046.md`

**שאלה 46.** מהי הפונקציונליות של הפונקציה `super()` בפייתון, במיוחד בהקשר של תכנות מונחה עצמים והיררכיות ירושה?

- א. הפונקציה `super()` משמשת להחזרת אובייקט מתווך (proxy object) אשר מפצל קריאות למתודות למחלקת ההורה או למחלקה אחות, ובכך מאפשר גישה למתודות בירושה שייתכן שנדרסו במחלקה הנוכחית.
- ב. היא משמשת כמנגנון לעקיפת דריסה של מתודות במחלקות צאצא, המבטיח שמתודה ממחלקת האב לא תידרס באף מחלקת צאצא.
- ג. פייתון משתמשת ב-`super()` כדי לזהות ולקרוא באופן אוטומטי לבנאי של מחלקת הבסיס, ללא קשר לאיזו מתודה או תכונה נמצאת בשימוש.
- ד. הפונקציה מאתחלת את כל המשתנים במחלקת ההורה בערכי ברירת מחדל, ללא קשר לאופן שבו הם אותחלו במחלקת הצאצא.

**תשובה נכונה: א**

**הסבר:**

הפונקציה `super()` בפייתון היא כלי חשוב בתכנות מונחה עצמים, במיוחד בעבודה עם היררכיית מחלקות וירושה מרובה.

*   **אפשרות א** נכונה: `super()` יוצרת אובייקט מתווך (proxy) שמנתב מחדש קריאות למתודות אל מחלקת ההורה.
*   **אפשרות ב** אינה נכונה: `super()` אינו עוקף דריסה של מתודות, אלא להיפך, משמש עבור קריאה למתודות שנדרסו במחלקת ההורה.
*   **אפשרות ג** אינה נכונה: `super()` אינו משמש לקריאה אוטומטית לבנאי של מחלקת הבסיס. קריאה זו מתבצעת בירושה באמצעות קריאה ל-`super().__init__()`.
*   **אפשרות ד** אינה נכונה: `super()` אינו מאתחל משתנים במחלקת ההורה בערכי ברירת מחדל, אלא קורא למתודות של מחלקת הבסיס.

**כיצד `super()` פועל:**

1.  **גישה למחלקת ההורה:** `super()` מאפשר למחלקת צאצא לקרוא למתודות של מחלקת ההורה. זה שימושי במיוחד כאשר מחלקת צאצא דורסת מתודה של ההורה, אך עדיין נדרש לבצע את הגרסה המקורית של המתודה מההורה.
2.  **ירושה מרובה:** במקרה של ירושה מרובה, `super()` עוזר לנהל קריאות למתודות בסדר הנכון (סדר רזולוציית המתודות - Method Resolution Order - MRO).
3.  **אתחול מחלקת ההורה:** לעיתים קרובות נעשה שימוש בבנאים `__init__()` כדי לקרוא לבנאי של מחלקת ההורה ולבצע את האתחול שלו.

**דוגמה:**

```python
class Parent:
    def __init__(self, name: str):
        self.name = name
        # מחלקת הורה: name {self.name}
        print(f"Parent class: name {self.name}")

    def speak(self):
      # מחלקת הורה: מדבר
      print("Parent class: Speaks")

class Child(Parent):
    def __init__(self, name: str, age: int):
      super().__init__(name)  # קוראים לבנאי של מחלקת ההורה
      self.age = age
      # מחלקת צאצא: age = {self.age}
      print(f"Child class: age = {self.age}")
    def speak(self):
        super().speak()
        # מחלקת צאצא: מדבר
        print("Child class: Speaks")

child = Child("Bob", 25)
# פלט:
# Parent class: name Bob
# Child class: age = 25
child.speak()
# פלט:
# Parent class: Speaks
# Child class: Speaks
```

**בתוצאה:**
*   `super().__init__(name)` במחלקת הצאצא מאפשר קריאה למתודה `__init__()` ממחלקת ההורה.
*   `super().speak()` במחלקת הצאצא קורא למתודה `speak` ממחלקת ההורה, ואז מדפיס משהו משלו.

לפיכך, **אפשרות א** היא התשובה הנכונה.
