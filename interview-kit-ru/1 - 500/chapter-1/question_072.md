**שאלה 072.** מהן מונדות בפייתון, וכיצד הן משמשות? ספק דוגמה לשימוש במונדת `Maybe` לטיפול בשגיאות אפשריות או בהיעדר ערכים.

*   A. מונדות הן טיפוסי נתונים מובנים בפייתון לאחסון רצפים של מספרים.
*   B. מונדות הן דרך לארגון קוד, כדי להשתמש ביותר משתנים מבלי לשנות את ביצועי התוכנית.
*   C. מונדות הן מבנה מופשט המשמש בתכנות פונקציונלי לבניית שרשרת חישובים, ומונדת `Maybe` משמשת לטיפול בשגיאות אפשריות או בהיעדר ערכים.
*   D. מונדות הן אובייקטים המשמשים לתיאור היררכיית מחלקות בפייתון.

**תשובה נכונה: C**

**הסבר:**

מונדות בפייתון (כמו גם בשפות תכנות אחרות המושפעות מגישה פונקציונלית) הן קונספט מופשט המאפשר לבנות ולשרשר פעולות וחישובים באופן מובנה. מונדות מספקות דרך להרכיב חישובים, במיוחד כאשר חישובים אלו עשויים להיות בלתי מוגדרים או עשויים לכלול תופעות לוואי (side effects).

*   **קונספטים מרכזיים של מונדות:**
    *   **מכילים:** מונדות בדרך כלל מייצגות מכילים (containers) לערכים.
    *   **הרכבה (Composition):** מונדות מאפשרות לשלב (להרכיב) פונקציות בשרשרת, כך שהפלט של פונקציה אחת הופך לקלט עבור הבאה בתור.
    *   **ניהול תופעות לוואי:** מונדות מסייעות לבודד ולנהל תופעות לוואי (לדוגמה, שגיאות, קלט/פלט) בתוך שרשרת החישובים.
*   **מונדת `Maybe` (או `Option`):**
    *   משמשת לייצוג ערך שעשוי להיות קיים או לא (כלומר, זהו או ערך מטיפוס מסוים, או "שום דבר" (`None`)).
    *   מאפשרת לטפל בשגיאות אפשריות או בהיעדר ערכים (לדוגמה, כאשר תוצאת חלוקה עשויה להיות חלוקה באפס).
    *   מסייעת למנוע חריגות (exceptions), אם אחד משלבי החישוב עשוי להחזיר ערך לא תקין.

**דוגמה (מטקסט השאלה):**

```python
class Maybe:
    def __init__(self, value=None):
        # בנאי המחלקה
        self.value = value

    def bind(self, func):
        # שיטה המקשרת Maybe לפונקציה אחרת
        # אם הערך הנוכחי הוא None, החזר Maybe() (כלומר None)
        if self.value is None:
            return Maybe()
        # אחרת, קרא לפונקציה func עם הערך הנוכחי והחזר את התוצאה עטופה ב-Maybe
        else:
            return func(self.value)

def safe_divide(x, y):
    # פונקציה המחלקת x ב-y באופן בטוח
    # אם y שווה לאפס, החזר Maybe() (כלומר None)
    if y == 0:
        return Maybe()
    # אחרת, החזר את התוצאה עטופה ב-Maybe
    else:
        return Maybe(x / y)

# אתחול עם Maybe(10), ושרשור פעולות באמצעות bind
result = Maybe(10).bind(lambda x: safe_divide(x, 2)).bind(lambda x: safe_divide(x, 0))

# הדפסת הערך
print(result.value)  # None
```

**ניתוח הדוגמה:**

1.  `class Maybe:`: יצירת מחלקה המממשת את מונדת `Maybe`.
2.  `__init__`: בנאי המחלקה.
3.  `bind(self, func)`: שיטה המקשרת `Maybe` לפונקציה אחרת. אם הערך הנוכחי של `Maybe` הוא `None`, אז מחזירה `Maybe()` (שמשמעו `None`). אחרת - תקרא לפונקציה `func` עם הערך הנוכחי.
4.  `safe_divide(x, y)`: פונקציה המחלקת את `x` ב-`y` באופן בטוח, ומחזירה `Maybe()` אם `y` שווה לאפס.
5.  `result`: מאותחל עם `Maybe(10)`, ולאחר מכן מבוצעת שרשרת פעולות, כאשר כל `bind` מקשר את תוצאת השלב הקודם לחישוב הבא.
6.  מכיוון שהחלוקה באפס תוביל להחזרת `None`, וכל קריאות ה-`bind` הבאות יחזירו גם הן `None`, בסופו של דבר `result.value` יהיה שווה ל-`None`.

**ניתוח האפשרויות:**

*   **A. מונדות — זהו טיפוס נתונים מובנה בפייתון לאחסון רצפים של מספרים:** לא נכון. מונדות הן קונספט מופשט, לא טיפוס נתונים מובנה.
*   **B. מונדות — זוהי דרך לארגון קוד, כדי להשתמש ביותר משתנים מבלי לשנות את ביצועי התוכנית:** לא נכון. מונדות אינן לניהול כמות משתנים.
*   **C. מונדות — זהו מבנה מופשט המשמש בתכנות פונקציונלי לבניית שרשרת חישובים, ומונדת `Maybe` משמשת לטיפול בשגיאות אפשריות או בהיעדר ערכים:** נכון.
*   **D. מונדות - אלו הם אובייקטים המשמשים לתיאור היררכיית מחלקות בפייתון:** לא נכון.

*   מונדת `Maybe` מסייעת לטפל בשגיאות אפשריות או בהיעדר ערכים, ומבטיחה קוד בטוח יותר.
