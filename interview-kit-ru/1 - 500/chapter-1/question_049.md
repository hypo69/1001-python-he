**שאלה 49.** כיצד Python מנהלת זיכרון, במיוחד בנוגע ליצירה והשמדה של אובייקטים?

- A. Python משתמשת במודל ניהול זיכרון ידני, שבו מפתחים נדרשים להקצות ולשחרר זיכרון במפורש, באמצעות קריאות מערכת.
- B. היא מיישמת מערכת אוטומטית המשתמשת בשילוב של ספירת הפניות (reference counting) ומאסף אשפה (garbage collector) לניהול זיכרון, המטפלת באופן דינמי בהקצאה ובשחרור של אובייקטים.
- C. ניהול זיכרון ב-Python מתבצע באמצעות החלפת קבצים (file swapping) כפויה, שבה נתונים נשמרים זמנית על דיסק במהלך הביצוע.
- D. Python מאפשרת למתכנתים להתאים אישית את אלגוריתם הקצאת הזיכרון בזמן אמת, תוך אופטימיזציה של הביצועים עבור סוגים ספציפיים של מבני נתונים.

**תשובה נכונה: B**

**הסבר:**

Python משתמשת בניהול זיכרון אוטומטי, הכולל שני מנגנונים עיקריים:

1.  **ספירת הפניות:** Python עוקבת אוטומטית אחר מספר ההפניות לכל אובייקט בזיכרון. כאשר מספר ההפניות לאובייקט יורד לאפס, האובייקט נחשב לבלתי-שימושי ומוכן להסרה.
2.  **מאסף אשפה:** ל-Python יש גם מאסף אשפה, הפועל מעת לעת ומאתר אובייקטים שכבר אין אליהם הפניות (לדוגמה, הפניות מעגליות שספירת הפניות אינה יכולה לזהות), ומשחרר את הזיכרון שהם תופסים.

*   **אפשרות A** אינה נכונה: Python אינה משתמשת בניהול זיכרון ידני, כמו בשפות C או C++.
*   **אפשרות B** נכונה: Python אכן משתמשת במערכת אוטומטית הכוללת ספירת הפניות ומאסף אשפה.
*   **אפשרות C** אינה נכונה: Python אינה משתמשת בהחלפת קבצים לניהול זיכרון.
*   **אפשרות D** אינה נכונה: Python אינה מאפשרת למתכנתים להתאים אישית את אלגוריתם הקצאת הזיכרון בזמן אמת.

**הסבר מפורט:**

*   **ספירת הפניות:**
    *   כאשר אובייקט נוצר, מונה ההפניות מאופס ל-1.
    *   בעת הקצאת אובייקט למשתנה חדש או העברתו לפונקציה, מונה ההפניות גדל.
    *   כאשר משתנה המפנה לאובייקט יוצא מתחום ההיקף או מוקצה לו ערך אחר, מונה ההפניות קטן.
    *   כאשר מונה ההפניות מגיע לאפס, Python משחררת אוטומטית את הזיכרון הנצרך על ידי האובייקט.
*   **מאסף אשפה:**
    *   ספירת הפניות אינה יכולה להתמודד עם מצבים שבהם אובייקטים מפנים זה לזה באופן מעגלי (הנקראים הפניות מעגליות). במקרה זה, האובייקטים עשויים לא להימחק, גם אם אין אליהם הפניות ישירות.
    *   מאסף האשפה סורק מעת לעת את הזיכרון בחיפוש אחר הפניות מעגליות כאלו ומוחק אובייקטים המעורבים במחזורים אלו, כאשר הם אינם בשימוש יותר.

**יתרונות ניהול זיכרון אוטומטי:**

1.  **פישוט פיתוח:** מפתחים אינם צריכים להקצות או לשחרר זיכרון באופן מפורש, מה שמפחית את מורכבות הקוד ואת הסיכון לדליפות זיכרון.
2.  **הגברת מהימנות:** ניהול זיכרון אוטומטי מפחית את מספר השגיאות הקשורות לניהול זיכרון.
3.  **יעילות:** ניהול הזיכרון האוטומטי ב-Python יעיל מספיק ברוב המקרים, ומאפשר חיסכון בזיכרון.

לפיכך, **אפשרות B** היא התשובה הנכונה.
