### `question_032.md`

**שאלה 32.** מהן ההשלכות והיישומים הטיפוסיים של מילת המפתח `yield` במימוש גנרטורים בפייתון, בהתחשב בתועלת שלה בניהול זיכרון ובקרת זרימה במערכי נתונים גדולים?

- א. מילת המפתח `yield` גורמת לפונקציה להחזיר גנרטור הניתן לאיטרציה, ומאפשרת לפייתון לייצר (generate) איברים באופן עצל (lazily), אחד בכל פעם ורק לפי הצורך, ובכך לחסוך בזיכרון.
- ב. היא גורמת לסיום מיידי של ביצוע פונקציית הגנרטור, ומשחררת את כל המשאבים שהיא רכשה במהלך פעולתה.
- ג. `Yield` ממירה כל פונקציית פייתון לתת-שגרה מרובת תהליכים, המסוגלת לפעול במקביל לפונקציות אחרות.
- ד. היא משמשת ככלי איתור באגים (debugging) בפייתון, המאפשר למפתחים לעקוב אחר ערכים המוחזרים מפונקציות בזמן ריצה, מבלי להשפיע על ביצוע הפונקציה.

**תשובה נכונה: א**

**הסבר:**

מילת המפתח `yield` ממלאת תפקיד חשוב בהגדרת גנרטורים בפייתון. גנרטורים הם סוג מיוחד של פונקציות המחזירות *איטרטורים*. הם מאפשרים ליצור רצף של ערכים באופן עצל, כלומר, הערכים מיוצרים רק לפי דרישה (כאשר ניגשים אליהם), ולא מראש.

*   **אפשרות א'** נכונה: `yield` הופך פונקציה לגנרטור שמחזיר ערכים בעצלתיים אחד בכל פעם, ובכך חוסך זיכרון.
*   **אפשרות ב'** אינה נכונה: `yield` אינו מסיים את ביצוע הפונקציה, אלא משעה אותה ושומר את המצב, תוך החזרת ערך.
*   **אפשרות ג'** אינה נכונה: `yield` אינו הופך פונקציה למרובת תהליכים.
*   **אפשרות ד'** אינה נכונה: `yield` אינו כלי לאיתור באגים.

**הבדלים עיקריים מ-`return`:**

*   `return`: מסיים את ביצוע הפונקציה ומחזיר ערך בודד. לאחר `return`, הפונקציה מפסיקה לפעול.
*   `yield`: משעה את ביצוע הפונקציה, מחזיר את הערך הנוכחי ושומר את מצב הפונקציה. בבקשה הבאה לאיבר הבא, הפונקציה מחדשת את ביצועה מהנקודה בה היא עצרה ב-`yield`.

**יתרונות הגנרטורים:**

*   **חיסכון בזיכרון:** גנרטורים אינם מאחסנים את כל הערכים בזיכרון בו-זמנית, אלא מייצרים אותם לפי דרישה. זה מועיל במיוחד בעבודה עם מערכי נתונים גדולים.
*   **חישובים עצלים (Lazy evaluation):** ערכים מיוצרים רק כאשר הם אכן נדרשים.
*   **נוחות באיטרציה:** גנרטורים מחזירים איטרטורים שניתן להשתמש בהם בלולאות `for`.

**דוגמה:**

```python
def my_generator(n: int) -> int:
    for i in range(n):
        yield i * 2

gen = my_generator(5)

for value in gen:
    print(value)
    # Output:
    # 0
    # 2
    # 4
    # 6
    # 8
```

**בסיכום:**

הפונקציה `my_generator` משתמשת ב-`yield` כדי לייצר ערכים בזה אחר זה. בכל בקשה לערך הבא באמצעות לולאת `for`, הפונקציה מחדשת את ביצועה ומחזירה את הערך הבא. בכך, כל הערכים אינם נשמרים בזיכרון, מה שחוסך מקום.

לפיכך, **אפשרות א'** היא התשובה הנכונה.
