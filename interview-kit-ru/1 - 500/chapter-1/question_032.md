### `question_032.md`

**שאלה 32.** מהן ההשלכות והיישומים הטיפוסיים של מילת המפתח `yield` במימוש מחוללי (generators) פייתון, בהתחשב בתועלתה בניהול זיכרון ובבקרת זרימה במערכי נתונים גדולים?

- א. מילת המפתח `yield` גורמת לפונקציה להחזיר מחולל (generator), שניתן לבצע עליו איטרציה. זה מאפשר לפייתון לייצר (generate) איברים באופן הדרגתי, אחד אחד ורק לפי הצורך, ובכך לחסוך בזיכרון.
- ב. היא גורמת לסיום מיידי של ביצוע פונקציית המחולל, ושחרור כל המשאבים שהיא צברה במהלך פעולתה.
- ג. `Yield` ממירה כל פונקציית פייתון לשגרת משנה (subroutine) מרובת-תהליכים, המסוגלת לפעול במקביל לפונקציות אחרות.
- ד. היא משמשת ככלי איתור באגים (debugging) בפייתון, המאפשר למפתחים לעקוב אחר ערכים המוחזרים מפונקציות בזמן הריצה, מבלי להשפיע על ביצוע הפונקציה.

**תשובה נכונה: א**

**הסבר:**

מילת המפתח `yield` ממלאת תפקיד חשוב בהגדרת מחוללים בפייתון. מחוללים הם סוג מיוחד של פונקציות שמחזירות *איטרטורים* (iterators). הם מאפשרים ליצור סדרת ערכים באופן עצלני (lazy), כלומר, הערכים מיוצרים (generated) רק לפי דרישה (כאשר ניגשים אליהם), ולא מראש.

*   **אפשרות א'** נכונה: `yield` הופכת פונקציה למחולל שמחזיר ערכים באופן הדרגתי, אחד אחד, ובכך חוסך בזיכרון.
*   **אפשרות ב'** אינה נכונה: `yield` אינה מסיימת את ביצוע הפונקציה, אלא משעה אותה ושומרת את מצבה, תוך החזרת ערך.
*   **אפשרות ג'** אינה נכונה: `yield` אינה הופכת פונקציה למרובת-תהליכים.
*   **אפשרות ד'** אינה נכונה: `yield` אינה כלי לאיתור באגים.

**הבדלים עיקריים מ-`return`:**

*   `return`:  מסיימת את ביצוע הפונקציה ומחזירה ערך יחיד. לאחר `return`, הפונקציה מפסיקה את פעולתה.
*   `yield`:  משעה את ביצוע הפונקציה, מחזירה את הערך הנוכחי ושומרת את מצב הפונקציה. בבקשה הבאה לאיבר הבא, הפונקציה מחדשת את ביצועה מהמקום בו היא עצרה ב-`yield`.

**יתרונות המחוללים:**

*   **חיסכון בזיכרון:** מחוללים אינם שומרים את כל הערכים בזיכרון בו-זמנית, אלא מייצרים אותם לפי דרישה. זה שימושי במיוחד בעבודה עם מערכי נתונים גדולים.
*   **חישוב עצלני:** ערכים מיוצרים רק כאשר הם נחוצים באמת.
*   **נוחות איטרציה:** מחוללים מחזירים איטרטורים, שניתן להשתמש בהם בלולאות `for`.

**דוגמה:**

```python
def my_generator(n: int) -> int:
    for i in range(n):
        yield i * 2

gen = my_generator(5)

for value in gen:
    print(value)
    # פלט:
    # 0
    # 2
    # 4
    # 6
    # 8
```

**בתוצאה:**

הפונקציה `my_generator` משתמשת ב-`yield` כדי לייצר ערכים אחד אחרי השני. בכל בקשה לערך הבא באמצעות לולאת `for`, הפונקציה מחדשת את ביצועה ומחזירה את הערך הבא. בו-זמנית, כל הערכים אינם מאוחסנים בזיכרון, ובכך נחסך מקום.

לכן, **אפשרות א'** היא התשובה הנכונה.
