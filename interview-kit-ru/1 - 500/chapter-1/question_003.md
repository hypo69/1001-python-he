### `question_003.md`

**שאלה 3.** מהן ההשלכות של שימוש בארגומנטים ברי-שינוי כברירת מחדל בהגדרות פונקציות ב-Python, במיוחד במונחים של שימור אובייקט הפונקציה בין קריאות מרובות?

- א. ארגומנטים ברי-שינוי כברירת מחדל מועילים מכיוון שהם מאפשרים לעדכן ולשמור את ערכי ברירת המחדל של הפונקציה בין קריאות מרובות לפונקציה, המשקפים את השינויים האחרונים.
- ב. שימוש בארגומנטים ברי-שינוי כברירת מחדל עלול להוביל להתנהגות בלתי צפויה או לשגיאות, מכיוון ששינויים בארגומנטים אלו נשמרים בין קריאות פונקציה עוקבות, אלא אם כן הם אותחלו במפורש.
- ג. ארגומנטים שאינם ברי-שינוי כברירת מחדל, בניגוד לברי-שינוי, מאטים באופן משמעותי את ביצוע הפונקציות, ומאלצים את Python ליצור מחדש את ערך ברירת המחדל בכל קריאה לפונקציה.
- ד. ארגומנטים ברי-שינוי כברירת מחדל מונעים דליפות זיכרון על ידי איפוס אוטומטי של מצב הפונקציה לאחר כל קריאה, ומבטיחים ששום נתונים שיירים אינם נשמרים.

**תשובה נכונה: B**

**הסבר:**

שימוש באובייקטים ברי-שינוי (כגון רשימות או מילונים) כערכי ברירת מחדל לפרמטרים של פונקציות ב-Python עלול להוביל להתנהגות שאינה ברורה מאליה. הבעיה נעוצה בכך שערכי ברירת המחדל מחושבים *פעם אחת בלבד*, בעת הגדרת הפונקציה, ולא בכל קריאה אליה. הדבר גורם לכך שאותו אובייקט ברי-שינוי משמש כברירת מחדל בכל הקריאות, מה שעלול להוביל לתוצאות בלתי צפויות כאשר הפונקציה משנה את ערך ברירת המחדל הזה.

*   **אפשרות א'** שגויה: השינויים בערכים ברי-השינוי כברירת מחדל נשמרים בין הקריאות, ולא מתעדכנים. זו הסיבה לבעיות, ולא יתרון.
*   **אפשרות ב'** נכונה: זהו תיאור נכון של הבעיה: שינויים בערכי ברירת המחדל נשמרים בקריאות עתידיות לפונקציה, מה שעלול להוביל לשגיאות.
*   **אפשרות ג'** שגויה: ערכי ברירת מחדל שאינם ברי-שינוי (כגון מספרים, מחרוזות, טאפלים) אינם גורמים לבעיות של שימור מצב.
*   **אפשרות ד'** שגויה: ערכים ברי-שינוי כברירת מחדל דווקא יכולים להוביל לדליפות זיכרון.

**דוגמה:**

```python
def append_to_list(item: int, my_list: list[int] = []):
    """מוסיף אלמנט לרשימה, תוך שימוש בארגומנט ברי-שינוי כברירת מחדל."""
    my_list.append(item)
    print(my_list)


append_to_list(1)       # פלט: [1]
append_to_list(2)       # פלט: [1, 2]  (בלתי צפוי!)
append_to_list(3, [])   # פלט: [3] (רגיל, רשימה הועברה במפורש)
append_to_list(4)       # פלט: [1, 2, 4] (בלתי צפוי!)
```

**התוצאה:**

*   בקריאה הראשונה לפונקציה `append_to_list(1)`, נוצרת רשימה `[]`, ו-`1` מתווסף אליה, הפלט הוא `[1]`.
*   בקריאה השנייה `append_to_list(2)`, **נעשה שימוש באותה רשימה שנוצרה קודם לכן**, לכן `2` מתווסף אליה, והפלט הוא `[1, 2]`.
*   בקריאה `append_to_list(3, [])`, מועברת לפונקציה רשימה חדשה, וערך ברירת המחדל אינו בשימוש, הפלט הוא `[3]`.
*   בקריאה האחרונה, שוב נעשה שימוש ברשימה הישנה משתי הקריאות הראשונות, ו-4 מתווסף אליה.

כדי להימנע מבעיה כזו, נהוג להשתמש בערך `None` כערך ברירת מחדל ולבצע בדיקה עבור `None` בתוך הפונקציה:
```python
def append_to_list_correct(item: int, my_list: list[int] | None = None):
    """מוסיף אלמנט לרשימה, תוך שימוש ב-None כארגומנט ברירת מחדל."""
    if my_list is None:
        my_list = []
    my_list.append(item)
    print(my_list)
append_to_list_correct(1)
append_to_list_correct(2)
append_to_list_correct(3,[])
append_to_list_correct(4)
# פלט:
# [1]
# [2]
# [3]
# [4]
```

לפיכך, **אפשרות ב'** נכונה, מכיוון שהיא מתארת במדויק את הבעיה ואת השלכותיה.
