### `question_003.md`

**שאלה 3.** מהן ההשלכות של שימוש בארגומנטים ברי שינוי כברירת מחדל בהגדרות פונקציות בפייתון, במיוחד בהיבט של שמירת אובייקט הפונקציה בין קריאות מרובות?

- א. ארגומנטים ברי שינוי כברירת מחדל מועילים, שכן הם מאפשרים לעדכן ולשמור את ערכי ברירת המחדל של הפונקציה בין קריאות מרובות, ומשקפים את השינויים האחרונים.
- ב. שימוש בארגומנטים ברי שינוי כברירת מחדל עלול להוביל להתנהגות בלתי צפויה או לשגיאות, שכן שינויים בארגומנטים אלה נשמרים בין קריאות פונקציה עוקבות, אם לא אופסו במפורש.
- ג. ארגומנטים שאינם ברי שינוי כברירת מחדל, בניגוד לארגומנטים ברי שינוי, מאטים משמעותית את ביצוע הפונקציות, ומאלצים את פייתון ליצור מחדש את ערך ברירת המחדל בכל קריאה לפונקציה.
- ד. ארגומנטים ברי שינוי כברירת מחדל מונעים דליפות זיכרון על ידי איפוס אוטומטי של מצב הפונקציה לאחר כל קריאה, מה שמבטיח שלא יישמרו נתונים שיוריים.

**תשובה נכונה: ב**

**הסבר:**

שימוש באובייקטים ברי שינוי (כגון רשימות או מילונים) כערכי ברירת מחדל עבור פרמטרים של פונקציות בפייתון עלול להוביל להתנהגות שאינה ברורה מאליה. הבעיה טמונה בכך שערכי ברירת המחדל מחושבים רק *פעם אחת*, בעת הגדרת הפונקציה, ולא בכל קריאה שלה. זה גורם לכך שאותו אובייקט בר שינוי משמש כברירת מחדל בכל הקריאות, מה שעלול להוביל לתוצאות בלתי צפויות כאשר הפונקציה משנה את ערך ברירת מחדל זה.

*   **אפשרות א** אינה נכונה: שינויים בערכי ברירת מחדל ברי שינוי נשמרים בין קריאות, ולא מעודכנים. זוהי הסיבה לבעיות, לא יתרון.
*   **אפשרות ב** נכונה: זהו תיאור מדויק של הבעיה: שינויים בערכי ברירת המחדל נשמרים בין קריאות עתידיות של הפונקציה, מה שעלול להוביל לשגיאות.
*   **אפשרות ג** אינה נכונה: ערכי ברירת מחדל שאינם ברי שינוי (כגון מספרים, מחרוזות, טאפלים) אינם גורמים לבעיות של שמירת מצב.
*   **אפשרות ד** אינה נכונה: ערכי ברירת מחדל ברי שינוי דווקא עלולים להוביל לדליפות זיכרון.

**דוגמה:**

```python
def append_to_list(item: int, my_list: list[int] = []):
    """מוסיף פריט לרשימה, תוך שימוש בארגומנט בר שינוי כברירת מחדל."""
    my_list.append(item)
    print(my_list)


append_to_list(1)       # פלט: [1]
append_to_list(2)       # פלט: [1, 2]  (בלתי צפוי!)
append_to_list(3, [])   # פלט: [3] (רגיל, רשימה הועברה במפורש)
append_to_list(4)       # פלט: [1, 2, 4] (בלתי צפוי!)
```

**תוצאה:**

*   בקריאה הראשונה לפונקציה `append_to_list(1)`, נוצרת רשימה `[]`, ו-`1` מתווסף אליה, הפלט הוא `[1]`.
*   בקריאה השנייה `append_to_list(2)`, **נעשה שימוש באותה רשימה שנוצרה קודם**, לכן `2` מתווסף אליה, והפלט הוא `[1, 2]`.
*   בקריאה `append_to_list(3, [])`, מועברת לפונקציה רשימה חדשה, וערך ברירת המחדל אינו בשימוש, הפלט `[3]`.
*   בקריאה האחרונה, שוב נעשה שימוש ברשימה הישנה משתי הקריאות הראשונות, ויוסף אליה 4.

כדי להימנע מבעיה כזו, לעיתים קרובות משתמשים בערך `None` כערך ברירת מחדל ומבצעים בדיקה עבור `None` בתוך הפונקציה:
```python
def append_to_list_correct(item: int, my_list: list[int] | None = None):
    """מוסיף פריט לרשימה, תוך שימוש ב- None כארגומנט ברירת מחדל."""
    if my_list is None:
        my_list = []
    my_list.append(item)
    print(my_list)
append_to_list_correct(1)
append_to_list_correct(2)
append_to_list_correct(3,[])
append_to_list_correct(4)
# פלט:
# [1]
# [2]
# [3]
# [4]
```

לפיכך, **אפשרות ב** נכונה, שכן היא מתארת במדויק את הבעיה ואת השלכותיה.
