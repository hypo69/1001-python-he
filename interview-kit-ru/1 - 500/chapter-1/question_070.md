**שאלה 070.** מהי קומפוזיציה של פונקציות, וכיצד ניתן לממש אותה בפייתון? ספק דוגמת קוד המדגימה קומפוזיציה של פונקציות, כאשר התוצאה של פונקציה אחת מועברת כארגומנט לפונקציה אחרת.

-   A. קומפוזיציה של פונקציות היא תהליך של יצירת אובייקטים ממחלקות, המשתמשים בערכים שהתקבלו מאובייקטים אחרים.
-   B. קומפוזיציה של פונקציות היא תהליך שבו פונקציה קוראת לעצמה מספר פעמים.
-   C. קומפוזיציה של פונקציות היא שילוב של שתיים או יותר פונקציות, שבו תוצאת ביצוע פונקציה אחת מועברת כארגומנט לפונקציה אחרת.
-   D. קומפוזיציה של פונקציות היא דרך ליצור פונקציות שיכולות להתבצע בחוטים (threads) נפרדים.

**תשובה נכונה: C**

**הסבר:**

קומפוזיציה של פונקציות (function composition) הוא תבנית פונקציונלית המאפשרת לשלב שתיים או יותר פונקציות כך שתוצאת הביצוע של פונקציה אחת מועברת כארגומנט לפונקציה אחרת. זה מאפשר ליצור פונקציות חדשות, מורכבות יותר, מפונקציות פשוטות וניתנות לשימוש חוזר.

*   **מושגי יסוד של קומפוזיציה של פונקציות:**
    *   **שרשרת חישובים:** התוצאה של פונקציה אחת הופכת לערך הקלט של הפונקציה הבאה.
    *   **יישום רציף:** הפונקציות מיושמות אחת אחרי השנייה בסדר מסוים.
    *   **פונקציה חדשה מפונקציות פשוטות:** קומפוזיציה מאפשרת ליצור פונקציות מורכבות מפונקציות פשוטות יותר.
    *   **יכולת שימוש חוזר:** קומפוזיציה מאפשרת להשתמש מחדש בפונקציות שנוצרו קודם לכן.
*   **מימוש קומפוזיציה בפייתון:**
    *   בפייתון ניתן לממש קומפוזיציה ידנית באמצעות קריאות מקוננות לפונקציות או באמצעות אופרטור `|` (החל מפייתון 3.9), או באמצעות שימוש בפונקציות למבדה.

**דוגמה (מטקסט השאלה):**

```python
def double(x):
    return 2 * x

def square(x):
    return x ** 2

def compose(f, g):
    return lambda x: f(g(x))

composed = compose(square, double)
print(composed(5))  # 100
```

**תיאור הדוגמה:**

1.  **`double(x)`:** פונקציה שמכפילה את המספר `x` ב-2.
2.  **`square(x)`:** פונקציה שמעלה את המספר `x` בחזקת 2.
3.  **`compose(f, g)`:** פונקציה המממשת קומפוזיציה.
    *   מקבלת את הפונקציות `f` ו-`g` כארגומנטים.
    *   מחזירה פונקציית למבדה `lambda x: f(g(x))`, שמפעילה תחילה את פונקציית `g` על הארגומנט `x`, ולאחר מכן מפעילה את פונקציית `f` על התוצאה של `g(x)`.
4.  **`composed = compose(square, double)`:** נוצרת פונקציה חדשה `composed`, שהיא הקומפוזיציה של `square` ו-`double`. היא שקולה ל-`lambda x : square(double(x))`.
5.  `composed(5)`: מקבלת את הארגומנט `5` ומבצעת את הפונקציות בסדר `(5*2)**2`, כלומר, `double(5)` (ששווה ל-`10`) ולאחר מכן `square(10)` (ששווה ל-`100`).

**דוגמאות נוספות:**
```python

# דוגמת קומפוזיציה באמצעות קריאות מקוננות לפונקציות
def add1(x):
   return x + 1
def multiply_by_3(x):
   return x*3

def composed_func_manual(x):
    return add1(multiply_by_3(x))

print(composed_func_manual(5)) # פלט: 16

# דוגמת קומפוזיציה באמצעות שימוש ב-reduce ופונקציית lambda
from functools import reduce

def compose_with_reduce(*funcs):
    return lambda x: reduce(lambda acc, f: f(acc), funcs, x)


composed2 = compose_with_reduce(add1, multiply_by_3)
print(composed2(5)) # פלט: 16
```
**ניתוח האפשרויות:**
*   **A. קומפוזיציה של פונקציות היא תהליך של יצירת אובייקטים ממחלקות, המשתמשים בערכים שהתקבלו מאובייקטים אחרים.:** לא נכון. קומפוזיציה של פונקציות אינה קשורה ליצירת אובייקטים ממחלקות.
*   **B. קומפוזיציה של פונקציות היא תהליך שבו פונקציה קוראת לעצמה מספר פעמים.:** לא נכון. זהו תיאור של רקורסיה.
*   **C. קומפוזיציה של פונקציות היא שילוב של שתיים או יותר פונקציות, שבו תוצאת ביצוע פונקציה אחת מועברת כארגומנט לפונקציה אחרת.:** נכון.
*   **D. קומפוזיציה של פונקציות היא דרך ליצור פונקציות שיכולות להתבצע בחוטים (threads) נפרדים.:** לא נכון.

**לסיכום:**

*   שימוש בקומפוזיציה תורם למודולריות ולשימוש חוזר בקוד.
