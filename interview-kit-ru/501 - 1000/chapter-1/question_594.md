**שאלה 594.** מהם ההבדלים העיקריים בין שיטות מופע, שיטות מחלקה ושיטות סטטיות ב-Python, וכיצד הן משמשות בהקשר של מחלקות?

-   A. שיטות מופע, שיטות מחלקה ושיטות סטטיות שוות לחלוטין ויכולות לשמש זו במקום זו בכל הקשר.
-   B. שיטות מופע מתייחסות למופע ספציפי של המחלקה ומקבלות את `self` כפרמטר ראשון. שיטות סטטיות אינן מקבלות את `self` ואינן קשורות למופע כלשהו של המחלקה. שיטות מחלקה מקבלות פרמטר `cls` ויכולות לשנות את מצב המחלקה.
-   C. שיטות מופע, שיטות מחלקה ושיטות סטטיות הן פשוט דרכים שונות לכתוב את אותו הדבר, ומהדר Python מטפל בהן באופן זהה.
-   D. שיטות מחלקה יכולות להיקרא דרך מופע של המחלקה, שיטות סטטיות - רק דרך המחלקה, ושיטות מופע - ישירות דרך פונקציה, ללא יצירת מופע.

**תשובה נכונה: B**

**הסבר:**

ב-Python, שיטות מחלקה מייצגות דרכים שונות לעבודה עם מחלקות ואובייקטים, ולכל סוג יש את ייעודו ומאפייניו הייחודיים.

*   **שיטות מופע (Instance methods):**
    *   **פרמטר ראשון `self`:** מקבלות פרמטר ראשון חובה `self`, שהוא הפניה למופע המחלקה.
    *   **גישה למצב המופע:** יש להן גישה למאפיינים ולשיטות אחרות של מופע ספציפי של המחלקה.
    *   **תלויות במופע:** עובדות עם מצב מופע ספציפי של המחלקה.
    *   נקראות דרך המופע: `instance.method()`.

*   **שיטות סטטיות (Static methods):**
    *   **דקורטור `@staticmethod`:** מסומנות בדקורטור `@staticmethod`.
    *   **אין פרמטר חובה `self`:** אינן מקבלות פרמטר חובה `self` (אין להן גישה למאפייני המופע).
    *   **אינן תלויות במופע:** אינן יכולות לשנות את מצב המופע או המחלקה.
    *   **שיטות אוטונומיות:** מתפקדות כפונקציות רגילות הקשורות למחלקה (אך לא למופעים שלה).
    *   **נקראות דרך המחלקה:** `Class.method()` או `instance.method()`.

*   **שיטות מחלקה (Class methods):**
    *   **דקורטור `@classmethod`:** מסומנות בדקורטור `@classmethod`.
    *   **פרמטר ראשון `cls`:** מקבלות פרמטר ראשון חובה `cls`, שהוא הפניה למחלקה.
    *   **גישה למצב המחלקה:** יכולות לגשת ולשנות את מצב המחלקה.
    *   **תלויות במחלקה:** יכולות לעבוד עם מאפייני המחלקה.
    *   **נקראות דרך המחלקה:** `Class.method()` או `instance.method()`.

**דוגמאות:**

```python
class CoffeeShop:
    specialty = 'espresso'

    def __init__(self, coffee_price):
        self.coffee_price = coffee_price

    # instance method
    def make_coffee(self):
        print(f'Making {self.specialty} for ${self.coffee_price}')

    # static method
    @staticmethod
    def check_weather():
        print('Its sunny')

    # class method
    @classmethod
    def change_specialty(cls, specialty):
        cls.specialty = specialty
        print(f'Specialty changed to {specialty}')

# Создаем экземпляр (יוצרים מופע)
coffee_shop = CoffeeShop('5')

# Вызов метода экземпляра (с помощью экземпляра) (קריאה לשיטת מופע (באמצעות מופע))
coffee_shop.make_coffee()
# Вывод: Making espresso for $5 (פלט: Making espresso for $5)

# Вызов статического метода (с помощью экземпляра) (קריאה לשיטה סטטית (באמצעות מופע))
coffee_shop.check_weather()
# Вывод: Its sunny (פלט: Its sunny)

# Вызов метода класса (с помощью экземпляра) (קריאה לשיטת מחלקה (באמצעות מופע))
coffee_shop.change_specialty('drip coffee')
# Вывод: Specialty changed to drip coffee (פלט: Specialty changed to drip coffee)

# Вызов метода экземпляра после вызова метода класса (קריאה לשיטת מופע לאחר קריאה לשיטת מחלקה)
coffee_shop.make_coffee()
# Вывод: Making drip coffee for $5 (פלט: Making drip coffee for $5)
# Вызов статического метода (с помощью класса) (קריאה לשיטה סטטית (באמצעות מחלקה))
CoffeeShop.check_weather()
# Вывод: Its sunny (פלט: Its sunny)
# Вызов метода класса (с помощью класса) (קריאה לשיטת מחלקה (באמצעות מחלקה))
CoffeeShop.change_specialty('latte')
# Вывод: Specialty changed to latte (פלט: Specialty changed to latte)
print(coffee_shop.specialty) # Выведет latte (ידפיס latte)
```
**ניתוח דוגמאות:**
1.  **`CoffeeShop`:** מחלקה עם מאפיין `specialty`, המוגדר כברירת מחדל ל-`espresso`.
2.  **`make_coffee(self)`:** שיטת מופע. ניגשת למאפיינים `specialty` ו-`coffee_price` של המופע דרך `self`.
3.  **`check_weather()`:** שיטה סטטית עם הדקורטור `@staticmethod`. אין לה גישה למופע או למאפייני המחלקה.
4.  **`change_specialty(cls, specialty)`:** שיטת מחלקה עם הדקורטור `@classmethod`. היא משנה את מאפיין המחלקה `specialty` באמצעות הפרמטר `cls`.
5.  בהמשך הדוגמה מוצגות קריאות לשיטות אלו באמצעות מופע המחלקה והמחלקה עצמה.
