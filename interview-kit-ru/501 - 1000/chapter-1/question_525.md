**שאלה 525.** מהי רקורסיית זנב (tail recursion)?

-   א. זהו סוג של רקורסיה שבו פונקציה קוראת לעצמה מספר פעמים בתוך קריאה אחת.
-   ב. זהו סוג של רקורסיה שבו פונקציה קוראת לעצמה, ואחרי קריאה זו מבצעת פעולות נוספות.
-   ג. זהו סוג של רקורסיה שבו הפעולה האחרונה המבוצעת בפונקציה היא קריאה לעצמה.
-   ד. זהו סוג של רקורסיה שבו פונקציה קוראת לפונקציה רקורסיבית אחרת.

**תשובה נכונה: ג**

**הסבר:**

רקורסיית זנב (tail recursion) היא סוג מיוחד של רקורסיה שבו הקריאה הרקורסיבית לפונקציה היא הפעולה ה*אחרונה* המבוצעת באותה פונקציה. משמעות הדבר היא שלאחר הקריאה הרקורסיבית לא מבוצעות פעולות נוספות, ותוצאת הקריאה מוחזרת מיד.

*   **מאפיינים של רקורסיית זנב:**
    *   הקריאה הרקורסיבית היא הפעולה האחרונה.
    *   אין צורך לאחסן תוצאות ביניים על גבי מחסנית הקריאות (call stack).

*   **אופטימיזציה:**
    *   מהדר (compiler) או מפרש (interpreter) יכול לבצע אופטימיזציה של רקורסיית זנב על ידי החלפתה באיטרציה (לולאה).
    *   אופטימיזציה זו מאפשרת למנוע גלישת מחסנית קריאות (stack overflow) במקרה של מספר רב של קריאות רקורסיביות.
    *   במהלך האופטימיזציה נעשה שימוש במסגרת מחסנית (stack frame) אחת עם משתנים לוקליים משתנים.

**ניתוח האפשרויות:**
*   **א. זהו סוג של רקורסיה שבו פונקציה קוראת לעצמה מספר פעמים בתוך קריאה אחת:** לא נכון.
*   **ב. זהו סוג של רקורסיה שבו פונקציה קוראת לעצמה, ואחרי קריאה זו מבצעת פעולות נוספות:** לא נכון.
*   **ג. זהו סוג של רקורסיה שבו הפעולה האחרונה המבוצעת בפונקציה היא קריאה לעצמה:** נכון. זוהי ההגדרה המדויקת של רקורסיית זנב.
*   **ד. זהו סוג של רקורסיה שבו פונקציה קוראת לפונקציה רקורסיבית אחרת:** לא נכון.

**דוגמאות:**

*   **רקורסיית זנב (דוגמה בפסאודו-קוד):**

    ```
    function tail_recursive(n, accumulator) {
      if (n == 0) {
         return accumulator;
      } else {
         return tail_recursive(n - 1, accumulator + n); // הפעולה האחרונה
      }
    }
    ```

*   **רקורסיה שאינה רקורסיית זנב (דוגמה בפסאודו-קוד):**

    ```
    function not_tail_recursive(n) {
      if (n == 0) {
        return 0;
      } else {
        return n + not_tail_recursive(n - 1); // קריאת הרקורסיה אינה הפעולה האחרונה
      }
    }
    ```

**לסיכום:**
*   רקורסיית זנב מאפשרת כתיבת קוד בסגנון רקורסיבי תוך הימנעות מבעיות פוטנציאליות של גלישת מחסנית.
*   עם זאת, לא כל שפות התכנות תומכות באופטימיזציה של רקורסיית זנב, כולל Python.

לפיכך, התשובה הנכונה היא **ג. זהו סוג של רקורסיה שבו הפעולה האחרונה המבוצעת בפונקציה היא קריאה לעצמה.**
