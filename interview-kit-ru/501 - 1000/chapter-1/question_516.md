**שאלה 516:** מה תהיה תוצאת הרצת הקוד הבא?

```python
my_list = [1, 2, 3]
my_list.append(my_list)
print(my_list)
```

א. [1, 2, 3, [1, 2, 3]]
ב. [1, 2, 3, [...]]
ג. שגיאת גלישת מחסנית
ד. תוצג שגיאת סוג

**תשובה נכונה: ב**

**הסבר (עבור question_516.md, בדגש על מוּטָבִּילִיוּת):**

שאלה זו בוחנת את ההבנה של מוּטָבִּילִיוּת של רשימות וכיצד Python מטפלת בהפניות מחזוריות.

1.  **רשימה ראשונית:** `my_list` מאותחלת כ-[1, 2, 3].
2.  **שינוי הרשימה (מוּטָצְיָה):** הקריאה ל-`my_list.append(my_list)` מוסיפה את *הרשימה עצמה* `my_list` לסופה. חשוב לציין שרשימות ב-Python הן *מוּטָבִּילִיוֹת*, כלומר ניתן לשנות אותן לאחר יצירתן. לפיכך, `my_list` מכילה כעת הפניה לעצמה, מה שיוצר הפניה מחזורית.
3.  **ייצוג בהדפסה:** כאשר `my_list` מודפסת למסך, Python מזהה את התלות המחזורית הזו. כדי למנוע הדפסה אינסופית של רשימות מקוננות, היא משתמשת בסימן שלוש נקודות `[...]` כדי לייצג את הרשימה המקוננת המפנה לרשימה האב.

**ניתוח האפשרויות:**

*   **א. [1, 2, 3, [1, 2, 3]]**: שגוי. Python לא תפרט את הקינון (nesting) באופן אינסופי עקב ההפניה המחזורית.
*   **ב. [1, 2, 3, [...]]**: נכון. זוהי הדרך שבה Python מייצגת רשימה מחזורית.
*   **ג. שגיאת גלישת מחסנית**: שגוי. Python תוכננה כך שתטפל נכון במקרים כאלה.
*   **ד. תוצג שגיאת סוג**: שגוי. הקוד תקין מבחינה תחבירית (syntax) וסמנטית (semantics).
