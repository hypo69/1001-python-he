**שאלה 569.** מהי תבנית העיצוב הפונקציונלית 'Compose' (קומפוזיציה), וכיצד היא משמשת בפייתון? ספק דוגמה המדגימה שילוב של מספר פונקציות לכדי פונקציה אחת באמצעות קומפוזיציה.

-   A. תבנית 'Compose' היא דרך ליצור אובייקטים חדשים על בסיס אובייקטים קיימים, ללא שינוי קוד המקור שלהם.
-   B. תבנית 'Compose' היא דרך לשלב מספר פונקציות לשרשרת, כך שהתוצאה של קריאה לפונקציה אחת הופכת לקלט עבור הפונקציה הבאה.
-   C. תבנית 'Compose' היא דרך לשנות את התנהגות אובייקט בזמן ריצה, מבלי ליצור מחלקות חדשות.
-   D. תבנית 'Compose' היא דרך ליצור פונקציות רקורסיביות שיכולות לקרוא לעצמן.

**תשובה נכונה: B**

**הסבר:**

תבנית 'Compose' (קומפוזיציה) בתכנות פונקציונלי מאפשרת לשלב מספר פונקציות לכדי פונקציה אחת, תוך יישום עוקב שלהן אחת על השנייה. התוצאה של קריאה לפונקציה אחת הופכת לקלט עבור הבאה, מה שיוצר שרשרת של טרנספורמציות או פעולות.

*   **מושגי יסוד בתבנית 'Compose':**
    *   **יישום עוקב:** פונקציות מיושמות בזו אחר זו, מעבירות את התוצאה מאחת לשנייה.
    *   **יצירת פונקציות חדשות:** שילוב של פונקציות פשוטות ליצירת פונקציות מורכבות יותר.
    *   **שיפור הקריאות:** מאפשר פירוק פעולות מורכבות לפונקציות פשוטות וניתנות לשימוש חוזר.
    *   **סגנון דקלרטיבי:** מאפשר תיאור פעולות מורכבות באופן דקלרטיבי, ציון כיצד הפונקציות צריכות לשלב, ולא כיצד בדיוק יש ליישם זאת.
    *   **קומפוזיציה של פונקציות מסדר גבוה:** קומפוזיציה מיושמת לעיתים קרובות באמצעות פונקציות מסדר גבוה, שיכולות לעבוד עם פונקציות אחרות.

*   **שימוש בפייתון:**
    *   בפייתון, ניתן להשתמש בפונקציה `functools.reduce` או פשוט לשלב קריאות לפונקציות בשרשרת כדי ליישם קומפוזיציה.

**דוגמאות:**

```python
from functools import reduce

def add_one(x):
    return x + 1

def multiply_by_two(x):
    return x * 2

def compose(*funcs):
    def composed_function(x):
        # יישום פונקציות ברצף, התוצאה של כל פונקציה היא הקלט לבאה
        return reduce(lambda acc, f: f(acc), funcs, x)
    return composed_function

# שיטה 1: שימוש בפונקציית compose מותאמת אישית
# הפונקציות ייושמו בסדר הזה: add_one, multiply_by_two, add_one
composed_function = compose(add_one, multiply_by_two, add_one)
print(composed_function(5))  # תדפיס: 13  ((5 + 1) * 2 + 1)

# שיטה 2: שילוב פונקציות בשרשרת באופן ידני
# יישום פונקציות בסדר הנכון ידנית
print(add_one(multiply_by_two(add_one(5)))) # תדפיס: 13

# שיטה 3: ללא reduce, כאשר מספר הפונקציות ידוע
# יצירת פונקציה מורכבת ידנית עבור מספר פונקציות קבוע
def composed_function2(x):
  return add_one(multiply_by_two(x))
print(composed_function2(5)) # תדפיס: 11
```

**ניתוח הדוגמאות:**

1.  `add_one(x)`: הפונקציה מוסיפה 1 למספר הנתון.
2.  `multiply_by_two(x)`: הפונקציה מכפילה את המספר ב-2.
3.  `compose(*funcs)`: פונקציה שמיישמת את תבנית 'Compose':
    *   מקבלת מספר שרירותי של פונקציות (הפונקציות לקומפוזיציה) דרך `*funcs`.
    *   מחזירה את `composed_function`, אשר עוברת איטרציה על כל הפונקציות, מיישמת אותן על הערך ההתחלתי.
4.  **שיטה 1:**
    *   `composed_function` נוצרת באמצעות `compose(add_one, multiply_by_two, add_one)`, כאשר הפונקציות ייושמו בסדר ההופעה (add_one, multiply_by_two, add_one).
    *   `composed_function(5)`: מקבלת `5` ומיישמת עליו את הפונקציות. `5 + 1 = 6`, `6 * 2 = 12`, `12 + 1 = 13`.
5.  **שיטה 2:** קומפוזיציה שנוצרה ידנית על ידי קריאה לפונקציות בסדר הדרוש. `5 + 1 = 6`, `6 * 2 = 12`, `12 + 1 = 13`.
6.  **שיטה 3:** קומפוזיציה ללא reduce, אם מספר הפונקציות ידוע. `5 * 2 = 10` ו-`10 + 1 = 11`.

**ניתוח האפשרויות:**
*   **A. תבנית 'Compose' היא דרך ליצור אובייקטים חדשים על בסיס אובייקטים קיימים, ללא שינוי קוד המקור שלהם.:** שגוי.
*   **B. תבנית 'Compose' היא דרך לשלב מספר פונקציות לשרשרת, כך שהתוצאה של קריאה לפונקציה אחת הופכת לקלט עבור הפונקציה הבאה.:** נכון.
*   **C. תבנית 'Compose' היא דרך לשנות את התנהגות אובייקט בזמן ריצה, מבלי ליצור מחלקות חדשות.:** שגוי.
*   **D. תבנית 'Compose' היא דרך ליצור פונקציות רקורסיביות שיכולות לקרוא לעצמן.:** שגוי.

**לסיכום:**
