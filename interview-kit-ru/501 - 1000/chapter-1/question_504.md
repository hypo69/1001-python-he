### `question_504.md`
**שאלה 504:**
```python
def foo():
    try:
        return 1
    finally:
        return 2
result = foo()
print(result)
```
מה תהיה התוצאה של הרצת קוד זה?

A. 1
B. 2
C. None
D. תתרחש שגיאה

**תשובה נכונה: B**

**הסבר:**

שאלה זו עוסקת בהבנת האופן שבו פועל בלוק `finally` במבנה `try...finally` בפייתון. כך זה עובד:

1.  **בלוק `try`:** ראשית, מתבצע הקוד בתוך בלוק `try`. במקרה זה, מדובר ב-`return 1`. בדרך כלל, כאשר מתבצעת פקודת `return`, הפונקציה מסיימת את פעולתה ומחזירה ערך.
2.  **בלוק `finally`:** אולם, לפני שהפונקציה מסתיימת סופית, מתבצע הבלוק `finally`. זה קורה **תמיד**, ללא קשר לשאלה האם אירע חריג בבלוק `try` או לא, ואף אם בבלוק `try` בוצעה פקודת `return`.
3.  **דריסת ערך ההחזרה:** בתוך בלוק `finally` נמצאת גם הפקודה `return 2`. פקודת `return` זו דורסת את הערך שהיה אמור להיות מוחזר מבלוק `try`. לפיכך, הפונקציה **תמיד** מחזירה את הערך מבלוק `finally`.

**ניתוח האפשרויות:**
*   **A. 1:** שגוי, שכן הערך מבלוק `try` נדרס.
*   **B. 2:** נכון, שכן דווקא הערך מבלוק `finally` מוחזר על ידי הפונקציה.
*   **C. None:** שגוי, שכן הפונקציה תמיד מחזירה ערך (מספר).
*   **D. תתרחש שגיאה:** שגוי, הקוד ירוץ ללא שגיאות.

**סיכום:**

כתוצאה מהרצת הקוד `foo()`, הפונקציה תחזיר את הערך 2, שהוחזר מבלוק `finally`, גם אם בבלוק `try` היה `return 1`. לפיכך, הערך שהוחזר מבלוק `try` מותעלם.

**נקודות חשובות נוספות:**
*   בלוק `finally` חיוני לשחרור משאבים (קבצים, חיבורים לבסיסי נתונים וכו') שנלכדו בבלוק `try`, ללא קשר לשאלה האם התרחשו חריגים או לא.
*   החזרה מבלוק `finally` עלולה להסוות שגיאות שנוצרו בבלוק `try`. לכן חשוב להשתמש בו בזהירות ובהבנה מלאה של ההשלכות.

**דוגמה למקרה מורכב יותר עם חריג:**
```python
def foo2():
    try:
        1 / 0
        return 1
    except ZeroDivisionError:
        return 3
    finally:
        return 2
result2 = foo2()
print(result2)
#פלט: 2
```

גם אם בבלוק `try` ייווצר חריג `ZeroDivisionError` והוא יטופל בבלוק `except`, עדיין הפונקציה תחזיר את הערך 2, המוחזר מבלוק `finally`.

**לסיכום:** התשובה הנכונה היא **B. 2**. בלוק `finally` תמיד מתבצע ודורס את ערך ההחזרה.
