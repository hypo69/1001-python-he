### `question_591.md`

**שאלה 591.** מהם דקורטורים (Decorators) ב-Python וכיצד הם מיושמים? ספק/י דוגמת קוד המדגימה יצירה ושימוש בדקורטור פשוט.

-   א. דקורטורים הם פונקציות המשמשות לאחסון נתונים אודות פונקציות אחרות.
-   ב. דקורטורים הם דרך ליצירת פונקציות אנונימיות, המשמשות לחישובים קצרים.
-   ג. דקורטורים הם פונקציות המקבלות פונקציה אחרת כארגומנט, מרחיבות את הפונקציונליות שלה ומחזירות פונקציה חדשה (עוטף/Wrapper).
-   ד. דקורטורים הם תחביר מיוחד לקריאה למתודות הורה (Parent Methods) במחלקות.

**תשובה נכונה: ג**

**הסבר:**

דקורטורים ב-Python הם דרך עוצמתית ונוחה לשנות או להרחיב התנהגות של פונקציות או מתודות מבלי לשנות את קוד המקור שלהן. הם מבוססים על קונספט של פונקציות מסדר גבוה (Higher-Order Functions), שבו פונקציות יכולות לקבל פונקציות אחרות כארגומנטים או להחזיר פונקציות.

*   **מאפיינים עיקריים של דקורטורים:**
    *   **פונקציית-עוטף (Wrapper Function):** דקורטור הוא פונקציה שמקבלת פונקציה אחרת ומחזירה "עוטף" - פונקציה חדשה שקוראת לפונקציה המקורית, ומוסיפה לה פונקציונליות חדשה.
    *   **`@decorator_name`:** תחביר מיוחד ליישום דקורטור, שהוא Sugar תחבירי (Syntactic Sugar).
    *   **שימוש חוזר בקוד:** דקורטורים תורמים לשימוש חוזר בקוד.
    *   **שינוי התנהגות:** דקורטורים מאפשרים להוסיף לוגיקה חדשה לפונקציות או מתודות ללא צורך בשינוי קוד המקור שלהן.

*   **כיצד פועל דקורטור:**
    1.  נוצרת פונקציית-דקורטור (בדרך כלל פונקציה שמקבלת פונקציה ומחזירה פונקציה).
    2.  הדקורטור מיושם על פונקציה באמצעות הסימן `@` לפני שם הדקורטור (לדוגמה `@my_decorator`).
    3. בעת קריאה לפונקציה המקושטת/מעוטרת, ייקרא תחילה העוטף, ובתוכו הפונקציה המקורית.

**דוגמאות:**

```python
# דוגמה ליצירת דקורטור פשוט
def logging(func):
  def log_function_called():
     print(f'{func.__name__} called.')
     func()
  return log_function_called

@logging # מקשטים (מעטרים) את הפונקציה באמצעות @logging
def my_name():
  print('chris')

@logging # מקשטים (מעטרים) את הפונקציה באמצעות @logging
def friends_name():
  print('naruto')

my_name()
friends_name()
# פלט:
# my_name called.
# chris
# friends_name called.
# naruto

# דוגמה לשימוש ב-*args, **kwargs

def logger(func):
    def wrapper(*args, **kwargs):
        print(f'Вызывается функция {func.__name__} с аргументами {args}, и {kwargs}')
        result = func(*args, **kwargs)
        print(f'функция {func.__name__} завершила работу')
        return result
    return wrapper
@logger
def my_function(x, y = 10):
    return x*y
my_function(2, 3)
# פלט:
# Вызывается функция my_function с аргументами (2,), и {'y': 3}
# функция my_function завершила работу
```

**ניתוח דוגמאות:**

1.  **`logging(func)`:**
    *   הדקורטור `logging`, המקבל כארגומנט את הפונקציה `func`.
    *   `log_function_called()`: פונקציית-עוטף, המדפיסה הודעה על קריאה לפונקציה וקוראת לפונקציה המקורית `func`.
    *   `@logging` מיישם את הדקורטור על הפונקציות `my_name` ו-`friends_name`.
2.  **`my_name()` ו-`friends_name()`:** שתי פונקציות שעליהן מיושם הדקורטור.
3. בעת קריאה ל-`my_name()` או `friends_name()` מודפסת הודעה על קריאה לפונקציה, ולאחר מכן מבוצע קוד הפונקציה.
4. **`logger(func)`**: דקורטור, המקבל פונקציה כארגומנט.
    *   `wrapper(*args, **kwargs)`: עוטף המדפיס מידע על קריאה לפונקציה, וכן את הארגומנטים, ולאחר קריאה לפונקציה מדפיס גם הודעה על סיום. (שים/שימי לב: המחרוזות המודפסות נשארות בשפת המקור כפי שהופיעו בקוד).
    *   `@logger`: מיישם את הדקורטור על `my_function`.

**ניתוח האפשרויות:**
*   **א. דקורטורים הם פונקציות המשמשות לאחסון נתונים אודות פונקציות אחרות.:** לא נכון.
*   **ב. דקורטורים הם דרך ליצירת פונקציות אנונימיות, המשמשות לחישובים קצרים.:** לא נכון.
*   **ג. דקורטורים הם פונקציות המקבלות פונקציה אחרת כארגומנט, מרחיבות את הפונקציונליות שלה ומחזירות פונקציה חדשה (עוטף/Wrapper).:** נכון.
*   **ד. דקורטורים הם תחביר מיוחד לקריאה למתודות הורה (Parent Methods) במחלקות.:** לא נכון.

**לסיכום:**
*   דקורטורים מאפשרים להרחיב פונקציונליות של פונקציות קיימות ללא שינוי הקוד שלהן.
*   הם משמשים להוספת Logים (רישום אירועים), בדיקת הרשאות גישה, שמירת תוצאות בזיכרון מטמון (Caching) ומשימות נפוצות אחרות.
*   `@` משמש ליישום דקורטור על פונקציה, ובכך מאפשר לממש לוגיקה "תוך כדי תנועה" (on the fly).

לפיכך, התשובה הנכונה היא **ג. דקורטורים הם פונקציות המקבלות פונקציה אחרת כארגומנט, מרחיבות את הפונקציונליות שלה ומחזירות פונקציה חדשה (עוטף/Wrapper).**
