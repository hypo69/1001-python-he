**שאלה 555.** מהי רקורסיה בפייתון, ובאילו מקרים השימוש בה מתאים? הצג דוגמת קוד המדגימה פונקציה רקורסיבית.

-   א. רקורסיה היא תהליך שבו פונקציה אחת קוראת לפונקציה אחרת, אשר בתורה קוראת לראשונה.
-   ב. רקורסיה היא תהליך שבו פונקציה קוראת לעצמה. זה שימושי לפתרון בעיות שניתן לפרק לתת-בעיות בעלות אופי דומה.
-   ג. רקורסיה היא כאשר פונקציה קוראת ללולאה, והלולאה קוראת לפונקציה.
-   ד. רקורסיה היא כאשר פונקציה משתמשת במשתנים גלובליים במקום במשתנים מקומיים.

**תשובה נכונה: ב**

**הסבר:**

רקורסיה בתכנות היא שיטה לפתרון בעיה שבה פונקציה קוראת לעצמה. הדבר שימושי במיוחד בעת פתרון בעיות שניתן לפרק לתת-בעיות קטנות יותר, הנפתרות באותו אופן.

*   **עקרונות יסוד של רקורסיה:**
    *   **מקרה בסיס:** חייב להתקיים מקרה בסיס (או תנאי עצירה) המסיים את הקריאות הרקורסיביות, ובכך מונע רקורסיה אינסופית.
    *   **מקרה רקורסיבי:** הפונקציה קוראת לעצמה עם תת-קבוצה קטנה יותר של הבעיה (עם ארגומנטים ששונו).
    *   כל קריאה רקורסיבית חייבת לקרב את הפתרון למקרה הבסיס.

*   **מתי מתאים להשתמש ברקורסיה:**
    *   לפתרון בעיות שניתן לבטא באופן טבעי בצורה רקורסיבית (למשל, מעבר על מבני עץ, חישוב עצרת, סדרות רקורסיביות).
    *   כאשר רקורסיה הופכת את הקוד לקריא ומובן יותר בהשוואה לגישה איטרטיבית.
    *   כאשר לבעיה יש מבנה דמוי-עצמי.

**דוגמה (מטקסט השאלה):**

```python
def factorial(n):
    if n == 0 or n == 1:
        return 1
    else:
        return n * factorial(n - 1)

print(factorial(5))  # 120
```

**תיאור הדוגמה:**

*   **`factorial(n)`:** פונקציה רקורסיבית המחשבת את העצרת של המספר `n`.
    *   **מקרה בסיס:** `if n == 0 or n == 1: return 1`: אם `n` שווה ל-0 או 1, הפונקציה מחזירה 1 (עצרת 0 ו-1 שווה ל-1).
    *   **מקרה רקורסיבי:** `else: return n * factorial(n - 1)`: אחרת, הפונקציה קוראת לעצמה עם הארגומנט `n - 1` ומכפילה את התוצאה ב-`n`.
