-   א. רקורסיה היא תהליך שבו פונקציה קוראת לפונקציה אחרת, שבתורה קוראת לראשונה.
-   ב. רקורסיה היא תהליך שבו פונקציה קוראת לעצמה. זה שימושי לפתרון בעיות שניתן לפרק לתת-בעיות בעלות אופי דומה.

**תשובה נכונה: ב**

**הסבר:**

רקורסיה בתכנות היא שיטה לפתרון בעיה שבה פונקציה קוראת לעצמה. זה שימושי במיוחד בעת פתרון בעיות שניתן לפרק לתת-בעיות קטנות יותר הנפתרות באותה שיטה.

*   **עקרונות יסוד של רקורסיה:**
    *   **מקרה בסיס:** חייב להיות מקרה בסיס (או תנאי עצירה) שמסיים את הקריאות הרקורסיביות ומונע רקורסיה אינסופית.
    *   **מקרה רקורסיבי:** הפונקציה קוראת לעצמה עם תת-קבוצה קטנה יותר של הבעיה (עם ארגומנטים ששונו).
    *   כל קריאה רקורסיבית חייבת לקרב את הפתרון למקרה הבסיס.

*   **מתי מתאים להשתמש ברקורסיה:**
    *   לפתרון בעיות שניתן לבטא באופן טבעי בצורה רקורסיבית (לדוגמה, מעבר על עצים, חישוב פקטוריאל, סדרות רקורסיביות).
    *   כאשר רקורסיה הופכת את הקוד לקריא ומובן יותר בהשוואה לגישה איטרטיבית.
    *   כאשר לבעיה יש מבנה דמוי עצמו.

**דוגמה (מטקסט השאלה):**

```python
def factorial(n):
    if n == 0 or n == 1:
        return 1
    else:
        return n * factorial(n - 1)

print(factorial(5))  # 120
```

**תיאור הדוגמה:**

*   **`factorial(n)`:** פונקציה רקורסיבית המחשבת את הפקטוריאל של המספר `n`.
    *   **מקרה בסיס:** `if n == 0 or n == 1: return 1`: אם `n` שווה ל-0 או ל-1, הפונקציה מחזירה 1 (הפקטוריאל של 0 ו-1 הוא 1).
    *   **מקרה רקורסיבי:** `else: return n * factorial(n - 1)`: אחרת, הפונקציה קוראת לעצמה עם הארגומנט `n - 1` ומכפילה את התוצאה ב-`n`.
