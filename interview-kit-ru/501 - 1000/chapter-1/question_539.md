### `question_539.md`

**שאלה 539.** מה תהיה התוצאה שתתקבל בעת הרצת קוד הפייתון הבא, המממש את אלגוריתם מיון ערימה?

```python
import heapq

def heap_sort(arr):
    heapq.heapify(arr)
    sorted_arr = []
    while arr:
        sorted_arr.append(heapq.heappop(arr))
    return sorted_arr

# דוגמה לשימוש
arr = [64, 34, 25, 12, 22, 11, 90]
print("Original array:", arr)
sorted_arr = heap_sort(arr)
print("Sorted array:", sorted_arr)
```

-   A. `Original array: [64, 34, 25, 12, 22, 11, 90] Sorted array: [90, 64, 34, 25, 22, 12, 11]`
-   B. `Original array: [64, 34, 25, 12, 22, 11, 90] Sorted array: [11, 12, 22, 25, 34, 64, 90]`
-   C. `Original array: [64, 34, 25, 12, 22, 11, 90] Sorted array: [11, 90, 12, 64, 22, 34, 25]`
-   D. `Original array: [64, 34, 25, 12, 22, 11, 90] Sorted array: [64, 34, 25, 22, 12, 11, 90]`

**תשובה נכונה: B**

**הסבר:**

הקוד שניתן מממש את אלגוריתם מיון ערימה (heap sort) תוך שימוש במודול `heapq` בפייתון.

*   **אלגוריתם מיון ערימה:**
    *   משתמש במבנה נתונים "ערימה" (heap), המהווה עץ בינארי שמקיים את מאפיין הערימה (כל צומת גדול או שווה לצאצאיו).
    *   בונים ערימה מהמערך.
    *   שולפים בהדרגה את האיבר הקטן ביותר (או הגדול ביותר) מהערימה, ומוסיפים אותו למערך הממוין.
*   **תיאור הקוד:**
    *   `heap_sort(arr)`: פונקציה המקבלת מערך `arr` כארגומנט.
        *   `heapq.heapify(arr)`: ממירה את המערך לערימת מינימום (min-heap), כאשר האיבר המינימלי נמצא תמיד בשורש.
        *   `sorted_arr = []`: יוצרת רשימה ריקה לשמירת האיברים הממוינים.
        *   `while arr`: לולאה הנמשכת כל עוד הערימה אינה ריקה.
        *   `sorted_arr.append(heapq.heappop(arr))`: שולפת את האיבר המינימלי מהערימה ומוסיפה אותו לרשימה הממוינת.
        *   `return sorted_arr`: מחזירה את הרשימה הממוינת.

**ניתוח האפשרויות:**
*   **A. `Original array: [64, 34, 25, 12, 22, 11, 90] Sorted array: [90, 64, 34, 25, 22, 12, 11]`:** לא נכון, זהו מיון יורד.
*   **B. `Original array: [64, 34, 25, 12, 22, 11, 90] Sorted array: [11, 12, 22, 25, 34, 64, 90]`:** נכון. התוצאה היא מיון בסדר עולה.
*   **C. `Original array: [64, 34, 25, 12, 22, 11, 90] Sorted array: [11, 90, 12, 64, 22, 34, 25]`:** לא נכון.
*   **D. `Original array: [64, 34, 25, 12, 22, 11, 90] Sorted array: [64, 34, 25, 22, 12, 11, 90]`:** לא נכון.

**לסיכום:**
*   הקוד מממש נכונה את אלגוריתם מיון ערימה לצורך מיון מערך בסדר עולה.
*   מיון ערימה הוא אלגוריתם מיון יעיל בעל סיבוכיות זמן מובטחת של O(n log n).
*   נעשה שימוש במודול הסטנדרטי `heapq`, המספק כלים לעבודה עם ערימות.

לפיכך, התשובה הנכונה היא **B. `Original array: [64, 34, 25, 12, 22, 11, 90] Sorted array: [11, 12, 22, 25, 34, 64, 90]`**.

### `question_538.md`

**שאלה 538.** מה תהיה התוצאה שתתקבל בעת הרצת קוד הפייתון הבא, המממש את אלגוריתם מיון בסיס?

```python
def radix_sort(arr):
    max_val = max(arr)
    exp = 1
    while max_val // exp > 0:
        counting_sort(arr, exp)
        exp *= 10
    return arr

def counting_sort(arr, exp):
    n = len(arr)
    output = [0] * n
    count = [0] * 10

    for i in range(n):
        index = (arr[i] // exp) % 10
        count[index] += 1

    for i in range(1, 10):
        count[i] += count[i - 1]

    for i in range(n - 1, -1, -1):
        index = (arr[i] // exp) % 10
        output[count[index] - 1] = arr[i]
        count[index] -= 1

    for i in range(n):
        arr[i] = output[i]


# דוגמה לשימוש
arr = [170, 45, 75, 90, 802, 24, 2, 66]
print("Original array:", arr)
sorted_arr = radix_sort(arr)
print("Sorted array:", sorted_arr)
```

-   A. `Original array: [170, 45, 75, 90, 802, 24, 2, 66] Sorted array: [802, 170, 90, 75, 66, 45, 24, 2]`
-   B. `Original array: [170, 45, 75, 90, 802, 24, 2, 66] Sorted array: [2, 24, 45, 66, 75, 90, 170, 802]`
-   C. `Original array: [170, 45, 75, 90, 802, 24, 2, 66] Sorted array: [2, 170, 24, 45, 66, 75, 90, 802]`
-   D. `Original array: [170, 45, 75, 90, 802, 24, 2, 66] Sorted array: [170, 45, 2, 24, 66, 75, 90, 802]`

**תשובה נכונה: B**

**הסבר:**

הקוד שניתן מממש את אלגוריתם מיון בסיס (radix sort), שהוא אלגוריתם מיון שאינו מבוסס השוואה.

*   **אלגוריתם מיון בסיס:**
    *   ממיין איברים החל מהספרה הפחות משמעותית (ספרת היחידות) וממשיך הלאה לספרות המשמעותיות יותר.
    *   משתמש במיון ספירה יציב לצורך מיון האיברים לפי הספרה.
*   **תיאור הקוד:**
    *   `radix_sort(arr)`: פונקציה המקבלת מערך `arr` כארגומנט.
        *   `max_val = max(arr)`: מוצאת את הערך המקסימלי לצורך קביעת מספר הספרות.
        *   `exp = 1`: מאתחלת את המשתנה `exp` המייצג את הספרה (1, 10, 100, ...).
        *   `while max_val // exp > 0`: הלולאה עוברת על כל הספרות.
        *   `counting_sort(arr, exp)`: ממיין את המערך לפי הספרה הנוכחית.
    *   `counting_sort(arr, exp)`: פונקציה המבצעת מיון ספירה עבור הספרה הנוכחית.
        *   `n = len(arr)`: אורך המערך.
        *   `output = [0] * n`: יוצרת רשימה לשמירת האיברים הממוינים.
        *   `count = [0] * 10`:  יוצרת רשימה לספירת התדירות של ספרות בספרה הנוכחית (0-9).
        *   `for i in range(n)`: לולאה הסופרת את התדירות של ספרות בספרה הנוכחית.
        *   `for i in range(1, 10)`: לולאה הממירה את מערך ה-`count` כך שיאחסן את המיקומים של הספרות ב-`output`.
        *   `for i in range(n - 1, -1, -1)`: לולאה הממקמת את האיברים במיקום הנכון, ומקטינה את `count[index]`.
        *   `for i in range(n)`: מעתיקה את הערכים מ-`output` ל-`arr`.
**ניתוח האפשרויות:**
*   **A. `Original array: [170, 45, 75, 90, 802, 24, 2, 66] Sorted array: [802, 170, 90, 75, 66, 45, 24, 2]`:** לא נכון. זהו מיון בערך יורד.
*   **B. `Original array: [170, 45, 75, 90, 802, 24, 2, 66] Sorted array: [2, 24, 45, 66, 75, 90, 170, 802]`:** נכון. התוצאה היא מיון בסדר עולה.
*   **C. `Original array: [170, 45, 75, 90, 802, 24, 2, 66] Sorted array: [2, 170, 24, 45, 66, 75, 90, 802]`:** לא נכון.
*   **D. `Original array: [170, 45, 75, 90, 802, 24, 2, 66] Sorted array: [170, 45, 2, 24, 66, 75, 90, 802]`:** לא נכון.

**לסיכום:**
*   הקוד מממש נכונה את אלגוריתם מיון בסיס לצורך מיון מערך בסדר עולה.
*   מיון בסיס מתאים למיון איברים שערכיהם נמצאים בטווח מסוים, ויכול להיות יעיל יותר מאלגוריתמים מבוססי השוואה אם טווח הערכים אינו גדול מדי.

לפיכך, התשובה הנכונה היא **B. `Original array: [170, 45, 75, 90, 802, 24, 2, 66] Sorted array: [2, 24, 45, 66, 75, 90, 170, 802]`**.
