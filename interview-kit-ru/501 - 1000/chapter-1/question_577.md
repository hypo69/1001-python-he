### `question_577.md`

**שאלה 577.** מהו מימוּא (memoization) בתִכנוּת, וכיצד ניתן למַמֵש אותו בפייתון? הציגו דוגמה לקוד המדגים שימוש במימוּא לצורך אופטימיזציה של פונקציה רקורסיבית.

-   A. מימוּא הוא טכניקה המשמשת לשמירת נתונים במסד נתונים.
-   B. מימוּא הוא טכניקת אופטימיזציה שבה תוצאות חישובים קודמים נשמרות ונעשה בהן שימוש חוזר, כדי להימנע מחישובים חוזרים של ערכים זהים, במיוחד בפונקציות רקורסיביות.
-  C. מימוּא הוא טכניקה שבה נתונים אינם משתנים, מה שמאפשר הימנעות משגיאות.
-  D. מימוּא הוא דרך הצפנת נתונים לפני שמירתם על דיסק.

**תשובה נכונה: B**

**הסבר:**

מימוּא (memoization) היא טכניקת אופטימיזציה שעיקרה הוא שמירת תוצאות קריאות לפונקציות (בדרך כלל עם ארגומנטים ספציפיים) במבנה נתונים כלשהו, כדי להימנע מביצוע חוזר של חישובים אלה בקריאות הבאות עם אותם ארגומנטים. זוהי כלי חשוב להגברת הביצועים, במיוחד בפונקציות רקורסיביות, שבהן חישובים חוזרים יכולים להיות יקרים מאוד מבחינת משאבים.

*   **מושגי יסוד של מימוּא:**
    *   **אגירת תוצאות (Caching):** שמירת תוצאות קריאה לפונקציה במטמון (cache).
    *   **שימוש חוזר בתוצאות:** בקריאה חוזרת לפונקציה עם אותם ארגומנטים, נעשה שימוש בתוצאה השמורה במטמון.
    *   **האצת חישובים:** מאפשר להימנע מחישוב חוזר של ערכים שכבר חושבו.
    *   **אופטימיזציה של פונקציות רקורסיביות:** מאפשר להאיץ באופן משמעותי את פעולת פונקציות רקורסיביות, תוך הימנעות מחישובים חוזרים.

*   **מימוש בפייתון:**
    *   בפייתון ניתן למַמֵש מימוּא באמצעות המפענח (decorator) `functools.lru_cache()`.
    *   המפענח `@functools.lru_cache()` מוסיף אוטומטית שמירת תוצאות במטמון לפונקציה.

**דוגמאות:**

```python
from functools import lru_cache

# דוגמה לחישוב מספרי פיבונאצ'י באמצעות מימוּא
@lru_cache(maxsize=None)
def fibonacci(n):
    if n <= 1:
        return n
    return fibonacci(n - 1) + fibonacci(n - 2)


print(fibonacci(35))  # 9227465

# דוגמה ללא מימוּא
def fibonacci_no_memo(n):
    if n <= 1:
        return n
    return fibonacci_no_memo(n - 1) + fibonacci_no_memo(n - 2)

print(fibonacci_no_memo(30)) # 832040, קריאה זו מתבצעת הרבה יותר לאט

# דוגמה לשימוש ב-`lru_cache` עם maxsize
@lru_cache(maxsize=3)
def sum_cached(x, y):
  print(f"Calculating {x} + {y}")
  return x + y

sum_cached(1,2) # Calculating 1 + 2
sum_cached(1,3) # Calculating 1 + 3
sum_cached(1,4) # Calculating 1 + 4
sum_cached(1,2) # אינו מחושב (נלקח מהמטמון)
sum_cached(1,5) # Calculating 1 + 5, במקרה זה (1,3) יוצא מהמטמון
sum_cached(1,2) # מחושב שוב מכיוון ש (1,2) כבר אינו במטמון

```
**ניתוח הדוגמאות:**

1.  **`@lru_cache(maxsize=None)`:** זהו המפענח (decorator) שמממש מימוּא עבור הפונקציה `fibonacci(n)`. `maxsize=None` פירושו שהמטמון לא יהיה מוגבל במספר האובייקטים.
    *   הפונקציה `fibonacci(n)` נקראת עם הארגומנט `35`.
2.  **`fibonacci_no_memo(n)`:** דוגמה לפונקציה `fibonacci` ללא שימוש במימוּא.
3.  **`sum_cached(x, y)`:** דוגמה עם `lru_cache(maxsize=3)` - המטמון ישמור ערכים עבור עד 3 הקריאות האחרונות. אם נקראת הפונקציה עם ערכים חדשים, הישנים יימחקו מהמטמון לפי מדיניות LRU (Least Recently Used - הפריט שהיה בשימוש הכי פחות לאחרונה).

**ניתוח האפשרויות:**
*   **A. מימוּא הוא טכניקה המשמשת לשמירת נתונים במסד נתונים.:** שגוי.
*   **B. מימוּא הוא טכניקת אופטימיזציה שבה תוצאות חישובים קודמים נשמרות ונעשה בהן שימוש חוזר, כדי להימנע מחישובים חוזרים של ערכים זהים, במיוחד בפונקציות רקורסיביות.:** נכון.
*   **C. מימוּא הוא טכניקה שבה נתונים אינם משתנים, מה שמאפשר הימנעות משגיאות.:** שגוי, זהו תיאור אי-שינוי (Immutability).
*   **D. מימוּא הוא דרך הצפנת נתונים לפני שמירתם על דיסק.:** שגוי.

**לסיכום:**
*   מימוּא מאפשר להאיץ באופן משמעותי את החישובים, במיוחד בפונקציות רקורסיביות.
*   המפענח `functools.lru_cache()` מספק דרך פשוטה להוספת מימוּא לפונקציות פייתון.
*   ניתן להגדיר את `lru_cache`, למשל, על ידי הגבלת מספר התוצאות הנשמרות, כדי לא למלא את הזיכרון.

לפיכך, התשובה הנכונה היא **B. מימוּא הוא טכניקת אופטימיזציה שבה תוצאות חישובים קודמים נשמרות ונעשה בהן שימוש חוזר, כדי להימנע מחישובים חוזרים של ערכים זהים, במיוחד בפונקציות רקורסיביות.**
