**שאלה 573.** מהו תבנית התכנון הפונקציונלי "Functor" (פונקטור), וכיצד היא משמשת ב-Python? הבא דוגמה לשימוש בפונקטור להחלת פונקציה על ערך בתוך הקשר (קונטקסט).

-   א. פונקטור הוא פונקציה לביצוע חישובים בלולאה.
-   ב. פונקטור הוא דרך לארגון קוד, שבה הנתונים משתנים, אך הפונקציות אינן משתנות.
-   ג. פונקטור הוא הקשר התומך בפעולת `map` להחלת פונקציה על הערך הפנימי שלו.
-   ד. פונקטור הוא תחביר מיוחד לתיאור מחלקות ב-Python.

**התשובה הנכונה: ג**

**הסבר:**

בתכנות פונקציונלי, פונקטור (Functor) הוא הקשר (קונטקסט) התומך בפעולת `map`. הוא מאפשר להחיל פונקציה על ערך הנמצא "בתוך" הקשר זה, מבלי לשנות את הקשר עצמו. פונקטורים מאפשרים הפשטה מסוג הנתונים וממבנה הנתונים, ומספקים דרך אוניברסלית להחלת פונקציות.

*   **מושגים עיקריים של פונקטור:**
    *   **הקשר (קונטקסט):** מייצג מיכל או עטיפה שמכילה ערך.
    *   **פעולת `map`:** שיטה (או פונקציה) המאפשרת להחיל פונקציה על הערך הכלול בתוך ההקשר.
    *   **שימור ההקשר:** תוצאת פעולת `map` נשארת באותו הקשר כמו הערך המקורי.
    *   **הפשטה:** מאפשרת עיבוד ערכים בתוך סוגי הקשרים שונים באופן אחיד.

*   **יישום ב-Python:**
    *   ב-Python, פונקטורים יכולים להיות מיושמים באמצעות מחלקות שמגדירות שיטה שניתן לקרוא לה `map`.
    *   ניתן ליישם זאת כשיטת מופע של מחלקה או כפונקציה שמקבלת אובייקט מחלקה כארגומנט.

**דוגמאות:**

```python
# דוגמה ליישום Functor באמצעות מחלקה
class Maybe:
    def __init__(self, value):
        self.value = value

    def map(self, func):
        if self.value is None:
            return Maybe(None)
        return Maybe(func(self.value))

#שימוש ב-Maybe כ-Functor

maybe_number = Maybe(5)
squared = maybe_number.map(lambda x: x**2)
print(squared.value) # פלט: 25

nothing = Maybe(None)
squared_nothing = nothing.map(lambda x: x**2)
print(squared_nothing.value) # פלט: None

#דוגמה ל-Functor עם שימוש ברשימה
class ListFunctor:
    def __init__(self, value):
        self.value = value
    def map(self, func):
         return ListFunctor([func(x) for x in self.value])

list_functor = ListFunctor([1, 2, 3])
mapped_list = list_functor.map(lambda x: x*2)

print(mapped_list.value) # פלט: [2, 4, 6]

def  map_functor(func, obj):
  if  hasattr(obj, "map"):
    return obj.map(func)
  elif isinstance(obj, list):
    return [func(item) for item in obj]
  else:
      return None

# קוראים לפונקציה map_functor עבור רשימה ועבור אובייקט Maybe
list_functor = [1,2,3]
print(map_functor(lambda x:x*2, list_functor)) # פלט: [2, 4, 6]
maybe_num = Maybe(5)
print(map_functor(lambda x:x*2, maybe_num).value) # פלט: 10
```

**ניתוח דוגמאות:**

1.  **`class Maybe`:** מיישמת פונקטור שיכול לאחסן ערך או כלום (`None`).
    *   `__init__(self, value)`: מקבלת ערך התחלתי.
    *   `map(self, func)`: מחילה את הפונקציה `func` על הערך השמור, אם קיים.
2.  **שימוש ב-Maybe כ-Functor:**
    *   `maybe_number` נוצר עם המספר 5.
    *   `maybe_number.map(lambda x: x**2)` מחילה את הפונקציה `x ** 2` על הערך 5.
    *   `nothing` נוצר עם הערך `None`.
    *   `nothing.map(lambda x: x**2)` מחזירה `Maybe(None)`, מכיוון שהערך הוא `None`.
3.  **`class ListFunctor`:** מיישמת פונקטור עבור רשימות.
    *   `__init__`: בנאי.
    *   `map(self, func)`: מחילה את הפונקציה `func` על כל איבר ברשימה, ומחזירה `ListFunctor` חדש עם הרשימה החדשה.
4.  **`map_functor`:**
    *   בודקת אם יש שיטה `map` באובייקט, וקוראת לה.
    *   אם האובייקט הוא רשימה, מוחלת פעולת map על כל איבר.
    *   במקרה אחר - מחזירה `None`.
5.  מחילה את `map_functor` על רשימה ועל אובייקט Maybe.

**ניתוח אפשרויות:**
*   **א. פונקטור הוא פונקציה לביצוע חישובים בלולאה.:** לא נכון. פונקטור הוא מושג, ולא רק פונקציה, וכן אינו קשור ללולאות.
*   **ב. פונקטור הוא דרך לארגון קוד, שבה הנתונים משתנים, אך הפונקציות אינן משתנות.:** לא נכון. זהו תיאור של אי-שינוי (Immutability), לא של פונקטור.
*   **ג. פונקטור הוא הקשר התומך בפעולת `map` להחלת פונקציה על הערך הפנימי שלו.:** נכון.
*   **ד. פונקטור הוא תחביר מיוחד לתיאור מחלקות ב-Python.:** לא נכון.

*   `map` היא הפעולה המרכזית של פונקטור, המאפשרת לבצע טרנספורמציה על ערכים בתוך מיכל.
