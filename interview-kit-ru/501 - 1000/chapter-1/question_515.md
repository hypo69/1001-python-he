### question_515.md

**שאלה 515.** כיצד פועלת הצהרת `finally` בבלוק `try-except`?

- א. היא מבצעת את הקוד שבתוכה רק במקרה ששום חריג לא אירע בבלוק `try`.
- ב. היא מבוצעת ללא קשר לשאלה אם אירע חריג או לא, ואפילו אם נתקלה בהצהרת `return`.
- ג. היא משמשת להשלכת חריג, אם הוא לא הושלך בבלוק `try`.
- ד. היא מונעת הפצה נוספת של כל חריג שאירע בבלוק `try`.

**תשובה נכונה: ב**

**הסבר:**

הצהרת `finally` בבלוק `try-except` מיועדת לביצוע קוד אשר *חייב* להתבצע *תמיד*, ללא קשר לשאלה אם אירע חריג בבלוק `try` או לא, או אפילו אם בוצעה הצהרת `return`.

*   **אפשרות א'** אינה נכונה: לשם כך משתמשים ב-`else`, לא ב-`finally`.
*   **אפשרות ב'** נכונה: הקוד בבלוק `finally` מתבצע ללא קשר לאופן שבו בוצע בלוק `try`.
*   **אפשרות ג'** אינה נכונה: לשם השלכת חריג משתמשים ב-`raise`, לא ב-`finally`.
*   **אפשרות ד'** אינה נכונה: `finally` אינה מונעת הפצת חריגים, אלא רק מבטיחה את ביצוע הקוד.

**כיצד פועל `try...except...finally`:**

1.  מבוצע הקוד בבלוק `try`.
2.  אם בבלוק `try` מתרחש חריג, פייתון מחפש בלוק `except` מתאים לטיפול בו.
3.  **ללא קשר** לשאלה אם אירע חריג והאם נמצא בלוק `except`, *תמיד* מבוצע הקוד בבלוק `finally`.
4.  אם לא היה חריג, בלוק `finally` יבוצע לאחר בלוק `try`.
5.  בלוק `finally` מבוצע גם אם בלוק `try` מכיל הצהרת `return`, או אפילו אם ה-`except` מכיל הצהרת `return`.

**שימושים טיפוסיים ב-`finally`:**

*   **שחרור משאבים:** שחרור משאבים חיצוניים, כגון קבצים, סוקטים, או חיבורים למסדי נתונים.
*   **הבטחת סיום תקין:** ביצוע פעולות סופיות, אשר מובטח שיבוצעו, ללא קשר לביצוע מוצלח או לא מוצלח של הקוד.

**דוגמה:**

```python
def my_function(num: int) -> int | None:
  """בדיקת זוגיות של מספר וטיפול בחריגים באמצעות try-except-finally"""
  try:
    result: int = 10 // num
    print(f"תוצאת החלוקה: {result}")
    if num > 5:
       return result
  except ZeroDivisionError:
    print("לא ניתן לחלק באפס")
    return None # מחזירים None אם אירעה שגיאה
  finally:
      print("בלוק finally תמיד מבוצע")


res1 = my_function(2)
print(f"תוצאה: {res1}")

res2 = my_function(0) # יתרחש ZeroDivisionError
print(f"תוצאה: {res2}")

res3 = my_function(6)
print(f"תוצאה: {res3}")

# פלט:
# תוצאת החלוקה: 5
# בלוק finally תמיד מבוצע
# תוצאה: 5
# לא ניתן לחלק באפס
# בלוק finally תמיד מבוצע
# תוצאה: None
# תוצאת החלוקה: 1
# בלוק finally תמיד מבוצע
# תוצאה: 1
```

**לסיכום:**
* בעת קריאה ל-`my_function(2)`, הקוד מתבצע ללא שגיאות, ובלוק `finally` מבוצע ולאחר מכן מוחזרת התוצאה.
* בעת קריאה ל-`my_function(0)` מתרחש `ZeroDivisionError`, הקוד עובר לבלוק except, שם מטופל החריג ומוחזר `None`, וכן מבוצע הקוד מבלוק `finally`.
* בעת קריאה ל-`my_function(6)` מוחזר `result`, וכן מבוצע בלוק `finally`.

לפיכך, **אפשרות ב'** מהווה את התשובה הנכונה.
**השאלה המקורית:**

מה תהיה תוצאת ביצוע הקוד הבא?

```python
my_list = [1, 2, 3]
my_list.append(my_list)
print(my_list)
```

א. [1, 2, 3, [1, 2, 3]]
ב. [1, 2, 3, [...]]
ג. שגיאת גלישת מחסנית
ד. יושלך חריג מסוג Type Error

**תשובה נכונה: ב**

**הסבר:**

שאלה זו בודקת הבנה של אופן הפעולה של השיטה `append` וכיצד פייתון מטפל בהפניות מחזוריות ברשימות.

1.  **רשימה ראשונית:** `my_list` נוצר בתור `[1, 2, 3]`.
2.  **הוספת הרשימה לעצמה:** השיטה `append(my_list)` מוסיפה את *הרשימה עצמה* `my_list` ל*סוף* הרשימה `my_list`. פעולה זו יוצרת רשימה מקוננת והפניה מחזורית, מכיוון שהאיבר האחרון של `my_list` מפנה ל-`my_list` עצמה.
3.  **פלט:** כאשר אנו מדפיסים את `my_list`, פייתון מזהה את ההפניה המחזורית ובמקום לנסות לפרוש את הקינון לאינסוף, הוא מסמן הפניה זו בתור `[...]`.

**ניתוח האפשרויות:**
*   **א. [1, 2, 3, [1, 2, 3]]**: לא נכון, פייתון לא יפרוש את הקינון לאינסוף.
*   **ב. [1, 2, 3, [...]]**: נכון, פייתון מזהה הפניה מחזורית ומחליף אותה ב-`[...]`.
*   **ג. שגיאת גלישת מחסנית**: לא נכון, פייתון מטפל בהפניות מחזוריות בצורה שמאפשרת להימנע מגלישת מחסנית.
*   **ד. יושלך חריג מסוג Type Error**: לא נכון, הקוד מתבצע ללא שגיאות טיפוס.
