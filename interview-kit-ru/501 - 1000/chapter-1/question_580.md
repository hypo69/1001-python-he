**שאלה 580.** מהו ה-GIL (Global Interpreter Lock) ב-Python ואיזה תפקיד הוא ממלא בתכנות מרובה תהליכונים?

-   א. ה-GIL הוא כלי לניהול משאבי זיכרון ב-Python, המונע דליפות זיכרון.
-   ב. ה-GIL הוא מיוטקס (mutex) המשמש לסנכרון תהליכונים ב-Python, המגביל את הגישה לאובייקטי Python ומונע ביצוע בו זמני של מספר תהליכונים של בייט-קוד.
-   ג. ה-GIL הוא מנגנון לאופטימיזציית קוד Python, המאפשר לקמפל אותו לקוד מכונה בעל ביצועים טובים יותר.
-   ד. ה-GIL הוא דרך לארגן חישובים מקביליים ב-Python באמצעות מספר תהליכים.

**תשובה נכונה: ב**

**הסבר:**

ה-GIL (Global Interpreter Lock) — הוא מנעול גלובלי של המפרש, המהווה מיוטקס (mutex) המשמש ב-CPython (המימוש הראשי של Python). מיוטקס זה מבטיח שבכל רגע נתון רק תהליכון Python אחד יכול לבצע בייט-קוד Python, מה שמשמעותו שריבוי תהליכונים אינו מתבצע בפועל בצורה מקבילית אמיתית.

*   **תפקידים עיקריים של ה-GIL:**
    *   **סנכרון תהליכונים:** ה-GIL מבטיח שרק תהליכון אחד יוכל לגשת לאובייקטי Python ולבצע בייט-קוד. זה מונע תנאי מרוץ ונזק לאובייקטים בעת גישה מקבילית.
    *   **מיוטקס (mutex):** ה-GIL מייצג מיוטקס המשמש לחסימה ושחרור גישה לאובייקטי Python.
    *   **הגבלת מקביליות:** ה-GIL אינו מאפשר לבצע קוד Python באופן מקבילי אמיתי במספר תהליכונים, גם אם נעשה שימוש בספריות של ריבוי תהליכונים.
*   **השפעה על ריבוי תהליכונים:**
    *   ה-GIL מגביל ריבוי תהליכונים ב-Python, מכיוון שרק תהליכון אחד יכול לבצע בייט-קוד בכל רגע נתון.
    *   משמעות הדבר היא שריבוי תהליכונים ב-Python עשוי שלא לספק את העלייה הרצויה בביצועים במקרה של משימות עתירות עיבוד (CPU-bound).
    *   עם זאת, ה-GIL אינו מונע מקביליות של פעולות קלט/פלט (I/O-bound), כאשר התהליכונים מבלים את רוב הזמן בהמתנה להשלמת קלט/פלט, ה-GIL יכול לשחרר את המנעול, ובכך לאפשר לתהליכונים אחרים לעבוד.

**דוגמאות:**

נניח שיש לנו שני תהליכונים שאמורים לעבד נתונים במקביל:

```python
import threading
import time

def worker(name):
    print(f"Thread {name} started")
    time.sleep(2) # הדמיה של עבודה ארוכה
    print(f"Thread {name} finished")

threads = []
for i in range(2):
    thread = threading.Thread(target=worker, args=(i,))
    threads.append(thread)
    thread.start()

for thread in threads:
    thread.join()

print("All threads finished")
```

בדוגמה זו, למרות שאנו מפעילים שני תהליכונים, הקוד יבוצע באופן סדרתי, ולא מקבילי. ה-GIL יבטיח שבכל רגע נתון רק תהליכון אחד פועל.

**ניתוח האפשרויות:**
*   **א. ה-GIL הוא כלי לניהול משאבי זיכרון ב-Python, המונע דליפות זיכרון.:** לא נכון.
*   **ב. ה-GIL הוא מיוטקס (mutex) המשמש לסנכרון תהליכונים ב-Python, המגביל את הגישה לאובייקטי Python ומונע ביצוע בו זמני של מספר תהליכונים של בייט-קוד.:** נכון.
*   **ג. ה-GIL הוא מנגנון לאופטימיזציית קוד Python, המאפשר לקמפל אותו לקוד מכונה בעל ביצועים טובים יותר.:** לא נכון.
*   **ד. ה-GIL הוא דרך לארגן חישובים מקביליים ב-Python עם שימוש במספר תהליכים.:** לא נכון.

**לסיכום:**
*   ה-GIL הוא מנגנון ב-Python להגנה מפני גישה מקבילית למשאבים משותפים בריבוי תהליכונים.
*   בגלל ה-GIL, ריבוי תהליכונים אינו מאפשר לבצע קוד Python באופן מקבילי אמיתי, ויש להשתמש בו רק במשימות מסוג I/O-bound (פעולות קלט/פלט).
*   למקביליות של משימות מסוג CPU-bound, יש להשתמש בריבוי תהליכים (multiprocessing).

לפיכך, התשובה הנכונה היא **ב. ה-GIL הוא מיוטקס (mutex) המשמש לסנכרון תהליכונים ב-Python, המגביל את הגישה לאובייקטי Python ומונע ביצוע בו זמני של מספר תהליכונים של בייט-קוד.**
