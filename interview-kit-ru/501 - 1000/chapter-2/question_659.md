**שאלה 659.** ב-Python, מה המשמעות של שימוש בקו תחתון כפול (__) בתחילת ובסוף שם משתנה (לדוגמה, `__some_variable__`)?

-   A. משתנים עם קו תחתון כפול בתחילת ובסוף השם מיועדים לאחסון נתונים רגישים.
-   B. משתנים עם קו תחתון כפול בתחילת ובסוף השם משמשים ליצירת משתנים סטטיים במחלקות.
-   C. משתנים עם קו תחתון כפול בתחילת ובסוף השם (שיטות ותכונות "דאנדר" או "קסם") משמשים לשיטות ותכונות מיוחדות, שיכולות להיקרא אוטומטית ב-Python, ומספקות מידע אודות האובייקט.
-   D. משתנים עם קו תחתון כפול בתחילת ובסוף השם משמשים רק למשתנים פנימיים.

**תשובה נכונה: C**

**הסבר:**

ב-Python, שמות עם קו תחתון כפול בתחילת ובסוף (לדוגמה, `__init__`, `__str__`, `__name__`, `__len__`) נקראים שיטות ותכונות "דאנדר" (dunder, קיצור של "double underscore") או "קסם" (magic). הם משמשים לשיטות ותכונות מיוחדות בעלות משמעות ייחודית בשפת Python.

*   **מאפיינים עיקריים של שיטות ותכונות "דאנדר":**
    *   **ייעוד מיוחד:** מגדירים את התנהגות האובייקטים במצבים שונים ומשמשים ליישום פעולות יסודיות של השפה.
    *   **קריאה אוטומטית:** לעתים קרובות נקראים אוטומטית על ידי מפרש ה-Python בעת ביצוע פעולות מסוימות.
    *   **ניהול התנהגות:** מאפשרים למתכנתים להתאים אישית את התנהגות האובייקטים והמחלקות.
    *   **משמעות מוגדרת מראש:** הקווים התחתונים הכפולים מצביעים על כך שלשיטה יש משמעות מיוחדת עבור מפרש ה-Python.
    *   **ניהול:** מנוהלים ומוגדרים על ידי python.

*   **דוגמאות לשיטות "דאנדר":**
    *   `__init__` - בנאי המחלקה, נקרא בעת יצירת אובייקט.
    *   `__str__` - נקרא על ידי הפונקציה `str()` וצריך להחזיר ייצוג מחרוזתי של האובייקט.
    *   `__len__` - נקרא על ידי הפונקציה `len()` וצריך להחזיר את האורך.
    *   `__add__`, `__mul__`, `__sub__` - נקראות בעת שימוש באופרטורים `+`, `*`, `-`.
*   **דוגמאות לתכונות "דאנדר":**
    *   `__name__` - שם המודול.
    *   `__module__` - שם המודול שבו מוגדרת המחלקה.
    *   `__doc__` - מחרוזת התיעוד עבור האובייקט.
    *   `__file__` - נתיב הקובץ.
    *   `__dict__` - תכונות האובייקט.
    *   `__class__` - מחלקת האובייקט.

**דוגמאות:**
```python
class MyClass:
    def __init__(self, name):
        self.name = name
    def __str__(self):
      return f"Object: {self.name}"

    def __len__(self):
        return len(self.name)
obj = MyClass("test")
print(f"Объект: {obj}")  # יודפס: אובייקט: test, מכיוון ש-`__str__` הוגדר
print(f"Длина объекта: {len(obj)}") # יודפס 4, מכיוון ש-`__len__` הוגדר
print(f"Имя класса: {obj.__class__}")# יודפס שם המחלקה: <class '__main__.MyClass'>
print(f"Документация класса: {MyClass.__doc__}") # יודפס תיעוד המחלקה: None
print(f"Атрибуты обьекта: {obj.__dict__}")  # יודפס: תכונות האובייקט: {'name': 'test'}

def my_func():
    """מחרוזת תיעוד עבור my_func"""
    pass
print(f"Имя модуля: {my_func.__module__}") # יודפס שם המודול: __main__
print(f"Документация  функции: {my_func.__doc__}") # יודפס תיעוד הפונקציה: Docstring עבור my_func
```

*   **C. משתנים עם קו תחתון כפול בתחילת ובסוף השם (שיטות ותכונות "דאנדר" או "קסם") משמשים לשיטות ותכונות מיוחדות, שיכולות להיקרא אוטומטית ב-Python, ומספקות מידע אודות האובייקט.:** נכון.

**לסיכום:**

*   ברוב המקרים, אין להשתמש בשיטות ותכונות `__dunder__` ליצירת משתנים מותאמים אישית, אלא לדריסה (override) של ההתנהגות הסטנדרטית של אובייקטים.
