### `question_695.md`

**שאלה 695.** נתון מערך `capacity`, המייצג את הקיבולת של `n` תיקים, ומערך `rocks`, כאשר `rocks[i]` הוא מספר האבנים שכבר נמצאות בתיק ה-`i`. כמו כן, נתון מספר שלם `additionalRocks`, המייצג את מספר האבנים שניתן להוסיף לכל תיק. פתח אלגוריתם המחשב את מספר התיקים המקסימלי שניתן למלא, על ידי חלוקת `additionalRocks`.

**דוגמאות:**

```
קלט: capacity = [2,3,4,5], rocks = [1,2,4,4], additionalRocks = 2
פלט: 3

קלט: capacity = [10,2,2], rocks = [2,2,0], additionalRocks = 100
פלט: 3
```

- א. כדי לפתור בעיה זו יש למיין תחילה את המערך capacity, ולאחר מכן למלא את התיקים ברצף עד שנגמרות האבנים.
- ב. כדי לפתור בעיה זו יש למיין תחילה את ההפרש בין capacity ל-rocks, ולאחר מכן למלא את התיקים ברצף עד שנגמרות האבנים.
- ג. כדי לפתור בעיה זו, יש לעבור על כל הקומבינציות כדי לקבל את התוצאה האופטימלית ביותר, מה שיהפוך את הבעיה ללא יעילה.
- ד. כדי לפתור בעיה זו יש להשתמש באלגוריתם חמדן, אשר ממלא תחילה את התיקים עם ההפרש הקטן ביותר בין הקיבולת למספר האבנים בהם.

**תשובה נכונה: ב**

**הסבר:**

כדי לפתור בעיה זו ניתן להשתמש באלגוריתם חמדן, אשר מורכב מהשלבים הבאים: נוצרת רשימה חדשה עם ההפרש בין capacity ל-rocks, רשימה זו ממוינת בסדר עולה, ולאחר מכן, האבנים מחולקות ברצף לתיקים בהתאם לגודל ההפרש, עד שנגמרות האבנים הנוספות `additionalRocks`.

* **אלגוריתם (גישה חמדנית):**
    1. **חישוב ההפרש:** יצירת רשימה שבה כל איבר הוא ההפרש בין `capacity[i]` ל-`rocks[i]`, כלומר כמה אבנים צריך להוסיף לכל תיק.
    2. **מיון:** מיון רשימה זו בסדר עולה.
    3. **איטרציה וחלוקה:** איטרציה על הרשימה הממוינת ובכל איטרציה הוספת כמות האבנים הנדרשת לתיק, אם יש מספיק `additionalRocks`.
        * אם אין מספיק `additionalRocks` למילוי כל התיקים, עצירת הלולאה.
    4. **ספירה:** ספירת מספר התיקים שהצלחנו למלא.
* **מדוע זה עובד:**
    * הגישה החמדנית מתבססת על כך שאנו ממלאים תחילה את התיקים הדורשים את המספר הקטן ביותר של אבנים.
    * בכל שלב של בחירת הפתרון הטוב ביותר מקומית (מילוי התיקים עם ההפרש המינימלי), נקבל פתרון אופטימלי גלובלית.

* **יתרונות האלגוריתם:**
    * **אופטימליות:** הגישה החמדנית מספקת פתרון אופטימלי לבעיה הנתונה.
    * **סיבוכיות לינארית:** סיבוכיות הזמן היא O(n log n) עקב הצורך במיון הרשימה.
    * **פשטות:** קל ליישום.

**דוגמאות (פסאודו-קוד):**
```
function maximum_filled_bags(capacity, rocks, additionalRocks):
  differences = []
  for i in range(length(capacity)):
    differences.append(capacity[i] - rocks[i])
  sort differences
  count = 0
  for i in differences:
    if additionalRocks >= differences[i]:
        additionalRocks -= differences[i]
        count += 1
  return count
```
**דוגמאות ליישום בפייתון:**

```python
def maximum_filled_bags(capacity, rocks, additionalRocks):
    differences = []
    for i in range(len(capacity)):
        differences.append(capacity[i] - rocks[i])
    differences.sort()
    count = 0
    for diff in differences:
        if additionalRocks >= diff:
            additionalRocks -= diff
            count += 1
        else:
            break
    return count

capacity1 = [2,3,4,5]
rocks1 = [1,2,4,4]
additionalRocks1 = 2
print(f"קלט: capacity = {capacity1}, rocks = {rocks1}, additionalRocks = {additionalRocks1}")
print(f"פלט: {maximum_filled_bags(capacity1, rocks1, additionalRocks1)}") # פלט: 3

capacity2 = [10,2,2]
rocks2 = [2,2,0]
additionalRocks2 = 100
print(f"קלט: capacity = {capacity2}, rocks = {rocks2}, additionalRocks = {additionalRocks2}")
print(f"פלט: {maximum_filled_bags(capacity2, rocks2, additionalRocks2)}") # פלט: 3
```
