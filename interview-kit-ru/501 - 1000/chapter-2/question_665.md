**שאלה 665.** מהם JSON ו-XML, ובמה שונים זה מזה כתבניות לייצוג נתונים?

-   א. JSON ו-XML הם טיפוסי נתונים ב-Python.
-   ב. JSON ו-XML הם תבניות המשמשות אך ורק לתיאור ממשקים גרפיים.
-   ג. JSON הוא תבנית טקסטואלית קלת-משקל, ו-XML הוא תבנית טקסטואלית לייצוג נתונים מובנים, המשתמשת בתגיות.
-   ד. JSON משמש אך ורק להעברת נתונים ברשת, ו-XML משמש אך ורק לאחסון נתונים בקבצים.

**תשובה נכונה: ג**

**הסבר:**

JSON (JavaScript Object Notation) ו-XML (eXtensible Markup Language) הן שתי תבניות פופולריות לייצוג נתונים, המשמשות לאחסון והעברה של מידע מובנה. למרות ששתי התבניות מיועדות לייצוג נתונים, יש להן הבדלים משמעותיים בתחביר, במבנה ובתחומי היישום.

*   **JSON (JavaScript Object Notation):**
    *   **קל-משקל:** תבנית טקסטואלית עם תחביר פשוט וברור.
    *   **מבנה:** מבוסס על זוגות "מפתח-ערך", רשימות, אובייקטים וטיפוסי נתונים פרימיטיביים.
    *   **אוניברסלי:** קל לקריאה ולייצור הן על ידי מכונות והן על ידי בני אדם.
    *   **שימוש:** משמש לעיתים קרובות בפיתוח ווב להעברת נתונים בין השרת ללקוח.
    *   **תחביר:** משתמש בסוגריים מסולסלים `{}` עבור אובייקטים, סוגריים מרובעים `[]` עבור מערכים, נקודתיים `:` עבור זוגות "מפתח-ערך", וגרשיים `"` עבור מחרוזות.
    *   **טיפוסי נתונים:** תומך במחרוזות, מספרים, ערכים בוליאניים, `null`, מערכים ואובייקטים.

*   **XML (eXtensible Markup Language):**
    *   **גמישות:** תבנית טקסטואלית למבנוּת נתונים ותיאור מסמכים.
    *   **תגיות:** משתמש בתגיות פותחות וסוגרות לייצוג אלמנטים ומאפיינים.
    *   **מבנה מורכב:** התחביר מורכב יותר מזה של JSON, כולל תגיות מקוננות, מאפיינים וכו'.
    *   **שימוש:** מיושם בתחומים שונים, כגון קובצי תצורה, שירותי ווב והעברת נתונים.
    *   **תחביר:** משתמש בתגיות פותחות וסוגרות `<tag>content</tag>`, מאפיינים `<tag attribute="value">content</tag>`, וכן בהערות.

**הבדלים בין JSON ל-XML:**

| מאפיין         | JSON                                     | XML                                                                              |
|---------------|------------------------------------------|----------------------------------------------------------------------------------|
| **תחביר**     | פשוט, מבוסס על אובייקטים ומערכים       | מורכב יותר, מבוסס על תגיות ומאפיינים                                              |
| **קריאות**    | קל לקריאה על ידי בני אדם                 | פחות קריא מ-JSON, אך קריא יותר מתבניות בינאריות.                                   |
| **גודל**      | בדרך כלל קטן יותר                        | בדרך כלל גדול יותר מ-JSON עקב תגיות עודפות                                        |
| **תחומי יישום**| פיתוח ווב, העברת נתונים, קובצי תצורה     | קובצי תצורה, שירותי ווב, העברת מסמכים (לדוגמה, XML).                               |
| **תמיכה**     | נתמך נרחב בווב                           | פחות מ-JSON, אך עדיין נתמך בתוכנות שונות.                                         |
| **מורכבות ניתוח**| פשוט יותר                               | מורכב יותר מ-JSON                                                               |

**דוגמאות:**
```python
# דוגמת JSON (מחרוזת)
json_data = '{"name": "John Doe", "age": 30, "city": "New York"}'
print(f"דוגמת JSON: {json_data}")

# דוגמת XML (מחרוזת)
xml_data = """
<person>
  <name>Jane Doe</name>
  <age>25</age>
  <city>London</city>
</person>
"""
print(f"דוגמת XML: {xml_data}")

import json
# דוגמה להמרה של מילון למחרוזת JSON
my_dict = {"a" : 1, "b" : 2, "c": [1,2,3]}
json_from_dict = json.dumps(my_dict)
print(f"המרת מילון ל-JSON: {json_from_dict}") # {"a": 1, "b": 2, "c": [1, 2, 3]}
```
**ניתוח האפשרויות:**
*   **א. JSON ו-XML הם טיפוסי נתונים ב-Python.:** שגוי.
*   **ב. JSON ו-XML — זה פורמטים שמשמשים אך ורק לתיאור ממשקים גרפיים.:** שגוי.
*   **ג. JSON — זה פורמט טקסטואלי קל-משקל, ו-XML — פורמט טקסטואלי להצגת נתונים מובנים שמשתמש בתגים.:** נכון.
*   **ד. JSON משמש אך ורק להעברת נתונים ברשת, ו-XML משמש אך ורק לאחסון נתונים בקבצים.:** שגוי.

**לסיכום:**
*   JSON פשוט וקל-משקל יותר מ-XML.
*   XML גמיש ומורכב יותר מ-JSON.
*   הבחירה ביניהם תלויה במשימה.

לפיכך, התשובה הנכונה היא **ג. JSON — זה פורמט טקסטואלי קל-משקל, ו-XML — פורמט טקסטואלי להצגת נתונים מובנים שמשתמש בתגים.**
