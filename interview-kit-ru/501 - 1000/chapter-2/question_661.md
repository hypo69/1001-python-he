**שאלה 662.** מהי מטרת השימוש במילת המפתח `async` בפייתון?

-   א. מילת המפתח `async` משמשת ליצירת תוכניות מרובות-נימים בפייתון.
-   ב. מילת המפתח `async` משמשת ליצירת לולאות עם בדיקת תנאי אוטומטית.
-   ג. מילת המפתח `async` משמשת ליצירת פונקציות אסינכרוניות (קורוטינות), שיכולות להשהות את ביצוען ולא לחסום את התהליך הראשי.
-   ד. מילת המפתח `async` משמשת ליצירת גנרטורים המחזירים סדרת מספרים אינסופית.

**תשובה נכונה: ג**

**הסבר:**

מילת המפתח `async` בפייתון משמשת להגדרת פונקציות אסינכרוניות (הידועות גם כקורוטינות או שגרות שיתופיות). פונקציות אסינכרוניות מאפשרות כתיבת קוד שיכול לבצע פעולות קלט/פלט (I/O) ארוכות ללא חסימת תהליך הביצוע הראשי של התוכנית, מה שהופך אותן לשימושיות עבור בקשות רשת, עבודה עם מסדי נתונים ומשימות אחרות הדורשות המתנה לא-חוסמת.

*   **מאפיינים עיקריים של `async`:**
    *   **פונקציה אסינכרונית (קורוטינה):** מילת המפתח `async` משמשת להגדרת פונקציות שיכולות להשהות את ביצוען מבלי לחסום את התהליך הראשי.
    *   **שימוש יחד עם `await`:** משמשות יחד עם `await` להשהיה וחידוש הביצוע של הפונקציה.
    *   **קוד לא-חוסם:** `async` משמשת לכתיבת קוד שאינו חוסם את תהליך הביצוע הראשי.
    *   **תחביר:** `async def my_function(): ...`.

*   **לשם מה משתמשים ב-`async`:**
    *   **קלט/פלט אסינכרוני:** מאפשר ביצוע פעולות קלט/פלט אסינכרוניות, כגון בקשות לשרתים, עבודה עם קבצים וכו'.
    *   **ביצוע מקבילי:** בשילוב עם `asyncio`, מאפשר ביצוע מקביל של מספר פונקציות אסינכרוניות, מה שמעלה את הביצועים.
    *   **אי-חסימה:** אינה חוסמת את תהליך הביצוע, ומאפשרת לבצע משימות אחרות בזמן שממתינים להשלמת חישובים אסינכרוניים.

**דוגמאות:**
```python
import asyncio
# פונקציה אסינכרונית המשתמשת ב-await
async def fetch_data():
    print("Fetching data...")
    await asyncio.sleep(2)  # הדמיה של פעולה ארוכה
    print("Data fetched")
    return "Data"

async def main():
    result = await fetch_data()
    print(result)

# הפעלת הפונקציה האסינכרונית
asyncio.run(main())
# פלט:
# Fetching data...
# Data fetched
# Data
# דוגמה לשימוש ב-asyncio.gather לקריאה מקבילית של קורוטינות
async def download_data(url):
    print(f"Downloading from {url}")
    await asyncio.sleep(1) # הדמיית הורדה
    print(f"Download from {url} done!")
    return f"data from {url}"

async def main2():
   urls = ["http://example.com/1", "http://example.com/2", "http://example.com/3"]
   tasks = [download_data(url) for url in urls]
   results = await asyncio.gather(*tasks)
   print(results)
asyncio.run(main2())
# פלט (בסדר קרוב למקבילי):
# Downloading from http://example.com/1
# Downloading from http://example.com/2
# Downloading from http://example.com/3
# Download from http://example.com/1 done!
# Download from http://example.com/2 done!
# Download from http://example.com/3 done!
# ['data from http://example.com/1', 'data from http://example.com/2', 'data from http://example.com/3']
```

**ניתוח האפשרויות:**
*   **א. מילת המפתח `async` משמשת ליצירת תוכניות מרובות-נימים בפייתון.:** שגוי. ריבוי-נימים בפייתון אינו קשור ל-`async`.
*   **ב. מילת המפתח `async` משמשת ליצירת לולאות עם בדיקת תנאי אוטומטית.:** שגוי.
*   **ג. מילת המפתח `async` משמשת ליצירת פונקציות אסינכרוניות (קורוטינות), שיכולות להשהות את ביצוען ולא לחסום את התהליך הראשי.:** נכון.
*   **ד. מילת המפתח `async` משמשת ליצירת גנרטורים המחזירים סדרת מספרים אינסופית.:** שגוי. גנרטורים משתמשים ב-`yield`.

**בסיכומו של דבר:**
*   `async` מאפשר יצירת פונקציות אסינכרוניות, המתבצעות מבלי לחסום את התהליך הראשי.
*   משמש ליישום קוד לא-חוסם, לדוגמה בעבודה עם קלט/פלט.
*   עובד בשילוב עם `await`, וגם בשילוב עם ספריית `asyncio`.

לפיכך, התשובה הנכונה היא **ג. מילת המפתח `async` משמשת ליצירת פונקציות אסינכרוניות (קורוטינות), שיכולות להשהות את ביצוען ולא לחסום את התהליך הראשי.**
