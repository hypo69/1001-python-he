### `question_679.md`

**שאלה 679.** באיור מוצג עץ חיפוש בינארי (BST), המשמש למשחק "נחש את המספר". פתח אלגוריתם לחיפוש המספר הסודי, באמצעות חיפוש בינארי, ויישם API המחזיר:

-   `-1`, אם המספר הסודי גדול מהמספר שנבחר.
-   `1`, אם המספר הסודי קטן מהמספר שנבחר.
-   `0`, אם המספר הסודי והמספר שנבחר זהים.

**הצגת העץ ב-ASCII:**

```
      7
     / \
    3   9
   / \ / \
  1  5  8  10
   \   / \
    2 4   6
```
*   **תנאי הבעיה:** נתון לכם עץ חיפוש בינארי. אתם בוחרים מספר ולאחר מכן, בהתבסס על השוואה למספר הסודי, האלגוריתם אומר לכם אם המספר הסודי גדול או קטן יותר, ועליכם לנחש את המספר הזה.
*   **ממשק:**
    *   האלגוריתם צריך להיות מסוגל לקבל מספר ולהחזיר:
    *   `-1`: אם המספר הסודי גדול מהמספר שנבחר.
    *   `1`: אם המספר הסודי קטן מהמספר שנבחר.
    *   `0`: אם המספר הסודי והמספר שנבחר זהים.
*   **קלט:** `n = 10, pick = 6`
*   **פלט:** `6`

-   א. יש לעבור על כל הצמתים בעץ בסדר עולה עד למציאת המספר המבוקש.
-   ב. לחיפוש המספר הסודי יש להשתמש באלגוריתם חיפוש לעומק (DFS).
-   ג. לחיפוש המספר הסודי יש להשתמש באלגוריתם חיפוש לרוחב (BFS).
-   ד. לחיפוש המספר הסודי יש להשתמש באלגוריתם חיפוש בינארי, המאפשר לשלול צמתים שאינם מועמדים לערך המבוקש, בהתאם לתוצאות ההשוואה.

**תשובה נכונה: ד**

**הסבר:**

שימוש בעץ חיפוש בינארי (BST) ואלגוריתם חיפוש בינארי הוא הדרך האופטימלית לחיפוש המספר הסודי בבעיה זו, מכיוון שהוא מאפשר ניפוי צמתים מיותרים בכל שלב ובעל סיבוכיות זמן לוגריתמית O(log n).

*   **אלגוריתם (חיפוש בינארי):**
    1.  **התחלה:** מתחילים את החיפוש משורש העץ (צומת `7` בדוגמה).
    2.  **השוואה למספר הסודי:** משווים את הערך הנוכחי של הצומת למספר הסודי `pick`.
    3.  **קטן יותר:** אם הערך הנוכחי של הצומת גדול מהמספר הסודי, עוברים לתת-עץ השמאלי.
    4.  **גדול יותר:** אם הערך הנוכחי של הצומת קטן מהמספר הסודי, עוברים לתת-עץ הימני.
    5.  **התאמה:** אם הערך הנוכחי של הצומת שווה למספר הסודי, החיפוש הסתיים ומחזירים את הערך הנוכחי.
    6.  **חזרה:** ממשיכים בצעדים 2-5 עד למציאת המספר הסודי, או עד שמגיעים לצומת שאין לו ילדים, ואז מחזירים `-1`.

*   **מימוש API:**
    *   ה-API צריך לקבל מספר (הצומת שיש לבדוק) ולהשוות אותו למספר הסודי.
    *   בהתאם להשוואה, ה-API מחזיר `-1`, `1` או `0`.

*   **מדוע האפשרויות האחרות אינן מתאימות:**
    *   **אפשרות א** - מעבר ליניארי על העץ - אינו יעיל, מכיוון שאינו משתמש בתכונות עץ החיפוש ובעל סיבוכיות `O(n)`.
    *   **אפשרויות ב ו-ג** - אלגוריתמי מעבר על העץ לעומק או לרוחב אינם יעילים לבעיה זו ובעלי סיבוכיות `O(n)`.

**דוגמאות (פסאודו-קוד):**

```
function guess_number(root, pick)
    current_node = root

    while current_node is not None:
        if current_node.val == pick:
            return current_node.val  # נמצא
        if pick < current_node.val:
            current_node = current_node.left
        elif pick > current_node.val:
            current_node = current_node.right

    return -1
function check_number(number_to_check,  pick):
  if number_to_check > pick:
        return -1
    elif number_to_check < pick:
         return 1
    else:
         return 0

```
**ניתוח האפשרויות:**
*   **א. יש לעבור על כל הצמתים בעץ בסדר עולה עד למציאת המספר המבוקש.:** לא נכון, לכך משמש חיפוש בינארי.
*   **ב. לחיפוש המספר הסודי יש להשתמש באלגוריתם חיפוש לעומק (DFS).:** לא נכון, מכיוון ש-DFS אינו האלגוריתם האופטימלי ביותר לחיפוש ב-BST.
*   **ג. לחיפוש המספר הסודי יש להשתמש באלגוריתם חיפוש לרוחב (BFS).:** לא נכון, מכיוון ש-BFS אינו האלגוריתם האופטימלי ביותר לחיפוש ב-BST.
*   **ד. לחיפוש המספר הסודי יש להשתמש באלגוריתם בינארי, המאפשר לשלול צמתים שאינם מועמדים לערך המבוקש, בהתאם לתוצאות ההשוואה.:** נכון.

**לסיכום:**
*   חיפוש בינארי מאפשר חיפוש יעיל של האיבר המבוקש בעץ בינארי, שהוא ממוין (BST).
*   לאלגוריתם יש סיבוכיות זמן O(log n), במקום O(n) במעבר מלא.
*   שיטת השוואת האיברים בכל שלב מאפשרת לזנוח חלק לא מתאים של העץ.

לפיכך, התשובה הנכונה היא **ד. לחיפוש המספר הסודי יש להשתמש באלגוריתם בינארי, המאפשר לשלול צמתים שאינם מועמדים לערך המבוקש, בהתאם לתוצאות ההשוואה.**
