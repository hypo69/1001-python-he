### `question_676.md`

**שאלה 676.** נתונים לך `n` ערים המחוברות במספר מסוים של טיסות. המערך `flights`, כאשר `flights[i] = [from_i, to_i, price_i]`, מציין שיש טיסה מהעיר `from_i` לעיר `to_i` בעלות `price_i`. נתונים גם שלושה מספרים שלמים `src`, `dst`, ו-`k`. תאר אלגוריתם למציאת העלות הזולה ביותר לטיסה מהעיר `src` לעיר `dst` עם לא יותר מ-`k` עצירות. אם מסלול כזה אינו קיים, יש להחזיר `-1`.

**דוגמה:**

```
Ввод: n = 4, flights = [[0,1,100],[1,2,100],[2,0,100],[1,3,600],[2,3,200]], src = 0, dst = 3, k = 1
Вывод: 700

Ввод: n = 3, flights = [[0,1,100],[1,2,100],[0,2,500]], src = 0, dst = 2, k = 1
Вывод: 200
```

-   A. לא ניתן לפתור בעיה זו.
-   B. ניתן להשתמש באלגוריתם Dijkstra ובאלגוריתם חיפוש לרוחב (BFS) עם הגבלה על מספר העצירות למציאת המסלול הזול ביותר.
-   C. ניתן להשתמש רק באלגוריתם חיפוש לעומק (DFS) למציאת המסלול הזול ביותר.
-   D. רק אלגוריתם חיפוש לרוחב (BFS) מתאים לפתרון בעיה זו.

**תשובה נכונה: B**

**הסבר:**

בעיית מציאת המסלול הקצר ביותר בין שני צמתים בגרף עם הגבלה על מספר הצמתים המתווכים ניתנת לפתרון באמצעות שילוב של אלגוריתם חיפוש לרוחב (BFS) ועקרונות תכנות דינמי.

*   **האלגוריתם (שילוב של BFS ותכנות דינמי):**
    1.  **אתחול:**
        *   יוצרים מילון למעקב אחר עלות הטיסה לכל עיר. תחילה, העלות שווה לאינסוף עבור כל הערים, למעט נקודת ההתחלה (`src`) שעבורה העלות היא 0.
        *   יוצרים תור (`queue`) עבור BFS, שבתחילה מכיל רק את נקודת ההתחלה `src` ומספר עצירות שווה ל-`0`.
    2.  **חיפוש לרוחב:**
        *   עוברים על כל המסלולים עד שהתור מתרוקן.
        *   בכל צעד, מוציאים את העיר הנוכחית `from_city` ואת מספר העצירות `stops`.
        *   **בדיקת עצירה:** אם מספר העצירות עולה על `k`, מדלגים על מסלול זה.
        *   **ערים שכנות:** מחפשים את כל הערים השכנות `to_city` שאליהן ניתן להגיע מ-`from_city`.
        *   **עדכון עלות:** מחשבים את עלות הטיסה ומשווים אותה לעלות הטיסה שכבר נשמרה עבור העיר `to_city`, ושומרים את העלות הקטנה יותר.
        *   **הוספה לתור:** מוסיפים את `to_city` לתור אם מספר העצירות אינו עולה על `k`.
    3.  **תוצאה:** לאחר מעבר על כל הגרף, אם עלות הטיסה ליעד (`dst`) שונה מאינסוף, מחזירים אותה, אחרת מחזירים `-1`.

*   **מדוע Dijkstra אינו מתאים:**
    *   אלגוריתם Dijkstra מיועד למציאת המסלול הקצר ביותר בעלות מינימלית, ואינו מיועד לעבודה עם הגבלה על מספר העצירות.

*   **מדוע BFS עדיף על DFS:**
    *   DFS עלול להיכנס לנתיב שגוי ולהעמיק בו מאוד לפני שהוא חוזר, בעוד ש-BFS יעבור על כל האפשרויות עם מספר עצירות מינימלי קודם.

**דוגמאות (פסאודו-קוד):**

```
function find_cheapest_price(n, flights, src, dst, k):
    distances = {city: infinity for city in range(n)}
    distances[src] = 0
    queue = [(src, 0)]  # (city, stops)
    while queue is not empty:
        from_city, stops = queue.pop(0)
        if stops > k:
            continue
        for from_i, to_i, price in flights:
            if from_i == from_city:
              if distances[to_i] > distances[from_city] + price:
                distances[to_i] = distances[from_city] + price
                queue.append((to_i, stops +1))

    if distances[dst] is not infinity:
        return distances[dst]
    else:
        return -1
```

**ניתוח האפשרויות:**

*   **B. ניתן להשתמש באלגוריתם Dijkstra ובאלגוריתם חיפוש לרוחב (BFS) עם הגבלה על מספר העצירות.:** נכון, אך אלגוריתם Dijkstra בפני עצמו אינו מתאים לבעיה זו. האפשרות מתייחסת כנראה לשילוב של גישות.
*   **C. ניתן להשתמש רק באלגוריתם חיפוש לעומק (DFS).:** לא נכון. DFS אינו הגישה האופטימלית ביותר.
*   **D. רק אלגוריתם חיפוש לרוחב (BFS) מתאים לפתרון בעיה זו.:** לא נכון. ניתן להשתמש ב-BFS במקרה זה, אך נדרשת גם לוגיקה עם הגבלה על מספר העצירות ותכנות דינמי לבחירת המסלול האופטימלי.
