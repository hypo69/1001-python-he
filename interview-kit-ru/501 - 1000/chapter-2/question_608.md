**שאלה 608.** בהקשר של תכנות מונחה עצמים בפייתון, מה מייצגת מילת המפתח `self` ומה תפקידה בעבודה עם מחלקות והמתודות שלהן?

-   A. `self` היא מילת מפתח המשמשת לגישה למשתנים גלובליים מתוך מחלקה.
-   B. `self` היא מילת מפתח המשמשת ליצירת מחלקות חדשות.
-   C. `self` היא מילת מפתח המשמשת כפרמטר ראשון של מתודות מופע, ומייצגת הפניה למופע המחלקה שעבורו נקראת המתודה.
-   D. `self` היא מילת מפתח המשמשת לקריאת מתודות של מחלקת אב.

**תשובה נכונה: C**

**הסבר:**

בפייתון, מילת המפתח `self` ממלאת תפקיד מרכזי בתכנות מונחה עצמים, ומאפשרת גישה למאפיינים ולמתודות של מופע ספציפי של המחלקה.

*   **פונקציות עיקריות של `self`:**
    *   **הפניה למופע:** `self` היא הפניה לאובייקט (מופע) המחלקה שעבורו נקראת המתודה.
    *   **פרמטר ראשון של מתודה:** `self` הוא תמיד הפרמטר הראשון של כל מתודת *מופע* של המחלקה (הבנאי `__init__` גם הוא מתודת מופע, אך לא מתודות סטטיות).
    *   **גישה למאפייני מופע:** `self` משמשת לגישה ושינוי של מאפייני מופע שנוצרו באמצעות `__init__`.
    *   **קריאת מתודות מופע:** באמצעות `self` ניתן לקרוא למתודות אחרות השייכות לאותו מופע מחלקה.
    *   **הבחנה:** מסייעת להבדיל בין מאפייני מופע לבין משתנים מקומיים של מתודות.

*   **שימוש ב-`self` בבנאי:**
    *   במתודה `__init__` (הבנאי), `self` מפנה ל*מופע המחלקה שזה עתה נוצר*.
    *   `self` משמשת לאתחול מאפייני המופע, למשל `self.x = 10`.

*   **שימוש ב-`self` במתודות אחרות:**
    *   במתודות אחרות, `self` מפנה ל*מופע קיים* של המחלקה.
    *   `self` משמשת לגישה למאפיינים ולקריאת מתודות אחרות של המופע: למשל `self.print()`.

**דוגמאות:**

```python
class Circle:
    def __init__(self, radius):
        self.radius = radius  # שימוש ב-self למאפיין מופע

    def get_area(self):
        return 3.14 * self.radius ** 2  # שימוש ב-self לגישה למאפיין
    def print_info(self):
        print(f"Circle radius: {self.radius}")
        self.get_area() # קריאת מתודה אחרת באמצעות self.

c = Circle(5)  # יצירת מופע של המחלקה
print(f"Area: {c.get_area()}")  # גישה למתודה דרך המופע
c.print_info() # גישה למתודה דרך המופע, הקוראת למתודה אחרת באמצעות self.

# דוגמה ללא self
class MyClass:
    def __init__(x): # שגיאה, self חייב להיות הפרמטר הראשון
       pass
    #def get_x(self) # מתודה ללא self
    #    return self.x
try:
     MyClass(10)
except TypeError as e:
    print(e) # __init__() missing 1 required positional argument: 'x'


class MyClass2:
   def __init__(self, x):
        self.x = x
   def get_x():  # ללא self, זו אינה מתודת מופע
      return 10

obj2 = MyClass2(5)

try:
  obj2.get_x()  # שגיאה, obj2.get_x() נקראת כמתודת מופע, אך אינה כזו
except TypeError as e:
   print(e) # get_x() takes 0 positional arguments but 1 was given

print(MyClass2.get_x()) # קריאה ישירה למתודת המחלקה (זו שללא self)
```
**ניתוח האפשרויות:**
*   **A. `self` - זהו מילת מפתח, המשמשת לגישה למשתנים גלובליים מתוך מחלקה.:** שגוי.
*   **B. `self` - זהו מילת מפתח, המשמשת ליצירת מחלקות חדשות.:** שגוי.
*  **C. `self` - זהו מילת מפתח, המשמשת כהפניה למופע הנוכחי (אובייקט) של המחלקה ומשמשת לגישה למאפיינים ולמתודות של מופע זה.:** נכון.
*  **D. `self` - זהו מילת מפתח, המשמשת לקריאת מתודות של מחלקת אב.:** שגוי. לקריאה למחלקת אב משתמשים ב-`super()`.

**לסיכום:**
*   `self` מספקת הפניה למופע המחלקה מתוך מתודות.
*  `self` מאפשרת גישה למאפיינים וקריאה למתודות אחרות של אותו מופע.
* `self` הוא ארגומנט הכרחי למתודות מופע.

לפיכך, התשובה הנכונה היא **C. `self` - זהו מילת מפתח, המשמשת כהפניה למופע הנוכחי (אובייקט) של המחלקה ומשמשת לגישה למאפיינים ולמתודות של מופע זה.**
