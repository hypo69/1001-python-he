### `question_625.md`

**שאלה 625.** מה ההבדל בין המתודות `append()` ו-`extend()` בעבודה עם רשימות בפייתון?

-   A. המתודה `append()` מוסיפה את כל האיברים של אובייקט איטרבילי לסוף הרשימה, בעוד שהמתודה `extend()` מוסיפה רק איבר אחד.
-   B. המתודה `extend()` מוסיפה את כל האיברים של אובייקט איטרבילי לסוף הרשימה, בעוד שהמתודה `append()` מוסיפה איבר יחיד לסוף הרשימה (גם אם הוא רשימה).
-   C. המתודות `append()` ו-`extend()` עושות את אותו הדבר, אך `extend()` עובדת מהר יותר.
-   D. המתודות `append()` ו-`extend()` אינן משנות את הרשימה אלא יוצרות רשימה חדשה.

**תשובה נכונה: B**

**הסבר:**

בפייתון, המתודות `append()` ו-`extend()` משמשות להוספת איברים לרשימה, אך הן עושות זאת באופן שונה. הבנת הבדל זה חשובה לבחירה נכונה של המתודה בעת שינוי רשימות.

*   **המתודה `append()`:**
    *   **מוסיפה איבר יחיד:** מוסיפה *איבר* יחיד לסוף הרשימה.
    *  **סוג נתונים שרירותי:** האיבר יכול להיות מכל סוג נתונים, כולל רשימה.
    *   **רשימה כאיבר:** אם מוסיפים רשימה באמצעות `append()`, היא תתווסף כאיבר *יחיד* לרשימה.
    * **משנה את הרשימה המקורית:** המתודה `append()` משנה את הרשימה עצמה שעליה היא מופעלת ומחזירה `None`.

*   **המתודה `extend()`:**
    *  **מוסיפה איברים של אובייקט איטרבילי:** מוסיפה לסוף הרשימה איברים מתוך *אובייקט איטרבילי* אחר (למשל, רשימה, טאפל, מחרוזת).
    *   **פורשת את הקולקציה:** פורשת את האובייקט האיטרבילי ומוסיפה את כל איבריו לסוף הרשימה.
    *   **משנה את הרשימה המקורית:** המתודה `extend()` משנה את הרשימה עצמה שעליה היא מופעלת ומחזירה `None`.
    *   `extend()` אינה עובדת עם ערכים בודדים, אלא מצפה רק לאובייקטים איטרביליים.

**דוגמאות:**

```python
# דוגמה 1: שימוש ב-append()
my_list = [1, 2, 3]
my_list.append(4)
print(f"append(4): {my_list}") # פלט: [1, 2, 3, 4]

my_list.append([5, 6]) # מוסיפים רשימה שלמה כאיבר יחיד
print(f"append([5,6]): {my_list}") # פלט: [1, 2, 3, 4, [5, 6]]

# דוגמה 2: שימוש ב-extend()
my_list2 = [1, 2, 3]
my_list2.extend([4, 5])
print(f"extend([4,5]): {my_list2}") # פלט: [1, 2, 3, 4, 5]

# דוגמה 3: TypeError בעת extend עם מספר
try:
   my_list3 = [1, 2, 3]
   my_list3.extend(4) # TypeError, int אינו איטרבילי
except TypeError as e:
    print(e) # 'int' object is not iterable

# דוגמה 4: extend() עם מחרוזות
my_list4 = [1,2,3]
my_list4.extend("abc") # יוסיף כל תו כאיבר נפרד
print(f"extend string: {my_list4}") # יוביל לפלט [1, 2, 3, 'a', 'b', 'c']
```

**ניתוח האפשרויות:**
*  **A. המתודה `append()` מוסיפה את כל האיברים של אובייקט איטרבילי לסוף הרשימה, בעוד שהמתודה `extend()` מוסיפה רק איבר אחד.:** שגוי.
*  **B. המתודה `extend()` מוסיפה את כל האיברים של אובייקט איטרבילי לסוף הרשימה, בעוד שהמתודה `append()` מוסיפה איבר יחיד לסוף הרשימה (גם אם הוא רשימה).:** נכון.
*  **C. המתודות `append()` ו-`extend()` עושות את אותו הדבר, אך `extend()` עובדת מהר יותר.:** שגוי.
*  **D. המתודות `append()` ו-`extend()` אינן משנות את הרשימה אלא יוצרות רשימה חדשה.:** שגוי.

**לסיכום:**
*  `append()` מוסיפה *איבר יחיד* לסוף הרשימה.
*   `extend()` מוסיפה *איברים מתוך אובייקט איטרבילי* לסוף הרשימה.
*  שתי המתודות משנות את הרשימה המקורית.
*  הבחירה הנכונה במתודה תלויה בשאלה האם אנו מוסיפים איבר יחיד או רצף של איברים.

לכן, התשובה הנכונה היא **B. המתודה `extend()` מוסיפה את כל האיברים של אובייקט איטרבילי לסוף הרשימה, בעוד שהמתודה `append()` מוסיפה איבר יחיד לסוף הרשימה (גם אם הוא רשימה).**
