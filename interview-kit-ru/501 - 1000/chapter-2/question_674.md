**שאלה 674.** מהו עקרון DRY ("Don't Repeat Yourself") בתכנות, וכיצד ניתן ליישם עקרון זה באמצעות תכנות מונחה עצמים (OOP) ותכנות פונקציונלי (FP) ב-Python?

-   א. DRY הוא עקרון הממליץ לכתוב כמה שיותר קוד כדי להבטיח את מהימנותו.
-   ב. DRY הוא עקרון פיתוח הקובע שלכל פיסת קוד צריכה להיות מקור אמת יחיד, והוא צריך להיות נגיש וניתן לשינוי בקלות. ב-OOP, עקרון זה מיושם באמצעות ירושה, פולימורפיזם והפשטה, ואילו ב-FP - באמצעות פונקציות מסדר גבוה, סגורים (closures) וביטויי למדא.
-   ג. DRY הוא עקרון תכנון מסדי נתונים כדי שכל הנתונים יישמרו במקום אחד.
-   ד. DRY פירושו ויתור על חלוקת הקוד, לפישוט העבודה איתו.

**תשובה נכונה: ב**

**הסבר:**

עקרון DRY (Don't Repeat Yourself) - "אל תחזור על עצמך" - הוא עקרון יסוד בפיתוח תוכנה, הקורא להימנע משכפול קוד. מהותו נעוצה בכך שלכל חלק בקוד צריכה להיות רק הצגה אחת ויחידה במערכת, כדי להבטיח תחזוקה והכנסת שינויים קלה.

*   **מאפיינים עיקריים של עקרון DRY:**
    *   **מקור אמת יחיד:** כל חלק בלוגיקה צריך להיות מוגדר במקום אחד ולא לחזור על עצמו במקומות שונים.
    *   **הימנעות משכפול:** הפחתת כמות הקוד המשוכפל.
    *   **פישוט התחזוקה:** פישוט הכנסת שינויים, מכיוון שנדרש לשנות רק במקום אחד.
    *   **שיפור הקריאות:** הופך את הקוד למובן יותר, מכיוון שהחזרות על עצמו אינן מסיחות את הדעת מהלוגיקה.
*   **יישום DRY ב-OOP:**
    *   **ירושה:** הוצאת מאפיינים ושיטות משותפים למחלקות בסיס, ותתי-מחלקות מוסיפות או דורסות אותם.
    *   **פולימורפיזם:** מאפשר ליצור היררכיות מחלקות ולטפל בהן בצורה אחידה.
    *   **הפשטה:** הסתרת פרטי מימוש והוצאתם למודולים או שיטות נפרדות.

*   **יישום DRY ב-FP:**
    *   **פונקציות מסדר גבוה:** מאפשרות ליצור פונקציות כלליות יותר, תוך שימוש חוזר בלוגיקה.
    *   **סגורים (Closures):** מאפשרים לפונקציות לשמור מצב וליצור פונקציות גמישות יותר וניתנות לשימוש חוזר.
    *   **ביטויי למדא:** מאפשרים ליצור פונקציות קטנות למשימות קצרות.
    *   **קומפוזיציה:** חיבור פונקציות פשוטות ליצירת מורכבות יותר, ובכך הימנעות משכפול קוד.

**דוגמאות:**
```python
# OOP: דוגמה ליישום DRY באמצעות ירושה
class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def introduce(self):
        print(f"My name is {self.name} and I am {self.age} years old.")

class Student(Person):
    def __init__(self, name, age, major):
        super().__init__(name, age)
        self.major = major

    def introduce(self):
        super().introduce()
        print(f"I am majoring in {self.major}.")

class Teacher(Person):
    def __init__(self, name, age, department):
        super().__init__(name, age)
        self.department = department

    def introduce(self):
        super().introduce()
        print(f"I teach in the {self.department} department.")

# דוגמה 1: תכנות פונקציונלי ופונקציות מסדר גבוה.
def add(x, y):
    return x + y
def multiply(x, y):
    return x * y

def apply_operation(func, x, y): # פונקציה מסדר גבוה - מקבלת פונקציה כארגומנט
    return func(x,y)

print(apply_operation(add, 2,3)) # 5
print(apply_operation(multiply, 2,3)) # 6
# דוגמה 2 : תכנות פונקציונלי - קומפוזיציה ולמדא
def add_one(x):
    return x + 1
def multiply_by_two(x):
    return x * 2

composed_func = lambda x: add_one(multiply_by_two(x)) # קומפוזיציה עם למדא
print(composed_func(5))
```

**ניתוח אפשרויות התשובה:**

*   **א. DRY הוא עקרון הממליץ לכתוב כמה שיותר קוד כדי להבטיח את מהימנותו.:** לא נכון.
*   **ב. DRY הוא עקרון פיתוח הקובע שלכל פיסת קוד צריכה להיות מקור אמת יחיד, והוא צריך להיות נגיש וניתן לשינוי בקלות. ב-OOP, עקרון זה מיושם באמצעות ירושה, פולימורפיזם והפשטה, ואילו ב-FP - באמצעות פונקציות מסדר גבוה, סגורים (closures) וביטויי למדא.:** נכון.
*   **ג. DRY הוא עקרון תכנון מסדי נתונים כדי שכל הנתונים יישמרו במקום אחד.:** לא נכון.
*   **ד. DRY פירושו ויתור על חלוקת הקוד, לפישוט העבודה איתו.:** לא נכון.

**לסיכום:**
*   עקרון DRY מאפשר לכתוב קוד גמיש יותר, בר-שימוש חוזר ונוח לתחזוקה.
*   ב-OOP, הדבר מושג באמצעות ירושה ופולימורפיזם, ואילו ב-FP - באמצעות שימוש בפונקציות מסדר גבוה וקומפוזיציה.
*   הימנעות משכפול קוד הופכת את הקוד למובן וצפוי יותר.

לכן, התשובה הנכונה היא **ב. DRY הוא עקרון פיתוח הקובע שלכל פיסת קוד צריכה להיות מקור אמת יחיד, והוא צריך להיות נגיש וניתן לשינוי בקלות. ב-OOP, עקרון זה מיושם באמצעות ירושה, פולימורפיזם והפשטה, ואילו ב-FP - באמצעות פונקציות מסדר גבוה, סגורים (closures) וביטויי למדא.**
