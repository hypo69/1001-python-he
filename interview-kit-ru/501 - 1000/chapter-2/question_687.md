### `question_678.md`

**שאלה 678.** פתחו אלגוריתם שמקבל כקלט שתי מחרוזות (`text1` ו-`text2`) ומחזיר את האורך של תת-הרצף המשותף הארוך ביותר שלהן (LCS, longest common subsequence).

*   **תת-רצף:** אוסף של תווים, שאינם בהכרח סמוכים, המופיעים משמאל לימין באותו סדר כמו במחרוזת המקורית.

**דוגמאות:**

```
קלט: text1 = "abcde", text2 = "ace"
פלט: 3
הסבר: "ace" הוא ה-LCS

קלט: text1 = "abc", text2 = "def"
פלט: 0
```

-   א. לפתרון הבעיה ניתן להשתמש באלגוריתם חיפוש לעומק (DFS).
-   ב. לפתרון בעיה זו ניתן להשתמש באלגוריתם חמדן, על ידי השוואת תווים לפי הסדר.
-   ג. לפתרון הבעיה ניתן להשתמש באלגוריתם תכנות דינמי, הבנה טבלה המשקפת את אורכי ה-LCS עבור כל תת-המחרוזות האפשריות.
-   ד. לפתרון הבעיה ניתן להשתמש בחיפוש בינארי.

**תשובה נכונה: ג**

**הסבר:**

בעיית מציאת תת-הרצף המשותף הארוך ביותר (LCS, Longest Common Subsequence) היא בעיה קלאסית הנפתרת באמצעות תכנות דינמי (dynamic programming). שיטה זו מאפשרת לחלק את הבעיה לתת-בעיות קטנות יותר, תוך שימוש חוזר בתוצאות חישובים קודמים.

*   **אלגוריתם (תכנות דינמי):**
    1.  **יצירת טבלה:** נוצרת טבלה `dp` בגודל `(len(text1) + 1) x (len(text2) + 1)`, כאשר `dp[i][j]` תאחסן את האורך של תת-הרצף המשותף הארוך ביותר עבור תת-המחרוזות `text1[:i]` ו-`text2[:j]`.
    2.   **אתחול:** השורה הראשונה והעמודה הראשונה בטבלה מתמלאות באפסים, שכן תת-הרצף המשותף של מחרוזת ריקה עם כל מחרוזת אחרת הוא תמיד בעל אורך `0`.
    3.  **מילוי הטבלה:**
           *   מבצעים איטרציה על שורות ועמודות הטבלה, החל מאינדקס `1`.
           *   אם `text1[i - 1] == text2[j - 1]`, כלומר התווים הנוכחיים זהים, אזי `dp[i][j] = dp[i - 1][j - 1] + 1` (אורך ה-LCS שווה לאורך ה-LCS של השורות הקודמות + 1).
          *   אם התווים אינם זהים, אזי `dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])` (הערך המקסימלי בין ה-LCS של `text1[:i-1]` ו-`text2[:j]` לבין ה-LCS של `text1[:i]` ו-`text2[:j-1]`).
    4.  **תוצאה:** אורך ה-LCS עבור המחרוזות `text1` ו-`text2` מאוחסן בתא `dp[len(text1)][len(text2)]`.
    *    לאחר בניית הטבלה כולה, מחזירים את הערך מהתא האחרון בטבלה `dp[len(text1)][len(text2)]`.

* **יתרונות אלגוריתם התכנות הדינמי:**
     *  **פתרון מיטבי:** מבטיח מציאת הפתרון המיטבי ביותר.
    *   **יעילות:** בעל סיבוכיות זמן של O(m*n), כאשר m ו-n הם אורכי המחרוזות `text1` ו-`text2`.
     *   **הימנעות מחישובים חוזרים:** משתמש בערכים שחושבו קודם לכן כדי למצוא תת-רצפים משותפים ארוכים.

*   **מדוע גישות אחרות אינן מתאימות:**
    *   **DFS:** DFS עלול להיכנס לנתיבים לא מיטביים ובעל סיבוכיות גבוהה.
    *   **אלגוריתם חמדן:** אלגוריתמים חמדנים אינם מבטיחים מציאת פתרון מיטבי עבור LCS.
    *    **חיפוש בינארי:** אינו מתאים למציאת תת-רצף משותף ארוך ביותר.

**דוגמה (פסאודו-קוד):**

```
function longest_common_subsequence(text1, text2):
    m = length(text1)
    n = length(text2)
    dp = таблица(m+1, n+1)
    for i from 0 to m:
      dp[i][0] = 0
    for j from 0 to n:
      dp[0][j] = 0
    for i from 1 to m:
       for j from 1 to n:
          if text1[i-1] == text2[j-1]:
             dp[i][j] = dp[i-1][j-1] + 1
          else:
            dp[i][j] = max(dp[i-1][j], dp[i][j-1])
    return dp[m][n]
```
**דוגמאות מימוש ב-Python:**
```python
def longest_common_subsequence(text1, text2):
    m = len(text1)
    n = len(text2)
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if text1[i - 1] == text2[j - 1]:
                dp[i][j] = dp[i - 1][j - 1] + 1
            else:
                dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])
    return dp[m][n]

print(f"אורך ה-LCS עבור 'abcde' ו- 'ace': {longest_common_subsequence('abcde', 'ace')}")  # פלט: אורך ה-LCS עבור 'abcde' ו- 'ace': 3
print(f"אורך ה-LCS עבור 'abc' ו- 'def': {longest_common_subsequence('abc', 'def')}")  # פלט: אורך ה-LCS עבור 'abc' ו- 'def': 0
print(f"אורך ה-LCS עבור 'AGGTAB' ו- 'GXTXAYB': {longest_common_subsequence('AGGTAB', 'GXTXAYB')}")  # פלט: אורך ה-LCS עבור 'AGGTAB' ו- 'GXTXAYB': 4
```
