### `question_682.md`

**שאלה 682.** נתונים שני מערכים, שבהם כל איבר מייצג קטע (אינטרוול) בצורת זוג מספרים [התחלה, סוף]. פתחו אלגוריתם שיחזיר מערך חדש, המכיל את קטעי החיתוך של שני המערכים המקוריים.

**דוגמאות:**
```
Ввод: firstList = [[0,2],[5,10],[13,23],[24,25]], secondList = [[1,5],[8,12],[15,24],[25,26]]
Вывод: [[1,2],[5,5],[8,10],[15,23],[24,24],[25,25]]

Ввод: firstList = [[1,3],[5,9]], secondList = []
Вывод: []
```

- א. כדי לפתור את הבעיה יש להשתמש באלגוריתם מיון, ולאחר מכן לעבור על כל האיברים.
- ב. כדי לפתור את הבעיה משתמשים בשיטת מעבר על כל הקומבינציות של איברי שתי הרשימות ולאחר מכן סינונם.
- ג. כדי לפתור את הבעיה יש להשתמש בשני מצביעים, תוך כדי איטרציה (מעבר) על שתי הרשימות וחישוב חיתוך האינטרוולים הנוכחיים, כאשר יש לקדם תמיד את המצביע המצביע על סוף הקטע המוקדם יותר.
- ד. כדי לפתור את הבעיה יש להשתמש רק באלגוריתם רקורסיבי.

**תשובה נכונה: C**

**הסבר:**

לפתרון בעיית מציאת חיתוכי אינטרוולים בשני מערכים, השיטה האופטימלית היא שימוש בשיטת שני המצביעים. שיטה זו מאפשרת לעבור ביעילות על הרשימות ולמצוא חיתוכים בזמן לינארי (O(n+m)).

*   **אלגוריתם (שיטת שני המצביעים):**
    1.  **אתחול:**
          *   יוצרים שני מצביעים `i` ו-`j`, שמצביעים בתחילה על תחילת כל אחת מהרשימות (`firstList` ו-`secondList`).
        *   יוצרים רשימה ריקה `result`, שאליה ייכתבו התוצאות.
    2.  **איטרציה (מעבר):** כל עוד שני המצביעים לא חרגו מגבולות הרשימות שלהם:
       *  **חיתוך:** מחשבים את חיתוך האינטרוולים הנוכחיים:
           *  `start = max(firstList[i][0], secondList[j][0])`
           *  `end = min(firstList[i][1], secondList[j][1])`
          *  אם  `start <= end`, פירוש הדבר שהקטעים נחתכים. יוצרים קטע חדש `[start, end]` ומוסיפים אותו לרשימה `result`.
        * **קידום המצביעים:**
            *   משווים את קצוות סופי האינטרוולים הנוכחיים `firstList[i][1]` ו-`secondList[j][1]`.
             *  מקַדמים את מצביע הרשימה שסופה קטן יותר (`i++` או `j++`).
    3.  **החזרת תוצאה:**  לאחר המעבר על כל האיברים, מחזירים את הרשימה `result`.

*  **יתרונות האלגוריתם:**
    * **סיבוכיות לינארית:** סיבוכיות הזמן היא O(n+m), כאשר n ו-m הן אורכי הרשימות, וזאת יעיל יותר מ-O(n*m) של שיטת המעבר על כל הקומבינציות.
    *  **אופטימליות:** האלגוריתם מובטח למצוא את כל החיתוכים.
     * **יעילות:** האלגוריתם אינו דורש מיון, והוא פשוט וקל להבנה.

**דוגמאות (פסאודו-קוד):**
```
function interval_intersection(list1, list2):
  i = 0
  j = 0
  result = []
  while i < length(list1) and j < length(list2):
    start = max(list1[i][0], list2[j][0])
    end = min(list1[i][1], list2[j][1])
    if start <= end:
        result.append([start, end])
    if list1[i][1] < list2[j][1]:
      i+=1
    else:
       j+=1
  return result
```
