**שאלה 601.** מה ההבדל בין מילונים (dictionaries) בפייתון לבין JSON (JavaScript Object Notation)?

-   א. מילונים הם אובייקטים בלתי משתנים, בעוד ש-JSON הם אובייקטים משתנים.
-   ב. מילונים הם סוג נתונים בפייתון, המייצג אוסף של זוגות מפתח-ערך עם אינדקס, בעוד ש-JSON הוא פורמט טקסטואלי להחלפת נתונים, העוקב אחר כללי תחביר מוגדרים.
-   ג. מילונים מיועדים להעברת נתונים ברשת, בעוד ש-JSON משמש לאחסון נתונים בקבצים.
-   ד. מילונים הם סוג נתונים המשמש לאחסון נתונים במסד נתונים, בעוד ש-JSON משמש להצגתם בדפדפן.

**תשובה נכונה: ב**

**הסבר:**

מילונים (dictionaries) ו-JSON (JavaScript Object Notation) הם שתי דרכים שונות לייצוג נתונים, המשמשות לעתים קרובות יחד, אך בעלות מאפיינים וייעודים שונים.

*   **מילונים (dictionaries) בפייתון:**
    *   **סוג נתונים של פייתון:** מייצגים סוג נתונים מובנה לאחסון זוגות "מפתח-ערך".
    *   **אוסף בלתי מסודר:** סדר האלמנטים במילון אינו מובטח (אך מובטח החל מפייתון 3.7).
    *   **מפתחות:** המפתחות חייבים להיות ייחודיים ואובייקטים בלתי משתנים (לדוגמה, מחרוזות, מספרים, טאפלים).
    *   **ערכים:** הערכים יכולים להיות מכל סוג נתונים.
    *   **יכולת שינוי:** מילונים הם אובייקטים משתנים (mutable).
    *   **תחביר:** משתמשים בסוגריים מסולסלים: `{"key1": "value1", "key2": 2}`.

*   **JSON (JavaScript Object Notation):**
    *   **פורמט נתונים טקסטואלי:** מייצג פורמט טקסטואלי, המבוסס על תחביר של אובייקטים ב-JavaScript.
    *   **להחלפת נתונים:** מיועד לאחסון והעברת נתונים, למשל, בעת אינטראקציה בין שרת ווב לדפדפן, או לכתיבה לקבצים.
    *   **פורמט נתונים:** נתוני JSON הם פשוט מחרוזת, התואמת לפורמט מוגדר (עם שימוש בסוגריים מסולסלים, מירכאות, וכללים אחרים).
    *   **סוגי נתונים:** JSON תומך במספר סוגי נתונים פשוטים:
        *   מחרוזות (strings), לדוגמה `"hello"`.
        *   מספרים (numbers), לדוגמה `10`, `3.14`.
        *   ערכים בוליאניים (boolean), לדוגמה `true`, `false`.
        *   `null`.
        *   מערכים (array), לדוגמה `[1, 2, 3]`.
        *   אובייקטים (object), התואמים למילונים, לדוגמה `{"key": "value"}`.

**ההבדל בין מילונים ל-JSON:**
*   מילון - זוהי מבנה נתונים בפייתון, JSON - זהו פורמט טקסטואלי.
*   מילון הוא משתנה, בעוד ש-JSON הוא מחרוזת, ולכן, בלתי משתנה.
*   JSON הוא בלתי תלוי בשפה, בעוד שמילונים משמשים בפייתון.
*   ניתן להמיר מילונים ל-JSON ולהפך, באמצעות המודולים `json` או `pickle`.

**דוגמאות:**
```python
# דוגמה למילון
my_dict = {"name": "John Doe", "age": 30, "city": "New York"}
print(f"Словарь: {my_dict}, тип:{type(my_dict)}")  # יוצג: מילון: {'name': 'John Doe', 'age': 30, 'city': 'New York'} , סוג:<class 'dict'>

# דוגמה ל-JSON (מחרוזת)
json_string = '{"name": "Jane Doe", "age": 25, "city": "London"}'
print(f"JSON строка: {json_string}, тип:{type(json_string)}") # יוצג מחרוזת JSON: {"name": "Jane Doe", "age": 25, "city": "London"}, סוג:<class 'str'>

# דוגמה להמרה ממילון ל-JSON
import json

dict_to_convert = {"a" : 1, "b" : 2, "c" : [1,2,3]}
json_string = json.dumps(dict_to_convert) # ממיר את המילון למחרוזת json
print(f"Словарь в формате JSON: {json_string}, тип:{type(json_string)}") # יוצג: מילון בפורמט JSON: {"a": 1, "b": 2, "c": [1, 2, 3]}, סוג:<class 'str'>

# דוגמה להמרה מ-JSON למילון
import json
json_string = '{"a": 1, "b": 2, "c": [1, 2, 3]}'
dict_from_json = json.loads(json_string)
print(f"JSON строка в словаре: {dict_from_json}, тип:{type(dict_from_json)}") # יוצג: מחרוזת JSON במילון: {'a': 1, 'b': 2, 'c': [1, 2, 3]}, סוג:<class 'dict'>
```
**ניתוח האפשרויות:**
*   **א. מילונים הם אובייקטים בלתי משתנים, בעוד ש-JSON הם אובייקטים משתנים.:** שגוי.
*   **ב. מילונים הם סוג נתונים בפייתון, המייצג אוסף של זוגות מפתח-ערך עם אינדקס, בעוד ש-JSON הוא פורמט טקסטואלי להחלפת נתונים, העוקב אחר כללי תחביר מוגדרים.:** נכון.
*   **ג. מילונים מיועדים להעברת נתונים ברשת, בעוד ש-JSON משמש לאחסון נתונים בקבצים.:** שגוי.
*   **ד. מילונים הם סוג נתונים המשמש לאחסון נתונים במסד נתונים, בעוד ש-JSON משמש להצגתם בדפדפן.:** שגוי.

**לסיכום:**
*   מילונים ו-JSON משרתים מטרות שונות.
*   מילונים – זוהי מבנה נתונים בפייתון, בעוד ש-JSON - זהו פורמט טקסטואלי להחלפת נתונים.

לפיכך, התשובה הנכונה היא **ב. מילונים הם סוג נתונים בפייתון, המייצג אוסף של זוגות מפתח-ערך עם אינדקס, בעוד ש-JSON הוא פורמט טקסטואלי להחלפת נתונים, העוקב אחר כללי תחביר מוגדרים.**
