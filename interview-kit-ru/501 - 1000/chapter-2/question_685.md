**שאלה 685.** נתונים מערך של מספרים שלמים `nums` ומספר שלם `k`, המייצג את גודל תת-מערך. יש לפתח אלגוריתם שמחשב את הערך המקסימלי עבור כל חלון גולש בגודל `k` במערך `nums`.

**דוגמאות:**
```
קלט: nums = [1,3,-1,-3,5,3,6,7], k = 3
פלט: [3,3,5,5,6,7]

קלט: nums = [1], k = 1
פלט: [1]
```

*   א. כדי לפתור את הבעיה, יש צורך למיין את כל תת-המערכים ולקחת את הערך המקסימלי.
*   ב. כדי לפתור את הבעיה, יש צורך להשתמש באלגוריתם שעובר על כל איברי המערך ומשתמש בשתי לולאות מקוננות כדי לעבור על תת-המערכים בגודל k.
*   ג. כדי לפתור את הבעיה, יש צורך להשתמש בתור לאחסון k איברים בלבד, ויש לחשב את הערך המקסימלי בכל פעם שנוסף איבר חדש ומוסר איבר שיצא מגבולות k.
*   ד. כדי לפתור את הבעיה, ניתן להשתמש בדק (תור דו-כיווני), שבו נאחסן רק אינדקסים של איברים המהווים מועמדים למקסימום, ובכך לאפשר מעקב אחר אינדקסים של איברים, ובו בזמן להיות מסוגלים להוסיף ולהסיר איברים ביעילות מתחילת או מסוף הדק.

**תשובה נכונה: ד**

**הסבר:**

כדי לפתור את הבעיה של מציאת המקסימום בכל חלון גולש בגודל `k` במערך ביעילות, יש להשתמש במבנה הנתונים "דק" (תור דו-כיווני). שימוש בדק מאפשר מעקב אחר אינדקסים של מועמדים למקסימום בחלון.

*   **אלגוריתם המשתמש בדק (תור דו-כיווני):**
    1.  **אתחול:**
        *   יוצרים דק ריק.
        *   התוצאה היא רשימה.
    2.  **איטרציה על המערך:**
        *   עוברים על המערך משמאל לימין.
        *   **הסרת איברים ישנים:** אם האיבר הראשון בדק נמצא מחוץ לחלון הנוכחי, מסירים אותו מתחילת הדק.
        *   **הסרת איברים קטנים יותר:** מסירים מהסוף את כל האינדקסים של איברים הקטנים מהאיבר הנוכחי.
        *   **הוספת האינדקס הנוכחי:** מוסיפים את האינדקס הנוכחי של האיבר לסוף הדק.
        *   **רישום המקסימום:** כאשר אורך החלון הגיע ל-k, רושמים את הערך המקסימלי מתחילת הדק למערך התוצאה, מכיוון שזהו האינדקס של האיבר הגדול ביותר בחלון הנוכחי.
    3.  **החזרת התוצאה:** מחזירים את התוצאה - רשימת המקסימומים.

*   **יתרונות אלגוריתם זה:**
    *   **סיבוכיות:** מבטיח סיבוכיות זמן לינארית O(n), שכן כל איבר במערך מתווסף לדק ומוסר ממנו לא יותר מפעם אחת.
    *   **יעילות:** מאפשר מעקב אחר מועמדים למקסימום בחלון, מבלי לעבור עליהם מחדש, ובו בזמן מבטיח סיבוכיות קבועה עבור קבלת האיבר המקסימלי בחלון.

*   **מדוע אפשרויות אחרות אינן מתאימות:**
    *   **אפשרות א:** מיון כל תת-המערכים הוא פתרון לא יעיל ויארך O(n^2 log n).
    *   **אפשרות ב:** שימוש בשתי לולאות מקוננות למעבר על תת-מערכים יוביל לסיבוכיות O(n*k), ולא O(n log n).
    *   **אפשרות ג:** שימוש בתור לא יאפשר קבלת הערך המקסימלי בזמן O(1).

**דוגמאות (פסאודו-קוד):**
```
function max_sliding_window(nums, k)
  deque = []
  result = []
  for i = 0 to nums.length - 1:
    # Remove indices outside the window
      while deque is not empty and deque[0] < i - k + 1
        deque.pop_front()

     # remove elements less than current
     while deque is not empty and nums[deque[-1]] <= nums[i]:
         deque.pop_back()
      deque.append(i) # add new index
      if i >= k - 1:
         result.append(nums[deque[0]]) # add to results if window is at least k
   return result
```
**דוגמאות מימוש בפייתון:**

```python
from collections import deque

def max_sliding_window(nums, k):
    d = deque()
    res = []
    for i, x in enumerate(nums):
        while d and d[0] < i - k + 1:
            d.popleft()
        while d and nums[d[-1]] <= x:
            d.pop()
        d.append(i)
        if i >= k - 1:
            res.append(nums[d[0]])
    return res

nums = [1, 3, -1, -3, 5, 3, 6, 7]
k = 3
print(f"קלט: nums = {nums}, k = {k}")
print(f"פלט: {max_sliding_window(nums,k)}")  # Output: [3, 3, 5, 5, 6, 7]

nums2 = [1]
k2 = 1
print(f"קלט: nums = {nums2}, k = {k2}")
print(f"פלט: {max_sliding_window(nums2, k2)}")  # Output: [1]
```
