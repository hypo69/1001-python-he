**שאלה 649.** מהי המרת טיפוסים (type casting) ב-Python, ובמה היא שונה מהמרת טיפוסים מרומזת? מהן אננוטציות טיפוסים ב-Python, וכיצד הן משמשות?

- A. המרת טיפוסים ב-Python מתבצעת באופן מרומז בלבד, Python מזהה אוטומטית את טיפוס האובייקט במהלך ריצת התוכנית. אננוטציות טיפוסים אינן בשימוש ב-Python.
- B. המרת טיפוסים ב-Python מתבצעת באופן מפורש בלבד, ואננוטציות משמשות רק לתיאור טיפוסי משתנים בהערות, אך אינן משפיעות על בדיקתן.
- C. המרת טיפוסים – היא המרה של אובייקט מטיפוס נתונים אחד לאחר. המרת טיפוסים מרומזת מתבצעת אוטומטית על ידי מפרש Python, ואילו המרה מפורשת (המרת טיפוס) מתבצעת על ידי המפתח באמצעות פונקציה מוגדרת מראש, ובמהלכה עלול להתרחש אובדן נתונים. אננוטציות טיפוסים – הן היכולת לציין את הטיפוס הצפוי של משתנה, פרמטר של פונקציה או ערך מוחזר, לשם שיפור קריאות ודיבאגינג של קוד.
- D. המרת טיפוסים ב-Python אינה נדרשת, מכיוון ש-Python היא שפה עם טיפוסיות דינמית, ואננוטציות משמשות ליצירת מחלקות חדשות.

**תשובה נכונה: C**

**הסבר:**

ב-Python, המרת טיפוסים (type casting) היא תהליך שינוי טיפוס הנתונים של אובייקט. Python תומכת הן בהמרה מרומזת (אוטומטית) של טיפוסים והן בהמרה מפורשת (המרת טיפוס) באמצעות פונקציות מובנות. בנוסף, Python תומכת באננוטציות טיפוסים.

*   **המרת טיפוסים מרומזת (Implicit type conversion):**
    *   **המרה אוטומטית:** מפרש Python יכול לבצע אוטומטית המרת טיפוסים כאשר הדבר הגיוני, לדוגמה, בפעולות חשבוניות עם טיפוסים מספריים שונים.
    *   **שימור נתונים:** בדרך כלל מתבצעת באופן שלא יאפשר אובדן נתונים.
    *   **דוגמה:**
        *   `10 + 5.5` (מספר שלם מומר למספר עשרוני).
*   **המרת טיפוסים מפורשת (Explicit type conversion/type casting):**
    *   **המרה כפויה:** מבוצעת על ידי המפתח, המשתמש בפונקציות מובנות להמרה לטיפוס הרצוי.
    *   **אובדן נתונים אפשרי:** עלולה להוביל לאובדן נתונים, במיוחד בעת המרה לטיפוס "קטן" יותר.
    *   **דוגמאות:**
        *   `int("123")` (מחרוזת מומרת למספר שלם).
        *   `str(42)` (מספר שלם מומר למחרוזת).
        *   `float(5)` (מספר שלם מומר למספר עשרוני).
        *   `bool(0)` - (מספר שלם מומר לערך בוליאני).
*   **אננוטציות טיפוסים (Type hints):**
    *   **תיאור טיפוסים:** אננוטציות מאפשרות למתכנת לציין את הטיפוסים הצפויים של משתנים, פרמטרים של פונקציות או ערכים מוחזרים.
    *   **משפרות קריאות:** הופכות את הקוד למובן יותר ומסייעות למנוע שגיאות טיפוס.
    *   **אינן משפיעות על הביצוע:** אננוטציות אינן משפיעות על ביצוע הקוד, ומשמשות בעיקר לניתוח סטטי (לדוגמה, על ידי לינטרים).
    *   **תחביר:**
        *   `variable: type = value` - לאננוטציה של משתנים
        *   `def function(param1:type, param2:type) -> return_type:` - לאננוטציה של פונקציות

**דוגמאות:**

```python
# Example 1: Implicit type conversion
x = 10
y = 3.14
result = x + y # The integer is converted to a float to perform the addition.
print(f"Result of int + float addition: {result}, type: {type(result)}") # Output: Result of int + float addition: 13.14, type: <class 'float'>


# Example 2: Explicit type conversion
a = "123"
b = int(a)
print(f"Original string: {a}, type: {type(a)}")
print(f"Converted number: {b}, type: {type(b)}")

c = 123
d = str(c)
print(f"Original number: {c}, type: {type(c)}")
print(f"Converted string: {d}, type: {type(d)}")

# Example 3: Explicit conversion with a list
e = (1,2,3)
e_list = list(e)
print(f"Original tuple : {e}, type: {type(e)}")
print(f"Converted list : {e_list}, type: {type(e_list)}")


# Example 4: Data loss during explicit type casting
f = 3.999
int_from_float = int(f)
print(f"Original float number: {f}, type: {type(f)}")
print(f"Converted int number: {int_from_float}, type: {type(int_from_float)}") # Rounding occurs during conversion.


# Example 5: Type annotation
price: int = 5 # Annotate the variable with expected type int.
def indent_right(s: str, width: int) -> str: # Annotate the function returning str.
     return " " * width + s

print(f"Result : {indent_right('test', 5)}") # Output:      test
```
