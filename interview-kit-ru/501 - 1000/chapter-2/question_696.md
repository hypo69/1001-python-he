### `question_680.md`

**שאלה 680.** נתון מערך `capacity`, המייצג את הקיבולת של `n` שקים, וכן מערך `rocks`, כאשר `rocks[i]` הוא מספר האבנים שכבר נמצאות בשק ה-`i`. בנוסף, נתון מספר שלם `additionalRocks`, המייצג את מספר האבנים שניתן להוסיף לשקים כלשהם. פתחו אלגוריתם שמחשב את המספר המרבי של שקים שניתן למלא, על ידי חלוקת `additionalRocks`.

**דוגמאות:**

```
קלט: capacity = [2,3,4,5], rocks = [1,2,4,4], additionalRocks = 2
פלט: 3

קלט: capacity = [10,2,2], rocks = [2,2,0], additionalRocks = 100
פלט: 3
```

- A. לפתרון בעיה זו יש למיין תחילה את מערך capacity, ולאחר מכן למלא את השקים באופן סדרתי עד שיאזלו האבנים.
- B. לפתרון בעיה זו יש למיין תחילה את ההפרש בין capacity ל- rocks, ולאחר מכן למלא את השקים באופן סדרתי עד שיאזלו האבנים.
- C. לפתרון בעיה זו, יש לעבור על כל הקומבינציות על מנת לקבל את התוצאה האופטימלית ביותר, מה שיהפוך את הבעיה לבלתי יעילה.
- D. לפתרון בעיה זו יש להשתמש באלגוריתם חמדן, שממלא תחילה את השקים עם ההפרש הנמוך ביותר בין הקיבולת למספר האבנים שבהם.

**תשובה נכונה: B**

**הסבר:**

לפתרון בעיה זו ניתן להשתמש באלגוריתם חמדן (greedy algorithm), שמתבסס על הגישה הבאה: יוצרים רשימה חדשה עם ההפרש בין capacity ל- rocks, אותה ממיינים בסדר עולה, ולאחר מכן, האבנים מחולקות באופן סדרתי לשקים בהתאם לגודל ההפרש הדרוש להם, עד שאבני ה- `additionalRocks` הנוספות אוזלות.

*   **אלגוריתם (גישה חמדנית):**
    1.  **חישוב ההפרש:** יוצרים רשימה שבה כל איבר הוא ההפרש בין `capacity[i]` ל- `rocks[i]`, כלומר, כמה אבנים צריך להוסיף לכל שק.
    2.  **מיון:** ממיינים רשימה זו בסדר עולה.
    3.  **איטרציה והקצאה:** מבצעים איטרציה על הרשימה הממוינת ובכל איטרציה מוסיפים את כמות האבנים הדרושה לשק, אם כמות ה- `additionalRocks` מספקת.
        *   אם כמות ה- `additionalRocks` אינה מספקת למילוי כל השקים, עוצרים את הלולאה.
    4.  **ספירה:** סופרים כמה שקים הצלחנו למלא.
*   **מדוע זה עובד:**
    *   הגישה החמדנית מתבססת על כך שאנו ממלאים תחילה את השקים הדורשים את כמות האבנים הנמוכה ביותר.
    *   בכל שלב של בחירת הפתרון הטוב ביותר מקומית (למלא את השקים לפי המינימום הנדרש), אנו נקבל את הפתרון האופטימלי גלובלית.

*   **יתרונות האלגוריתם:**
    *   **אופטימליות:** הגישה החמדנית מבטיחה פתרון אופטימלי לבעיה הנתונה.
    *   **סיבוכיות זמן:** סיבוכיות הזמן היא O(n log n) בשל הצורך במיון הרשימה.
    *   **פשטות:** קל ליישום.

**דוגמאות (פסאודו-קוד):**
```
function maximum_filled_bags(capacity, rocks, additionalRocks):
  differences = []
  for i in range(length(capacity)):
    differences.append(capacity[i] - rocks[i])
  sort differences
  count = 0
  for i in differences:
    if additionalRocks >= differences[i]:
        additionalRocks -= differences[i]
        count += 1
  return count
```
**דוגמאות ליישום בפייתון:**

```python
def maximum_filled_bags(capacity, rocks, additionalRocks):
    differences = []
    for i in range(len(capacity)):
        differences.append(capacity[i] - rocks[i])
    differences.sort()
    count = 0
    for diff in differences:
        if additionalRocks >= diff:
            additionalRocks -= diff
            count += 1
        else:
            break
    return count

capacity1 = [2,3,4,5]
rocks1 = [1,2,4,4]
additionalRocks1 = 2
print(f"קלט: capacity = {capacity1}, rocks = {rocks1}, additionalRocks = {additionalRocks1}")
print(f"פלט: {maximum_filled_bags(capacity1, rocks1, additionalRocks1)}") # פלט: 3

capacity2 = [10,2,2]
rocks2 = [2,2,0]
additionalRocks2 = 100
print(f"קלט: capacity = {capacity2}, rocks = {rocks2}, additionalRocks = {additionalRocks2}")
print(f"פלט: {maximum_filled_bags(capacity2, rocks2, additionalRocks2)}") # פלט: 3
```
