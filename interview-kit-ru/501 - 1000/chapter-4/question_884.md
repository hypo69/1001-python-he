### `question_884.md`

**שאלה 884.** מה יקרה לנפח הזיכרון שהוקצה לרשימה (`list`) ב-Python אם יוסרו ממנה *מרבית* הפריטים (לדוגמה, הסרת 999,900 פריטים מרשימה שהכילה במקור 1,000,000 פריטים)?

א. Python תקטין באופן אוטומטי את נפח הזיכרון שהוקצה למינימום ההכרחי לאחסון הפריטים הנותרים.
ב. Python תקטין את נפח הזיכרון שהוקצה לגודל פרופורציונלי למספר הפריטים הנותרים.
ג. נפח הזיכרון שהוקצה יישאר כפי שהיה, מקביל לגודל הרשימה המקורי, עד שהרשימה תיבנה מחדש (לדוגמה, על ידי השמה `lst = lst[:]` או `lst = list(lst)`).
ד. Python תשחרר את כל הזיכרון שהוקצה, והרשימה תהפוך לבלתי נגישה.

**התשובה הנכונה: ג**

**הסבר:**

ב-Python, רשימות משנות את גודלן באופן דינמי, אך אינן מנהלות באופן אקטיבי את הזיכרון שהוקצה להן בכל עת.

*   **הקצאת זיכרון:** כאשר נוצרת רשימה, Python מקצה נפח זיכרון מסוים לאחסון הפריטים. נפח זה עשוי להיות גדול יותר מההכרחי לאחסון הפריטים הנוכחיים, וזאת כדי לאפשר הוספה יעילה של פריטים חדשים.
*   **הסרת פריטים:** כאשר פריטים מוסרים מרשימה, Python לא תמיד מקטינה מיד את נפח הזיכרון שהוקצה. פעולה זו נועדה לייעול ביצועים, כדי להימנע מפעולות תכופות של הקצאת זיכרון ושחרורו.

*   **אפשרות א' שגויה:** הזיכרון אינו מוקטן באופן אוטומטי.
*   **אפשרות ב' שגויה:** הקטנת הזיכרון מתרחשת רק בעת יצירה מפורשת של רשימה חדשה.
*   **אפשרות ג' נכונה:** תיאור מדויק של התנהגות Python.
*   **אפשרות ד' שגויה:** הרשימה נשארת נגישה, אך תופסת יותר מקום בזיכרון.

**כיצד לשחרר זיכרון שאינו בשימוש:**

כדי "לכווץ" רשימה ולשחרר זיכרון שאינו בשימוש, ניתן ליצור רשימה חדשה המכילה רק את הפריטים הנותרים. ניתן לעשות זאת במספר דרכים:

```python
my_list = list(range(1000000)) # Create a list with 1 million elements
del my_list[100:] # Remove most elements

# Method 1: Create a slice copy
my_list = my_list[:]  # Create a new list with only the remaining elements

# Method 2: Create a new list from iterable
my_list = list(my_list)
```
**לסיכום:**

נפח הזיכרון שהוקצה לרשימה אינו מוקטן באופן אוטומטי לאחר הסרת פריטים. הזיכרון ישוחרר רק לאחר יצירת הרשימה מחדש, למשל באמצעות פרוסה (slice) או יצירת רשימה חדשה.

לפיכך, אפשרות ג' היא הנכונה.
