**שאלה 582.** כיצד פייתון מנהל זיכרון ומה התפקיד שממלא אוסף האשפה?

-   A. בפייתון ניהול הזיכרון מתבצע באופן ידני לחלוטין על ידי המפתח.
-   B. בפייתון ניהול הזיכרון מתבצע אוטומטית, תוך שימוש באוסף אשפה מובנה והקצאת זיכרון בערימה.
-   C. בפייתון הזיכרון מוקצה במחסנית, ונדרש שימוש ב-API ברמה נמוכה כדי לנהל את הזיכרון.
-   D. בפייתון אוסף האשפה משמש רק לניהול זיכרון שהוקצה עבור מחרוזות.

**תשובה נכונה: B**

**הסבר:**

בפייתון, ניהול הזיכרון ממוכן, ומפתחים אינם צריכים להקצות או לשחרר זיכרון באופן ידני. פייתון משתמש בהקצאת זיכרון דינמית, ולשחרור זיכרון שאינו בשימוש הוא משתמש במנגנון איסוף אשפה.

*   **ניהול זיכרון אוטומטי:**
    *   פייתון מנהל באופן אוטומטי את הקצאת ושחרור הזיכרון עבור אובייקטים.
    *   מפתחים אינם צריכים להקצות זיכרון עבור אובייקטים באופן ידני; פעולה זו מבוצעת על ידי מפרש פייתון.
*   **ערימת זיכרון (Heap):**
    *   כל הזיכרון הדרוש ליצירת אובייקטים ומבני נתונים מוקצה בערימה.
    *   הערימה היא אזור זיכרון המוקצה באופן דינמי, בו מאוחסנים כל אובייקטי פייתון.
    *   גישה ישירה לערימה אינה מסופקת למתכנתי פייתון; היא מנוהלת על ידי המפרש.
*   **אוסף אשפה (Garbage Collector):**
    *   לפייתון יש אוסף אשפה מובנה המשחרר אוטומטית זיכרון התפוס על ידי אובייקטים שאינם בשימוש עוד בתוכנית.
    *   אוסף האשפה עוקב אחר הפניות לאובייקטים ומשחרר זיכרון שתפוס על ידי אובייקטים שלגביהם לא נותרו הפניות.
    *   פייתון משתמש גם במנגנון לטיפול בהפניות מחזוריות (ראו שאלה `question_557.md`).
*   **API בסיסי:**
    *   לפייתון יש API בסיסי לעבודה עם זיכרון ברמה נמוכה יותר, אך ברוב המקרים, מפתחי פייתון אינם נדרשים להשתמש במנגנונים אלה.

**דוגמאות:**

```python
#דוגמה להקצאת זיכרון אוטומטית
my_list = [1,2,3] # Python מקצה זיכרון אוטומטית עבור הרשימה
def create_objects():
  a = 10 # Python מקצה זיכרון אוטומטית עבור המספר השלם
  b = "test" # Python מקצה זיכרון אוטומטית עבור המחרוזת
  c = [a,b] # Python מקצה זיכרון אוטומטית עבור הרשימה
create_objects() # המשתנים יימחקו עם סיום פעולת הפונקציה, מאחר שאין להם הפניות.

# דוגמה לפעולת GC

import gc
import time

def create_objects_with_ref():
  a = [1,2,3]
  return a

list1 = create_objects_with_ref()
print(gc.get_count()) # (0,0,0)

list2 = list1 # יצירת הפניה ל-list1
print(gc.get_count()) # (0, 0, 0)

list1 = None # מחיקת הפניה אחת לרשימה
print(gc.get_count()) # (0, 0, 1)
time.sleep(2) # המתנה להפעלת אוסף האשפה
print(gc.get_count()) # (0, 0, 0)
```
**ניתוח הדוגמאות:**

1.  **דוגמה להקצאת זיכרון אוטומטית:**
    *   פייתון מקצה זיכרון אוטומטית עבור רשימה, מספר שלם, מחרוזת.
    *   פייתון גם משחרר זיכרון אוטומטית לאחר שהמשתנים בפונקציות אינם נדרשים יותר.
2.  **דוגמה לפעולת GC:**
    *   `gc.get_count()` מחזיר את מספר הדורות השונים של אובייקטים שאוסף האשפה עוקב אחריהם.
    *   לאחר מחיקת הפניה לרשימה, אוסף האשפה יגלה זאת (אך לא מייד) וישחרר את הזיכרון כאשר ימצא לנכון.

**ניתוח האפשרויות:**

*   **D. בפייתון אוסף האשפה משמש רק לניהול זיכרון שהוקצה עבור מחרוזות.:** שגוי.

**לסיכום:**
*   פייתון ממכן את הקצאת ושחרור הזיכרון, מפשט את תהליך הפיתוח.
*   אוסף האשפה המובנה משחרר זיכרון אוטומטית מאובייקטים שאינם בשימוש.
*   פייתון משתמש בזיכרון בערימה לאחסון אובייקטים.
*   מפתחי פייתון אינם עובדים ברמה נמוכה עם זיכרון ברוב המקרים.
