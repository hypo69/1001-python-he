**שאלה 887.** כיצד פועל ניהול הזיכרון האוטומטי (איסוף זבל) ב-Python?

A. ב-Python ניהול הזיכרון מבוסס אך ורק על ספירת התייחסויות, שבה הזיכרון משוחרר באופן מיידי כאשר מספר ההתייחסויות לאובייקט הופך לאפס, ללא התחשבות בתלויות מחזוריות אפשריות.

B. ב-Python נעשה שימוש בשילוב של מנגנון ספירת התייחסויות, שמשחרר זיכרון כאשר מונה ההתייחסויות מתאפס, ואספן זבל מחזורי, שסורק מעת לעת את האובייקטים בזיכרון כדי לגלות ולהסיר תלויות מחזוריות, שאינן מאפשרות שחרור זיכרון אוטומטי.

C. ב-Python ניהול הזיכרון האוטומטי מבוסס על ניתוח הישגוּת של אובייקטים, החל מאובייקטי שורש, וכל האובייקטים שאינם ניתנים להשגה מוסרים על ידי אספן הזבל, ללא שימוש בספירת התייחסויות.

D. Python מספק ניהול זיכרון ידני באמצעות פונקציות `malloc` ו-`free`, כמו בשפת C, ואיסוף זבל אוטומטי משמש רק לצורך איתור שגיאות (debugging).

**תשובה נכונה: B**

**הסבר:**

Python משתמשת בניהול זיכרון אוטומטי, מה שמפשט משמעותית את פיתוח היישומים. תהליך זה כולל שני מנגנונים עיקריים:

1.  **ספירת התייחסויות (Reference Counting):**
    *   לכל אובייקט ב-Python יש מונה התייחסויות, שגדל בעת יצירת התייחסות חדשה לאובייקט וקטן בעת הסרת התייחסות.
    *   כאשר מונה ההתייחסויות של אובייקט מגיע לאפס, משמעות הדבר היא שאין יותר התייחסויות לאובייקט, ו-Python משחררת באופן מיידי את הזיכרון שהאובייקט תופס.

2.  **אספן זבל מחזורי (Cyclic Garbage Collector):**
    *   ספירת התייחסויות אינה יכולה לטפל במקרים שבהם אובייקטים מתייחסים זה לזה, ויוצרים *תלויות מחזוריות*. במקרה זה, מוני ההתייחסויות של האובייקטים לעולם לא יגיעו לאפס, גם אם האובייקטים אינם בשימוש יותר.
    *   אספן הזבל המחזורי סורק מעת לעת את האובייקטים בזיכרון בחיפוש אחר תלויות מחזוריות כאלו.
    *   אם הוא מגלה מחזורים המורכבים מאובייקטים שאינם ניתנים יותר להשגה מהתוכנית הראשית (כלומר, אינם בשימוש), הוא מסיר אובייקטים אלו ומשחרר את הזיכרון שהם תופסים.

*   **אפשרות A אינה נכונה:** תלויות מחזוריות מתעלמות, מה שיוביל לדליפת זיכרון.
*   **אפשרות B נכונה:** זהו תיאור מלא ומדויק.
*   **אפשרות C אינה נכונה:** אלגוריתם ההישגוּת משמש ככלי עזר, אך העיקרי הוא ספירת התייחסויות.
*   **אפשרות D אינה נכונה:** ב-Python אין ניהול זיכרון ידני.

**דוגמה:**

```python
import gc

class MyObject:
    pass

obj1 = MyObject()
obj2 = MyObject()

obj1.reference = obj2
obj2.reference = obj1  # Cyclic reference created!

del obj1
del obj2 # The objects are still in memory because of the cyclic reference

print(gc.collect()) # אספן הזבל המחזורי יסיר אובייקטים אלו
```
**בתוצאה:**

ב-Python נעשה שימוש בניהול זיכרון אוטומטי באמצעות שילוב של ספירת התייחסויות ואספן זבל מחזורי. ספירת התייחסויות משחררת זיכרון כאשר אין יותר התייחסויות לאובייקט, בעוד שאספן הזבל המחזורי מגלה ומסיר תלויות מחזוריות, ומונע דליפות זיכרון.

לפיכך, אפשרות B היא התשובה הנכונה.
