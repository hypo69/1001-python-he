### `question_880.md`

**שאלה 880.** מהו Django ORM, ואילו יתרונות *עיקריים* הוא מעניק למפתחים בעבודה עם מסדי נתונים בפרויקטי Django?

A. Django ORM הוא שפת שאילתות מיוחדת, שפותחה עבור Django, המאפשרת לבצע פעולות מורכבות במסדי נתונים שאינן זמינות ב-SQL.
B. Django ORM הוא כלי ליצירה אוטומטית של תבניות HTML על בסיס נתונים ממסד הנתונים.
C. Django ORM הוא מערכת לפריסה אוטומטית של יישומי Django על השרת.
D. Django ORM הוא כלי למיפוי אובייקט-רלציוני, המאפשר לעבוד עם מסד הנתונים באמצעות מחלקות Python (מודלים), ומספק הפשטה מעל SQL, מפשט את ביצוע השאילתות ויצירת טבלאות.

**התשובה הנכונה: D**

**הסבר:**

Django ORM (ממפה אובייקט-רלציוני) הוא רכיב מפתח בפרימוורק Django, אשר מפשט את האינטראקציה עם מסדי הנתונים.

1.  **הגדרת Django ORM**:
    *   ORM הוא שיטת תכנות שממירה נתונים בין מערכות טיפוסים לא תואמות, תוך שימוש בפרדיגמה מונחית עצמים.
    *   Django ORM מאפשר ליצור אינטראקציה עם מסדי נתונים באמצעות מחלקות Python, במקום כתיבת שאילתות SQL.

2.  **יתרונות Django ORM:**
    *   **הפשטה מ-SQL**: מפתחים אינם צריכים לכתוב שאילתות SQL באופן ידני. Django ORM מייצר באופן אוטומטי שאילתות SQL על בסיס קוד Python.
    *   **פישוט העבודה עם מסד הנתונים**: מפתחים יכולים להגדיר את מבנה מסד הנתונים (טבלאות, שדות, קשרים) באמצעות מחלקות Python (מודלים).
    *   **יצירה אוטומטית של טבלאות:** Django ORM יכול ליצור באופן אוטומטי טבלאות במסד הנתונים על בסיס מודלים שהוגדרו.
    *   **אבטחה:** Django ORM מספק הגנה מפני הזרקות SQL.
    *   **ניידות:** ה-ORM מאפשר לעבור בקלות בין מערכות ניהול מסדי נתונים (СУБД) שונות.
    *   **שיטות נוחות לשאילתות:** ה-ORM מספק שיטות נוחות לביצוע שאילתות למסד הנתונים, כגון `filter()`, `get()`, `update()`, `delete()`.

*   **אפשרות A אינה נכונה:** Django ORM אינו שפת שאילתות, אלא ממשק.
*   **אפשרות B אינה נכונה:** תבניות נוצרות באמצעות Django templates.
*   **אפשרות C אינה נכונה:** הפריסה מתבצעת באמצעות כלים אחרים.
*   **אפשרות D נכונה:** מתארת את הייעוד של Django ORM.

**לסיכום:**

Django ORM מספק הפשטה מעל SQL, ומאפשר למפתחים לעבוד עם מסדי נתונים באמצעות מחלקות Python. זה מפשט את ביצוע השאילתות, יצירת טבלאות, משפר את האבטחה ומפשט את העברת היישום בין מערכות ניהול מסדי נתונים (СУБД).

לפיכך, אפשרות D היא הנכונה.
