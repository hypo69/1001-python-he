### `question_839.md`

**שאלה 839.** מהו תפקיד המתודה `__new__()` בפייתון ובאילו מקרים השימוש בה חשוב במיוחד?

A. המתודה `__new__()` אחראית על אתחול המאפיינים של מופע מחלקה שכבר נוצר.
B. המתודה `__new__()` אחראית על שחרור הזיכרון שתפס מופע המחלקה, בעת מחיקתו.
C. המתודה `__new__()` אחראית על יצירת מופע חדש של המחלקה, תוך הקצאת זיכרון לאובייקט. היא נקראת *לפני* המתודה `__init__()`.
D. המתודה `__new__()` משמשת להגדרה של מתודות סטטיות במחלקה.

**תשובה נכונה: C**

**הסבר:**

בפייתון, המתודה `__new__()` היא מתודה מיוחדת שאחראית על *יצירת* מופע חדש של מחלקה. היא נקראת *לפני* המתודה `__init__()`, שאחראית על *אתחול* המופע שכבר נוצר.

*   **`__new__()`**:
    *   מקצה זיכרון עבור אובייקט המחלקה החדש.
    *   מקבלת כארגומנט ראשון את ה*מחלקה* (`cls`), שעבורה נוצר המופע.
    *   חייבת להחזיר מופע חדש של המחלקה (או אובייקט מטיפוס אחר, במידה ונדרש לשנות את תהליך היצירה).

*   **`__init__()`**:
    *   מאתחלת את המאפיינים של מופע מחלקה *שכבר קיים*.
    *   מקבלת כארגומנט ראשון את *מופע* המחלקה (`self`).
    *   אינה אמורה להחזיר ערך כלשהו (מחזירה `None` כברירת מחדל).

**מתי חשוב להשתמש ב-`__new__()`:**

1.  **עבודה עם טיפוסים בלתי ניתנים לשינוי (immutable types):** בעת ירושה מטיפוסים בלתי ניתנים לשינוי (למשל, `str`, `int`, `tuple`), יש לדרוס את `__new__()` על מנת לשלוט ביצירת האובייקט, מכיוון ש-`__init__()` אינה יכולה לשנות אובייקט שכבר נוצר.

2.  **שליטה ביצירת האובייקט:** `__new__()` מאפשרת שליטה מלאה על תהליך יצירת האובייקט. הדבר שימושי במיוחד כאשר יש צורך, למשל, לממש את תבנית "Singleton" (בה נוצר מופע יחיד בלבד של המחלקה).

*   **אפשרות A אינה נכונה:** `__init__()` אחראית על האתחול, לא `__new__()`.
*   **אפשרות B אינה נכונה:** `__del__()` אחראית על שחרור הזיכרון.
*   **אפשרות C נכונה:** זהו התיאור הנכון של תפקיד המתודה `__new__()`.
*   **אפשרות D אינה נכונה:** `@staticmethod` או `@classmethod` משמשים עבור מתודות סטטיות.

**דוגמה:**

```python
class Singleton:
    _instance = None

    def __new__(cls, *args, **kwargs):
        if not cls._instance:
            cls._instance = super().__new__(cls)  # Call base class's __new__
        return cls._instance

    def __init__(self, value):
        if not hasattr(self, 'initialized'):
            self.value = value
            self.initialized = True

s1 = Singleton(10)
s2 = Singleton(20) # Does nothing, since s1 is the instance
print(s1.value) # Output 10, s2 doesn't change anything
```

**לסיכום:**

המתודה `__new__()` אחראית על יצירת מופע חדש של מחלקה, תוך הקצאת הזיכרון הנדרש. היא נקראת *לפני* המתודה `__init__()` וחשובה במיוחד בעבודה עם טיפוסים בלתי ניתנים לשינוי ובמצבים בהם נדרשת שליטה על תהליך יצירת האובייקט.

לפיכך, אפשרות C היא התשובה הנכונה.
