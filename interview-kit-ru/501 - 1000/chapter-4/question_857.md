### `question_857.md`

**שאלה 857.** מה יודפס על המסך כתוצאה מביצוע קוד הפייתון הבא?

```python
list_example = [1, 2, 3, 4, 5]
another_list = list_example
another_list[0] = 100
print(list_example)
```

A.  `[1, 2, 3, 4, 5]`
B.  `[100, 2, 3, 4, 5]`
C.  `[1, 100, 3, 4, 5]`
D.  `Error`

**תשובה נכונה: B**

**הסבר:**

קוד זה מדגים את המושג של *העתקה שטחית* (shallow copying) ויכולת השינוי (mutability) של רשימות בפייתון.

1.  **`list_example = [1, 2, 3, 4, 5]`:**
    *   נוצרת רשימה בשם `list_example`, המכילה מספרים שלמים.

2.  **`another_list = list_example`:**
    *   למשתנה `another_list` מושמת *לא עותק*, אלא *הפניה* לאותה רשימה בדיוק אליה מפנה `list_example`. זו נקראת *העתקה שטחית*. שני המשתנים מצביעים כעת על אותו אובייקט בזיכרון.

3.  **`another_list[0] = 100`:**
    *   משתנה האיבר באינדקס 0 ברשימה אליה מפנה `another_list`. היות ש-`another_list` ו-`list_example` מפנים לאותה רשימה *בדיוק*, שינוי ב-`another_list` ישתקף גם ב-`list_example`.

4.  **`print(list_example)`:**
    *   מודפסת על המסך התכולה של הרשימה אליה מפנה `list_example`.

*   **אפשרות A אינה נכונה:** הרשימה `list_example` שונתה.
*   **אפשרות B נכונה:** תשובה זו משקפת את ההבנה הנכונה.
*   **אפשרות C אינה נכונה:** שונה האיבר באינדקס 0, לא 1.
*   **אפשרות D אינה נכונה:** הקוד אינו מחזיר שגיאות.

**מושגי מפתח:**

*   **העתקה שטחית (Shallow Copying):** נוצר משתנה חדש שמפנה לאותו אובייקט כמו המשתנה המקורי. שינויים המבוצעים דרך משתנה אחד משתקפים באחר.
*   **יכולת שינוי (Mutability):** רשימות הן אובייקטים ברי שינוי, כלומר, ניתן לשנות את תכולתן לאחר יצירתן.

**כתוצאה מכך:**

היות ש-`another_list` היא רק הפניה נוספת לאותה רשימה כמו `list_example`, שינוי הערך של `another_list[0]` מוביל לשינוי של הרשימה *המקורית* `list_example`. לכן יודפס על המסך `[100, 2, 3, 4, 5]`.

לפיכך, אפשרות B היא הנכונה.
