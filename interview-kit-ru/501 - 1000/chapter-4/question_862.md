**שאלה 862.** כיצד ניתן להשתמש במשתנים בוליאניים (`bool`) כדי לפשט ביטויים לוגיים ב-Python, והצג תרחיש לדוגמה שבו הדבר יהיה מועיל במיוחד?

א. משתנים בוליאניים אינם יכולים לפשט ביטויים לוגיים, מכיוון שהם רק מאחסנים את הערכים `True` או `False`.
ב. משתנים בוליאניים רק מסבכים ביטויים לוגיים על ידי הוספת בדיקות מיותרות.
ג. משתנים בוליאניים יכולים להפוך ביטויים לוגיים לקריאים ומובנים יותר, במיוחד בעת אחסון תוצאות של תנאים מורכבים או בדיקות ביניים.
ד. משתנים בוליאניים משמשים רק לשליטה בלולאות ואינם משפיעים על ביטויים לוגיים.

**התשובה הנכונה: ג**

**הסבר:**

משתנים בוליאניים (`bool`) ממלאים תפקיד חשוב בפישוט ובשיפור קריאות של ביטויים לוגיים ב-Python. הם מאפשרים לאחסן את התוצאה של פעולות לוגיות ולהשתמש בה בבדיקות עתידיות, מה שהופך את הקוד למובן ומובנה יותר.

*   **כיצד `bool` מפשטים ביטויים לוגיים:**
    *   **אחסון תוצאות של תנאים מורכבים:** ניתן להשתמש במשתנה בוליאני כדי לאחסן את התוצאה של ביטוי לוגי מורכב, במקום לחזור על ביטוי זה מספר פעמים.
    *   **שיפור קריאות:** הקצאת שמות בעלי משמעות למשתנים בוליאניים הופכת את הקוד לקריא ומובן יותר, במיוחד עבור תנאים מורכבים.
    *   **פישוט בדיקות מורכבות:** משתנים בוליאניים מאפשרים לפצל בדיקות מורכבות לחלקים קטנים יותר, מובנים מבחינה לוגית.
*   **דוגמאות:**

```python
#Without using boolean
def is_valid_user_no_bool(age, is_member, has_active_subscription):
    if age >= 18 and (is_member or has_active_subscription):
        return True
    else:
        return False

# Using boolean
def is_valid_user(age, is_member, has_active_subscription):
    is_adult = age >= 18 # אחסון ביטויים בוליאניים
    is_entitled = is_member or has_active_subscription

    if is_adult and is_entitled:
        return True
    else:
        return False

# Example usage:
age = 25
is_member = True
has_active_subscription = False

if is_valid_user(age, is_member, has_active_subscription):
    print("User is valid") # פלט: User is valid
else:
    print("User is not valid")
```

**במקרים בהם הדבר מועיל במיוחד:**

1.  **תנאים מורכבים:** כאשר נדרש לשלב מספר תנאים באמצעות אופרטורים לוגיים (`and`, `or`, `not`).
2.  **בדיקות חוזרות:** כאשר יש צורך לבדוק את אותו התנאי במספר מקומות בקוד.
3.  **פונקציות עם מספר תנאים:** בפונקציות שבהן התוצאה תלויה בשילוב של תנאים שונים.

*   **אפשרות א שגויה:** משתנים בוליאניים *מאפשרים* לפשט ביטויים לוגיים.
*   **אפשרות ב שגויה:** להיפך, הקוד הופך למובן יותר.
*   **אפשרות ג נכונה:** משתנים בוליאניים *משפרים* את קריאותם של ביטויים לוגיים מורכבים.
*   **אפשרות ד שגויה:** משתנים בוליאניים אינם מוגבלים לשליטה בלולאות.

**לסיכום:**

שימוש במשתנים בוליאניים מאפשר להפוך ביטויים לוגיים לקריאים ומובנים יותר, במיוחד בעת עבודה עם תנאים מורכבים ובדיקות חוזרות. הקצאת שמות בעלי משמעות למשתנים בוליאניים משפרת את הבנת הלוגיקה של הקוד.

לפיכך, אפשרות ג היא הנכונה.
