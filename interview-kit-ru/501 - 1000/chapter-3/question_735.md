**שאלה 735.** נתון מספר שלם `n`. פתח אלגוריתם בפייתון, אשר מחשב את סכום כל המספרים הזוגיים בטווח שבין 1 ל-`n` (כולל).

**דוגמאות:**
```
קלט: n = 6
פלט: 12
הסבר: 2 + 4 + 6 = 12

קלט: n = 5
פלט: 6
הסבר: 2 + 4 = 6
```

- א. כדי לפתור את הבעיה יש לעבור על כל המספרים מ-1 ועד n ולסכום אותם.
- ב. כדי לפתור את הבעיה יש להשתמש בלולאת `for` ובאופרטור תנאי `if` כדי לסנן את המספרים הזוגיים ולסכום אותם לאחר מכן.
- ג. כדי לפתור את הבעיה ניתן להשתמש בנוסחת סכום סדרה חשבונית לחישוב סכום המספרים הזוגיים עד n.
- ד. כדי לפתור את הבעיה מתאים רק אלגוריתם רקורסיבי.

**תשובה נכונה: ג**

**הסבר:**

לפתרון הבעיה של מציאת סכום כל המספרים הזוגיים עד למספר נתון `n`, הגישה האופטימלית היא שימוש בנוסחת סדרה חשבונית. גישה זו מבטיחה חישוב תוצאה בזמן קבוע, מה שיעיל הרבה יותר מאיטרציה על המספרים.

*   **סדרה חשבונית:**
    *   **מספרים זוגיים:** המספרים הזוגיים מ-1 ועד `n` מהווים סדרה חשבונית.
    *   **הפרש (d):** ההפרש בין איברים סמוכים בסדרה שווה ל-2.
    *   **איבר ראשון (a):** האיבר הראשון הוא תמיד `2`.
*   **נוסחת סכום סדרה חשבונית:**
    *   סכום סדרה חשבונית מחושב לפי הנוסחה: `sum = (k/2)*(a + last)`, כאשר k הוא מספר האיברים, `a` הוא האיבר הראשון, ו-`last` הוא האיבר האחרון.
    *   כדי לקבל את האיבר האחרון, ניתן להשתמש גם בנוסחה `last = a + (k-1) * d`.
    *   במקרה שלנו:
        *   `k` (מספר האיברים) הוא `N // 2` (כאשר N הוא המספר הנתון).
        *   `a` = 2
        *   `d` = 2, ו-`last = a + (N//2 - 1)*2` = `2 + (N//2 - 1)*2` = `N` (אם N זוגי, האיבר האחרון בסדרה הוא N, ואם N אי-זוגי, האיבר האחרון הוא N-1, ומספר האיברים יהיה זהה בשני המקרים, כלומר N // 2).
    *   הסכום `sum = (N // 2) * (2 + last)`. בהתחשב בכך שהאיבר האחרון הוא `2 * (N // 2)`, הסכום הוא `sum = (N // 2) * (2 + 2 * (N // 2))` = `(N // 2) * 2 * (1 + N // 2)` = `(N // 2) * (N // 2 + 1) * 2`. רגע, הנוסחה שם נראית שגויה קצת - last הוא 2 * (N // 2), כלומר N אם N זוגי, ו N-1 אם N אי זוגי. אם נציב זאת, הנוסחה `(N // 2) * (2 + N)` נכונה רק אם last הוא N, וזה רק כאשר N זוגי. הנוסחה הנכונה לסכום כל הזוגיים עד N היא `(N // 2) * (N // 2 + 1)`. נתקן את ההסבר בהתאם.
    *   הסכום: האיבר הראשון הוא 2. מספר האיברים הזוגיים עד `N` הוא `N // 2`. האיבר האחרון הוא `2 * (N // 2)`.
    *   נוסחת הסכום: `sum = (מספר איברים / 2) * (איבר ראשון + איבר אחרון)`.
    *   `sum = ( (N // 2) / 2 ) * ( 2 + 2 * (N // 2) )`
    *   `sum = (N // 4) * (2 * (1 + N // 2))`
    *   `sum = (N // 2) * (1 + N // 2)`
    *   או, אם נסמן `k = N // 2` (מספר האיברים הזוגיים), הסכום הוא `k * (k + 1)`.
    *   לדוגמה, N=6: k=3. סכום = 3 * (3+1) = 12.
    *   לדוגמה, N=5: k=2. סכום = 2 * (2+1) = 6.
    *   הנוסחה `(N // 2) * (N // 2 + 1)` נראית נכונה ומתאימה לדוגמאות. ההסבר הרוסי על how last is calculated seemed a bit off. The sum of the first k even numbers (2, 4, ..., 2k) is `k * (k+1)`. If N is given, k is `N // 2`. So the sum is `(N // 2) * (N // 2 + 1)`. Let's use this corrected understanding for the Hebrew explanation.

*   **שימוש בנוסחה:** נשתמש בנוסחה המתוקנת `(N // 2) * (N // 2 + 1)` לחישוב סכום כל המספרים הזוגיים עד `n` (כולל).
*   **יתרונות האלגוריתם:**
    *   **סיבוכיות קבועה:** האלגוריתם בעל סיבוכיות O(1), מאחר שאינו משתמש בלולאות, אלא מחשב מיידית את התוצאה באמצעות נוסחה.
    *   **יעילות:** זהו הפתרון היעיל ביותר למציאת סכום מספרים זוגיים, מאחר שהוא מנצל את התכונות של סדרה חשבונית.

**דוגמאות (פסאודו-קוד):**

```
function even_sum(n):
    // חישוב סכום כל המספרים הזוגיים עד n
    k = n // 2  // מספר האיברים הזוגיים
    sum = k * (k + 1) // נוסחת סכום k האיברים הזוגיים הראשונים
    return sum
```

**דוגמאות מימוש בפייתון:**

```python
def evenSumTillN(n):
    # מחשב את סכום כל המספרים הזוגיים בטווח 1 עד n (כולל)
    # משתמש בנוסחה של סכום סדרה חשבונית
    k = n // 2  # מספר האיברים הזוגיים עד n
    sum = k * (k + 1)
    return sum

print(f"קלט: n = {6}")
print(f"פלט: {evenSumTillN(6)}")  # ידפיס: פלט: 12
print(f"קלט: n = {5}")
print(f"פלט: {evenSumTillN(5)}")  # ידפיס: פלט: 6
print(f"קלט: n = {10}")
print(f"פלט: {evenSumTillN(10)}")   # ידפיס: פלט: 30
print(f"קלט: n = {1}")
print(f"פלט: {evenSumTillN(1)}") # ידפיס: פלט: 0
```
