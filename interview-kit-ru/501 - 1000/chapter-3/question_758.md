**שאלה 758.** נתון מערך של מספרים שלמים `nums` ומספר שלם `k`. פתח אלגוריתם למציאה והחזרת `k` האלמנטים בעלי התדירות הגבוהה ביותר במערך `nums`. סדר האלמנטים בתוצאה אינו משנה.

* מובטח שהתשובה (קבוצת האלמנטים) תהיה ייחודית (כלומר, שני אלמנטים אינם יכולים להופיע באותה כמות פעמים).

**דוגמאות:**

```
קלט: nums = [1,1,1,2,2,3], k = 2
פלט: [1,2]

קלט: nums = [1], k = 1
פלט: [1]
```

- א. לפתרון הבעיה יש להשתמש בחיפוש בינארי על המערך, כדי למצוא את האלמנטים בעלי התדירות הגבוהה.
- ב. לפתרון הבעיה יש למיין את המערך, ולבחור את k האלמנטים מההתחלה.
- ג. לפתרון הבעיה יש להשתמש במילון לספירת תדירות האלמנטים ואלגוריתם מיון.
- ד. לפתרון הבעיה יש להשתמש בטבלת גיבוב (מילון) לספירת תדירות האלמנטים, וכן בתור עדיפויות (heap) לחילוץ `k` האלמנטים השכיחים ביותר.

**תשובה נכונה: D**

**הסבר:**

לפתרון בעיית מציאת `k` האלמנטים השכיחים ביותר במערך, הגישה האופטימלית היא שימוש בשילוב של טבלת גיבוב (מילון) ותור עדיפויות (heap). טבלת הגיבוב מאפשרת לספור במהירות את תדירות ההופעות של האלמנטים, ותור העדיפויות מאפשר לחלץ k אלמנטים בעלי התדירות המקסימלית.

* **אלגוריתם (טבלת גיבוב ותור עדיפויות):**
    1.  **ספירת תדירות:** יוצרים מילון (טבלת גיבוב) `counts`, שבו המפתח הוא אלמנט המערך, והערך הוא מספר הופעותיו.
        *   עוברים בלולאה על המערך `nums`, וסופרים את מספר ההופעות של כל אלמנט.
    2. **יצירת תור:** יוצרים תור עדיפויות (ערימת מינימום) `heap`. הפריוריטי הוא מספר ההופעות.
        *   עוברים בלולאה על המילון, ומכניסים לתור זוג (מספר, אלמנט) ומגבילים את גודל התור ל-`k`, תוך הוצאת אלמנטים עם התדירות המינימלית במידת הצורך (`heapq` מאפשר זאת באמצעות `heappushpop`).
    3.  **החזרת תוצאה:** מוציאים אלמנטים מהתור ויוצרים מהם רשימה.

* **יתרונות האלגוריתם:**
    *   **סיבוכיות לינארית:** טבלת הגיבוב מאפשרת לספור הופעות בזמן לינארי O(n). ותור העדיפויות מאפשר קבלת טופ `k` אלמנטים בסיבוכיות O(n log k) (פחות מ-O(n log n)).
    *   **יעילות:** מאפשר לא למיין את כל האלמנטים, אלא לקבל רק את הנדרשים ביותר.
    *   **שימוש בטבלאות גיבוב:** מבטיח מהירות גישה גבוהה לאלמנט.
    *   **תור עדיפויות:** מאפשר שליפת אלמנטים נדרשים מהתור לפי עדיפות.

**דוגמאות מימוש ב-Python:**
```python
import heapq
from collections import Counter
def find_top_k_frequent(nums, k):
    counts = Counter(nums)  # סופר את תדירות האלמנטים
    heap = []
    for value, count in counts.items():
        if len(heap) < k:
            heapq.heappush(heap, (count, value))
        elif heap[0][0] < count:
            heapq.heappushpop(heap, (count, value))

    return [value for count, value in heap]

nums1 = [1,1,1,2,2,3]
k1 = 2
print(f"קלט: nums = {nums1}, k = {k1}")
print(f"פלט: {find_top_k_frequent(nums1, k1)}") # ידפיס: [2, 1]

nums2 = [1]
k2 = 1
print(f"קלט: nums = {nums2}, k = {k2}")
print(f"פלט: {find_top_k_frequent(nums2, k2)}")  # ידפיס: [1]

nums3 = [1,1,2,2,3,4,5,5]
k3 = 3
print(f"קלט: nums = {nums3}, k = {k3}")
print(f"פלט: {find_top_k_frequent(nums3, k3)}") # ידפיס: [5, 2, 1] (או סדר אחר)

```
