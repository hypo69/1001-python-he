**שאלה 725.** נתון מערך של מספרים שלמים `coins`, המייצג את הערכים הנקובים של מטבעות, ומספר שלם `amount`, המייצג את סכום היעד. פתח/י אלגוריתם המחזיר את המספר המינימלי של מטבעות הדרוש ליצירת סכום היעד `amount`, תוך שימוש במטבעות מהמערך `coins`, כאשר מספר המטבעות מכל ערך נקוב אינו מוגבל. אם לא ניתן להשיג את הסכום `amount`, החזר/י `-1`.

**דוגמאות:**

```
קלט: coins = [1, 2, 5], amount = 11
פלט: 3
הסבר: 11 = 5 + 5 + 1

קלט: coins = [2], amount = 3
פלט: -1

קלט: coins = [1], amount = 0
פלט: 0
```

-   א. כדי לפתור את הבעיה ניתן להשתמש באלגוריתם חמדן ולעבור על המטבעות בסדר יורד.
-   ב. כדי לפתור את הבעיה יש להשתמש בסריקה ממצה של כל הצירופים האפשריים ולהחזיר את המספר המינימלי של מטבעות.
- ג. כדי לפתור את הבעיה ניתן להשתמש בתכנות דינמי, לבנות טבלה עם המספר המינימלי של מטבעות, לצורך קבלת הסכום הנוכחי.
- ד. כדי לפתור את הבעיה ניתן להשתמש באלגוריתם חיפוש לרוחב (BFS) ולעבור על כל הצירופים האפשריים.

**תשובה נכונה: ג**

**הסבר:**

לפתרון הבעיה של מציאת המספר המינינלי של מטבעות עבור סכום נתון, תוך שימוש במספר בלתי מוגבל של מטבעות, מתאים אלגוריתם של תכנות דינמי. שיטה זו מאפשרת שימוש חוזר בפתרונות שחושבו קודם עבור סכומים קטנים יותר.

*   **אלגוריתם (תכנות דינמי):**
    1.  **יצירת מערך:** נוצר מערך `dp` בגודל `amount + 1`, כאשר `dp[i]` יתאים למספר המינימלי של מטבעות, הדרושים לקבלת הסכום `i`.
    2.  **אתחול:** מאתחלים את המערך `dp` בערכי אינסוף (למעט `dp[0] = 0`, מכיוון שסכום אפס ניתן להשיג ב-`0` מטבעות).
    3.  **איטרציה על סכומים:** עוברים באיטרציה על הסכומים מ-1 ועד `amount`, ועבור כל סכום עוברים באיטרציה על המערך `coins`:
        *   **בדיקה:** אם הערך הנקוב של המטבע `coin` קטן או שווה לסכום הנוכחי `i`, אז אנחנו יכולים להשתמש במטבע זה, ולעדכן את הערך של `dp[i]` כמקסימום מבין `dp[i]` ו-`dp[i - coin] + 1`.
    4.  **תוצאה:** לאחר האיטרציות, מחזירים את הערך `dp[amount]`. אם הערך של `dp[amount]` עדיין שווה לאינסוף (או לסימן אחר של היעדר פתרון), אז מחזירים `-1`, מכיוון שלא ניתן להשיג את `amount` מרשימת המטבעות.

*   **יתרונות האלגוריתם:**
    *   **אופטימליות:** מוצא את הפתרון האופטימלי ביותר.
    *   **תכנות דינמי:** משתמש בטבלה לאחסון תוצאות ביניים ונמנע מחישובים חוזרים.
    *   **יעילות:** מבטיח סיבוכיות אופטימלית, עבור בעיה זו.
*   **מדוע גישות אחרות אינן מתאימות:**
    *   **אלגוריתם חמדן:** אלגוריתם חמדן אינו מבטיח מציאת המספר המינימלי של מטבעות, במיוחד עבור סטים מסוימים של ערכים נקובים.
    *   **סריקה ממצה:** סריקה ממצה של כל האפשרויות אינה יעילה מאוד ובעלת סיבוכיות גבוהה.
    *   **BFS:** BFS יכול להתאים, אך תכנות דינמי מאפשר מציאת פתרון אופטימלי יותר מבחינת זמן.

**דוגמאות (פסאודו-קוד):**
```
function find_minimum_coins(coins, amount):
     dp =  an array of length amount+1 and each position set to infinity except 0 set to 0.
     for i from 1 to amount:
          for coin from coins:
                if coin <= i:
                    dp[i] = min(dp[i], dp[i-coin] +1)
      if dp[amount] is not infinity:
            return dp[amount]
      return -1
```
**דוגמאות מימוש ב-Python:**

```python
def find_minimum_coins(coins, amount):
    dp = [float('inf')] * (amount + 1)
    dp[0] = 0
    for i in range(1, amount + 1):
        for coin in coins:
            if coin <= i:
                dp[i] = min(dp[i], dp[i - coin] + 1)
    return dp[amount] if dp[amount] != float('inf') else -1

coins1 = [1, 2, 5]
amount1 = 11
print(f"קלט: coins = {coins1}, amount = {amount1}")
print(f"פלט: {find_minimum_coins(coins1, amount1)}")  # פלט: 3

coins2 = [2]
amount2 = 3
print(f"קלט: coins = {coins2}, amount = {amount2}")
print(f"פלט: {find_minimum_coins(coins2, amount2)}")  # פלט: -1

coins3 = [1]
amount3 = 0
print(f"קלט: coins = {coins3}, amount = {amount3}")
print(f"פלט: {find_minimum_coins(coins3, amount3)}")  # פלט: 0
```
