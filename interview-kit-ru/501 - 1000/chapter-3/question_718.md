### `question_718.md`

**שאלה 718.** נתון מערך של מספרים שלמים `citations`, כאשר `citations[i]` מייצג את מספר הציטוטים של המאמר ה-`i`. פתח אלגוריתם לחישוב ה-h-אינדקס המקסימלי של מחבר.

*   **h-אינדקס** - הוא המספר הגדול ביותר `h` כזה, שלמחבר יש `h` מאמרים, שכל אחד מהם מצוטט לפחות `h` פעמים.

**דוגמאות:**
```
Ввод: citations = [3, 0, 6, 1, 5]
Вывод: 3

Ввод: citations = [1, 1]
Вывод: 1
```

-   א. כדי לפתור את הבעיה, יש לעבור על כל המספרים האפשריים, ולבדוק את התאמתם ל-h-אינדקס.
-   ב. כדי לפתור את הבעיה, יש למיין את המערך ולבדוק ערכים מתחילת הרשימה, עד שנמצא את האיבר הקטן ביותר.
-   ג. כדי לפתור את הבעיה, יש להשתמש במערך מונה, ולעקוב אחר מספר המאמרים עם מספר הציטוטים המתאים, ולאחר מכן לעבור על המערך בסדר הפוך למציאת ה-h-אינדקס המקסימלי.
-   ד. כדי לפתור את הבעיה, יש להשתמש בחיפוש בינארי.

**התשובה הנכונה: ג**

**הסבר:**

לפתרון בעיית מציאת h-אינדקס, הגישה האופטימלית היא שימוש במערך מונה וביצוע איטרציה עליו מהסוף למציאת האינדקס המקסימלי העומד בקריטריון ה-h-אינדקס.

*   **אלגוריתם (באמצעות מערך מונה):**
    1.  **אתחול:**
        *   נוצר מערך מונה `counters`, כאשר `counters[i]` - מספר המאמרים עם i ציטוטים, ואורך המערך שווה ל-`length(citations) +1`.
    2.  **ספירת מופעים:** מבצעים איטרציה על המערך `citations`. עבור כל איבר `x`:
        *   אם `x` קטן מאורך המערך `citations`, אז מגדילים את המונה ב-`1` באינדקס `x`.
        *   אחרת (אם `x` גדול או שווה לאורך המערך), אז מגדילים את המונה עבור `n`.
    3.  **יצירת מערך "לינארי":** יוצרים מערך חדש `m` שבו המספר `x` חוזר על עצמו `counters[x]` פעמים.
    4.  **מציאת h-אינדקס:**
        *   עוברים על `m` מהסוף (אינדקס `n` עד 1) ועבור כל אינדקס, בודקים שהערך גדול או שווה לאינדקס.
        *   אם התנאי מתקיים, אזי אינדקס זה הוא h-אינדקס (או אחת מהאפשרויות), ואנו מחזירים אותו.

*   **יתרונות האלגוריתם:**
    *   **סיבוכיות לינארית:** לאלגוריתם סיבוכיות זמן של `O(n)`, כאשר n - מספר האיברים במערך `nums`.
    *   **הסרת כפילויות:** המערך `m` משמש להסרת כפילויות ולבדיקה קלה יותר.
*   **מדוע אפשרויות אחרות אינן מתאימות:**
    *   **מיון:** מיון, גם אם ייושם, יאט את האלגוריתם, ומעבר על כל הערכים עם בדיקה לאחר מכן יהיה לא יעיל.
    *   **רקורסיה:** שימוש ברקורסיה אינו יעיל.
    *   **אלגוריתם חמדן:** אלגוריתמים חמדנים אינם אופטימליים לפתרון הבעיה.
    *   **חיפוש בינארי:** חיפוש בינארי אינו ישים עבור מערך לא ממוין, ולא יאפשר למצוא את ה-h-אינדקס.

**דוגמאות (פסאודו-קוד):**
```
function calculate_h_index(citations):
  n = length(citations)
  counts = new array with length n+1, with default 0

  for citation in citations:
      if citation < n:
          counts[citation] +=1
      else:
          counts[n] +=1
  m = new array
  for  i in range(n):
     add i to array m, counts[i] times.
  for  i from length(m) to 1:
      if m[n-1] >= i:
          return i
  return 0 # return 0 if h-index not found
```
**דוגמאות מימוש ב-Python:**
```python
def calculate_h_index(citations):
    n = len(citations)
    s = [0] * (n + 1)
    for x in citations:
        if x < n:
            s[x] += 1
        else:
            s[n] += 1

    m = []
    for x in range(len(s)):
        m.extend([x] * s[x])

    for n in range(1, len(m) + 1):
        if m[-n] >= n:
            return n
    return 0

citations1 = [3, 0, 6, 1, 5]
print(f"Ввод: citations = {citations1}")
print(f"Вывод: {calculate_h_index(citations1)}")  # Вывод: 3

citations2 = [1,1]
print(f"Ввод: citations = {citations2}")
print(f"Вывод: {calculate_h_index(citations2)}")  # Вывод: 1

```
