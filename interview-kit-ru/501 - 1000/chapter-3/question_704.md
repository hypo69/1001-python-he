**שאלה 704.** נתון מערך של מספרים שלמים `nums`, ממוין בסדר לא יורד (כלומר בסדר עולה, עם אפשרות לשוויון) ומספר שלם `target`. תכנן אלגוריתם למציאת המיקום הראשון והאחרון של אלמנט המטרה `target` במערך `nums`, עם סיבוכיות זמן של O(log n). אם אלמנט המטרה אינו קיים במערך, החזר `[-1, -1]`.

**דוגמאות:**
```
קלט: nums = [5,7,7,8,8,10], target = 8
פלט: [3,4]

קלט: nums = [5,7,7,8,8,10], target = 6
פלט: [-1,-1]
```
-   A.  לפתרון הבעיה יש לעבור על כל איברי המערך ולמצוא את האינדקסים הראשוני והאחרונים של הופעת ה-target.
-  B.  לפתרון הבעיה ניתן להשתמש רק בחיפוש בינארי, שימצא רק את האינדקס האמצעי, אם קיים איבר כזה.
- C. לפתרון הבעיה יש ליישם שני חיפושים בינאריים: אחד למציאת המיקום הראשון, והשני למציאת המיקום האחרון, ואם ה-element לא נמצא, יש להחזיר `[-1,-1]`.
- D. לפתרון הבעיה יש להשתמש באלגוריתם חיפוש ליניארי במערך.

**תשובה נכונה: C**

**הסבר:**

לפתרון בעיית מציאת המיקומים הראשוני והאחרונים של איבר במערך ממוין עם סיבוכיות זמן של `O(log n)`, הפתרון האופטימלי הוא שימוש בשני אלגוריתמים של חיפוש בינארי. האחד מוצא את המיקום הראשון, והשני – את המיקום האחרון.

*   **אלגוריתם (שני חיפושים בינאריים):**
    1.  **חיפוש בינארי ראשון (המיקום הראשון):**
         *   מחפשים איבר השווה ל-`target`, אך תוך כדי כך יש לנוע שמאלה עד שיימצא האיבר השמאלי ביותר.
        *   אם האיבר לא נמצא, אנו יוצאים מהאלגוריתם ומחזירים `[-1, -1]`.
     2.  **חיפוש בינארי שני (המיקום האחרון):**
          * אנו מוצאים את האיבר השווה ל-`target`, ותוך כדי כך ממשיכים לחפש לכיוון ימין עד למציאת האיבר הימני ביותר.
          *  אם האיבר לא נמצא, אנו יוצאים מהאלגוריתם.
      3.   **תוצאה:**  לאחר שני החיפושים הבינאריים, מוחזרת רשימה של האינדקסים הראשוני והאחרון.

*   **יתרונות האלגוריתם:**
    *   **סיבוכיות לוגריתמית:** מבטיח סיבוכיות זמן O(log n) עבור מציאת כל מיקום.
   *  **שימוש בתכונות המערך הממוין:**  מנצל את מיון המערך לחיפוש מהיר של האיבר הרצוי.
 *   **שני חיפושים בינאריים:** האלגוריתם משתמש בשני חיפושים בינאריים, מה שמבטיח מציאת האינדקס השמאלי והימני.

*   **מדוע אפשרויות אחרות אינן מתאימות:**
    * **חיפוש ליניארי:**  מעבר על המערך מתחילתו ועד סופו יגרום לסיבוכיות O(n).
     * **חיפוש בינארי יחיד:**  חיפוש יחיד לא יאפשר למצוא את המיקום הראשון והאחרון אם האיבר חוזר על עצמו.
     *  **מיון:**  מיון המערך בתחילת האלגוריתם אינו נחוץ ויאט את הביצוע.

**דוגמאות (פסאודו-קוד):**

```
function find_range(nums, target):

    # פונקציה למציאת הגבול השמאלי:
    function find_left_index(nums, target):
        low = 0
        high = length(nums) - 1
        index = -1

        while low <= high:
            mid = low + (high - low) // 2
            if nums[mid] >= target:
              if nums[mid] == target:
                   index = mid # שומרים היכן נמצא
              high = mid - 1
            else:
                low = mid + 1
        return index

   # פונקציה למציאת הגבול הימני:
    function find_right_index(nums, target):
        low = 0
        high = length(nums) - 1
        index = -1
        while low <= high:
           mid = low + (high - low) // 2
           if nums[mid] <= target:
            if nums[mid] == target:
              index = mid # שומרים היכן נמצא
            low = mid + 1
           else:
             high = mid - 1

        return index

    left = find_left_index(nums, target) # חיפוש אינדקס שמאלי
    if left == -1 :
        return [-1, -1] # האיבר לא נמצא
    right = find_right_index(nums, target) # חיפוש אינדקס ימני

    return [left, right]
```
