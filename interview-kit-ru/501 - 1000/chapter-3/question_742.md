**שאלה 742.** מהם דסקריפטורי נתונים (data descriptors) בפייתון, ולשם מה הם משמשים? ספק דוגמה למימוש דסקריפטור נתונים והסבר כיצד הוא עובד.

-  א. דסקריפטורי נתונים הם טיפוסי נתונים מיוחדים המיועדים לאחסון תיעוד.
-   ב. דסקריפטורי נתונים הם מנגנונים המאפשרים להרחיב פונקציונליות של פונקציות על ידי הוספת לוגיקה חדשה.
-  ג. דסקריפטורי נתונים הם דרך להגדיר מאפיינים מיוחדים שאת ערכיהם לא ניתן לשנות לאחר היצירה.
-  ד. דסקריפטורי נתונים הם מאפייני אובייקט, שהתנהגותם מוגדרת מחדש באמצעות מתודות מיוחדות, המאפשרות לשלוט בגישה, הצבה ומחיקה של ערכי המאפיינים.

**תשובה נכונה: ד**

**הסבר:**

דסקריפטורי נתונים (data descriptors) בפייתון הם מנגנון המאפשר להגדיר מחדש את ההתנהגות בעת גישה למאפייני אובייקט. הדבר מושג באמצעות מימוש אחת או יותר מהמתודות המיוחדות (`__get__`, `__set__`, `__delete__`) במחלקת הדסקריפטור. דסקריפטורי נתונים נמצאים בשימוש נרחב בפרויקטים, לארגון לוגיקת הגישה לנתונים.

*   **מאפיינים עיקריים של דסקריפטורי נתונים:**
    *  **מנגנון הגדרה מחדש:** מאפשרים להגדיר מחדש את ההתנהגות הסטנדרטית של מאפיין, מה שמאפשר להוסיף לוגיקה נוספת לפעולות עם מאפיינים.
   * **מתודות הדסקריפטור:** ממומשים באמצעות שימוש במתודות מיוחדות:
        *   `__get__(self, instance, owner)`: נקראת בעת ניסיון לקבל את ערך המאפיין.
        *    `__set__(self, instance, value)`: נקראת בעת הצבת ערך למאפיין.
        *    `__delete__(self, instance)`: נקראת בעת מחיקת מאפיין.
        *   `__set_name__(self, owner, name)`: נקראת בעת יצירת מופע של המחלקה (הופיעה בפייתון 3.6).
    *   **בקרת גישה:** מאפשר לשלוט בגישה, שינוי ומחיקה של ערכי המאפיינים.
     * **תאימות:** מאפשרים ליישם את אותו קוד בדיקה למאפיינים שונים.
    *   **שימוש חוזר:** יכולים לשמש למאפיינים שונים של המחלקה, תוך מימוש לוגיקה של שימוש חוזר.

*  **הבדל בין דסקריפטורי נתונים לדסקריפטורים ללא נתונים:**
     *  **דסקריפטורי נתונים** - מממשים את `__get__()` ואת `__set__()` (או `__delete__()`), ומאפשרים לשלוט בהצבה, גישה ומחיקה.
      * **דסקריפטורים ללא נתונים** - מממשים רק את המתודה `__get__()`.

**דוגמאות:**

```python
class StringChecker:
    def __get__(self, instance, owner):
        if instance is None:
            return self
        return instance.__dict__[self.name]

    def __set__(self, instance, str_value):
        if not isinstance(str_value, str):
            raise ValueError('יש לספק מחרוזת') # יפעיל ValueError, מאחר שאינו מחרוזת
        elif len(str_value) < 2:
            raise ValueError('נדרשות לפחות 2 אותיות') # יפעיל ValueError, מאחר שהאורך קטן מ-2
        instance.__dict__[self.name] = str_value

    def __set_name__(self, owner, name):
        self.name = name
class Employee:
    name = StringChecker()
    surname = StringChecker()
    patronymic = StringChecker()
    post = StringChecker()

    def __init__(self, name, surname, patronymic, post):
        self.name = name
        self.surname = surname
        self.patronymic = patronymic
        self.post = post

# Test
director = Employee('Иван', 'Николаевич', 'Прогин', 'Директор')
print(director.__dict__) # יציג: {'name': 'Иван', 'surname': 'Николаевич', 'patronymic': 'Прогин', 'post': 'Директор'}
try:
  director.name = 1 # יפעיל ValueError, מאחר שאינו מחרוזת
except ValueError as e:
  print(e) # יציג: יש לספק מחרוזת

try:
   director.name = 'A' # יפעיל ValueError, מאחר שהאורך קטן מ-2
except ValueError as e:
  print(e) # יציג: נדרשות לפחות 2 אותיות
```
**ניתוח האפשרויות:**

*   **א. דסקריפטורי נתונים הם טיפוסי נתונים מיוחדים המיועדים לאחסון תיעוד.:** לא נכון.
*   **ב. דסקריפטורי נתונים הם מנגנונים המאפשרים להרחיב פונקציונליות של פונקציות על ידי הוספת לוגיקה חדשה.:** לא נכון. דקורטורים משמשים להרחבת פונקציונליות של פונקציות, בעוד שדסקריפטורים משמשים לעבודה עם מאפיינים.
*  **ג. דסקריפטורי נתונים הם דרך להגדיר מאפיינים מיוחדים שאת ערכיהם לא ניתן לשנות לאחר היצירה.:** לא נכון, ניתן לשנות את ערכי המאפיינים.
* **ד. דסקריפטורי נתונים הם מאפייני אובייקט, שהתנהגותם מוגדרת מחדש באמצעות מתודות מיוחדות, המאפשרות לשלוט בגישה, הצבה ומחיקה של ערכי המאפיינים.:** נכון.

**לסיכום:**
*   דסקריפטורים מאפשרים התאמה אישית של הגישה למאפייני אובייקט.
*    משתמשים במתודות מיוחדות `__get__`, `__set__` ו-`__delete__` (וכן `__set_name__()`).
*   דסקריפטורים מאפשרים יצירת קוד קריא ותחזוקתי יותר.
*  יכולים לבצע בדיקות ופעולות נוספות בעת גישה למאפיינים.

לפיכך, התשובה הנכונה היא **ד. דסקריפטורי נתונים הם מאפייני אובייקט, שהתנהגותם מוגדרת מחדש באמצעות מתודות מיוחדות, המאפשרות לשלוט בגישה, הצבה ומחיקה של ערכי המאפיינים.**
