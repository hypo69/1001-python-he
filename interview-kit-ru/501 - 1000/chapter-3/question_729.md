**שאלה 729.** נתון גרף לא מכוון ומקושר (עץ) עם `n` צמתים, הממוספרים מ-`0` עד `n - 1`, עם שורש בצומת `0`. קשתות העץ מיוצגות על ידי המערך `edges`, כאשר `edges[i] = [a_i, b_i]` מציין קיום קשת בין צמתים `a_i` ו-`b_i`. לכל צומת יש תווית, המיוצגת על ידי תו ממחרוזת `labels`, כאשר `labels[i]` הוא התו של הצומת באינדקס `i`. פתח אלגוריתם לחישוב המערך `result` בגודל `n`, כאשר `result[i]` הוא כמות הצמתים בתת-העץ של הצומת `i`, שיש להם את אותה תווית כמו לצומת `i`.

**ייצוג ASCII של העץ:**
```
    0 ('a')
   /  \
  1 ('b')  2 ('a')
 /  \   / \
4 ('d') 5('c') 3('e') 6('d')
```
**דוגמאות:**
```
קלט: n = 7, edges = [[0,1],[0,2],[1,4],[1,5],[2,3],[2,6]], labels = "abaedcd"
פלט: [2,1,1,1,1,1,1]

קלט: n = 4, edges = [[0,1],[1,2],[0,3]], labels = "bbbb"
פלט: [4,2,1,1]
```

-   א. לפתרון הבעיה יש צורך למיין את העץ ולאחר מכן להשתמש בחיפוש לרוחב (BFS).
-   ב. לפתרון הבעיה ניתן להשתמש רק בחיפוש לעומק (DFS), ובנוסף צריך להחזיר את סכום אורכי שני המסלולים הארוכים ביותר העוברים דרך הצומת.
-   ג. לפתרון הבעיה יש צורך להשתמש במחסנית ולחצות את העץ בחיפוש לרוחב (BFS), תוך חישוב ושמירת התוצאה עבור כל תת-עץ.
-   ד. לפתרון הבעיה יש צורך להשתמש בחיפוש לעומק (DFS) רקורסיבי, ובנוסף לשמור את האורך המקסימלי בכל צומת, בעת בחינת תת-העצים.

**התשובה הנכונה: ב**

**הסבר:**

לפתרון בעיית ספירת כמות הצמתים בעלי תוויות זהות בתת-העץ, האופטימלי הוא שימוש באלגוריתם רקורסיבי של חיפוש לעומק (DFS). אלגוריתם זה מאפשר לחצות את העץ ביעילות ולאסוף נתונים על תוויות תואמות בכל תת-עץ.

*   **האלגוריתם (DFS רקורסיבי):**
    1.  **יצירת גרף:** על בסיס המערך `edges`, יוצרים ייצוג של העץ באמצעות מילון, כאשר המפתחות יהיו הצמתים, והערכים - רשימות של צמתי בנים.
    2.  **פונקציה רקורסיבית:** יוצרים פונקציה רקורסיבית שמקבלת את הצומת הנוכחי ואת המערך `result`, שבו תיכתב כמות הצמתים התואמים.
        *   מאתחלים מונה עבור הצומת הנוכחי כ-1 (הצומת עצמו).
        *   עבור כל צאצא של הצומת, קוראים לפונקציה באופן רקורסיבי, תוך שימוש ב-`parent` כדי לא לחזור אחורה, ולהימנע מלולאה אינסופית.
        *   אם התו של צומת הבן תואם לתו של הצומת הנוכחי, מוסיפים את כמות הצמתים מתת-העץ של צומת הבן.
        *   שומרים את התוצאה במערך `result`.
        *   מחזירים את סך כמות הצמתים.
    3.  **קריאה ראשונית:** קוראים לפונקציה הרקורסיבית עבור השורש (צומת 0), ובסוף מחזירים את הרשימה `result`.

*   **יתרונות האלגוריתם:**
    *   **חצייה רקורסיבית:** מאפשר לחצות את העץ לעומק בקלות.
    *   **יעילות:** מגיעה למורכבות זמן לינארית `O(n)`.
    *   **ספירה נכונה:** סופר צמתים בעלי תווית תואמת בתוך תת-העץ.

**דוגמאות (פסאודו-קוד):**
```
function find_subtree_counts(n, edges, labels):
    adj = create adjency list from edges
    result = new array[n] initialized with 0

    function dfs(node, parent):
        count = 1
        for child in adj[node]:
           if child != parent:
            # אם התו של צומת הבן תואם לתו של הצומת הנוכחי
            child_count = dfs(child, node)
            if labels[node] == labels[child]:
                count += child_count
        result[node] = count
        return count

    dfs(0, -1) # קריאה רקורסיבית מהשורש
    return result
```
**דוגמאות מימוש ב-Python:**
```python
def find_subtree_counts(n, edges, labels):
    adj = [[] for _ in range(n)]
    for u, v in edges:
        adj[u].append(v)
        adj[v].append(u)

    result = [0] * n
    def dfs(node, parent):
      count = 1
      for child in adj[node]:
        if child != parent:
          child_count = dfs(child, node)
          # אם התו של צומת הבן תואם לתו של הצומת הנוכחי
          if labels[node] == labels[child]:
                count += child_count
      result[node] = count
      return count

    dfs(0, -1) # קריאה רקורסיבית מהשורש
    return result


n1 = 7
edges1 = [[0,1],[0,2],[1,4],[1,5],[2,3],[2,6]]
labels1 = "abaedcd"
print(f"קלט: n = {n1}, edges = {edges1}, labels = '{labels1}'")
print(f"פלט: {find_subtree_counts(n1, edges1, labels1)}")  # ידפיס [2, 1, 1, 1, 1, 1, 1]

n2 = 4
edges2 = [[0,1],[1,2],[0,3]]
labels2 = "bbbb"
print(f"קלט: n = {n2}, edges = {edges2}, labels = '{labels2}'")
print(f"פלט: {find_subtree_counts(n2, edges2, labels2)}")  # ידפיס: [4, 2, 1, 1]
```
