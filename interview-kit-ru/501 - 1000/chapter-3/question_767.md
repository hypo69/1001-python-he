**שאלה 712.** נתון מערך של מספרים שלמים `nums` ומספר שלם `k`. פתח/י אלגוריתם בפייתון, שיחזיר `True` אם קיימים במערך שני אינדקסים ייחודיים `i` ו-`j`, עבורם מתקיימים התנאים:

-   `nums[i] == nums[j]`
-   `abs(i - j) <= k`

אחרת, החזר/י `False`.

**דוגמאות:**

```
קלט: nums = [1,2,3,1], k = 3
פלט: True
הסבר: קיימים אינדקסים i=0 ו-j=3, nums[0] == nums[3] ו-abs(0 - 3) <= 3

קלט: nums = [1,0,1,1], k = 1
פלט: True
הסבר: קיימים אינדקסים i=1 ו-j=2, nums[1] == nums[2] ו-abs(1 - 2) <= 1
```
- A. לפתרון הבעיה יש להשתמש בלולאה כפולה, כדי לבדוק כל זוג אינדקסים במערך.
- B. לפתרון הבעיה יש להשתמש באלגוריתם מיון, ולאחר מכן לבדוק את כל האלמנטים הסמוכים במערך.
- C. לפתרון הבעיה יש להשתמש בחלון גולש, כדי לבדוק כל אלמנט להתאמה לתנאי בחלון הנוכחי.
- D. לפתרון הבעיה יש להשתמש בטבלת גיבוב לאחסון אינדקס האלמנט, ולבדוק שההפרש בין האינדקס הנוכחי לערך מטבלת הגיבוב אינו עולה על הערך k.

**תשובה נכונה: D**

**הסבר:**

לפתרון בעיית חיפוש כפילויות במרחק שאינו עולה על `k`, אופטימלי הוא שימוש בטבלת גיבוב (מילון). המילון יאפשר לעקוב אחר אינדקסים של אלמנטים ולבדוק ביעילות את המרחק בין כפילויות, תוך מתן פתרון בזמן לינארי.

*   **אלגוריתם (טבלת גיבוב):**
    1.  **אתחול:** יוצרים מילון `seen_indices` לאחסון אינדקס ההופעה האחרונה של כל אלמנט.
    2.  **איטרציה:** עוברים על המערך `nums`:
        *   **בדיקה לכפילות:** בודקים האם האלמנט הנוכחי `nums[i]` קיים במילון `seen_indices`.
            *   אם קיים, מחשבים את הפרש האינדקסים `abs(i - seen_indices[nums[i]])`. אם ההפרש אינו עולה על `k`, מחזירים `True`.
        *   **עדכון האינדקס:** אם לא נמצאה כפילות או שהתנאי לא התקיים, מעדכנים את ערך האלמנט `seen_indices[nums[i]]` להיות האינדקס הנוכחי `i`.
    3.  **החזרת `False`:** אם לאחר מעבר על כל האלמנטים לא נמצאו זוגות מתאימים, מחזירים `False`.

*   **יתרונות האלגוריתם:**
    *   **סיבוכיות לינארית:** לאלגוריתם סיבוכיות זמן O(n), מכיוון שהוא עובר על המערך פעם אחת.
    *   **גיבוב:** טבלת הגיבוב מספקת חיפוש מהיר של אלמנטים.
    *   **פשטות:** הקוד תמציתי וקל ליישום.

**דוגמאות (פסאודו-קוד):**
```
function check_duplicates_in_range(nums, k):
    seen_indices =  empty  dictionary() # map to store last seen index of values
    for i from 0 to length(nums) -1:
       if nums[i] is in seen_indices
           if abs(i - seen_indices[nums[i]]) <= k:
                return True
        seen_indices[nums[i]] = i
    return False
```
**דוגמאות ליישום בפייתון:**

```python
def check_duplicates_in_range(nums, k):
    seen_indices = {} # מילון למעקב אחר האינדקס האחרון שנצפה
    for i, num in enumerate(nums):
        if num in seen_indices:
            if abs(i - seen_indices[num]) <= k:
                return True
        seen_indices[num] = i
    return False

nums1 = [1, 2, 3, 1]
k1 = 3
print(f"קלט: nums = {nums1}, k = {k1}")
print(f"פלט: {check_duplicates_in_range(nums1, k1)}")  # יציג: True

nums2 = [1, 0, 1, 1]
k2 = 1
print(f"קלט: nums = {nums2}, k = {k2}")
print(f"פלט: {check_duplicates_in_range(nums2, k2)}")  # יציג: True
```
**ניתוח האפשרויות:**
*   **A. לפתרון הבעיה יש להשתמש בלולאה כפולה, כדי לבדוק כל זוג אינדקסים במערך.:** לא נכון. לאלגוריתם כזה סיבוכיות O(n^2).
*   **B. לפתרון הבעיה יש להשתמש באלגוריתם מיון, ולאחר מכן לבדוק את כל האלמנטים הסמוכים במערך.:** לא נכון. מיון אינו נחוץ, ולאלגוריתם זה סיבוכיות O(n log n).
*   **C. לפתרון הבעיה יש להשתמש בחלון גולש, כדי לבדוק כל אלמנט להתאמה לתנאי בחלון הנוכחי.:** לא נכון. למרות שחלון גולש יכול להתאים, סיבוכיות האלגוריתם המשתמש במילון תהיה טובה יותר.
*   **D. לפתרון הבעיה יש להשתמש בטבלת גיבוב לאחסון אינדקס האלמנט, ולבדוק שההפרש בין האינדקס הנוכחי לערך מטבלת הגיבוב אינו עולה על הערך k.:** נכון.

**כתוצאה מכך:**
*   טבלת גיבוב (מילון) מאפשרת לשמור את אינדקס ההופעה האחרונה של אלמנט.
*   האלגוריתם משתמש במידע מטבלת הגיבוב לבדיקה מהירה של התנאי.
*   האלגוריתם בעל סיבוכיות לינארית ואופטימלי למשימה הנתונה.

לפיכך, התשובה הנכונה היא **D. לפתרון הבעיה יש להשתמש בטבלת גיבוב לאחסון אינדקס האלמנט, ולבדוק שההפרש בין האינדקס הנוכחי לערך מטבלת הגיבוב אינו עולה על הערך k.**
