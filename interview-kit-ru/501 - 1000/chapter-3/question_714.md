**שאלה 714.** נתון גרף קשיר ובלתי מכוון (עץ) הכולל `n` צמתים, ממוספרים מ-`0` עד `n - 1`, עם שורש בצומת `0`, ועם `n-1` קשתות. העץ מיוצג על ידי מערך `edges` כאשר `edges[i] = [a_i, b_i]` פירושו שישנה קשת בין הצמתים `a_i` ו-`b_i`. לכל צומת יש תווית, המיוצגת על ידי תו מתוך המחרוזת `labels`. יש לפתח אלגוריתם לחישוב מערך `result` בגודל `n`, כאשר `result[i]` הוא מספר הצמתים בתת-העץ של הצומת `i` אשר להם אותה תווית כמו לצומת `i`.

**ייצוג העץ ב-ASCII:**

```
    0 ('a')
   /  \
  1 ('b')  2 ('a')
 /  \   / \
4 ('d') 5('c') 3('e') 6('d')
```
**דוגמאות:**
```
קלט: n = 7, edges = [[0,1],[0,2],[1,4],[1,5],[2,3],[2,6]], labels = "abaedcd"
פלט: [2,1,1,1,1,1,1]

קלט: n = 4, edges = [[0,1],[1,2],[0,3]], labels = "bbbb"
פלט: [4,2,1,1]
```

-   A.  לפתרון הבעיה יש צורך למיין את העץ ולאחר מכן להשתמש בחיפוש לרוחב (BFS).
-   B. לפתרון הבעיה יש צורך להשתמש בחיפוש לעומק (DFS), ובמהלך זאת כל קריאה תחזיר מונה של צמתים בעלי אותו ערך.
-   C. לפתרון הבעיה יש צורך להשתמש במחסנית ולעבור על העץ בחיפוש לרוחב (BFS), חישוב ושמירת התוצאה עבור כל תת-עץ.
-   D. לפתרון הבעיה יש צורך להשתמש בחיפוש בינארי בשילוב עם תכנות דינמי.

**תשובה נכונה: B**

**הסבר:**

לפתרון בעיית ספירת מספר הצמתים עם תוויות תואמות בתת-עץ, האופטימלי ביותר הוא שימוש באלגוריתם חיפוש לעומק (DFS). במהלך זאת, בכל קריאה רקורסיבית נחזיר מידע על מספר ההתאמות, ונגדיל את המונים בכל רמה.

*   **אלגוריתם (DFS רקורסיבי):**
    1.  **יצירת גרף:** על בסיס המערך `edges`, ניצור ייצוג של העץ באמצעות מילון, כאשר המפתחות יהיו הצמתים, והערכים - רשימות של צמתים צאצאים.
    2.  **פונקציה רקורסיבית:**
        *   הפונקציה מקבלת את הצומת הנוכחי ואת המערך `result`, היכן שיירשם מס' הצמתים התואמים.
        *   בתחילה, נגדיר ערך התחלתי למונה עבור הצומת הנוכחי כ-1 (הצומת עצמו).
        *   **איטרציה על צמתים צאצאים:** נקרא רקורסיבית לפונקציה עבור כל צומת צאצא.
        *   **בדיקת התאמה:** אם ערך הצומת הצאצא תואם את ערך הצומת הנוכחי, נגדיל את המונה.
        *   **החזרת המונה:** נחזיר את המספר הכולל של צמתים תואמים.
    3.  **קריאה התחלתית:** נקרא לפונקציה הרקורסיבית עבור השורש (צומת `0`).
    4.  **החזרת התוצאה:** לאחר מעבר על כל הצמתים, נחזיר את הרשימה `result`.

*   **יתרונות האלגוריתם:**
    *   **מעבר על העץ:** רקורסיה מאפשרת מעבר קל על העץ לעומק.
    *   **יעילות:** מגיעה לסיבוכיות `O(n)`, כאשר `n` — מספר הצמתים בעץ.
    *   **ספירה נכונה:** סופר צמתים בעלי תווית תואמת בתת-העץ.
    *   **פשטות:** אלגוריתם קל ליישום.

**דוגמאות מימוש בפייתון:**
```python
def find_subtree_counts(n, edges, labels):
    adj = [[] for _ in range(n)]
    for u, v in edges:
        adj[u].append(v)
        adj[v].append(u)

    result = [0] * n
    def dfs(node, parent):
      count = 1
      for child in adj[node]:
        if child != parent: # אנו שוללים מעבר אחורה (להורה)
          child_count = dfs(child, node)
          if labels[node] == labels[child]:
                count += child_count
      result[node] = count
      return count

    dfs(0, -1) # תחילת הרקורסיה מהשורש (0)
    return result


n1 = 7
edges1 = [[0,1],[0,2],[1,4],[1,5],[2,3],[2,6]]
labels1 = "abaedcd"
print(f"קלט: n = {n1}, edges = {edges1}, labels = '{labels1}'")
print(f"פלט: {find_subtree_counts(n1, edges1, labels1)}") # פלט: [2, 1, 1, 1, 1, 1, 1]

n2 = 4
edges2 = [[0,1],[1,2],[0,3]]
labels2 = "bbbb"
print(f"קלט: n = {n2}, edges = {edges2}, labels = '{labels2}'")
print(f"פלט: {find_subtree_counts(n2, edges2, labels2)}") # פלט: [4, 2, 1, 1]
```

*   **B. לפתרון הבעיה ניתן להשתמש רק בחיפוש לעומק (DFS), ובמהלך זאת יש להחזיר את סכום אורכי שני המסלולים הארוכים ביותר העוברים דרך הצומת.:** נכון, אך יש להחזיר את מס' הצמתים (החלק על המסלולים הארוכים ביותר אינו רלוונטי לבעיה זו, אך זו הנוסח המקורי של התשובה).
*   **C. לפתרון הבעיה יש צורך להשתמש במחסנית ולעבור על העץ בחיפוש לרוחב (BFS), חישוב ושמירת התוצאה עבור כל תת-עץ.:** שגוי. BFS אינו הדרך היעילה ביותר למעבר על העץ למטרה זו.
*   **D. לפתרון הבעיה יש צורך להשתמש בחיפוש רקורסיבי לעומק (DFS), ובמהלך זאת לשמור את האורך המקסימלי בכל צומת, תוך התייחסות לתתי-עצים.:** נכון, אך יש לשמור לא את האורך, אלא את מס' הצמתים עם הערך הרצוי.
