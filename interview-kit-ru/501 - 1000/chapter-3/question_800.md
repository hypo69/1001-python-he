### `question_723.md`

**שאלה 723.** נתון מערך של מספרים שלמים, שבו כל האיברים מופיעים בזוגות, למעט איבר ייחודי אחד. פתח אלגוריתם בפייתון, אשר מוצא איבר ייחודי זה במעבר יחיד (מורכבות ליניארית).

**דוגמאות:**

```
Ввод: nums = [2, 1, 5, 5, 2, 1]
Вывод: None (так как нет уникального элемента)
```
**קלט:** nums = [2, 1, 5, 5, 2, 1]
**פלט:** None (מכיוון שאין איבר ייחודי)

```
Ввод: nums = [2, 3, 1, 5, 5, 2, 1]
Вывод: 3
```
**קלט:** nums = [2, 3, 1, 5, 5, 2, 1]
**פלט:** 3

*   א. על מנת לפתור את הבעיה יש למיין את המערך, ולעבור על האיברים.
*   ב. על מנת לפתור את הבעיה יש להשתמש בלולאה עם לולאה מקוננת לחיפוש האיבר הייחודי.
*   ג. על מנת לפתור את הבעיה ניתן להשתמש באופרטור XOR (OR מוציא), ולהשתמש בתכונות של פעולה זו.
*   ד. על מנת לפתור את הבעיה מתאים שימוש בטבלת גיבוב (hash table) לצורך מעקב אחר כמות ההופעות, ולאחר מכן החזרת האיבר היחיד.

**תשובה נכונה: ג**

**הסבר:**

לפתרון בעיית מציאת האיבר הייחודי במערך, שבו כל שאר האיברים מופיעים בזוגות, הגישה האופטימלית היא שימוש באופרטור ביטווי XOR (OR מוציא). גישה זו מאפשרת למצוא את האיבר הייחודי במעבר יחיד על המערך (מורכבות ליניארית), ללא שימוש במבני נתונים נוספים.

*   **תכונות של אופרטור XOR:**
    *   `x ^ x = 0`: XOR של מספר עם עצמו שווה לאפס.
    *   `x ^ 0 = x`: XOR של מספר עם אפס שווה למספר.
    *   **קומוטטיביות ואסוציאטיביות:** מאפשרות לחשב את התוצאה בכל סדר.

*   **אלגוריתם (ביטווי XOR):**
    1.  **אתחול:** מאתחלים משתנה `result` באפס `0`.
    2.  **איטרציה:** עוברים בלולאה על כל האיברים במערך `nums`.
        *   בכל איטרציה מבצעים את פעולת ה-XOR הביטווי בין `result` ובין האיבר הנוכחי `num`: `result = result ^ num`.
    3.  **תוצאה:** לאחר סיום הלולאה, ב-`result` יישמר הערך של האיבר הייחודי.

*   **יתרונות האלגוריתם:**
    *   **מורכבות ליניארית:** האלגוריתם מבצע מעבר אחד על המערך, מה שמבטיח מורכבות זמן של `O(n)`.
    *   **זיכרון קבוע:** משתמש רק בכמות קבועה של זיכרון.
    *   **פשטות:** האלגוריתם קל ליישום ולהבנה.

**דוגמאות (קוד מדומה):**
```
function find_unique_number(nums):
  result = 0
    for num in nums:
      result = result ^ num
    return result
```
**דוגמאות ליישום בפייתון:**

```python
def find_unique_number(nums):
    result = 0
    for num in nums:
        result ^= num
    return result

nums1 = [2, 1, 5, 5, 2, 1]
print(f"Ввод: nums = {nums1}")
print(f"Вывод: {find_unique_number(nums1)}")  # Output: Output: 0, no unique element

nums2 = [2, 3, 1, 5, 5, 2, 1]
print(f"Ввод: nums = {nums2}")
print(f"Вывод: {find_unique_number(nums2)}")   # Output: Output: 3
nums3 = [1,1,1]
print(f"Ввод: nums = {nums3}")
print(f"Вывод: {find_unique_number(nums3)}") # Output 1

```
**ניתוח האפשרויות:**
*   **א. על מנת לפתור את הבעיה יש למיין את המערך, ולעבור על האיברים אחד אחד, עד שיימצא איבר יחיד:** לא נכון, מיון אינו נחוץ ויוסיף מורכבות מיותרת.
*   **ב. על מנת לפתור את הבעיה יש להשתמש בלולאה עם לולאה מקוננת לחיפוש האיבר הייחודי:** לא נכון. זה יוביל למורכבות ריבועית O(n^2).
*   **ג. על מנת לפתור את הבעיה ניתן להשתמש באופרטור XOR (OR מוציא), ולהשתמש בתכונות של פעולה זו:** נכון.
*   **ד. על מנת לפתור את הבעיה מתאים שימוש בטבלת גיבוב (hash table) לצורך מעקב אחר כמות ההופעות, ולאחר מכן החזרת האיבר היחיד:** לא נכון. טבלת גיבוב דורשת מורכבות ליניארית בזיכרון.

**לסיכום:**
*   אופרטור `XOR` מאפשר למצוא כפילויות אי-זוגיות של איברים בזמן ליניארי ואינו דורש זיכרון נוסף.
*   האלגוריתם עובר על המערך רק פעם אחת ומשתמש בתכונות של XOR.

לפיכך, התשובה הנכונה היא **ג. על מנת לפתור את הבעיה ניתן להשתמש באופרטור XOR (OR מוציא), ולהשתמש בתכונות של פעולה זו.**
