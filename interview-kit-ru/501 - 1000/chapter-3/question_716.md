**שאלה 716.** פתחו אלגוריתם למציאת אורך התת-מחרוזת הארוכה ביותר המורכבת אך ורק מתווים שאינם חוזרים על עצמם במחרוזת נתונה.

**דוגמה:**

```
Ввод: s = "abcabcbb"
Вывод: 3
Объяснение: "abc" является самым длинным палиндромом.

Ввод: s = "bbbbb"
Вывод: 1
Объяснение: "b" является самым длинным палиндромом.
```

-   א. לפתרון הבעיה ניתן להשתמש באלגוריתם של מעבר מלא (brute force) על כל התת-מחרוזות ובדיקתן לאיתור יחידות (uniqueness).
-   ב. לפתרון הבעיה יש להשתמש באלגוריתם רקורסיבי, ולהחזיר את האורך המקסימלי בעת חזרה מהרקורסיה.
-   ג. לפתרון הבעיה יש להשתמש באלגוריתם תכנות דינמי (dynamic programming).
-   ד. לפתרון הבעיה ניתן להשתמש בגישת "חלון הזזה" (sliding window) וטבלת גיבוב (hash table) למעקב אחר אלמנטים שכבר נתקלו בהם, מה שמאפשר להשיג סיבוכיות זמן לינארית.

**תשובה נכונה: ד**

**הסבר:**

לפתרון הבעיה של מציאת אורך התת-מחרוזת הארוכה ביותר עם תווים שאינם חוזרים על עצמם, הגישה האופטימלית היא שיטת "חלון ההזזה" והשימוש בטבלת גיבוב (מילון) למעקב אחר התווים שכבר נתקלו בהם. גישה זו מאפשרת להשיג סיבוכיות זמן לינארית של `O(n)`.

*   **אלגוריתם (חלון הזזה עם טבלת גיבוב):**
    1.  **אתחול:**
        *   נוצר מילון ריק בשם `seen`, למעקב אחר האינדקס שבו תָו כבר נתקל בו.
        *   משתנה `max_length` מאותחל ל-0.
        *   משתנה `start` מאותחל ל-0 - אינדקס תחילת תת-המערך (חלון) הנוכחי.
    2.  **מעבר:** עוברים על כל התווים במחרוזת:
        *   **בדיקת הימצאות תו במילון:** אם התו הנוכחי `el` כבר קיים במילון `seen`, יש להשוות את `start` ואת `seen[el]+1`, מה שיאפשר לקבל את ערך ההתחלה הנכון עבור המחרוזת הנוכחית.
        *   **עדכון האינדקס:** שומרים את האינדקס `n` (של התו הנוכחי) במילון.
        *   **חישוב האורך המקסימלי:** מחשבים את אורך התת-מחרוזת הנוכחית, ומשווים אותו לאורך המקסימלי, ושומרים את הערך המקסימלי.
    3.  **תוצאה:** לאחר מעבר על כל המחרוזת מוחזר האורך המקסימלי.

*   **יתרונות האלגוריתם:**
    *   **סיבוכיות לינארית:** סיבוכיות הזמן היא `O(n)`, כאשר `n` הוא אורך המחרוזת.
    *   **זיכרון קבוע:** משתמש בכמות קבועה של זיכרון נוסף.
    *   **יעילות:** מספק מציאת פתרון מהירה.

**דוגמאות (פסאודו-קוד):**

**דוגמאות מימוש ב-Python:**
```python
def max_subs_len(s):
    seen = {}
    max_length = 0
    start = 0

    for n, el in enumerate(s):
        if el in seen:
            start = max(start, seen[el] + 1)
        seen[el] = n
        max_length = max(max_length, n - start + 1)

    return max_length
s1 = "abcabcbb"
print(f"קלט: s = '{s1}'")
print(f"פלט: {max_subs_len(s1)}")  # יציג: פלט: 3

s2 = "bbbbb"
print(f"קלט: s = '{s2}'")
print(f"פלט: {max_subs_len(s2)}")   # יציג: פלט: 1
s3 = "pwwkew"
print(f"קלט: s = '{s3}'")
print(f"פלט: {max_subs_len(s3)}")   # יציג: פלט: 3

```

**ניתוח האפשרויות:**
*   **א. לפתרון הבעיה יש לעבור על כל התת-מחרוזות האפשריות, ובמקביל לבדוק אותן לאיתור יחידות (uniqueness).:** לא נכון. גישה זו דורשת סיבוכיות של `O(n^3)`.
*   **ב. לפתרון הבעיה יש להשתמש באלגוריתם רקורסיבי, ולהחזיר את האורך המקסימלי בעת חזרה מהרקורסיה.:** לא נכון. רקורסיה אינה אופטימלית.
*   **ג. לפתרון הבעיה יש להשתמש באלגוריתם תכנות דינמי (dynamic programming).:** לא נכון. תכנות דינמי אינו הגישה האופטימלית.
*   **ד. לפתרון הבעיה ניתן להשתמש בגישת "חלון הזזה" (sliding window) וטבלת גיבוב (hash table) למעקב אחר אלמנטים שכבר נתקלו בהם, מה שמאפשר להשיג סיבוכיות זמן לינארית.:** נכון.
