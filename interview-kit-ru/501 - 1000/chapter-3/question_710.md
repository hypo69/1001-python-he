**שאלה 710.** נתונה מחרוזת `s`. פתח/י אלגוריתם למציאת והחזרת הפלינדרום הארוך ביותר במחרוזת.

*   פלינדרום הוא מחרוזת הנקראת באופן זהה משמאל לימין ומימין לשמאל.

**דוגמאות:**

```
קלט: s = "babad"
פלט: "bab"
הסבר: "aba" גם כן מהווה תשובה

קלט: s = "cbbd"
פלט: "bb"
```

-   א. כדי לפתור את הבעיה יש לעבור על כל תת-המחרוזות האפשריות, ובנוסף לבדוק כל אחת מהן האם היא פלינדרום.
-   ב. כדי לפתור את הבעיה יש להשתמש באלגוריתם מיון מהיר ולהשוות את תת-המחרוזות הממוינות.
-   ג. כדי לפתור את הבעיה יש להשתמש באלגוריתם תכנות דינמי, המוצא את כל הפלינדרומים, ושומר את הארוך ביותר.
-   ד. כדי לפתור את הבעיה, יש לבדוק כל עמדה במחרוזת ולהרחיב את הפלינדרום שמאלה וימינה ככל האפשר, ולשמור את הארוך ביותר.

**תשובה נכונה: ד**

**הסבר:**

למציאת תת-מחרוזת הפלינדרום הארוכה ביותר במחרוזת, הגישה האופטימלית תהיה שימוש בשיטה הסוקרת את כל הפלינדרומים האפשריים מכל עמדה, תוך שימוש בשיטת הרחבה, שלה סיבוכיות זמן ריבועית ואינה דורשת זיכרון נוסף.

*   **אלגוריתם (עם מרכוז והרחבה):**
    1.  **אתחול:** מגדירים את תת-מחרוזת ההתחלתית הארוכה ביותר כמחרוזת ריקה.
    2.  **איטרציה על המחרוזת:** עוברים על כל סימן במחרוזת.
        *   **מרכז כתו בודד:** מגדירים את התו הנוכחי כמרכז הפלינדרום, ובודקים את הפלינדרום המקסימלי באמצעות הרחבה לשני הכיוונים.
        *   **מרכז בין שני תווים:** מגדירים שהמרכז של הפלינדרום נמצא בין התו הנוכחי לתו הבא, ובאופן דומה מרחיבים לשני הכיוונים.
        *   **השוואה עם המקסימום:** אם הפלינדרום החדש ארוך יותר מהפלינדרום המקסימלי הנוכחי, אז שומרים את החדש.
    3.  **החזרת התוצאה:** לאחר מעבר על כל האלמנטים מחזירים את הפלינדרום הארוך ביותר.

*   **יתרונות האלגוריתם:**
    *   **סיבוכיות ריבועית:** לאלגוריתם סיבוכיות זמן של `O(n^2)`, שהיא הפתרון האופטימלי ביותר.
    *   **יעילות:** בודק רק את הצירופים ההכרחיים.
    *   **פשטות:** האלגוריתם קל ליישום.
    *   **זיכרון קבוע:** אינו דורש זיכרון נוסף.

**דוגמאות (פסאודו-קוד):**
```
function longest_palindrome(s):
    longest_palindrome_string = ""
    for i in range (length(s)):
       # בודקים פלינדרום, מרכז בנקודה i
        l1 = expand_around_center(s, i, i)
       # בודקים פלינדרום, מרכז בין i ל- i+1
       l2 = expand_around_center(s, i, i+1)
       longest_palindrome_string =  max(longest_palindrome_string, l1, l2)
     return longest_palindrome_string
 function expand_around_center(s, left, right):
   while left >=0 and right < len(s) and s[left] == s[right]:
     left = left-1
     right=right+1
   return s[left+1:right]
```

**דוגמאות ליישום ב-Python:**

```python
def longest_palindrome(s):
    longest_palindrome_string = ""

    def expand_around_center(s, left, right):
        while left >= 0 and right < len(s) and s[left] == s[right]:
            left -= 1
            right += 1
        return s[left+1:right]

    for i in range(len(s)):
        palindrome1 = expand_around_center(s, i, i)
        palindrome2 = expand_around_center(s, i, i+1)
        longest_palindrome_string = max(longest_palindrome_string, palindrome1, palindrome2, key=len)

    return longest_palindrome_string
print(f"קלט: s = 'babad'")
print(f"פלט: '{longest_palindrome('babad')}'") # יודפס: פלט: 'bab'

print(f"קלט: s = 'cbbd'")
print(f"פלט: '{longest_palindrome('cbbd')}'") # יודפס: פלט: 'bb'

print(f"קלט: s = 'testpalindromemordnilaptest'")
print(f"פלט: '{longest_palindrome('testpalindromemordnilaptest')}'") # יודפס: פלט: 'testpalindromemordnilaptest'

```

*   **ב. כדי לפתור את הבעיה יש להשתמש באלגוריתם מיון מהיר ולהשוות את תת-המחרוזות הממוינות.:** שגוי. מיון מהיר אינו יסייע בפתרון הבעיה.
