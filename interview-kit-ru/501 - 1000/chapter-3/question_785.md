**שאלה 785.** נתון מספר `n`, המייצג את מספר המועמדים לשתי משלחות חלל למאדים. כמו כן נתונה מטריצה `conflicts` בגודל `n x n`, כאשר `conflicts[i][j] = "+"`, אם השתתפות המועמדים `i` ו-`j` באותה משלחת אינה רצויה. יש לפתח אלגוריתם ב-Python שמפצל את המועמדים לשתי קבוצות כך שלא יהיו קונפליקטים בתוך אף קבוצה. אם הדבר אינו אפשרי, יש להדפיס `"No solution"`. במקרה שקיימות מספר אפשרויות חלוקה עם הבדל מינימלי בגודל הקבוצות, יש להציג את הפתרון שבו הקבוצה הראשונה קטנה יותר לקסיקוגרפית.

*   אם ישנם מספר פתרונות חלוקה בגודל קבוצות זהה - יש לבחור את זה שבו הקבוצה הראשונה קטנה יותר לקסיקוגרפית.
*   מספור המועמדים מתחיל מ-`1`.

- ב. לפתרון הבעיה יש לבצע מעבר על כל הקומבינציות האפשריות לחלוקה ל-2 קבוצות, לבדוק קונפליקט ולבחור את הטובה ביותר.

- ד. לפתרון הבעיה יש ליישם חיפוש בינארי תוך שימוש במערך מצב הקבוצות.

**תשובה נכונה: ג**

**הסבר:**

לפתרון בעיית חלוקת המועמדים לשתי קבוצות עם מינימום קונפליקטים והבדל מינימלי בגודל הקבוצות, הדרך האופטימלית תהיה שימוש באלגוריתם בקטראקינג. בקטראקינג מאפשר לבדוק את כל האפשרויות האפשריות לחלוקה, ומבטיח מציאת פתרון אופטימלי תחת האילוצים הנתונים.

*   **אלגוריתם (רקורסיבי עם בקטראקינג):**
    1.  **אתחול:**
        *   `best_group1` (רשימה) - לאחסון הקבוצה הראשונה הטובה ביותר, מאותחל ל-`null` (כדי לבדוק שעדיין אין פתרון טוב יותר).
        *   `min_diff` - לאחסון ההבדל המינימלי בין גודלי הקבוצות, מאותחל לאינסוף (infinity).
    2.  **פונקציה רקורסיבית:** ניצור פונקציה רקורסיבית שמקבלת:
        *   `index` - האינדקס הנוכחי של המועמד.
        *   `group1` - מועמדים בקבוצה הראשונה.
        *   `group2` - מועמדים בקבוצה השנייה.
    3.  **מקרה בסיס:** אם כל המועמדים עברו עיבוד, יש לחשב את ההבדל בין מספר המועמדים בקבוצות ולבדוק האם הוא קטן מ-`min_diff`:
        *   אם קטן - מצאנו פתרון טוב יותר.
        *   אם שווה, יש לבדוק לקסיקוגרפית, ואם הפתרון החדש קטן יותר, יש לעדכן את `best_group1`.
    4.  **קריאה רקורסיבית:** לכל מועמד בלולאה:
        *   תחילה, ננסה להוסיף את המועמד ל-`group1`, ונקרא רקורסיבית לפונקציה. (יש לוודא שהוא אינו בקונפליקט עם אף מועמד אחר ב-`group1`).
        *   לאחר החזרה, נסיר את המועמד מ-`group1` (כדי שניתן יהיה לנסות אפשרויות אחרות).
        *   ננסה להוסיף את המועמד ל-`group2`, אם לא היה קונפליקט, ונקרא רקורסיבית לפונקציה.
        *   לאחר החזרה, נסיר את המועמד מ-`group2`.
    5.  **קריאה ראשונית:** יש לקרוא לפונקציה הרקורסיבית עם פרמטרים התחלתיים (אינדקס `0`, רשימות `group1` ו-`group2` ריקות).
    6.  **תוצאה:** לאחר סיום הרקורסיה, יש להחזיר את `best_group1` ממוין, או `"No solution"` אם `best_group1` נשאר `null`.

*   **יתרונות האלגוריתם:**
    *   **מעבר על כל האפשרויות:** בקטראקינג מאפשר לבדוק את כל הקומבינציות האפשריות של קבוצות, ומבטיח מציאת פתרון.
    *   **גיזום:** האלגוריתם גוזם (מסנן) קומבינציות לא חוקיות.
    *   **אופטימליות:** האלגוריתם מבטיח בחירת פתרון עם ההבדל המינימלי בגודל הקבוצות ועם הסדר הלקסיקוגרפי המינימלי, אם ישנן מספר אפשרויות.

**דוגמאות (פסאודו-קוד):**

```
function find_best_teams(n, conflicts):
    best_group1 = null
    min_diff = infinity;
    function backtrack(index, group1, group2):
        if index == n
            diff = abs(length(group1) - length(group2))
            if diff < min_diff
                min_diff = diff;
                best_group1 = group1;
            elif diff = min_diff
                best_group1 = min(group1, best_group1)
            return
        current_player = index+1 # player index starts from 1 not from zero
        can_add_to_group1 = True
        for player_in_group1 in group1:
            if conflicts[player_in_group1-1][current_player-1] == '+':
                can_add_to_group1 = False
        if can_add_to_group1:
            group1.append(current_player)
            backtrack(index + 1, group1, group2)
            group1.pop()

        can_add_to_group2 = True
        for player_in_group2 in group2:
            if conflicts[player_in_group2-1][current_player-1] == '+':
                can_add_to_group2 = False
        if can_add_to_group2:
            group2.append(current_player)
            backtrack(index + 1, group1, group2)
            group2.pop()

    backtrack(0, [], [])
    return best_group1 if best_group1 is not null else "No solution" # Adjusted pseudocode check for null
    # Original pseudocode comment: return if best_group1 is null - then No solution, or sort and return it.

```

```python
def find_best_teams(n, conflicts):
    best_group1 = None
    min_diff = float('inf')
    def backtrack(index, group1, group2):
        nonlocal min_diff, best_group1
        if index == n:
           diff = abs(len(group1) - len(group2))
           if diff < min_diff:
               min_diff = diff
               best_group1 = sorted(group1) # שמירת רשימה ממוינת עבור השוואה לקסיקוגרפית עתידית והפלט הסופי
           elif diff == min_diff:
               # יש לבדוק אם group1 הנוכחית (ממוינת לצורך השוואה) קטנה יותר מ-best_group1 השמור
               # יש לוודא ש-best_group1 כבר מאותחל כדי למנוע שגיאת השוואה עם None
               current_sorted_group1 = sorted(group1)
               if best_group1 is None or current_sorted_group1 < best_group1:
                best_group1 = current_sorted_group1
           return

        current_player = index+1 # אינדקס השחקן מתחיל מ-1 ולא מ-0

        # נסה להוסיף לקבוצה 1
        can_add_to_group1 = True

             if conflicts[player_in_group1-1][current_player-1] == '+':
                can_add_to_group1 = False
                break # אין צורך להמשיך אם נמצא קונפליקט
```
