### `question_687.md`

**שאלה 687.** נתון מוצר תוכנה עם `n` גרסאות, ממוספרות מ-1 עד `n`. סופק לכם API בשם `bool isBadVersion(version)`, המחזיר `True` אם הגרסה `version` פגומה, ו-`False` אחרת. ידוע שכל הגרסאות שאחרי גרסה פגומה הן גם פגומות. פתחו אלגוריתם למציאת הגרסה הפגומה הראשונה באמצעות מספר מינימלי של קריאות ל-API.

**דוגמאות:**

```
קלט: n = 5, bad = 4
פלט: 4
הסבר:
call isBadVersion(3) -> False
call isBadVersion(5) -> True
call isBadVersion(4) -> True
4 היא הגרסה הפגומה הראשונה.

קלט: n = 10, bad = 6
פלט: 6
```

-   א. לפתרון הבעיה יש להשתמש בחיפוש ליניארי, על ידי בדיקה סדרתית של כל גרסה.
-   ב. לפתרון הבעיה יש להשתמש בלולאת while ולעבור על כל הערכים בסדר הפוך.
-   ג. לפתרון הבעיה יש להשתמש בחיפוש בינארי למציאת הגרסה הפגומה הראשונה, מכיוון שהחיפוש מתבצע ברצף ממוין.
-   ד. לפתרון הבעיה ניתן להשתמש באלגוריתם חמדן, שתמיד בוחר את הערך הפגום הראשון שנמצא.

**תשובה נכונה: ג**

**הסבר:**

לפתרון בעיית מציאת הגרסה הפגומה הראשונה, ברשימה ממוינת של גרסאות עם מספר מינימלי של בדיקות, יש ליישם אלגוריתם חיפוש בינארי, המאפשר לפסול אפשרויות מיותרות בכל שלב, ובכך להגיע לסיבוכיות לוגריתמית של O(log n).

*   **אלגוריתם (חיפוש בינארי):**
    1.  **אתחול:** מגדירים את `left` ל-1 ואת `right` ל-`n` (מספר הגרסאות המרבי).
    2.  **איטרציה (חיפוש בינארי):** כל עוד הגבול השמאלי `left` קטן או שווה לגבול הימני `right`:
        *   מוצאים את נקודת האמצע: `mid = left + (right - left) // 2`
        *   **בדיקה:** קוראים ל-`isBadVersion(mid)`:
            *   אם `isBadVersion(mid)` מחזיר `True`, פירוש הדבר ש-`mid` היא "גרסה פגומה" פוטנציאלית, ויש לצמצם את טווח החיפוש לחצי השמאלי `right = mid - 1`, תוך שמירת `mid` כתשובה פוטנציאלית.
            *   אם `isBadVersion(mid)` מחזיר `False`, פירוש הדבר ש-`mid` היא "גרסה תקינה", ויש לצמצם את טווח החיפוש לחצי הימני `left = mid + 1`.
    3.  **תוצאה:** לאחר לולאת `while`, מחזירים את ה-`index` האחרון שנמצא או את `left`.

*   **יתרונות האלגוריתם:**
    *   **סיבוכיות לוגריתמית:** האלגוריתם מספק סיבוכיות זמן של O(log n), מכיוון שבכל שלב אנו מקטינים את טווח החיפוש בחצי.
    *   **מספר בדיקות מינימלי:** משתמש במספר הנמוך ביותר של קריאות ל-`isBadVersion()`.
    *   **אופטימליות:** מאפשר למצוא את הגרסה ה"פגומה" הראשונה ברצף ממוין של גרסאות.

**דוגמאות (פסאודו קוד):**
```
function find_first_bad_version(n):
   left = 1
   right = n
   result = -1
   while left <= right:
     mid = left + (right - left) // 2
     if isBadVersion(mid):
       result = mid;
       right = mid - 1 # go to left half
     else:
       left = mid + 1  # go to right half
   return result
```

**דוגמאות מימוש בפייתון:**

```python
def isBadVersion(version, bad): # פונקציית עזר לבדיקת גרסה "פגומה".
  return version >= bad

def find_first_bad_version(n, bad):
    left = 1
    right = n
    result = -1
    while left <= right:
        mid = left + (right - left) // 2
        if isBadVersion(mid, bad):
            result = mid
            right = mid - 1
        else:
           left = mid + 1
    return result

n1 = 5
bad1 = 4
print(f"קלט: n = {n1}, bad = {bad1}")
print(f"פלט: {find_first_bad_version(n1, bad1)}") # יוצג: פלט: 4

n2 = 10
bad2 = 6
print(f"קלט: n = {n2}, bad = {bad2}")
print(f"פלט: {find_first_bad_version(n2, bad2)}") # יוצג: פלט: 6

```
