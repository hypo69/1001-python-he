**שאלה 774.** נתון מערך של מספרים שלמים `nums`, הממוין בסדר עולה, ולאחר מכן הוזז ביחס לאיבר ציר לא ידוע, כלומר ייתכן שהמערך יהיה במבנה `[nums[k], nums[k+1], ..., nums[n-1], nums[0], nums[1], ..., nums[k-1]]`. פתח אלגוריתם בפייתון למציאת האינדקס של מספר שלם נתון `target` במערך `nums` בזמן O(log n), או החזר `-1` אם האיבר אינו קיים.

**דוגמאות:**

```
קלט: nums = [4,5,6,7,0,1,2], target = 0
פלט: 4
הסבר: האיבר 0 נמצא באינדקס 4.

קלט: nums = [4,5,6,7,0,1,2], target = 3
פלט: -1
הסבר: האיבר 3 לא נמצא.
```

-   A. לפתרון הבעיה יש להשתמש בחיפוש לינארי, ולבדוק כל איבר בתורו.
-   B. לפתרון הבעיה יש להשתמש באלגוריתם מיון, ולאחר מכן ליישם חיפוש בינארי.
-   C. לפתרון הבעיה יש להשתמש בחיפוש בינארי, אך תוך התחשבות בכך שהמערך ממוין רק חלקית ולבדוק על פי איזה עיקרון ממוין תת-המערך.
-  D. לפתרון הבעיה מתאימה רק המתודה `index()`, המחזירה מיד את הערך אם הוא קיים, ומחזירה שגיאה אם הערך אינו קיים.

**תשובה נכונה: C**

**הסבר:**

לפתרון בעיית מציאת איבר במערך ממוין חלקית עם סיבוכיות זמן `O(log n)`, המתאים ביותר הוא התאמה של אלגוריתם החיפוש הבינארי, תוך התחשבות באפשרות קיום הזזה במערך.

*   **אלגוריתם (חיפוש בינארי מותאם):**
    1.  **אתחול גבולות:** קובעים מצביע לגבול השמאלי `left = 0` ומצביע לגבול הימני `right` על סוף המערך `len(nums)-1`.
    2.  **חיפוש בינארי:** כל עוד הגבול השמאלי אינו עולה על הגבול הימני:
         *   מחשבים את אמצע המערך: `mid = left + (right-left) // 2`
           *   **ערך מטרה:** אם הערך באמצע `nums[mid]` שווה לערך המטרה `target`, אז מחזירים את האינדקס `mid`.
           *   **החלק הימני ממוין:** בודקים האם המחצית הימנית של המערך ממוינת `nums[mid] <= nums[right]`.
              *  אם כן:
                   *  אם `target` נמצא בחצי הימני, אז עוברים לחצי הימני `left = mid + 1`.
                   *  אחרת - עוברים לחצי השמאלי `right = mid - 1`.
        * **החלק השמאלי ממוין:** אם החלק הימני אינו ממוין, אז החלק השמאלי בוודאות ממוין.
             * אם `target` נמצא בחצי השמאלי, אז עוברים לחצי השמאלי `right = mid - 1`.
          *   אחרת - עוברים לחצי הימני `left = mid + 1`.
    3. **תוצאה:** לאחר סיום הלולאה, אם האיבר לא נמצא, אז מחזירים `-1`.

*   **יתרונות האלגוריתם:**
    *   **סיבוכיות לוגריתמית:** מבטיח סיבוכיות זמן O(log n), שכן גודל אזור החיפוש מצטמצם פי 2 בכל שלב.
    *  **טיפול בהזזה:** מתחשב במיון החלקי של המערך.

**דוגמאות (פסאודו-קוד):**
```
function search_rotated_array(nums, target):
     left = 0
    right = length(nums) - 1

    while left <= right:
        mid = left + (right - left) // 2
        if nums[mid] == target
             return mid;
        if nums[mid] <= nums[right]:
          if  target >= nums[mid] and target <= nums[right]:
                left = mid + 1;
            else:
                right = mid - 1;
       else:
            if  target >= nums[left] and target < nums[mid]:
                right = mid - 1;
           else
               left = mid + 1
    return -1;
```
**דוגמאות ליישום בפייתון:**

```python
def search_rotated_array(nums, target):
    left = 0
    right = len(nums) - 1

    while left <= right:
        mid = left + (right - left) // 2
        if nums[mid] == target:
            return mid
        if nums[mid] <= nums[right]:
           if target >= nums[mid] and target <= nums[right]:
               left = mid + 1
           else:
               right = mid - 1

        else:
            if target >= nums[left] and target < nums[mid]:
               right = mid - 1
            else:
                left = mid + 1
    return -1

nums1 = [4, 5, 6, 7, 0, 1, 2]
target1 = 0
print(f"קלט: nums = {nums1}, target = {target1}")
print(f"פלט: {search_rotated_array(nums1, target1)}")   # יציג: פלט: 4

nums2 = [4,5,6,7,0,1,2]
target2 = 3
print(f"קלט: nums = {nums2}, target = {target2}")
print(f"פלט: {search_rotated_array(nums2, target2)}") #  יציג: פלט: -1

nums3 = [1,2,3]
target3 = 2
print(f"קלט: nums = {nums3}, target = {target3}")
print(f"פלט: {search_rotated_array(nums3, target3)}")   # יציג: פלט: 1

nums4 = [5, 1, 2, 3, 4]
target4 = 3
print(f"קלט: nums = {nums4}, target = {target4}")
print(f"פלט: {search_rotated_array(nums4, target4)}") # יציג: פלט: 3
```
