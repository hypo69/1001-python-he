### `question_006.md` (OCP - יישום)

**שאלה 006.** ברשותך מערכת רישום יומנים (logging) שכותבת הודעות לקובץ. בתחילה, היא תומכת רק בפורמט טקסטואלי. מהי הדרך *הטובה ביותר* להוסיף תמיכה בכתיבת יומנים בפורמט JSON, תוך *שמירה* על עקרון הפתיחות/סגירות (OCP)?

A. להוסיף לקלאס הרישום הקיים מבנה תנאי (`if/else`) כדי לקבוע את הפורמט ולכתוב לקובץ בפורמט הרצוי.
B. ליצור תת-קלאס שידרוס את מתודת הכתיבה לקובץ ויוסיף לוגיקה עבור פורמט JSON.
C. ליצור קלאס חדש המממש ממשק `Logger`, ולהשתמש בקלאס זה לכתיבת יומנים בפורמט JSON, מבלי לשנות את קלאס רישום היומנים הטקסטואלי הקיים.
D. להשתמש בדקורטור להוספת פונקציונליות של רישום יומנים ב-JSON לקלאס הקיים.

**תשובה נכונה: C**

**הסבר:**

כדי לשמור על עקרון הפתיחות/סגירות (OCP), הדרך הטובה ביותר להוסיף פונקציונליות חדשה היא מבלי לשנות קוד קיים. במקרה זה, משמעות הדבר היא שאין לשנות את הקלאס הקיים שכותב יומנים בפורמט טקסטואלי.

*   **אפשרות A (שגויה):** שינוי הקלאס הקיים באמצעות אופרטורים תנאיים מפר את עקרון ה-OCP.
*   **אפשרות B (שגויה):** דריסת מתודות, אף שהיא מהווה הרחבה, עדיין משנה את התנהגות קלאס הבסיס.
*   **אפשרות C (נכונה):** יצירת קלאס חדש לרישום יומנים ב-JSON, המממש ממשק משותף, מאפשרת להרחיב את פונקציונליות מערכת רישום היומנים ללא שינוי בקוד הקיים.
*   **אפשרות D (שגויה):** שימוש בדקורטור יכול להוות אפשרות, אך במקרה זה הוספת קלאס חדש עם ממשק מדגימה באופן ברור יותר את שמירת עקרון ה-OCP.

**דוגמה:**

```python
from abc import ABC, abstractmethod

class Logger(ABC):
    @abstractmethod
    def log(self, message):
        pass

class TextLogger(Logger):
    def log(self, message):
        with open("log.txt", "a") as f:
            f.write(message + "\n")

class JsonLogger(Logger):
    import json
    def log(self, message):
        with open("log.json", "a") as f:
            json.dump({"message": message}, f)
            f.write("\n")
```
**כתוצאה מכך:**

יצירת קלאס חדש לרישום יומנים ב-JSON, המממש את הממשק המשותף `Logger`, היא הדרך הטובה ביותר לשמור על עקרון הפתיחות/סגירות (OCP).

לפיכך, אפשרות C היא הנכונה.
