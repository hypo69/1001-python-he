# ייבוא מודולים ב-Python

ייבוא ספריות ב-Python מאפשר שימוש במודולים ופונקציות חיצוניים או עצמיים, המוגדרים בקבצים או בספריות אחרות. הדבר מסייע בארגון הקוד, שיפור קריאותו ומניעת כפילויות.

### מהם מודולים?
מודול ב-Python הוא פשוט קובץ בעל סיומת `.py`, המכיל קוד (פונקציות, מחלקות, משתנים וכו'). כאשר ברצונך להשתמש בקוד ממודול אחר, יש לייבא אותו לקובץ הנוכחי.

### מודולים חיצוניים ופנימיים
1.  **מודולים חיצוניים** – אלו מודולים שאינם חלק מהספרייה הסטנדרטית של Python ואשר יש להתקין בנפרד. לדוגמה, מודולים לעבודה עם שרתי ווב, עיבוד נתונים או למידת מכונה, כגון `numpy`, `requests` ואחרים.

2.  **מודולים פנימיים** – אלו מודולים שכבר כלולים בספרייה הסטנדרטית של Python. לדוגמה, מודולים לעבודה עם קבצים, זמן, פעולות מתמטיות: `os`, `math`, `datetime`.

### התקנת מודולים חיצוניים באמצעות pip
כדי להתקין מודול חיצוני, נשתמש בפקודה `pip` – זהו כלי לניהול חבילות Python. לדוגמה, כדי להתקין את ספריית `requests`, יש להריץ את הפקודה הבאה בטרמינל:
```bash
pip install requests
```
פקודה זו תוריד ותתקין את הספרייה בפרויקט שלך.

### ייבוא מודולים
כאשר ברצונך להשתמש במודול ב-Python, יש לייבא אותו:
1.  **ייבוא המודול כולו**:
    ```python
    import os
    ```
    לאחר מכן תוכל להשתמש בכל הפונקציות והמשתנים מהמודול, למשל:
    ```python
    os.listdir()  # רשימת קבצים בספרייה
    ```

2.  **ייבוא אלמנטים ספציפיים מהמודול**:
    ```python
    from math import sqrt
    ```
    כעת ניתן להשתמש בפונקציה `sqrt` ללא צורך לפנות למודול `math`.

3.  **ייבוא עם כינוי**:
    ```python
    import pandas as pd
    ```
    במקרה זה, כדי להשתמש בפונקציות של ספריית `pandas`, תכתוב `pd`, ולא את השם המלא `pandas`.

### ייבוא מודולים עצמיים
אם אתה כותב מספר קבצים באותו פרויקט, ניתן ליצור מודולים עצמיים. לדוגמה, אם יש לך קובץ `utils.py` עם פונקציות שימושיות, תוכל לייבא אותו לקבצים אחרים כך:
```python
import utils
```
או לייבא פונקציה ספציפית:
```python
from utils import my_function
```

### מדוע ליצור מודול אחד ולייבא אותו לחלקים אחרים בפרויקט?
1.  **ארגון הקוד**: חלוקת הלוגיקה למודולים שונים מסייעת למנוע אי-סדר בפרויקט גדול. לדוגמה, ניתן ליצור מודול לעבודה עם נתונים, אחר לעיבוד בקשות, ושלישי עבור הממשק.

2.  **שימוש חוזר בקוד**: כאשר הלוגיקה מחולקת למודולים, ניתן להשתמש בה בחלקים שונים של הפרויקט. הדבר מפחית כפילויות בקוד ומקל על התחזוקה.

3.  **קריאות ותחזוקה**: כאשר הקוד מחולק למודולים, מפתחים אחרים (או אתה בעצמך בעתיד) יוכלו להבין ולתחזק את הפרויקט בקלות רבה יותר.

דוגמה:
```python
# קובץ utils.py
def greet(name):
    return f"Hello, {name}!"

# קובץ main.py
from utils import greet

print(greet("Alice"))
```

בדוגמה זו, הפונקציה `greet` מוגדרת במודול אחד (`utils.py`), אך נעשה בה שימוש במודול אחר (`main.py`), מה שמשפר את המבנה והופך את הקוד למודולרי יותר ונוח לשינוי.
