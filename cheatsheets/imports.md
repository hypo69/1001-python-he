# ייבוא מודולים בפייתון

ייבוא ספריות בפייתון מאפשר להשתמש במודולים ופונקציות חיצוניים או מודולים עצמיים, המוגדרים בקבצים או ספריות אחרים. פעולה זו מסייעת בארגון הקוד, שיפור קריאותו ומניעת כפילויות.

### מהם מודולים?
מודול בפייתון הוא פשוט קובץ עם סיומת `.py` המכיל קוד (פונקציות, מחלקות, משתנים וכו'). כאשר רוצים להשתמש בקוד ממודול אחר, יש לייבא אותו לקובץ הנוכחי.

### מודולים חיצוניים ופנימיים
1.  **מודולים חיצוניים** – אלו מודולים שאינם חלק מהספרייה הסטנדרטית של פייתון ויש להתקינם בנפרד. לדוגמה, מודולים לעבודה עם שרתי אינטרנט, עיבוד נתונים או למידת מכונה, כגון `numpy`, `requests` ואחרים.

2.  **מודולים פנימיים** – אלו מודולים שכבר כלולים בספרייה הסטנדרטית של פייתון. לדוגמה, מודולים לעבודה עם קבצים, זמן, פעולות מתמטיות: `os`, `math`, `datetime`.

### התקנת מודולים חיצוניים באמצעות pip
כדי להתקין מודול חיצוני, משתמשים בפקודה `pip` – כלי לניהול חבילות פייתון. לדוגמה, כדי להתקין את ספריית `requests`, יש לבצע את הפקודה הבאה בטרמינל:
```bash
pip install requests
```
פקודה זו תוריד ותתקין את הספרייה בפרויקט שלך.

### ייבוא מודולים
כאשר רוצים להשתמש במודול בפייתון, יש לייבא אותו:
1.  **ייבוא מודול שלם**:
    ```python
    import os
    ```
    לאחר מכן, ניתן להשתמש בכל הפונקציות והמשתנים מהמודול, לדוגמה:
    ```python
    os.listdir()  # רשימת קבצים בתיקייה
    ```

2.  **ייבוא אלמנטים ספציפיים מתוך מודול**:
    ```python
    from math import sqrt
    ```
    כעת ניתן להשתמש בפונקציה `sqrt` ללא צורך לפנות למודול `math`.

3.  **ייבוא עם כינוי (alias)**:
    ```python
    import pandas as pd
    ```
    במקרה זה, כדי להשתמש בפונקציות של ספריית `pandas`, תכתוב `pd` במקום את השם המלא `pandas`.

### ייבוא מודולים עצמיים
אם אתה כותב מספר קבצים בפרויקט אחד, ניתן ליצור מודולים משלך. לדוגמה, אם יש לך קובץ `utils.py` עם פונקציות שימושיות, אתה יכול לייבא אותו לקבצים אחרים כך:
```python
import utils
```
או לייבא פונקציה ספציפית:
```python
from utils import my_function
```

### למה כדאי ליצור מודול אחד ולייבא אותו לחלקים אחרים בפרויקט?
1.  **ארגון קוד**: הפרדת הלוגיקה למודולים שונים מסייעת למנוע בלגן בפרויקט גדול. לדוגמה, ניתן ליצור מודול לעבודה עם נתונים, מודול אחר לעיבוד בקשות, ושלישי לממשק המשתמש.

2.  **שימוש חוזר בקוד**: כאשר הלוגיקה מחולקת למודולים, ניתן להשתמש בה בחלקים שונים של הפרויקט. זה מקטין כפילויות קוד ומקל על תחזוקה.

3.  **קריאות ותחזוקה**: כאשר הקוד מחולק למודולים, מפתחים אחרים (או אתה בעתיד) יוכלו להבין ולתחזק את הפרויקט בקלות רבה יותר.

דוגמה:
```python
# utils.py
def greet(name):
    return f"Hello, {name}!"
    
# main.py
from utils import greet

print(greet("Alice"))
```

בדוגמה זו, הפונקציה `greet` מוגדרת במודול אחד (`utils.py`), אך נעשה בה שימוש במודול אחר (`main.py`), מה שמשפר את המבנה והופך את הקוד למודולרי יותר ונוח לשינויים.
