# מחלקות ב-python
```

מחלקות הן אחד המנגנונים העיקריים של תכנות מונחה עצמים (OOP) ב-Python. ניתן לחשוב על מחלקה כ"תבנית" או "שרטוט" ליצירת אובייקטים שיש להם תכונות (נתונים) ומתודות (פונקציות). אובייקטים שנוצרים על בסיס מחלקה נקראים מופעים (אינסטנסים) של המחלקה. מחלקות מאפשרות מבנה קוד, שיפור השימוש החוזר בו והקלת התחזוקה שלו.

### מבנה מחלקה

```python
class ClassName:
    # תכונות מחלקה
    def __init__(self, param1, param2):
        # בנאי (מאתחל) מחלקה
        self.param1 = param1
        self.param2 = param2

    # מתודות מחלקה
    def method(self):
        return f'{self.param1} ו {self.param2}'
```

1.  **בנאי** (`__init__`):
    הבנאי `__init__` הוא מתודה מיוחדת הנקראת אוטומטית בעת יצירת מופע חדש. הוא משמש לאתחול תכונות המופע.

    *   `self`: פרמטר המהווה הפניה למופע הנוכחי של המחלקה. ב-Python, חובה להעביר אותו כפרמטר הראשון בכל מתודות המחלקה (הוא אינו מועבר בעת קריאה למתודה).
    *   תכונות, כמו `param1` ו-`param2`, מוקצות למופע דרך `self`. תכונות אלו ניתנות לשימוש על ידי מתודות אחרות של המחלקה.

2.  **תכונות מחלקה**:
    תכונות הן משתנים השייכים למופעים של מחלקה זו. הן מוגדרות בתוך הבנאי (`__init__`) וניתן לגשת אליהן באמצעות הפניה למופע.

3.  **מתודות מחלקה**:
    מתודות הן פונקציות שיכולות לבצע מניפולציות על תכונות המופע. מתודות יכולות להשתמש בנתוני המופע, לשנות אותם או לבצע פעולות אחרות.

### יצירת מופע של מחלקה

כאשר מחלקה מוגדרת, ניתן ליצור מופעים של מחלקה זו. מופעים הם אינסטנסים של המחלקה.

```python
class Car:
    def __init__(self, make, model, year):
        self.make = make
        self.model = model
        self.year = year

    def description(self):
        return f'{self.year} {self.make} {self.model}'

# יצירת מופע
my_car = Car('Toyota', 'Corolla', 2020)
print(my_car.description())  # פלט: 2020 Toyota Corolla
```

*   בדוגמה זו, יצרנו מופע `my_car` של המחלקה `Car`. בעת יצירת המופע, מועברים ערכים עבור התכונות `make`, `model`, ו-`year`, הנשמרים במופע.
*   המתודה `description()` מאפשרת לקבל ייצוג מחרוזתי של המכונית.

### סוגי מתודות

1.  **מתודות מופע**: אלו מתודות רגילות הפועלות על מופעי המחלקה. הן מקבלות הפניה למופע כפרמטר ראשון (בדרך כלל זהו `self`).

    דוגמה:
    ```python
    def method(self):
        pass
    ```

2.  **מתודות מחלקה**: מתודות המקבלות את המחלקה עצמה כפרמטר ראשון. להגדרת מתודות כאלו משתמשים במקשט (`decorator`) `@classmethod`. הן יכולות לשנות את מצב המחלקה עצמה, ולא את המופעים הבודדים שלה.

    דוגמה:
    ```python
    class MyClass:
        @classmethod
        def class_method(cls):
            pass
    ```

3.  **מתודות סטטיות**: אלו מתודות שאינן משתמשות ב-`self` או ב-`cls` (כלומר, אין להן גישה לא למופע ולא למחלקה). מתודות סטטיות מוכרזות באמצעות המקשט `@staticmethod`. הן יכולות להיות שימושיות כאשר מתודה אינה תלויה במצב של אובייקט או מחלקה, אך קשורה ללוגיקה השייכת למחלקה.

    דוגמה:
    ```python
    class MyClass:
        @staticmethod
        def static_method():
            pass
    ```

### ירושה

אחד העקרונות המרכזיים של OOP הוא **ירושה**. מחלקה יכולה לרשת התנהגות ממחלקה אחרת, להרחיב או לשנות את הפונקציונליות שלה. זה מאפשר שימוש חוזר בקוד, הימנעות מכפילויות.

```python
class Animal:
    def speak(self):
        return 'קול של חיה'

class Dog(Animal):  # המחלקה Dog יורשת מהמחלקה Animal
    def speak(self):
        return 'האב'

# יצירת מופעים
dog = Dog()
print(dog.speak())  # פלט: האב
```

*   המחלקה `Dog` יורשת את המתודה `speak` מהמחלקה `Animal`, אך דורסת אותה כדי להחזיר את המחרוזת `'האב'`.

### פולימורפיזם

**פולימורפיזם** משמעותו יכולתם של אובייקטים ממחלקות שונות להשתמש במתודות זהות עם מימושים שונים. ב-Python, זה אפשרי בזכות ירושה ודריסת מתודות.

```python
class Cat(Animal):
    def speak(self):
        return 'מיאו'

# יצירת מופעים
cat = Cat()
print(cat.speak())  # פלט: מיאו
```

כאן, `Cat` גם דורסת את המתודה `speak`, אך מחזירה ערך אחר. זה מאפשר לקרוא למתודה `speak` ללא תלות בסוג האובייקט.

### אנקפסולציה

**אנקפסולציה** מאפשרת הסתרת פרטי מימוש פנימיים ומתן גישה לנתונים דרך מתודות ציבוריות. זה מסייע במניעת שימוש שגוי בנתונים.

```python
class Car:
    def __init__(self, make, model):
        self._make = make  # תכונה מוגנת
        self._model = model

    def get_make(self):
        return self._make

    def set_make(self, make):
        self._make = make

# יצירת מופע
my_car = Car('Toyota', 'Corolla')
print(my_car.get_make())  # פלט: Toyota
my_car.set_make('Honda')
print(my_car.get_make())  # פלט: Honda
```

כאן, התכונות `_make` ו-`_model` הן מוגנות (בדרך כלל ב-Python, קו תחתון מסמן שתכונות אלו אינן אמורות לשמש ישירות מחוץ למחלקה), אך ניתן לגשת אליהן ולשנות אותן באמצעות המתודות `get_make` ו-`set_make`.

### תכונות נוספות של מחלקות

1.  **דסטרקטור** (`__del__`):
    מתודה מיוחדת הנקראת כאשר מופע מושמד (למשל, בעת יציאה מתחום ההכרה). ניתן להשתמש בה לשחרור משאבים.

    דוגמה:
    ```python
    class MyClass:
        def __del__(self):
            print("המופע הושמד")

    obj = MyClass()
    del obj  # המופע יושמד והמתודה __del__ תיקרא
    ```

2.  **מתודות מיוחדות**:
    אלו מתודות מיוחדות עם שני קווים תחתונים (לדוגמה, `__init__`, `__str__`, `__repr__`, `__eq__`). הן מאפשרות לדרוס התנהגות סטנדרטית של פעולות, כגון יצירת מופעים, השוואה, הצגת מופעים בפורמט מחרוזתי, ועוד.

    דוגמה:
    ```python
    class Point:
        def __init__(self, x, y):
            self.x = x
            self.y = y

        def __repr__(self):
            return f'Point({self.x}, {self.y})'

    p = Point(3, 4)
    print(p)  # פלט: Point(3, 4)
    ```
