## איך להגדיר נקודות עצירה ולבצע דיבאגינג לקוד ב-Jupyter Notebooks. מדריך מקוצר

אף על פי שב-Jupyter Notebook אין אפשרות פשוטה להגדרת נקודת עצירה (breakpoint) בלחיצת עכבר, כמו ב-IDE מסוימים (למעט JupyterLab המודרני), יש לנו כלים רבי עוצמה אחרים לדיבאגינג יעיל.

### שיטה 1: שימוש ב-`ipdb` – כלי דיבאגינג אינטראקטיבי עבור IPython (מומלץ)

`ipdb` הוא גרסה משופרת של כלי הדיבאגינג הסטנדרטי של Python (`pdb`), המותאמת במיוחד לעבודה עם IPython (הליבה שבה Jupyter משתמש). הוא מספק הדגשת תחביר (syntax highlighting), השלמה אוטומטית (Tab) ונוחות נוספות.

**1. התקנה:**
אם `ipdb` עדיין אינו מותקן, פתח/י מסוף או הרץ/הריצי בתא של הנוטבוק:

```bash
pip install ipdb
# או באמצעות conda
# conda install ipdb -c conda-forge
```

**2. שימוש:**
   *   יבאו את `ipdb` בתא שבו נמצא הקוד שברצונכם לבצע עליו דיבאגינג.
   *   הכניסו את השורה `ipdb.set_trace()` בנקודת הקוד שבה ברצונכם להפסיק את הביצוע.

**3. אופן הפעולה:**
   כאשר ביצוע הקוד יגיע לשורה `ipdb.set_trace()`, הוא יושהה. מיד מתחת לתא הנוכחי תופיע שורת פקודה אינטראקטיבית של הדיבאגר, שמתחילה ב-`ipdb>`. כאן ניתן להזין פקודות לשליטה בתהליך הדיבאגינג.

**דוגמה 1: פונקציה פשוטה**

```python
import ipdb

def calculate_stats(data):
    if not data:
        return 0, 0

    total_sum = sum(data)
    count = len(data)

    # לפני חישוב הממוצע...
    print("Перед вычислением среднего...")
    # מציבים נקודת עצירה כאן כדי לבדוק את total_sum ואת count
    ipdb.set_trace()

    average = total_sum / count
    print("Среднее вычислено.")
    return total_sum, average

my_list = [10, 20, 30, 40, 50]
s, avg = calculate_stats(my_list)

print(f"Сумма: {s}, Среднее: {avg}")
```

**מה יקרה בעת הביצוע:**
הביצוע יושהה לפני השורה `average = total_sum / count`. תראו את הפלט `Перед вычислением среднего...` ולאחר מכן את ההנחיה `ipdb>`.

**פקודות `ipdb` עיקריות:**

*   `n` (next): בצע את השורה הבאה של הקוד (מבלי להיכנס לפונקציות שנקראות).
*   `s` (step): בצע את השורה הבאה של הקוד, *תוך כניסה* לפונקציות שנקראות.
*   `c` (continue): המשך ביצוע רגיל של הקוד עד ל-`ipdb.set_trace()` הבא או עד סוף התוכנית/תא.
*   `p <משתנה>` (print): הדפס את ערך המשתנה שצוין. לדוגמה, `p total_sum` או `p count`.
*   `l` (list): הצג מספר שורות קוד מסביב לנקודת העצירה הנוכחית.
*   `u` (up): עבר רמה אחת למעלה בערימת הקריאות (אם נכנסת לפונקציה).
*   `d` (down): עבר רמה אחת למטה בערימת הקריאות.
*   `q` (quit): קטע/י מיידית את ביצוע הקוד וצא/י מהדיבאגר.
*   `h` (help): הצג את רשימת פקודות הדיבאגר הזמינות.
*   `!` (python_code): בצע קוד Python שרירותי. לדוגמה, `!import math` או `!new_var = total_sum * 2`.

**דוגמה 2: דיבאגינג בתוך לולאה**

```python
import ipdb

def process_items(items):
    processed = []
    for i, item in enumerate(items):
        print(f"Обработка элемента {i}: {item}")
        value = item * item # ריבוע המספר
        if value > 500:
            # נעצור אם הריבוע גדול מ-500
            print("Значение превысило порог!")
            ipdb.set_trace()
        processed.append(value)
    return processed

data = [5, 10, 15, 25, 30]
result = process_items(data)
print(f"Результат: {result}")
```

בעת ביצוע תא זה, הדיבאגר `ipdb` יופעל כאשר `item` יהיה שווה ל-`25` (מכיוון ש-`25*25 = 625`, שגדול מ-500). תוכלו לבדוק את הערכים הנוכחיים של `i`, `item`, `value` ו-`processed`.

### שיטה 2: שימוש ב-`pdb` הסטנדרטי

אם אינכם מעוניינים להתקין את `ipdb` או מעדיפים להשתמש רק בספרייה הסטנדרטית של Python, ניתן להשתמש במודול המובנה `pdb`.

**1. שימוש:**
   *   יבאו את `pdb`.
   *   הכניסו את `pdb.set_trace()` בנקודת הקוד הרצויה.

**דוגמה:**

```python
import pdb

def greet(name, language="en"):
    if language == "en":
        greeting = f"Hello, {name}!"
    elif language == "es":
        greeting = f"Hola, {name}!"
    else:
        greeting = f"Hi, {name}!"
        # נציב נקודת עצירה כאן כדי לבדוק את 'language' ואת 'name'
        pdb.set_trace()
    return greeting

message = greet("World", language="fr") # משתמשים בשפה לא נתמכת
print(message)
```

**אופן הפעולה:**
בדומה ל-`ipdb`, הביצוע יושהה בשורה `pdb.set_trace()`, ותופיע ההנחיה `pdb>`. הפקודות (`n`, `s`, `c`, `p`, `l`, `q` וכו') זהות ברובן, אך הממשק עשוי להיות פחות נוח (לדוגמה, ללא הדגשת תחביר).

### שיטה 3: כלי הדיבאגינג הוויזואלי ב-JupyterLab (גישה מודרנית)

אם אתם משתמשים ב-**JupyterLab גרסה 3.0 ומעלה**, יש לכם גישה לכלי דיבאגינג ויזואלי מובנה, הדומה מאוד לכלי דיבאגינג ב-IDE מלאים.

**1. דרישות:**
   *   JupyterLab >= 3.0
   *   ליבה (Kernel) עם תמיכה בדיבאגינג. לרוב זו תהיה `ipykernel` גרסה 6.0 ומעלה, או `xeus-python`.

   ודאו שיש לכם את הגרסאות העדכניות:
   ```bash
   pip install -U jupyterlab "ipykernel>=6"
   # או עבור xeus-python (בשימוש לרוב עם conda/mamba)
   # mamba install xeus-python -c conda-forge
   ```
   *   **חשוב:** לאחר התקנה או שדרוג ייתכן שיידרש **להפעיל מחדש את JupyterLab**.

**2. שימוש:**
   *   **הפעלה:** ב-JupyterLab, בצד ימין (או שמאל, תלוי בתצורה שלכם) מצאו את סמל ה**חיפושית (🐞)** בסרגל הצד. לחצו עליו. אם הליבה שלכם תומכת בדיבאגינג, פנל הדיבאגינג יופעל עבור הנוטבוק הנוכחי. תראו מתג "Debugger" בפינה הימנית העליונה של הנוטבוק – ודאו שהוא מופעל.
   *   **הגדרת נקודת עצירה:** **לחצו בעכבר** על השדה האפור משמאל למספר השורה בתא הקוד. תופיע נקודה אדומה – זוהי נקודת העצירה שלכם. ניתן להגדיר מספר נקודות.
   *   **הרצה:** הריצו את התא כרגיל (Shift+Enter או כפתור Run).
   *   **דיבאגינג:** ביצוע הקוד יושהה בנקודת העצירה הראשונה שתגיע אליה. בפנל הדיבאגר תראו את ערימת הקריאות (Callstack), רשימת משתנים מקומיים וערכיהם, ורשימה של כל נקודות העצירה. השתמשו בכפתורים בפנל הדיבאגר:
         *   `Continue (▶️)`: המשך עד נקודת העצירה הבאה או עד הסוף.
         *   `Terminate (⏹️)`: הפסק את הביצוע.
         *   `Step Over (⏭️)`: עבור לשורה הבאה (מבלי להיכנס לפונקציות).
         *   `Step In (↘️)`: היכנס לפונקציה שנקראת.
         *   `Step Out (↖️)`: צא מהפונקציה הנוכחית לרמה אחת למעלה.

**דוגמה (לשימוש עם כלי הדיבאגינג הוויזואלי של JupyterLab):**

```python
# --- תא 1 ---
def complex_calculation(x, y):
    # הציבו נקודת עצירה בשורה הבאה (בלחיצת עכבר משמאל)
    temp_a = x * x
    temp_b = y + 5
    # הציבו נקודת עצירה נוספת כאן
    result = temp_a / temp_b
    return result

# --- תא 2 ---
a = 10
b = 0 # שגיאת חלוקה באפס פוטנציאלית
# הציבו נקודת עצירה כאן
final_value = complex_calculation(a, b - 5) # מעבירים y = -5 כדי למנוע חלוקה באפס מיד

print(f"Итоговое значение: {final_value}")
```

הריצו את תא 1 (כדי להגדיר את הפונקציה), לאחר מכן הפעילו את כלי הדיבאגינג, הציבו נקודות עצירה בתא 2 ובפונקציה `complex_calculation`, ולאחר מכן הריצו את תא 2. תוכלו לעבור על הביצוע צעד אחר צעד, תוך צפייה בשינוי ערכי המשתנים `a`, `b`, `x`, `y`, `temp_a`, `temp_b`, `result` בפנל הדיבאגר.

### שיטה 4: הפקודה המאגית `%debug` (דיבאגינג לאחר שגיאה)

שיטה זו שימושית מאוד כאשר תא כבר כשל עם שגיאה (זרק חריגה), ואתם רוצים להבין בדיוק מה קרה ברגע הכשל.

**1. שימוש:**
   *   הריצו את התא שגורם לשגיאה. תראו את הפלט הסטנדרטי של עקבת המחסנית (Traceback).
   *   בתא ה*בא*, הזינו ובצעו את הפקודה המאגית:
     ```python
     %debug
     ```

**2. אופן הפעולה:**
   Jupyter יפעיל את כלי הדיבאגינג (`ipdb`, אם מותקן, אחרת `pdb`) בדיוק בנקודת הקוד (ברכיב האחרון של ערימת הקריאות) שבה אירעה השגיאה. תגיעו למצב אינטראקטיבי של כלי הדיבאגינג (`ipdb>` או `pdb>`) ותוכלו לבחון את ערכי המשתנים שהובילו לשגיאה.

**דוגמה:**

*   **תא 1 (גורם לשגיאה):**
    ```python
    def risky_division(num, den):
        # שכחנו לבדוק את המכנה
        quotient = num / den
        return quotient

    numerator = 100
    denominator = 0 # זה יגרום ל-ZeroDivisionError

    result = risky_division(numerator, denominator)
    print(result)
    ```
    *(בעת ביצוע תא זה תקבלו `ZeroDivisionError`)*

*   **תא 2 (מופעל מיד לאחר השגיאה):**
    ```python
    %debug
    ```
    *(ביצוע תא זה יפתח את כלי הדיבאגינג `ipdb>` או `pdb>` בתוך הפונקציה `risky_division` בדיוק בשורה `quotient = num / den`. תוכלו להזין `p num` ו-`p den` כדי לראות את ערכיהם (100 ו-0).)*

### טיפים ושיטות עבודה מומלצות

*   **הסירו נקודות עצירה:** אל תשכחו למחוק או להעיר שורות `ipdb.set_trace()` / `pdb.set_trace()` לפני שמירת גרסה סופית של הנוטבוק או העברתו לאחרים. נקודות עצירה ויזואליות ב-JupyterLab לרוב אינן נשמרות בקובץ `.ipynb` עצמו, אך עדיף להסיר גם אותן.
*   **השתמשו עם `print()`:** לפעמים פשוט יותר להוסיף מספר פקודות `print()` להדפסת ערכי משתנים, מאשר להפעיל כלי דיבאגינג מלא. שלבו גישות.
*   **הבינו את תחום ההיקף (scope):** נקודת עצירה בתוך פונקציה תשהה את הביצוע רק כאשר פונקציה זו תיקרא.
*   **שקלו שימוש בלוגינג:** עבור תהליכים ארוכים או דיבאגינג בסביבת פרודקשן, מודול `logging` הסטנדרטי הוא לעתים קרובות כלי מתאים יותר מכלי דיבאגינג אינטראקטיבי.
