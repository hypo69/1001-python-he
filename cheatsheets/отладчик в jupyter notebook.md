## כיצד להגדיר נקודות עצירה (Breakpoints) ולבצע ניפוי שגיאות בקוד ב-Jupyter Notebooks. מדריך עזר.

אף על פי שב-Jupyter Notebook אין אפשרות פשוטה להגדיר נקודות עצירה בלחיצת עכבר, כמו בסביבות פיתוח משולבות (IDE) מסוימות (למעט ב-JupyterLab המודרני), לרשותנו עומדים כלים רבי עוצמה אחרים לניפוי שגיאות יעיל.

### שיטה 1: שימוש ב-`ipdb` — כלי ניפוי שגיאות אינטראקטיבי עבור IPython (מומלץ)

`ipdb` היא גרסה משופרת של כלי ניפוי השגיאות הסטנדרטי של Python (`pdb`), המותאמת במיוחד לעבודה עם IPython (הקרנל שבו Jupyter משתמש). היא מספקת הדגשת תחביר, השלמה אוטומטית (Tab) ונוחות נוספת.

**1. התקנה:**
אם `ipdb` אינו מותקן עדיין, פתח את הטרמינל או הפעל בתא ביופיטר נוטבוק:

```bash
pip install ipdb
# או באמצעות conda
# conda install ipdb -c conda-forge
```

**2. שימוש:**
*   ייבא את `ipdb` בתא שבו נמצא הקוד שברצונך לנפות את שגיאותיו.
*   הכנס את השורה `ipdb.set_trace()` במקום בקוד שבו ברצונך להשהות את הביצוע.

**3. אופן הפעולה:**
כאשר ביצוע הקוד יגיע לשורה `ipdb.set_trace()`, הוא יושהה. ממש מתחת לתא הנוכחי תופיע שורת פקודה אינטראקטיבית של כלי ניפוי השגיאות, המתחילה ב-`ipdb>`. כאן תוכל להזין פקודות לשליטה על תהליך ניפוי השגיאות.

**דוגמה 1: פונקציה פשוטה**

```python
import ipdb

def calculate_stats(data):
    if not data:
        return 0, 0

    total_sum = sum(data)
    count = len(data)

    print("לפני חישוב הממוצע...")
    # נגדיר כאן נקודת עצירה כדי לבדוק את total_sum ואת count
    ipdb.set_trace()

    average = total_sum / count
    print("הממוצע חושב.")
    return total_sum, average

my_list = [10, 20, 30, 40, 50]
s, avg = calculate_stats(my_list)

print(f"סכום: {s}, ממוצע: {avg}")
```

**מה יקרה בעת ההפעלה:**
הביצוע יושהה לפני השורה `average = total_sum / count`. תראה את הפלט "לפני חישוב הממוצע..." ולאחר מכן את ההנחיה `ipdb>`.

**פקודות `ipdb` עיקריות:**

*   `n` (next): בצע את השורה הבאה של הקוד (מבלי להיכנס לפונקציות שנקראות).
*   `s` (step): בצע את השורה הבאה של הקוד, *תוך כניסה* לפונקציות שנקראות.
*   `c` (continue): המשך ביצוע רגיל של הקוד עד לנקודת העצירה הבאה (`ipdb.set_trace()`) או עד סוף התוכנית/תא.
*   `p <משתנה>` (print): הדפס את ערך המשתנה שצוין. לדוגמה, `p total_sum` או `p count`.
*   `l` (list): הצג מספר שורות קוד סביב מיקום העצירה הנוכחי.
*   `u` (up): עבור רמה אחת למעלה במחסנית הקריאות (אם נכנסת לתוך פונקציה).
*   `d` (down): עבור רמה אחת למטה במחסנית הקריאות.
*   `q` (quit): הפסק את ביצוע הקוד באופן מיידי וצא מכלי ניפוי השגיאות.
*   `h` (help): הצג רשימת פקודות זמינות של כלי ניפוי השגיאות.
*   `!<קוד_פייתון>`: בצע קוד Python שרירותי. לדוגמה, `!import math` או `!new_var = total_sum * 2`.

**דוגמה 2: ניפוי שגיאות בתוך לולאה**

```python
import ipdb

def process_items(items):
    processed = []
    for i, item in enumerate(items):
        print(f"עיבוד אלמנט {i}: {item}")
        value = item * item # ריבוע המספר
        if value > 500:
            # נעצור אם הריבוע גדול מ-500
            print("הערך עלה על הסף!")
            ipdb.set_trace()
        processed.append(value)
    return processed

data = [5, 10, 15, 25, 30]
result = process_items(data)
print(f"תוצאה: {result}")
```

בעת הפעלת תא זה, כלי ניפוי השגיאות `ipdb` יופעל כאשר `item` יהיה שווה ל-`25` (מכיוון ש-`25*25 = 625`, שהוא גדול מ-500). תוכל לבדוק את הערכים הנוכחיים של `i`, `item`, `value` ו-`processed`.

### שיטה 2: שימוש ב-`pdb` הסטנדרטי

אם אינך רוצה להתקין את `ipdb` או שאתה מעדיף להשתמש רק בספרייה הסטנדרטית של Python, תוכל להשתמש במודול המובנה `pdb`.

**1. שימוש:**
*   ייבא את `pdb`.
*   הכנס את `pdb.set_trace()` במקום הרצוי בקוד.

**דוגמה:**

```python
import pdb

def greet(name, language="en"):
    if language == "en":
        greeting = f"Hello, {name}!"
    elif language == "es":
        greeting = f"Hola, {name}!"
    else:
        greeting = f"Hi, {name}!"
        # נגדיר כאן נקודת עצירה כדי לבדוק את 'language' ואת 'name'
        pdb.set_trace()
    return greeting

message = greet("World", language="fr") # נשתמש בשפה שאינה נתמכת
print(message)
```

**אופן הפעולה:**
בדומה ל-`ipdb`, הביצוע יושהה בשורה `pdb.set_trace()`, ותופיע ההנחיה `pdb>`. הפקודות (`n`, `s`, `c`, `p`, `l`, `q` וכו') זהות ברובן, אך הממשק עשוי להיות פחות נוח (לדוגמה, ללא הדגשת תחביר).

### שיטה 3: כלי ניפוי שגיאות ויזואלי ב-JupyterLab (גישה מודרנית)

אם אתה משתמש ב-**JupyterLab גרסה 3.0 ומעלה**, יש לך גישה לכלי ניפוי שגיאות ויזואלי מובנה, הדומה מאוד לכלי ניפוי שגיאות בסביבות פיתוח משולבות מלאות.

**1. דרישות:**
*   JupyterLab >= 3.0
*   קרנל עם תמיכה בניפוי שגיאות. בדרך כלל זה `ipykernel` גרסה 6.0 ומעלה, או `xeus-python`.

ודא שהגרסאות העדכניות מותקנות:
```bash
pip install -U jupyterlab "ipykernel>=6"
# או עבור xeus-python (משמש בדרך כלל עם conda/mamba)
# mamba install xeus-python -c conda-forge
```
*   **חשוב:** לאחר ההתקנה או העדכון, ייתכן שיהיה צורך **להפעיל מחדש את JupyterLab**.

**2. שימוש:**
*   **הפעלה:** ב-JupyterLab, בצד ימין (או שמאל, תלוי בתצורה שלך) מצא את סמל **החיפושית (🐞)** בסרגל הצד. לחץ עליו. אם הקרנל שלך תומך בניפוי שגיאות, חלונית ניפוי השגיאות תופעל עבור הנוטבוק הנוכחי. תראה מתג "Debugger" בפינה הימנית העליונה של הנוטבוק – ודא שהוא מופעל.
*   **הגדרת נקודת עצירה:** **לחץ בעכבר** על השדה האפור משמאל למספר השורה בתא הקוד. תופיע נקודה אדומה – זוהי נקודת העצירה שלך. תוכל להגדיר מספר נקודות.
*   **הפעלה:** הפעל את התא כרגיל (Shift+Enter או כפתור Run).
*   **ניפוי שגיאות:** ביצוע הקוד יעצור בנקודת העצירה הראשונה שאליה יגיע.
*   בחלונית ניפוי השגיאות תראה את מחסנית הקריאות (Callstack), רשימת המשתנים המקומיים וערכיהם, רשימת כל נקודות העצירה.
*   השתמש בלחצנים בחלונית ניפוי השגיאות:
    *   `Continue (▶️)`: המשך עד לנקודת העצירה הבאה או לסוף.
    *   `Terminate (⏹️)`: עצור את הביצוע.
    *   `Step Over (⏭️)`: עבור לשורה הבאה (מבלי להיכנס לפונקציות).
    *   `Step In (↘️)`: היכנס לתוך הפונקציה הנקראת.
    *   `Step Out (↖️)`: צא מהפונקציה הנוכחית רמה אחת למעלה.

**דוגמה (לשימוש עם כלי ניפוי השגיאות הוויזואלי של JupyterLab):**

```python
# --- תא 1 ---
def complex_calculation(x, y):
    # הגדר נקודת עצירה בשורה הבאה (בלחיצת עכבר משמאל)
    temp_a = x * x
    temp_b = y + 5
    # הגדר כאן נקודת עצירה נוספת
    result = temp_a / temp_b
    return result

# --- תא 2 ---
a = 10
b = 0 # שגיאה פוטנציאלית של חלוקה באפס
# הגדר כאן נקודת עצירה
final_value = complex_calculation(a, b - 5) # נעביר y = -5 כדי להימנע מחלוקה באפס מיד

print(f"ערך סופי: {final_value}")
```

הפעל את תא 1 (כדי להגדיר את הפונקציה), לאחר מכן הפעל את כלי ניפוי השגיאות, הגדר נקודות עצירה בתא 2 ובפונקציה `complex_calculation`, ולאחר מכן הפעל את תא 2. תוכל לעבור צעד אחר צעד בביצוע, ולעקוב אחר שינוי המשתנים `a`, `b`, `x`, `y`, `temp_a`, `temp_b`, `result` בחלונית ניפוי השגיאות.

### שיטה 4: פקודת הקסם `%debug` (ניפוי שגיאות לאחר שגיאה)

שיטה זו שימושית מאוד כאשר תא כבר קרס עם שגיאה (זרק חריגה), ואתה רוצה להבין מה בדיוק קרה ברגע הכשל.

**1. שימוש:**
*   הפעל את התא שגורם לשגיאה. תראה את פלט המעקב הסטנדרטי של מחסנית (Traceback).
*   בתא *הבא* הכנס ובצע את פקודת הקסם:
    ```python
    %debug
    ```

**2. אופן הפעולה:**
Jupyter יפעיל את כלי ניפוי השגיאות (`ipdb`, אם מותקן, אחרת `pdb`) בדיוק במקום בקוד (במסגרת האחרונה של מחסנית הקריאות) שבו אירעה החריגה. תמצא את עצמך במצב אינטראקטיבי של כלי ניפוי השגיאות (`ipdb>` או `pdb>`) ותוכל לבדוק את ערכי המשתנים שהובילו לשגיאה.

**דוגמה:**

*   **תא 1 (גורם לשגיאה):**
    ```python
    def risky_division(num, den):
        # שכחנו לבדוק את המכנה
        quotient = num / den
        return quotient

    numerator = 100
    denominator = 0 # זה יגרום ל-ZeroDivisionError

    result = risky_division(numerator, denominator)
    print(result)
    ```
    *(בעת הפעלת תא זה תקבל `ZeroDivisionError`)*

*   **תא 2 (מופעל מיד לאחר השגיאה):**
    ```python
    %debug
    ```
    *(הפעלת תא זה תפתח את כלי ניפוי השגיאות `ipdb>` או `pdb>` בתוך הפונקציה `risky_division` בדיוק בשורה `quotient = num / den`. תוכל להזין `p num` ו-`p den` כדי לראות את ערכיהם (100 ו-0).)*

### טיפים ושיטות עבודה מומלצות

*   **הסר נקודות עצירה:** אל תשכח להסיר או להעיר את השורות `ipdb.set_trace()` / `pdb.set_trace()` לפני שמירת הגרסה הסופית של הנוטבוק או העברתו לאחרים. נקודות עצירה ויזואליות ב-JupyterLab בדרך כלל אינן נשמרות בקובץ `.ipynb` עצמו, אך עדיף גם אותן להסיר.
*   **השתמש ב-`print()`:** לעיתים קל יותר להוסיף כמה פקודות `print()` כדי להדפיס ערכי משתנים מאשר להפעיל כלי ניפוי שגיאות מלא. שלב גישות.
*   **הבן את טווח הראות (Scope):** נקודת עצירה בתוך פונקציה תשהה את הביצוע רק כאשר פונקציה זו תיקרא.
*   **שקול תיעוד (Logging):** לתהליכים ארוכים או ניפוי שגיאות בסביבת ייצור, המודול הסטנדרטי `logging` הוא לעיתים קרובות כלי מתאים יותר מכלי ניפוי שגיאות אינטראקטיבי.
