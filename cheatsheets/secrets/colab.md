# 🔐 אחסון סודות ב-Google Colab: איך לא לחשוף את מפתח ה-API שלכם

האם תהיתם איפה להחביא טוקנים ומפתחות API כדי שלא יברחו מהמחברת שלכם? דמיינו: אתם עובדים על פרויקט מגניב ב-Colab, מתחברים ל-OpenAI או GitHub, ואז... באם! המפתח בולט בקוד כמו עץ חג מולד. מוכר? בואו נבין איך להימנע מצרות עם הפיצ'ר המובנה של Colab - "סודות".

---

## 🤯 למה להטמיע מפתחות בקוד זה רעיון גרוע?

נניח שכתבתם כך:

```python
api_key = "sk-1234567890-pineapple-pizza"  # 🚨 קריאת חירום!
```

ואז בטעות שיתפתם את המחברת או העליתם אותה לרפו ציבורי. מה יקרה? המפתח שלכם עלול:

1. **לדלוף** לאנשים זדוניים (ואז מישהו ייצור חתולים על חשבונכם).
2. **להיכנס ללוגים** של שירותים (כן, זה קורה!).
3. **לגרום לחסימת** החשבון (ספקי API לא אוהבים מפתחות שדלפו).

**הפתרון?** ה"סודות" המובנים של Colab - כמו כספת למפתחות הדיגיטליים שלכם. הם:
- לא נראים בקוד
- לא גלויים אפילו בשיתוף
- ניתנים לניהול בכמה קליקים

---

## 🧰 מה זה "סודות" ואיך הם עובדים?

דמיינו מעטפה קסומה שמכניסה אוטומטית את הנתונים הנכונים לקוד. בערך כך עובדים "סודות" - משתני סביבה ש:
- נשמרים בחשבון הגוגל שלכם
- נטענים בעת הפעלת המחברת
- נגישים דרך `os.environ` (כמו בפרויקטים מקומיים)

**הפיצ'ר:** אפילו אם תציגו למישהו את כל הקוד, המעטפה תישאר אטומה!

---

## 🛠️ מדריך שלב-שלב: איך להחביא "שלדים בארון"

### 1️⃣ פותחים את החדר הסודי

בצד שמאל ב-Colab לוחצים על אייקון התיקייה → בוחרים בלשונית **🔐 Secrets**. לפניכם הכספת האישית שלכם!

![ממשק הסודות ב-Google Colab](attachment:1b6f3ce9-49db-4ed9-b55e-0410350f9a38.png)

### 2️⃣ יוצרים רכיב סודי

1. לוחצים "הוסף סוד" (או על אייקון המפתח 🔑)
2. ממלאים:
   - **שם המשתנה** - לדוגמה `MY_SUPER_SECRET` (ללא רווחים, רישיות חשובות!)
   - **ערך** - הטוקן או המפתח עצמו

👉 *טיפ:* תנו שמות משמעותיים - `OPENAI_KEY` עדיף על `secret_123`.

### 3️⃣ משתמשים בקסם בקוד

כעת המפתח שלכם נגיש דרך הסביבה:

```python
import os

api_key = os.environ.get("OPENAI_API_KEY")  # 🪄 קסם!
```

או מחברים ישירות לספרייה:

```python
from openai import OpenAI

client = OpenAI(api_key=os.environ["OPENAI_API_KEY"])  # ✅ בטוח!
```

זה דומה לאיך ש-GitHub Actions או GitLab CI עובדים - הסודות קיימים רק בזמן הריצה.

## 🎯 איפה זה שימושי? 5 דוגמאות מובילות

## 🚨 כללי בטיחות: איך לא "להיחשף"

1. **הכפילו הגנה:** הפעילו הגבלות על מפתחות API (למשל הגבלת בקשות ב-OpenAI).
2. **אל תחלקו את הכספת:** אם מעבירים את החשבון, מחקו סודות מראש.
3. **השתמשו ב"פראצ'וט גיבוי":** 
```python
key = os.environ.get("API_KEY") or st.text_input("הזן מפתח")  # אם הסוד לא נמצא
```
4. **נקו אחריכם:** מחקו מפתחות לא בשימוש מהאחסון.
