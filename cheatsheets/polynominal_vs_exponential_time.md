### **זמן פולינומי**

**זמן פולינומי** הוא מונח המשמש בתורת הסיבוכיות החישובית לתיאור זמן הריצה של אלגוריתם, הגדל כפולינום (רב-איבר) של גודל הקלט. אם זמן הריצה של אלגוריתם ניתן לביטוי כ-\(O(n^k)\), כאשר \(n\) הוא גודל הקלט ו-\(k\) הוא קבוע, אז אלגוריתם כזה פועל בזמן פולינומי.

#### **דוגמאות:**
1.  **מיון רשימה**: אלגוריתמים כמו מיון מיזוג או מיון מהיר פועלים בזמן \(O(n \log n)\), שהוא זמן פולינומי.
2.  **מציאת המסלול הקצר ביותר בגרף**: אלגוריתם דיקסטרה פועל בזמן \(O(n^2)\) או \(O(n \log n)\) בהתאם למימוש, שגם הוא פולינומי.

#### **מאפיינים:**
*   אלגוריתמים הפועלים בזמן פולינומי נחשבים **יעילים** ו**ישימים בפועל**.
*   בעיות שניתן לפתור בזמן פולינומי שייכות ל**מחלקה P**.

---

### **זמן אקספוננציאלי**

**זמן אקספוננציאלי** הוא זמן הריצה של אלגוריתם, הגדל בצורה אקספוננציאלית בהתאם לגודל הקלט. אם זמן הריצה ניתן לביטוי כ-\(O(k^n)\), כאשר \(n\) הוא גודל הקלט ו-\(k\) הוא קבוע, אז אלגוריתם כזה פועל בזמן אקספוננציאלי.

#### **דוגמאות:**
1.  **בעיית הסוכן הנודד**: פתרון בשיטת החיפוש הממצה של כל המסלולים האפשריים דורש זמן \(O(n!)\), שהוא גרוע יותר מאקספוננציאלי.
2.  **מעבר על כל תת-הקבוצות**: אלגוריתם הבודק את כל תת-הקבוצות האפשריות של קבוצה בגודל \(n\) פועל בזמן \(O(2^n)\).

#### **מאפיינים:**
*   אלגוריתמים הפועלים בזמן אקספוננציאלי נחשבים **לא יעילים** עבור קלטים גדולים, שכן זמן הריצה הופך להיות גדול בצורה לא מעשית אפילו עבור \(n\) קטן יחסית.
*   בעיות שניתן לפתור רק בזמן אקספוננציאלי שייכות לעיתים קרובות למחלקות **קשות NP** (NP-hard) או **שלמות NP** (NP-complete).

---

### **השוואת זמן פולינומי וזמן אקספוננציאלי**

| **מאפיין**                     | **זמן פולינומי**                       | **זמן אקספוננציאלי**                     |
| :------------------------------ | :------------------------------------- | :--------------------------------------- |
| **גידול זמן הריצה**             | איטי (לדוגמה, \(n^2\), \(n^3\))        | מהיר (לדוגמה, \(2^n\), \(3^n\))          |
| **דוגמאות לבעיות**            | מיון, מציאת מסלול קצר ביותר           | בעיית הסוכן הנודד, מעבר על תת-קבוצות    |
| **יישימות בפועל**              | יעיל עבור נתונים גדולים               | לא ישים עבור נתונים גדולים             |
| **מחלקת סיבוכיות**             | P                                      | קשות NP (NP-hard), שלמות NP (NP-complete) |

---

### **מדוע זה חשוב?**

1.  **זמן פולינומי**:
    *   אלגוריתמים הפועלים בזמן פולינומי נחשבים **ישימים בפועל**, מכיוון שהם יכולים לעבד נפחי נתונים גדולים בזמן סביר.
    *   בעיות **מחלקה P** (הניתנות לפתרון בזמן פולינומי) הן הבסיס ליישומים רבים במדעי המחשב, כגון עיבוד נתונים, רשתות, קריפטוגרפיה ובינה מלאכותית.

2.  **זמן אקספוננציאלי**:
    *   אלגוריתמים הפועלים בזמן אקספוננציאלי **אינם מעשיים** אפילו עבור קלטים קטנים יחסית. לדוגמה, עבור \(n = 100\), \(2^n\) כבר עולה על מספר האטומים ביקום הנצפה.
    *   בעיות שניתן לפתור רק בזמן אקספוננציאלי דורשות לעיתים קרובות שימוש ב**שיטות קירוב**, **היוריסטיקות** או **חישובים מקביליים**.

### **דוגמה להמחשה**

דמיינו שיש לכם בעיה ואתם רוצים לפתור אותה עבור \(n = 10\) ו-\(n = 100\):

*   **זמן פולינומי (\(n^2\))**:
    *   עבור \(n = 10\): \(10^2 = 100\) פעולות.
    *   עבור \(n = 100\): \(100^2 = 10,000\) פעולות.

*   **זמן אקספוננציאלי (\(2^n\))**:
    *   עבור \(n = 10\): \(2^{10} = 1,024\) פעולות.
    *   עבור \(n = 100\): \(2^{100} \approx 1.26 \times 10^{30}\) פעולות.

כפי שניתן לראות, עבור \(n = 100\), אלגוריתם פולינומי יבצע 10,000 פעולות, שזה בהחלט ריאלי, ואילו אלגוריתם אקספוננציאלי ידרוש \(1.26 \times 10^{30}\) פעולות, שזה כמעט בלתי אפשרי.

לבניית גרפים המדגימים את ההבדל בין זמן פולינומי לזמן אקספוננציאלי, ניתן להשתמש בפונקציות מתמטיות שונות. להלן דוגמאות לפונקציות שניתן להשתמש בהן לצורך ויזואליזציה:

### **פונקציות פולינומיות**
1.  **פונקציה לינארית**:
    \( f(n) = n \)
    דוגמה: זמן הריצה של אלגוריתם המעבד כל איבר פעם אחת.

2.  **פונקציה ריבועית**:
    \( f(n) = n^2 \)
    דוגמה: זמן הריצה של אלגוריתם עם לולאות מקוננות, למשל, מיון בועות.

3.  **פונקציה מעוקבת**:
    \( f(n) = n^3 \)
    דוגמה: זמן הריצה של אלגוריתם המעבד נתונים תלת-ממדיים.

4.  **פונקציה לוגריתמית**:
    \( f(n) = \log n \)
    דוגמה: זמן הריצה של חיפוש בינארי.

5.  **פונקציה ליניארית-לוגריתמית**:
    \( f(n) = n \log n \)
    דוגמה: זמן הריצה של מיון מהיר או מיון מיזוג.

---

### **פונקציות אקספוננציאליות**
1.  **פונקציה אקספוננציאלית**:
    \( f(n) = 2^n \)
    דוגמה: זמן הריצה של אלגוריתם המעבר על כל תת-הקבוצות של קבוצה.

2.  **פונקציית עצרת**:
    \( f(n) = n! \)
    דוגמה: זמן הריצה של אלגוריתם המעבר על כל התמורות (לדוגמה, בעיית הסוכן הנודד).

3.  **פונקציה אקספוננציאלית עם בסיס אחר**:
    \( f(n) = 3^n \)
    דוגמה: זמן הריצה של אלגוריתם החוקר את כל הצירופים האפשריים.

### **דוגמת קוד לבניית גרפים (פייתון, Matplotlib)**

```python
import matplotlib.pyplot as plt
import numpy as np

# n range
n = np.linspace(1, 20, 100)

# Polynomial functions
linear = n
quadratic = n**2
cubic = n**3
logarithmic = np.log(n)
nlogn = n * np.log(n)

# Exponential functions
exponential = 2**n
factorial = [np.math.factorial(int(i)) for i in n]  # Factorial is defined only for integers

# Plotting graphs
plt.figure(figsize=(10, 6))

# Polynomial functions
plt.plot(n, linear, label='Linear: $f(n) = n$')
plt.plot(n, quadratic, label='Quadratic: $f(n) = n^2$')
plt.plot(n, cubic, label='Cubic: $f(n) = n^3$')
plt.plot(n, logarithmic, label='Logarithmic: $f(n) = \log n$')
plt.plot(n, nlogn, label='Linearithmic: $f(n) = n \log n$')

# Exponential functions
plt.plot(n, exponential, label='Exponential: $f(n) = 2^n$')
plt.plot(n, factorial, label='Factorial: $f(n) = n!$')

# Graph settings
plt.yscale('log')  # Logarithmic scale for convenience
plt.xlabel('Input size (n)')
plt.ylabel('Time complexity')
plt.title('Comparison of Polynomial and Exponential Time Complexity')
plt.legend()
plt.grid(True)
plt.show()
```
