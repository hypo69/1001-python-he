# סינגלטון (Singleton) ב-`Python`

ב-`Python`, סינגלטון הוא תבנית עיצוב המבטיחה שלמחלקה יהיה מופע (instance) יחיד בלבד, ומספקת נקודת גישה גלובלית למופע זה. משמעות הדבר היא שבכל ניסיון ליצור אובייקט חדש ממחלקה זו, תקבל תמיד את אותו האובייקט בדיוק.

סינגלטונים שימושיים כאשר יש צורך להגביל את מספר המופעים של מחלקה, למשל:

*   לניהול חיבור למסד נתונים (כדי להימנע מפתיחת חיבורים רבים).
*   לאחסון הגדרות תצורה גלובליות של היישום (כדי שכל חלקי היישום ישתמשו באותה תצורה).
*   לרישום אירועים (logging) (כדי שכל ההודעות יגיעו לקובץ יחיד).

להלן מספר דרכים למימוש סינגלטון ב-`Python`.

**דרכים למימוש סינגלטון:**

1.  **באמצעות דריסת שיטת `__new__`**

    *   השיטה `__new__` אחראית ליצירת מופע של המחלקה. על ידי דריסתה, ניתן לשלוט בתהליך זה.
    *   בדוגמה זו, נשמור את המופע היחיד של המחלקה במשתנה `_instance`.
    *   אם המופע עדיין לא קיים, ניצור אותו; אחרת, נחזיר את המופע הקיים.
    *   **`Python` Code:**

        ```python
        class Singleton:
            _instance = None  # שמירת המופע היחיד

            def __new__(cls, *args, **kwargs):
                """
                דורס את שיטת __new__ כדי לשלוט ביצירת המופע.

                Args:
                    cls: המחלקה שעבורה נוצר המופע.
                    *args: ארגומנטים מיקומיים עבור הבנאי.
                    **kwargs: ארגומנטים ממופים בשם עבור הבנאי.

                Returns:
                    המופע היחיד של המחלקה.
                """
                if not cls._instance: # אם המופע עדיין לא נוצר
                    cls._instance = super().__new__(cls, *args, **kwargs) # יצירת מופע חדש
                return cls._instance # החזרת המופע הקיים

        # דוגמת שימוש
        s1 = Singleton()
        s2 = Singleton()

        print(s1 is s2)  # הדפסת True, מאחר שזהו אותו אובייקט בדיוק
        ```

2.  **באמצעות דקורטור**

    *   דקורטור הוא פונקציה שמשנה התנהגות של מחלקה.
    *   בדוגמה זו, ניצור פונקציה-דקורטור `singleton` שמקבלת מחלקה ומחזירה גרסה עטופה שלה.
    *   בתוך הדקורטור, נשמור מופעים של מחלקות במילון `instances`.
    *   אם מופע של מחלקה עדיין לא נוצר, ניצור אותו ונשמור אותו במילון; אחרת, נחזיר את המופע הקיים.
    *   **`Python` Code:**

        ```python
        def singleton(cls):
            """
            דקורטור ליצירת סינגלטון.

            Args:
                cls: המחלקה שיש להפוך אותה לסינגלטון.

            Returns:
                המחלקה ששוונתה, שהינה סינגלטון.
            """
            instances = {} # שמירת מופעים של מחלקות

            def wrapper(*args, **kwargs):
                """
                פונקציית עטיפה המחזירה את המופע היחיד של המחלקה.

                Args:
                   *args: ארגומנטים מיקומיים עבור הבנאי.
                   **kwargs: ארגומנטים ממופים בשם עבור הבנאי.

                Returns:
                    המופע היחיד של המחלקה.
                """
                if cls not in instances: # אם המופע עדיין לא נוצר
                    instances[cls] = cls(*args, **kwargs) # יצירת מופע ושמירתו
                return instances[cls] # החזרת המופע הקיים
            return wrapper

        @singleton # יישום הדקורטור על המחלקה
        class MyClass:
            pass

        # דוגמת שימוש
        obj1 = MyClass()
        obj2 = MyClass()

        print(obj1 is obj2)  # הדפסת True, מאחר שזהו אותו אובייקט בדיוק
        ```

3.  **באמצעות מטאקלאס**

    *   מטאקלאס מאפשר לשלוט בתהליך יצירת המחלקות עצמן.
    *   בדוגמה זו, ניצור מטאקלאס `SingletonMeta`, שיפקח על יצירת המופעים.
    *   המטאקלאס ישמור מופעים של מחלקות במילון `_instances`.
    *   בעת יצירת מופע חדש, נבדוק האם הוא כבר קיים במילון; אם לא – ניצור אותו, אחרת נחזיר את המופע הקיים.
    *   **`Python` Code:**

        ```python
        class SingletonMeta(type):
            """
            מטאקלאס ליצירת סינגלטון.
            """
            _instances = {} # שמירת מופעים

            def __call__(cls, *args, **kwargs):
                """
                דורס את שיטת __call__ כדי לשלוט ביצירת המופע.

                Args:
                    cls: המחלקה שעבורה נוצר המופע.
                    *args: ארגומנטים מיקומיים עבור הבנאי.
                    **kwargs: ארגומנטים ממופים בשם עבור הבנאי.

                Returns:
                    המופע היחיד של המחלקה.
                """
                if cls not in cls._instances: # אם המופע עדיין לא נוצר
                    cls._instances[cls] = super().__call__(*args, **kwargs) # יצירת מופע חדש
                return cls._instances[cls] # החזרת המופע הקיים

        class Singleton(metaclass=SingletonMeta):
            """
            מחלקה שהינה סינגלטון.
            """
            pass

        # דוגמת שימוש
        s1 = Singleton()
        s2 = Singleton()

        print(s1 is s2)  # הדפסת True, מאחר שזהו אותו אובייקט בדיוק
             ```

4.  **באמצעות מודול**

    *   ב-`Python`, מודול כשלעצמו מהווה סינגלטון.
    *   ניתן ליצור אובייקט בתוך מודול, והוא יהיה המופע היחיד.
    *   **`Python` Code:**
        ```python
        # קובץ singleton.py
        class Singleton:
            pass

        instance = Singleton()
        ```
        ```python
        # בקובץ אחר
        from singleton import instance

        obj1 = instance
        obj2 = instance

        print(obj1 is obj2)  # הדפסת True, מאחר שזהו אותו אובייקט בדיוק
        ```

**יתרונות הסינגלטון:**

*   **הבטחת מופע יחיד:** סינגלטון מבטיח שלמחלקה יהיה רק מופע יחיד. זה שימושי לניהול משאבים שחייבים להיות ייחודיים.
*   **גישה גלובלית:** סינגלטון מספק נקודת גישה גלובלית למופע המחלקה, מה שמפשט את השימוש במופע זה בכל חלק בתוכנית.

**חסרונות הסינגלטון:**

*   **מצב גלובלי:** סינגלטון עלול להוביל לשימוש במצב גלובלי, מה שעלול לגרום לתופעות לוואי בלתי צפויות ולהקשות על בדיקות.
*   **הפרת עקרונות תכנות מונחה עצמים:** סינגלטון עלול להפר את עקרונות האחריות היחידה (Single Responsibility) ואנפסולציה (Encapsulation).

**מתי להשתמש בסינגלטון?**

*   כאשר אתה זקוק שאובייקט יתקיים כמופע יחיד (למשל, תצורה, לוגר, חיבור למסד נתונים).
*   כאשר נדרשת גישה גלובלית לאובייקט זה.
