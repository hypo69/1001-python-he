# **Amazin** (Maze Generator)

#### תיאור
המשחק מהווה מחולל מבוכים, אשר בכל פעם יוצר מבוך ייחודי עם מסלול יחיד ונכון. השחקן קובע את מימדי המבוך, והתוכנית בונה אותו תוך התחשבות בפרמטרים שהוגדרו.

- **מאפיינים:**
  - השחקן מזין את רוחב וגובה המבוך.
  - התוכנית מבטיחה קיום מסלול יחיד דרך המבוך.
  - המבוך מוצג על גבי המסך.

---

### הנחיות שלב אחר שלב ליישום:

#### 1. **אתחול המשחק**
   - להציע לשחקן להזין את מימדי המבוך (רוחב וגובה).
   - לוודא שהמימדים שהוזנו תקינים (לדוגמה, גדולים מאחד).

#### 2. **לוגיקת הליבה של יצירת המבוך**
   - ליצור מטריצה בגודל הנתון, המייצגת רשת של תאים.
   - להשתמש באלגוריתם ליצירת מבוכים, לדוגמה, *אלגוריתם חיפוש לעומק רקורסיבי (DFS)*:
     - להתחיל מתא התחלתי אקראי.
     - להתקדם לתאים סמוכים, תוך הסרת קירות בין התא הנוכחי לתא הבא.
     - אם כל השכנים כבר בוקרו, לחזור לתא הקודם ולהמשיך.
     - לסיים את התהליך כאשר כל התאים בוקרו.
   - להבטיח קיום מסלול יחיד דרך המבוך.

#### 3. **הצגה ויזואלית של המבוך**
   - להשתמש בתווים עבור התצוגה:
     - `+`, `-`, `|` עבור קירות.
     - רווחים עבור מעברים.
   - להציג את המבוך המוכן בצורה טקסטואלית.

#### 4. **פונקציות נוספות**
   - אפשרות להגדיר גודל מוגדר מראש (לדוגמה, 10x10), במקרה שהמשתמש הזין נתונים לא תקינים.
   - אזהרה לגבי מימדים גדולים מדי של המבוך על מנת למנוע עומס יתר על הזיכרון.

### דוגמה לפעולת התוכנית

1. **תחילת פעולה**:
   ```
   Введите ширину и высоту лабиринта:
   > 10 8
   ```

2. **פלט המבוך**:
   ```
   +--+--+--+--+--+--+--+--+--+--+
   |        |        |           |
   +  +--+  +  +--+  +  +--+--+  +
   |     |     |     |        |  |
   +--+  +  +--+  +  +  +--+  +  +
   |     |        |     |     |  |
   +--+--+--+--+--+--+--+--+--+--+
   ```

3. **יציאה מהתוכנית**:
   ```
   Сгенерировать новый лабиринт? (да/нет):
   > нет
   До свидания!
   ```
